| 슬롯 | 탭 ID | 탭 이름 | 아이템 | 레시피 수 |
|---|---|---|---|---:|
| 0 | `tab_estate` | 영지 제작 | `EMERALD` | 8 |
| 1 | `tab_trace` | 강화 흔적 | `PRISMARINE_CRYSTALS` | 3 |
| 2 | `tab_smelt` | 제련 | `BLAST_FURNACE` | 2 |
| 3 | `tab_refine` | 정제 | `BREWING_STAND` | 2 |

| 5 | `tab_alchemy_boost` | 연금술 (부스트) | `EXPERIENCE_BOTTLE` | 3 |
| 6 | `tab_cooking` | 요리 | `COOKED_BEEF` | 4 |
| 7 | `tab_ore_convert` | 광물 변환 | `DIAMOND` | 9 |

| 상태 | 아이템 |

## 3. 레시피 슬롯 (슬롯 9~17)

아이템: 완성품 아이콘

### 재료 충분 (등록 가능)
```
§f[완성품 이름]
§7──────────
§7생산량: §f[N]개 / 세트
§7소요 시간: §f[M]분 / 세트
§7──────────
§7필요 재료
§7 [재료명]: §f[보유] §7/ §f[필요]  §a✔
§7 [재료명]: §f[보유] §7/ §f[필요]  §a✔
§7──────────
§7효력: §7[효력 설명]
§7──────────
§7좌클릭  §f1세트 대기열 등록
§7우클릭  §f최대 등록 §8(최대 8세트)
```

```
§7──────────
§7생산량: §f[N]개 / 세트
§7소요 시간: §f[M]분 / 세트
§7──────────
§7필요 재료
§7 [재료명]: §f[보유] §7/ §f[필요]  §a✔
§7 [재료명]: §f[보유] §7/ §f[필요]  §c✘  §8([N]개 부족)
§7──────────
§7효력: §7[효력 설명]
```

## 4. 대기열 슬롯 (슬롯 18 / 21 / 24)

각 대기열은 독립 진행. 한 번 등록되면 완료 전까지 추가 불가.

### 빈 대기열

아이템: `WHITE_STAINED_GLASS_PANE`

```
§7대기열 [N]
§7──────────
§7비어있음
§8레시피를 클릭해 등록하세요
```

### 진행 중 대기열

아이템: 완성품 아이콘

```
§e대기열 [N]  §8진행 중
§7──────────
§a[완성품 이름]
§7세트: §f[완료됨] §7/ §f[총] 세트
§7남은 시간: §f[M]분 [S]초  §8(현재 세트 기준)
§7──────────
§8완료 시 창고 자동 입금
```

## 5. 가공기 페이지 네비게이션 (슬롯 39 / 40 / 41)

영지에 공방 가공기를 여러 대 배정한 경우 페이지로 구분. 가공기 1대 = 1페이지.

### 현재 가공기 표시 (슬롯 40)

아이템: `PAPER`

```
§f공방 가공기 [N] §7/ §f[M]대
§7Lv.[레벨]
```

### 이전 / 다음 버튼 (슬롯 39 / 41)

1페이지이면 슬롯 39, 마지막이면 슬롯 41 = `GRAY_STAINED_GLASS_PANE` (비활성).

```
§f◀ 이전 가공기
§f다음 가공기 ▶
```

## 6. 클릭 규칙

### 레시피 클릭 → 대기열 등록

| 클릭 | 동작 |
|---|---|
| 좌클릭 | 1세트 등록 → 빈 대기열 중 번호 가장 낮은 곳에 배정 |
| 우클릭 | 최대 등록 → 재료 충분하면 8세트, 아니면 가능한 세트 수 |

- 재료는 등록 즉시 소모 (창고 우선, 창고 부족 시 인벤 보충)
- 모든 대기열이 사용 중이면 채팅 알림:
```
§c대기열이 가득 찼습니다.
```

### 대기열 슬롯 클릭

클릭 무반응 (읽기 전용). 진행 중 대기열 취소 기능 없음.

## 7. 완료 처리

- 세트 완료 시: 완성품 수량 × 세트 수를 창고 DB에 즉시 적재
- 대기열 자동 초기화 (빈 상태로 전환)
- 플레이어 오프라인 상태에서도 처리
