# 웹 대시보드 페이지별 상세 스펙

> **[STATUS: DRAFT]** — 운영자 전용 웹 대시보드 페이지 설계.
>
> 기준일: 2026-05-23

---

## 전체 레이아웃 — 탭 기반 단일 화면

URL 직접 입력 없이 탭 클릭으로만 이동한다. 모든 탐색은 클릭 기반이다.

```
┌──────────────────────────────────────────────────────────────┐
│  🔴 Zenon 서버 관제                          [로그아웃]        │
├──────────────────────────────────────────────────────────────┤
│  [현황]  [서버]  [경제]  [필드]  [아이템]  [보스]  [DPS]  [플레이어]  │  ← 상단 탭
├──────────────────────────────────────────────────────────────┤
│                                                              │
│                   탭 콘텐츠 영역                              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 인터랙션 원칙

| 행동 | 결과 |
|---|---|
| 상단 탭 클릭 | 탭 콘텐츠 전환 (페이지 이동 없음) |
| 기간 버튼 클릭 (`7일` `14일` `30일` `전체`) | 해당 탭의 차트·표 전체 갱신 |
| 차트 막대·점 클릭 | 해당 날짜 상세 드릴다운 패널 열림 |
| 표 행 클릭 | 해당 항목 상세 슬라이드 패널 열림 |
| 보스명 클릭 | 해당 보스 상세 섹션으로 포커스 |
| 플레이어 행 클릭 | 플레이어 상세 패널 열림 |
| `X` 또는 바깥 클릭 | 열린 패널 닫힘 |

---

## 공통 규칙

- 토큰 없으면 로그인 화면 표시 (탭 없음).
- 일별 집계 데이터는 자정 스냅샷 기준이다.
- 실시간 데이터(현재 접속자, 최근 보스 클리어)는 30초 자동 갱신한다.
- 나머지 데이터는 5분 자동 갱신 또는 수동 새로고침 버튼.
- 차트 x축 기본값: 최근 14일. 기간 버튼으로 전환.
- 드릴다운 패널은 오른쪽에서 슬라이드인. 탭 전환 시 자동 닫힘.

## 탭 1 — 현황

### 목적

운영자가 서버 상태를 한눈에 파악하는 요약 화면.

### 레이아웃

```
┌─────────────────────────────────────────────┐
│  현재 접속자 수            서버 업타임        │
│  [N명 온라인]              [Xh Xm]           │
├─────────────────────────────────────────────┤
│  오늘 생성 골드  │ 오늘 소모 골드 │ 골드 순증감 │
│  [XXXXG]        │ [XXXXG]       │ [+/-XXXXG]  │
├─────────────────────────────────────────────┤
│  최근 보스 클리어 목록 (최대 10건)            │
│  [시각] [보스명] [클리어타임] [파티인원]      │
├─────────────────────────────────────────────┤
│  오늘 발행 아이템 (등급별)                    │
│  레어 N개 / 에픽 N개 / 유니크 N개 / 레전더리 N개 │
└─────────────────────────────────────────────┘
```

### 표시 데이터

| 항목 | 갱신 주기 | 비고 |
|---|---|---|
| 현재 접속자 수 | 30초 | 실시간 |
| 서버 업타임 | 30초 | 실시간 |
| 오늘 생성 골드 | 5분 | 당일 자정 이후 누계 |
| 오늘 소모 골드 | 5분 | 당일 자정 이후 누계 |
| 골드 순증감 | 5분 | 생성 - 소모 |
| 최근 보스 클리어 | 30초 | 최근 10건 목록 |
| 오늘 발행 아이템 | 5분 | 잠재 등급별 합계 |

## 탭 2 — 서버

### 목적

서버 퍼포먼스와 접속 패턴을 추적한다. TPS 저하·핑 급등·피크 시간대 과부하를 감지한다.

### 레이아웃

```
┌─ 실시간 상태 카드 (30초 갱신) ──────────────┐
│  현재 TPS  │  평균 핑  │  현재 접속자  │ 업타임 │
│  [XX.X]   │  [XXms]  │  [N명]       │ [Xh]  │
└─────────────────────────────────────────────┘

┌─ 필터 바 ──────────────────────────────────┐
│  기간: [오늘] [7일] [14일] [30일]           │
└─────────────────────────────────────────────┘

┌─ 섹션 A: 시간대별 동접 히트맵 ─────────────┐
│  y축: 요일  x축: 0~23시                     │
│  색상 밀도: 평균 동접수                      │
│  → 셀 클릭: 해당 시간대 상세 패널           │
│  (평균 동접, 최대 동접, 평균 TPS, 평균 핑)  │
└─────────────────────────────────────────────┘

┌─ 섹션 B: 일별 평균 동접 + 피크 동접 ───────┐
│  [꺾은선] 일별 평균 동접 (파란선)            │
│  [꺾은선] 일별 피크 동접 (빨간선)            │
│  → 점 클릭: 해당 날 시간대별 동접 패널       │
└─────────────────────────────────────────────┘

┌─ 섹션 C: TPS / 평균 핑 추이 ───────────────┐
│  [꺾은선] 일별 평균 TPS  (기준선: 20.0)     │
│  [꺾은선] 일별 평균 핑 (기준선: 운영자 설정) │
│  TPS 18 미만 구간 자동 붉은 음영 처리        │
└─────────────────────────────────────────────┘

┌─ 섹션 D: 피크 시간대 요약 ─────────────────┐
│  최다 동접 시간대 TOP 3 (시간, 평균 동접)    │
│  TPS 최저 구간 TOP 3 (시간, TPS 수치)        │
└─────────────────────────────────────────────┘
```

### 표시 데이터

| 항목 | 갱신 주기 | 비고 |
|---|---|---|
| 현재 TPS | 30초 | 실시간 |
| 현재 평균 핑 | 30초 | 접속자 평균 |
| 시간대별 동접 히트맵 | 5분 | 시간별 집계 |
| 일별 평균/피크 동접 | 자정 집계 | |
| 일별 평균 TPS/핑 | 자정 집계 | |

## 탭 3 — 경제

### 목적

골드·큐브·강화석 흐름을 일별로 추적하여 경제 이상을 감지한다.

### 레이아웃

```
┌─ 필터 바 ──────────────────────────────────┐
│  기간: [7일] [14일] [30일] [시즌 전체]      │
└─────────────────────────────────────────────┘

┌─ 섹션 A: 골드 총량 ─────────────────────────┐
│  서버 전체 골드 총량 현재값                   │
│  [꺾은선 그래프] 일별 골드 총량 추이          │
└─────────────────────────────────────────────┘

┌─ 섹션 B: 일별 골드 흐름 ───────────────────┐
│  [막대+꺾은선 복합 그래프]                    │
│  막대(생성) / 막대(소모) / 선(순증감)         │
│                                              │
│  수입 세부 내역 (파이 차트):                  │
│  몹 처치 / 보스 보상 / 작물 판매 /            │
│  경매 판매 수입 / 운영자 지급                 │
│                                              │
│  소모 세부 내역 (파이 차트):                  │
│  강화 비용 / 큐브 사용 / 작위 구매 /          │
│  편의 해금 / 경매소 수수료 / 기타             │
└─────────────────────────────────────────────┘

┌─ 섹션 C: 큐브 흐름 ────────────────────────┐
│  [막대 그래프] 일별 큐브 생성 vs 소모         │
│  큐브 조각 → 큐브 변환 수 (일별)             │
│  큐브 직접 사용 수 (일별)                    │
└─────────────────────────────────────────────┘

┌─ 섹션 D: 강화석 흐름 ──────────────────────┐
│  서버 전체 강화석 보유량 현재값 카드          │
│  (무기석 총량 / 방어구석 총량)               │
│  [꺾은선] 서버 전체 강화석 보유량 추이        │
│                                              │
│  [막대 그래프] 일별 강화석 적립 vs 소모       │
│  무기 강화석 / 방어구 강화석 구분             │
│  소모 세부: 강화 시도 / 경매 판매 분리        │
└─────────────────────────────────────────────┘

┌─ 섹션 E: 경제 이상 감지 지표 ──────────────┐
│  [표] 경보 기준 지표 현황                    │
│  (상세는 아래 "경제 이상 감지" 항목 참조)    │
└─────────────────────────────────────────────┘
```

### 경제 이상 감지 지표

| 지표 | 계산 방식 | 경보 기준 | 활성화 조건 | 의미 |
|---|---|---|---|---|
| 일별 골드 순증가율 | (당일 생성 - 당일 소모) / 전일 총량 | +5% 초과 | 시즌 5일 이후 | 골드 인플레 진행 중 |
| 상위 5인 골드 점유율 | 상위 5인 보유 / 전체 총량 | 60% 초과 | 시즌 7일 이후 | 골드 집중 |
| 큐브 소모/생성 비율 | 당일 큐브 소모 / 당일 큐브 생성 | 0.3 미만 | 시즌 5일 이후 | 큐브 과잉 축적 |
| 강화석 소모/생성 비율 | 당일 강화석 소모 / 당일 강화석 생성 | 0.2 미만 | 시즌 5일 이후 | 강화석 과잉 축적 |
| 레전더리 일발행 수 | 당일 잠재 레전더리 확정 건수 | 일 5건 초과 | 즉시 활성 | 큐브 과다 사용 또는 확률 오류 |
| 경매소 일 거래액 | 당일 `auction_sell` 골드 합계 | 전 7일 평균 대비 3배 초과 | 시즌 8일 이후 | 비정상 거래 가능성 |

경보 조건 충족 시 해당 지표 셀을 황색(주의) 또는 적색(위험)으로 강조한다.

> **활성화 조건**: 시즌 초반은 골드 총량 기저가 낮아 순증가율·점유율 지표가 항상 경보 상태가 됨. API의 `season_day` 값이 활성화 조건 미만이면 해당 지표는 회색(비활성) 표시.
> 경매 거래액은 전 7일 이동 평균 대비 상대 기준을 사용해 시즌 중반 거래 증가를 정상으로 처리한다.

### 플레이어별 골드 내역 (서브 테이블)

| 컬럼 | 설명 |
|---|---|
| 플레이어명 | UUID 기반 조회 |
| 현재 보유 골드 | |
| 누적 생성 골드 | 시즌 시작 이후 |
| 누적 소모 골드 | 시즌 시작 이후 |
| 최근 7일 순증감 | |

정렬: 기본값 현재 보유 골드 내림차순. 검색 필터 제공.

## 탭 4 — 필드

### 목적

필드별 자원 생산량과 몬스터 처치 수를 추적한다. 필드 난이도·밀도·수익성 밸런스 판단 근거를 제공한다.

### 레이아웃

```
┌─ 필터 바 ──────────────────────────────────┐
│  필드: [전체] [초원] [광산] [숲] [하수구] … │  ← 클릭 선택
│  기간: [오늘] [7일] [14일] [30일]           │
└─────────────────────────────────────────────┘

┌─ 섹션 A: 필드별 자원 생산 비교 (막대 그룹) ┐
│  x축: 필드명  y축: 생산량                    │
│  막대 그룹: 골드 │ 강화석 │ 큐브             │
│  → 막대 클릭: 해당 필드 상세 패널            │
└─────────────────────────────────────────────┘

┌─ 섹션 B: 필드별 몬스터 처치 수 ────────────┐
│  [막대] 필드별 총 처치 수                    │
│  [꺾은선] 일별 처치 수 추이 (필터 선택 시)  │
│  → 막대 클릭: 해당 필드 상세 패널            │
└─────────────────────────────────────────────┘

┌─ 섹션 C: 필드 밀도 이상 감지 ──────────────┐
│  [표] 필드별 1인당 시간당 처치 수            │
│  기준 대비 과다(🔴) / 적정(🟢) / 과소(🟡) 표시 │
│  과다: 리스폰 속도 대비 너무 빠른 클리어     │
│  과소: 플레이어가 기피하는 필드              │
└─────────────────────────────────────────────┘

     ┌─ 필드 상세 패널 (클릭 시) ──────────┐
     │  필드명                               │
     │  총 처치 수  │  총 생산 골드           │
     │  총 강화석   │  총 큐브                │
     │  1인당 시간당 처치 수                  │
     │  [꺾은선] 일별 처치 수 추이            │
     │  [꺾은선] 일별 골드 생산 추이          │
     │                            [X 닫기]  │
     └─────────────────────────────────────┘
```

### 필드 밀도 판단 기준

| 지표 | 설명 | 경보 기준 |
|---|---|---|
| 1인당 시간당 처치 수 | 해당 필드 총처치 ÷ 필드 체류 플레이어수 ÷ 시간 | 설계값 ±50% 이탈 |
| 골드/처치 비율 | 필드 총골드 ÷ 총처치 수 | 전 필드 평균 대비 2배 초과 |
| 필드 기피 지수 | 해당 필드 처치 수 ÷ 전체 평균 처치 수 | 0.3 미만 |

> 설계값은 보스/필드 문서(`zenon-rpg/docs/06_fields_bosses/`)에서 확정 후 채운다.

## 탭 5 — 아이템

### 목적

장비 및 소모재 발행 수를 추적하여 공급 과잉·부족을 판단한다.

### 레이아웃

```
┌─ 필터 바 ──────────────────────────────────┐
│  기간: [7일] [14일] [30일] [시즌 전체]      │
└─────────────────────────────────────────────┘

┌─ 섹션 A: 장비 등급별 누적 발행 수 ─────────┐
│  레어 / 에픽 / 유니크 / 레전더리              │
│  (잠재 확정 기준, 큐브 사용 결과)             │
│  [막대 그래프] 일별 발행 추이                │
└─────────────────────────────────────────────┘

┌─ 섹션 B: 큐브 사용 결과 분포 ──────────────┐
│  전체 큐브 사용 N회                          │
│  등급별 결과 분포 (파이 차트)                │
│  일별 큐브 사용 수 (막대)                    │
└─────────────────────────────────────────────┘

┌─ 섹션 C: 전승권 사용 수 ───────────────────┐
│  기본 전승 / 등급전승권 / 세부스탯전승권       │
│  일별 사용 수 (막대)                         │
└─────────────────────────────────────────────┘

┌─ 섹션 D: 강화 흔적 발행/소모 ──────────────┐
│  별의 흔적 / 달의 흔적 / 태양의 흔적         │
│  일별 발행(공방 제작) vs 소모(강화 사용)      │
└─────────────────────────────────────────────┘
```

## 탭 6 — 보스

### 목적

보스 클리어 데이터를 기반으로 너프·버프 판단 근거를 제공한다.

### 밸런스 판단에 필요한 최소 데이터

보스 너프/버프 판단에는 아래 데이터가 반드시 있어야 한다.

| 데이터 | 왜 필요한가 |
|---|---|
| 클리어 타임 | 설계 목표 타임 대비 얼마나 빠른지 판단 |
| 파티 구성 (무기 클래스별) | 특정 무기 조합이 과도하게 유리한지 확인 |
| 파티 인원 수 | 소수 인원 클리어가 가능한지 확인 |
| 평균 DPS (파티 전체 / 개인별) | 설계 DPS 기준 대비 초과 여부 |
| 최고 단일 타격 | 원샷 방지 기준(85% 클램프) 작동 여부 확인 |
| 클리어 실패 횟수 | 보스 난이도 체감 확인 |
| 타임아웃 소멸 횟수 | 보스 분류별 타임아웃 기준 미달 빈도 |

### 레이아웃

```
┌─ 필터 바 ──────────────────────────────────┐
│  보스: [전체] [필드보스1~5] [시즌보스1~6]   │
│        [시즌최종보스1~3]                     │
│  기간: [7일] [14일] [30일] [시즌 전체]      │
└─────────────────────────────────────────────┘

┌─ 보스 선택 버튼 열 ─────────────────────────┐
│  [전체] [들판 포식자] [폐광 골렘] [수로 군주]│  ← 클릭으로 선택
│  [균열 파수꾼] [시즌보스1~6] [최종보스1~3]   │
└─────────────────────────────────────────────┘

┌─ 선택 보스 요약 카드 ──────────────────────┐
│  총 클리어 N회  │  실패 N회  │  타임아웃 N회 │
│  평균 타임 Xm Xs  │  최단 타임 Xm Xs         │
└─────────────────────────────────────────────┘

┌─ 클리어 타임 히스토그램 ───────────────────┐
│  [히스토그램] 클리어 타임 분포               │
│  설계 목표 타임 기준선 표시                  │
│  → 막대 클릭: 해당 구간 클리어 목록 패널     │
└─────────────────────────────────────────────┘

┌─ 파티 구성 분포 (나란히 배치) ─────────────┐
│  [막대] 인원별        [막대] 무기 클래스 빈도 │
│  1인/2인/3인           검/도끼/창/석궁/낫/스태프│
│  → 막대 클릭: 해당 구성 클리어 목록 패널     │
└─────────────────────────────────────────────┘

┌─ DPS 요약 ────────────────────────────────┐
│  평균 파티 DPS (총합)                        │
│  [막대] 무기 클래스별 평균 개인 DPS          │
└─────────────────────────────────────────────┘

┌─ 클리어 기록 목록 (테이블) ────────────────┐
│  시각 │ 클리어타임 │ 인원 │ 파티 무기구성   │
│  파티 총 DPS                                │
│  → 행 클릭: 해당 클리어 상세 패널           │
│    (파티원별 DPS, 최고 단일타격, 데스카운트) │
└─────────────────────────────────────────────┘
```

### 보스별 설계 목표 타임 (기준선 표시용)

| 보스 유형 | 설계 목표 타임 | 타임아웃 |
|---|---|---|
| 필드보스 | 기준 그룹 TTK 8~13분 (`field_boss_stats_v1.md`) | 15분 |
| 시즌보스 1~6 | 실딜 8~12분, 총 전투 15분 내 (`season_boss_stats_v1.md`) | 15분 |
| 시즌최종보스 3종 | 22강 무기 + 레전더리 잠재 3인 기준 10분 내 | 10분 |

> 대시보드는 위 기준선을 표시하되, 오픈 후 1주차 평균 TTK 로그로 HP/패턴 보정 여부를 검토한다.

## 탭 7 — DPS

### 목적

무기 클래스별, 잠재 세팅별 실측 DPS를 비교하여 밸런스 판단 근거를 제공한다.

### 전제 조건

이 페이지는 `boss_combat_log` 테이블에 개인별 DPS 기록이 충분히 쌓인 뒤 의미 있다. 오픈 초기 7일은 데이터 수집 기간으로 본다.

### 레이아웃

```
┌─ 필터 바 ──────────────────────────────────┐
│  보스: [전체] [필드보스 합산] [시즌보스 합산]│
│  강화 구간: [전체] [15강↑] [18강↑] [20강↑] │
│  기간: [7일] [14일] [30일] [시즌 전체]      │
└─────────────────────────────────────────────┘

┌─ 섹션 A: 무기 클래스별 평균 DPS ───────────┐
│  [막대 그래프] 6개 무기 클래스 평균 DPS 비교 │
│  오차 범위(표준편차) 표시                    │
│  샘플 수 병기                               │
└─────────────────────────────────────────────┘

┌─ 섹션 B: 잠재 등급별 DPS 영향 ─────────────┐
│  잠재 없음 / 레어 / 에픽 / 유니크 / 레전더리 │
│  무기 클래스별 잠재 등급 평균 DPS 비교 테이블 │
└─────────────────────────────────────────────┘

┌─ 섹션 C: 직업각인 조합별 DPS ──────────────┐
│  각인 A vs 각인 B (무기별)                  │
│  샘플 수 10 미만은 참고값으로 표시           │
└─────────────────────────────────────────────┘
```

## 탭 8 — 플레이어

### 목적

개별 플레이어의 경제 활동과 성장 현황을 운영자가 확인한다.

### 레이아웃

```
┌─ 검색 + 정렬 바 ───────────────────────────┐
│  [검색창]        정렬: [골드▼] [IL] [등급]  │
└─────────────────────────────────────────────┘

┌─ 플레이어 목록 테이블 ─────────────────────┐
│  닉네임 │ 현재 골드 │ 누적 생성 │ 누적 소모  │
│  무기 클래스 │ 평균 IL │ 잠재 최고 등급       │
│  → 행 클릭: 오른쪽에 상세 패널 슬라이드인    │
└─────────────────────────────────────────────┘

     ┌─ 상세 패널 (행 클릭 시) ────────────┐
     │  닉네임  무기 클래스  Lv.            │
     │  ───────────────────────────────    │
     │  골드  보유 / 누적 생성 / 누적 소모   │
     │  강화  슬롯별 현재 강화 단계          │
     │  잠재  슬롯별 잠재 등급              │
     │  ───────────────────────────────    │
     │  최근 보스 클리어 10건               │
     │  최근 골드 이벤트 20건               │
     │                          [X 닫기]   │
     └─────────────────────────────────────┘
```

> 1차 버전에서는 목록 + 상세 패널(기본정보·강화·잠재)까지 구현한다. 골드 이벤트 상세는 2차.
