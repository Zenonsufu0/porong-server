## Phase 2. 디스코드 인증

> Discord 선입장 온보딩 플로우. 약관 동의(역할) → 마크 접속(제한 로비) → 인게임 `/인증` 코드 발급 → 봇 버튼/모달 검증(`/auth/verify`) → 역할 부여 + 화이트리스트. (코드 방향 = 인게임 발급 → 봇 검증, DL-138)

### 작업 목록

**2-1. 봇 기반 설정**
- Discord 봇 토큰 등록 (secrets 관리 주의)
- 채널 구조 구성:
  - `약관-인증`, `접속정보` (역할 제한), `필드보스-알림`, `시즌보스-알림`, `운영로그`

**2-2. 온보딩 플로우 구현**

```
디스코드 입장
→ 약관 동의 (버튼 인터랙션) → <서버>_인증전 역할 부여
→ 인게임 접속 (미인증 = 제한 로비 허용)
→ 인게임 /인증 → 게임서버가 코드 발급 (uuid 바인드·짧은 TTL·1회용)
→ 봇 인증 버튼/모달에 코드 입력
→ 봇 POST /auth/verify (X-API-Key, {code, discordId}) → 200 {ok, uuid, name}
→ 인증유저 역할 부여 + 화이트리스트(플레이 권한)
```
> DL-138. 닉네임 1회 입력 단계 폐기 — verify 응답 `name` 사용. 구 `/연동`(봇 발급 코드 입력)·role-queue 폴링 폐기.

**2-3. 역할 체계 구현**
- 권한 역할 (자동): 미인증 → 접속대기 → 인증유저 → 테스터 → 운영진
- 클래스 역할 (선택): 검사 / 도끼전사 / 창기사 / 석궁사 / 낫사냥꾼 / 지팡이술사
- 알림 구독 역할 (선택): 필드보스알림 / 시즌보스알림 / 업데이트알림

**2-4. PoroRPG 연동 처리**
- 인게임 코드 생성 → Discord 봇 검증 → DB 연동 확정

### 완료 기준
- 신규 유저가 Discord 플로우를 통해 인증유저 역할까지 독립적으로 완료 가능
- 인게임 `/인증` 코드 발급 → 봇 `/auth/verify` 검증 후 DB에 discord_id ↔ {uuid, name} 매핑 확인

## Phase 3. 전투 코어

> 6무기 클래스, 스킬 입력, 쿨타임, 자기 자원, 직업각인.

### 작업 목록

**3-1. 무기 6종 클래스 구현**

| 무기 | 내부 ID |
|---|---|
| 검 | `sword` |
| 도끼 | `axe` |
| 창 | `spear` |
| 석궁 | `crossbow` |
| 낫 | `scythe` |
| 스태프 | `staff` |

**3-2. 스킬 입력 체계**
- LC (기본기) / RC (이동기) / Shift+RC (특수기) / F (핵심기)
- 스킬타입 판별: 입력 이벤트 기반 서버-사이드 추적

**3-3. 쿨타임 시스템**
- 스킬별 독립 쿨타임 관리
- 쿨타임 감소% 잠재 적용 지원

**3-4. 자기 자원 (유지형)**
- 무기 무관 최대 6스택
- 스태프의 "마력 충전" 자원 (무기 고유 자원명, 영지 마력 시스템과 별개)

**3-5. 직업각인**
- 무기별 A/B 2종 고유 각인
- 공용각인: 1차 시즌 제외
- 레퍼런스: `zenon-rpg/docs/04_combat_weapon_skills/weapon_skills_v1.md`

**3-6. 금지 설계 검증 목록**
- 지속 장판 ❌
- 적 표식 ❌
- 방어 감소 ❌
- 받는 피해 증가 ❌
- 파티 전체 피해 증가 ❌
- 복잡한 스택 폭발 ❌

**3-7. 전투 피해 공식 구현**

```
유효 방어율 = DEF / (DEF + 200)
기본 피해   = ATK × (1 - 유효 방어율)
스킬 피해   = 기본 피해 × 스킬 배율
```

레퍼런스: `zenon-rpg/docs/04_combat_weapon_skills/combat_balance_v2.md`

### 완료 기준
- 6무기 각각 4개 스킬 입력 인식 및 스킬 발동 확인
- 직업각인 A/B 전환 동작 확인
- 피해 공식 단위 테스트 통과

## Phase 4. 장비 성장

> T1 장비 5슬롯, 강화, 잠재/큐브, 강화 흔적, 전승.

### 작업 목록

**4-1. T1 장비 5슬롯**
- 슬롯: 무기 / 투구 / 상의 / 하의 / 신발 (악세서리 없음)
- 장비 등급: 커먼~레전더리
- 장비 이름 변경권: 10,000G

**4-2. 강화 시스템 (1~25강)**
- 강화석: DB 가상 재화로 직접 적립/소모. 실물 아이템 드랍 없음
- 방어구 강화석 비용: `ceil(무기 강화석 ÷ 1.5)`
- 강화 성공률 표 적용: `economy_numbers_v2.md` T1 1~25강 기준
- 파괴/하락 없음
- 최대 25강. 시즌보스/최종보스 권장 스펙은 22강+ 선발대 기준

**4-3. 강화 흔적 3종**
- 별의 흔적 / 달의 흔적 / 태양의 흔적
- 강화 성공률 보정 아이템
- 영지 공방 가공기 제작 경로 (Phase 5 연동)

**4-4. 잠재능력/큐브 시스템**
- 등급: 커먼 / 레어 / 에픽 / 유니크 / 레전더리
- 3라인 구조: 1라인=현재 등급, 2·3라인=아래 등급 (이탈 2라인 10%/3라인 5%)
- 큐브 1회: 3라인 전부 재롤 + 등급 상승 시도
- 메모리얼 선택 화면: 현재 vs 신규 비교 선택
- 등급 보호: 결과가 현재보다 낮으면 자동 폐기 (큐브만 소모)
- 천장: 유니크→레전더리 1,000회 / 에픽→유니크 200회 / 레어→에픽 40회 / 시작→레어 10회
- 레퍼런스: `zenon-rpg/docs/02_database_api_stats/potential_options_v1.md`

**4-5. 전승 시스템**
- 기본 전승: 0G (무료)
- 등급전승권: 100,000G
- 세부스탯전승권: 100,000G

### 완료 기준
- 강화 1~25강 성공/실패 동작 확인
- 큐브 사용 → 메모리얼 화면 → 선택 적용 동작 확인
- 천장 카운터 DB 저장 및 로그아웃 후 유지 확인

- 포트 8765

**7-5. 경제 로그**

- 골드 거래 로그
