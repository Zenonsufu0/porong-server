## 3. 아이템 슬롯 lore

### 구매·판매 모두 있는 아이템
```
§f[아이템 이름]
§7──────────
§e1세트: §f[N]G §7([amount]개)
§7좌클릭  §f1세트 구매 §7([amount]개 / [N]G)
§7우클릭  §f[64/amount]세트 구매 §7(~64개 / [총G]G)
§7──────────
§2판매: §f[N]G §71개
§7Shift+클릭  §f1개 판매
```

> 우클릭 세트 수 = `max(1, 64 / amount)`. 결과적으로 약 64개를 한번에 구매. (DL-075)

### 구매 전용 (치장·특수)
```
§f[아이템 이름]
§7──────────
§e구매: §f[N]G
§7좌클릭  §f구매
```

품절 시: `§c품절` 표시, 클릭 비활성.

## 4. 구매 처리 (DL-075)

| 클릭 | 구매량 |
|---|---|
| 좌클릭 | 1세트 (config의 `amount`개) |
| 우클릭 | `max(1, 64 / amount)` 세트 — 약 64개 |

> 예시:
> - 황금사과(amount=1): 좌=1개, 우=64개
> - 빵(amount=16): 좌=16개, 우=64개 (4세트)
> - 화살(amount=64): 좌=64개, 우=64개 (1세트, 좌=우 동일)

골드 부족 시:
```
§c골드가 부족합니다. (보유: §f[N]G §c/ 필요: §f[N]G§c)
```

구매 아이템 적재: 인벤토리 우선 → 인벤 풀이면 창고 자동 적재.

## 6. 판매 서브 GUI (54슬롯)

현재 탭에서 판매 가능한 아이템만 표시. 인벤토리·창고 보유량 합산.

```
행\열   1     2     3     4     5     6     7     8     9
row0~4  [판매 가능 아이템 슬롯 × 45]
row5  [전량판매] ░   ░   [◀]  [페이지]  [▶]   ░     ░   [뒤로]
```

### 판매 아이템 슬롯 lore
```
§f[아이템 이름]
§2판매가: §f[N]G §71개
§7──────────
§7인벤토리: §f[N]개
§7창고: §f[N]개
§7합계: §f[N]개  →  §e[총G]G
§7──────────
§7좌클릭    §f1개 판매
§7우클릭    §f64개 판매
§7Shift+클릭  §f전량 판매
```

판매 차감 순서: 인벤토리 → 창고.

### 전량 판매 버튼 (슬롯 45)

아이템: `GOLD_BLOCK`

```
§e전량 판매
§7──────────
§7현재 탭 판매 가능 아이템 전부 판매
§7예상 수익: §e[N]G
§c[클릭 후 재클릭으로 확인]
```

전량 판매 클릭 시 슬롯 45 → `FIRE_CHARGE`(확인)로 교체 — 재클릭 시 실행, 다른 슬롯 클릭 시 취소.

## 8. 구현 참조

| 항목 | 내용 |
|---|---|
| 열기 트리거 | 영지 서브 허브 상점 구역 (slot 30~32, 39~41, 48~50, 중앙 40) 클릭 |

| 판매 차감 순서 | 인벤토리 → 창고 |
| 특수 아이템 구매 | `island_settings` 플래그 ON + `player_unlocks` 기록 |
