# 11. 운영자 웹 대시보드 — 전체 구조 개요

> **[STATUS: DRAFT]** — 운영자 전용 웹 대시보드 설계 문서. 구현 전 검토 필요.
>
> 기준일: 2026-05-23

---

## 0. 한 줄 정의

PoroRPG HTTP API(포트 8765)를 통해 운영자가 경제·전투 데이터를 시각적으로 모니터링하는 **읽기 전용 관제 도구**다.

---

## 1. 설계 원칙

| 원칙 | 내용 |
|---|---|
| 접근 제어 | 운영자 전용. 토큰 인증. 공개 접근 없음 |
| 데이터 원본 | PoroRPG HTTP API 단일 경로. DB 직접 접근 없음 |
| 갱신 방식 | 일별 집계 (자정 스냅샷). 실시간은 현재 접속자 수·TPS·핑·최근 보스 클리어만 예외 허용 |
| 범위 | 읽기 전용. 운영 개입 명령(킥·밴·수동 지급)은 대시보드 범위 제외 |
| 기술 복잡도 | 1차 버전은 단순하게. SPA 빌드 체인 최소화 |

---

## 2. 탭 구성 (단일 페이지, 8탭)

대시보드는 단일 SPA. URL 라우팅 없이 탭 클릭으로 전환.

| 탭 번호 | 탭 이름 | 주요 내용 |
|---|---|---|
| 탭 1 | 현황 | 현재 접속자 수·TPS·핑, 최근 보스 클리어, 당일 요약 수치 |
| 탭 2 | 서버 | 시간대별 동접 히트맵, 일별 TPS·핑·동접 추이, 피크 시간대 TOP3 |
| 탭 3 | 경제 | 골드 총량·일별 흐름·큐브·강화석, 경제 이상 감지 지표 |
| 탭 4 | 필드 | 필드별 골드/강화석/큐브 생산, 몬스터 처치 수, 밀도 이상 감지 |
| 탭 5 | 아이템 | 등급별 잠재 발행, 큐브 사용 결과, 전승권·강화 흔적 |
| 탭 6 | 보스 | 보스별 클리어 타임·파티 구성·DPS 분포 |
| 탭 7 | DPS | 무기 클래스별·잠재 등급별 평균 DPS |
| 탭 8 | 플레이어 | 검색·정렬, 개인 골드/강화/잠재 상세 패널 |

---

## 3. 기술 스택 (제안)

| 항목 | 선택 | 비고 |
|---|---|---|
| 프론트엔드 | 바닐라 JS + Chart.js 또는 경량 SPA | 빌드 체인 없는 방향 우선 |
| 백엔드 | 없음 (API 직접 호출) | PoroRPG API가 데이터 제공 |
| 인증 | Bearer 토큰 헤더 | 토큰은 서버 환경변수로 관리 |
| 호스팅 | 서버 내 정적 파일 서빙 또는 별도 포트 | 포트 8765와 분리 권장 |
| 차트 | Chart.js | 의존성 단순, CDN 가능 |

---

## 4. 데이터 흐름

```
운영자 브라우저
    │
    │  (HTTPS + Bearer Token)
    ▼
PoroRPG HTTP API (포트 8765)
    │
    │  (내부 Java 호출)
    ▼
SQLite poro.db
```

웹 대시보드는 API를 통해서만 데이터를 읽는다. DB 파일에 직접 접근하지 않는다.

## 6. 1차 버전 범위

포함:
- 탭 1 현황 (현재 접속자·최근 보스 클리어·당일 요약)
- 탭 2 서버 (TPS·핑·동접 히트맵·피크 시간대)
- 탭 3 경제 (골드·큐브·강화석 일별 그래프·이상 감지)
- 탭 4 필드 (필드별 자원 생산·몬스터 처치·밀도 감지)
- 탭 5 아이템 발행 (등급별·큐브 결과·전승권)
- 탭 6 보스 (클리어 타임·파티·DPS)
- 탭 7 DPS 분석 (무기 클래스별·잠재 등급별·각인 조합별)
- 탭 8 플레이어 조회 (검색·정렬·개인 장비/골드/잠재 상세)

제외 (1차):
- 실시간 스트리밍 (WebSocket)
- 운영 개입 기능 (킥·지급·밴)
- 모바일 최적화
