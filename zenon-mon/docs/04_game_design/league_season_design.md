> 허브 리그 구역(`hub_design.md` §6)의 게임 설계. 기록·진행은 ZenonMonCore가 담당.
> 연동: `SeasonManager` / `PlayerProgress` / `seasons.json`(`config_structure.md`).
> 전제: **모든 대전은 Cobblemon 포켓몬 배틀**(인간 PvP 아님, 결정 011).
> ⚠️ 초안. `TBD`/`예시`는 검토·확정 대상.
> ✅ **정규리그 코어 구현 완료(2026-06-07, 결정 036)**: §4 점수제+실시간 큐 매칭+lvl50 정규화 구현. **§7 블로커 해소(옵션 A)** — `BattleFormat.adjustLevel=50` + `BattleBuilder.pvp1v1`. 구현 모듈명은 설계의 `SeasonManager` 대신 **`LeagueManager`**(+`LeagueMenu`·`SeasonConfig`·`seasons.json`). 챔피언스리그(§6)·시즌 리셋(§8)은 후속. **인게임 미검증(2인 필요)**: 매칭/lvl50/점수.

## 1. 목표 / 원칙
- 짐 진행 이후의 **경쟁형/도전형 엔드게임**.
- 콘텐츠 3종으로 분리: **배틀타워(상시 PvE)** / **정규리그(점수제)** / **챔피언스리그(서버 마지막날 토너먼트)**.
- 친선대전은 비공식(기록 없음).
- 모든 대전은 포켓몬 배틀. 인간 직접 가해 없음.
- 밸런스(룰셋·점수·보상)는 `seasons.json`로 설정(하드코딩 금지).

## 3. 배틀타워 (탑 형식 PvE — 확정)
- **외형은 탑.** 입장하면 위로 올라가며 연속 트레이너 배틀에 도전.
- **총 50층(확정).** 1층부터 50층까지, **올라갈수록 난이도 상승**.
- **입장 자격: 관장 8명(8배지) 전원 클리어** (결정 028 — 기존 "배지 4개 이상"에서 상향. 엔드콘텐츠로 후반 배치). 세부: `battle_tower_design.md`.
- **진행 규칙(확정):**
  - 한 층의 트레이너를 **이기면 다음 층으로 진행**.
  - **지면 탑 밖으로 퇴장**(해당 시도 종료).
  - **진행상황은 저장 → 패배해도 초기화 안 함.** 재입장 시 마지막으로 도달/클리어한 지점부터 이어서 도전(클리어한 층은 다시 안 싸움). 즉 **체크포인트식**.
- **보상(확정): 각 층 클리어 시 보상 지급.** 층이 높을수록 보상↑. 보상은 **층당 최초 클리어 1회**(파밍 방지; 재도전 무보상) — `gym_badge_design.md` §6 초회 원칙과 동일 결.
- **트레이너 구성:** 층별로 고정 또는 진행도 tier 기반 풀. 50층 곡선에 맞춰 난이도 설계(`seasons.json`).
- **플레이어별 인스턴스**(동시 도전 가능, `gym_badge_design.md` §4-1과 동일 원리). 점유 락 불필요.
- 배틀 룰셋(확정): 6마리. **상대 NPC = Lv100 고정, 플레이어 = 실제 파티 레벨(정규화 없음).** 엔드콘텐츠라 직접 육성해 도전하는 의도 — **정규리그(§4)의 Lv50 정규화와 의도적으로 대비**. 층별 세부 = `battle_tower_design.md`.
- 시즌과 **독립(상시 콘텐츠)**.
- 저장: `PlayerProgress`에 `battleTower.highestClearedFloor`, 클리어 층 보상 수령 플래그.

**세부 설계 완료 → `battle_tower_design.md`**: 층별 트레이너 구성(50층 전수, 종/기술 jar 검증), 보상 곡선(§3-R: 층당 골드+10층 체크포인트+50층 특별 ~95,000 골드 예시), 50층=왕중왕전 예선 자격 후보. **남은 미정:** 보상 수치 운영 튜닝, NPC 메가/AI 실배틀 검증.

## 4. 정규리그 (점수제 래더 — 확정)
- **자격: 배지 8개(확정).** 시즌 내내 상시 참여.
- **방식: 점수제(확정).** 리그 배틀에서 **이기면 +점수, 지면 −점수.** 시즌 동안 누적 → 순위.
  - 점수폭(예시·확정 방향): **승 +10 / 패 −7** (`seasons.json`로 조정). 시작 점수 1000.
  - 점수 하한선 적용(예: 0 미만 방지) — TBD.
- **레벨: 레벨 50 고정.** 실제 파티 레벨과 무관하게 50으로 정규화해 대전(구현 §7: A/B/C).
- 형식: 6마리 싱글 가정(TBD).
- 시즌 종료 시 정규리그 순위 확정 → 챔피언스리그 시드/자격에 반영.

### 4-1. 매칭 시스템 (정규리그)
**실시간 큐 매칭만(확정). AI/비동기 상대 없음.**
- 프로젝트 서버 특성상 정규리그 시간대에 **실제 플레이어가 있다고 전제**한다. 상대가 없으면 **큐에서 대기**(AI 대체전 없음).

1. 플레이어가 정규리그장에서 **랭크 큐 참가**(또는 `/zenonmon` 메뉴).
2. 서버가 큐 내에서 **점수 근접한 실제 플레이어**와 페어링.
   - 매칭 윈도우 = 시작 ±50점에서 **대기 시간에 따라 점차 확대**(예: 10초마다 +25). 상한(±400) 도달 시 큐 내 가장 가까운 상대와 성사.
   - 상대가 전혀 없으면 매칭될 때까지 대기(취소 가능).
3. 양측 팀 lvl50 정규화 → 배틀 → 결과로 ±점수 반영(승자 +10 / 패자 −7).
4. 배틀 무효(접속 끊김 등) 처리: 끊은 쪽 패 처리 또는 노카운트 — TBD.

**어뷰징 방지:**
- **동일 상대 반복 파밍 제한**: 같은 상대와 단시간 재대전 시 점수 미반영 또는 쿨다운.
- 팀 동기화/검증은 서버 권위. 자해성 점수 조작(짜고 지기) 방지 장치 — TBD.

**저장:** `PlayerProgress`에 `rankedLeague.score`, `wins/losses`, 최근 상대 이력(쿨다운용).

## 5. 친선대전 (배틀 아레나 — 비공식)
- 유저 간 자유 대전. **포켓몬 6마리, 현재(실제) 레벨 그대로** 싸움.
- 정규화 없음, **기록/점수 없음**, 보상 없음. 순수 연습/친목.
- 자격 제한 없음.

## 6. 챔피언스리그 (서버 마지막날 토너먼트 — 확정 방향)
- **개최: 서버 마지막날(시즌 종료/서버 종료 시점) 1회성 이벤트.**
- **방식: 참가자 전체가 토너먼트(브래킷)로 대결 → 최후의 챔피언 1명 결정.**
- 정규리그와 **별개**: 정규리그=상시 점수제, 챔피언스리그=마지막날 단판 토너먼트.
- 참가 자격: 정규리그 참가자 전체(또는 상위 N) — TBD.
- 레벨/룰셋: 정규리그와 동일(lvl50, 메가만 허용 / 테라·다이맥스·Z무브 off).
- 우승자 = 서버 최종 챔피언 → **챔피언 홀 등재 + 동상 영구 보존**.

> "서버 마지막날" = 이 시즌/서버 라인의 종료 이벤트. 월드 와이프와는 분리(`world_policy.md` §6). 챔피언 이력은 다음 시즌으로 영구 보존.

## 7. 레벨 50 정규화 — ✅ 해소(옵션 A 채택, 결정 036)
> Cobblemon `BattleFormat.adjustLevel`(int) 확인 → GEN_9_SINGLES 복제 후 `adjustLevel=50`, `pvp1v1`에 전달. 아래는 검토 당시 옵션.

정규리그/챔피언스리그는 lvl50 고정이 목표. Cobblemon 월드 배틀이 기본적으로 실제 레벨을 쓰므로 구현 방식 확정 필요:
| 옵션 | 설명 | 확인 |
|---|---|---|
| A. Showdown 포맷 | MSD 번들 Showdown 시뮬의 Adjust Level 룰 사용 | Cobblemon이 월드 배틀에 노출하는지 검증 |
| B. ZenonMonCore 전용 진입점 | 리그 배틀을 별도 흐름으로 만들어 lvl50 적용 | 구현량 ↑ |
| C. 입장 시 클램프 | 배틀용 임시 팀을 lvl50로 스케일 | 스탯/기술 정합 주의 |
> 셋 다 안 되면 정규리그도 실제 레벨로 운영하는 폴백 검토. → Phase 6 전 기술 검증 항목.

## 9. ZenonMonCore 연동
- `SeasonManager` — 시즌 상태, 정규리그 점수/순위, 챔피언스리그 브래킷, 챔피언 이력.
- `PlayerProgress` 확장 — 정규리그 점수/순위/타이틀(`database_schema.md` 스키마 추가 필요).
- 배틀타워 — tier별 트레이너 풀 + 진행 기록(연승/최고층).
- `RewardManager` — 배틀타워/시즌/챔피언 보상.
- 배틀 결과 수신 = Cobblemon 배틀 이벤트(서버 권위 판정).
- 로깅 = 챔피언 확정/보상 = `AuditLog`.

## 10. seasons.json (향후, 스케치)
```jsonc
{
  "configVersion": 1,
  "battleTower": {
    "requireBadges": 4,
    "floors": 50,                   // 총 50층
    "floorRewardsOncePerFloor": true,
    "floorTrainers": [ /* 층(또는 tier) → 트레이너 */ ],
    "rewardCurve": {},              // 층 높을수록 보상↑
    "clearBonus": {}                // 50층 클리어 특별 보상
  },
  "rankedLeague": {
    "requireBadges": 8,
    "levelMode": "flat50",          // 레벨 50 고정(구현 §7)
    "format": "singles6",
    "startScore": 1000,
    "score": { "win": 10, "loss": -7, "floor": 0 },
    "matchmaking": {
      "mode": "realtime",           // 실시간만 (AI/비동기 없음)
      "windowStart": 50, "windowStepPerSec": 2.5, "windowMax": 400,
      "rematchCooldownSeconds": 600 // 동일 상대 파밍 방지
    },
    "ruleset": { "allowMega": true, "allowTera": false, "allowDynamax": false, "allowZMove": false, "bannedSpecies": [] }
  },
  "championsLeague": {
    "trigger": "server_last_day",   // 마지막날 1회
    "bracket": "single_elim",
    "participants": "all_ranked"    // 전체 or 상위 N
  }
}
```
