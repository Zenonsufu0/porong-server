## 1. 역할 체계

### 1-1. 권한 역할 (시스템 자동 부여)

인증 단계에 따라 자동 부여·승급된다. 플레이어가 직접 선택 불가.

| 역할 | 부여 조건 | 채널 접근 |
|---|---|---|
| `미인증` | 디스코드 입장 직후 기본 | 서버소개·규칙·약관인증 |
| `접속대기` | 약관 동의 + 닉네임 입력 완료 | + 접속정보·faq |
| `인증유저` | 인게임 코드 인증 완료 | + 잡담·스크린샷·파티모집·문의·버그제보·알림채널 |
| `테스터` | 운영진 수동 부여 | + 테스터채널 |
| `운영진` | 운영진 수동 부여 | 전체 |

### 1-2. 클래스 역할 (선택)

인증유저가 `/클래스선택`으로 선택. 파티 모집 시 멘션 대상.

| 역할 | 설명 |
|---|---|
| `검사` | 무기 클래스: sword |
| `도끼전사` | 무기 클래스: axe (MACE 기반) |
| `창기사` | 무기 클래스: spear |
| `석궁사` | 무기 클래스: crossbow |
| `낫사냥꾼` | 무기 클래스: scythe |
| `지팡이술사` | 무기 클래스: staff |

1인 1클래스. 선택 시 기존 클래스 역할 자동 제거.

### 1-3. 알림 구독 역할 (선택)

`/알림설정`으로 자유롭게 추가·제거.

| 역할 | 알림 대상 |
|---|---|
| `필드보스알림` | 필드보스 5분 전·등장 시 멘션 |
| `시즌보스알림` | 시즌보스 모집·공지 시 멘션 |
| `업데이트알림` | 점검·업데이트 공지 시 멘션 |

## 2. 온보딩 플로우

> **멀티서버 모델(DL-131):** 온보딩은 ①**공통 디스코드 인증**(규칙/이용약관 동의 + 마크
> 닉네임 1회 입력 → `접속대기`) → ②**서버별 약관동의 + 인게임 인증 → 서버별 화이트리스트**
> 2단계다. 아래는 그중 **RPG 서버 단계(②의 RPG 분기)**다. 닉네임은 ①에서 1회만 받고,
> RPG 화이트리스트는 RPG 약관동의 + 인게임 `/연동` 인증을 거친 유저에게만 부여한다.
> 역할 체계 SoT는 [`../roles_and_permissions.md`](../roles_and_permissions.md) §A·§D.

```
[공통] 디스코드 입장 (역할: 미인증)
    │  서버 규칙/이용약관 동의 + 모달: 마인크래프트 닉네임 1회 입력
    ▼
역할: 접속대기 (디스코드 인증 완료 — 아직 게임 화이트리스트 아님)
    │  서버 선택 패널에서 RPG 이모지 선택 (DL-132)
    ▼
역할: RPG접근 부여 → RPG 카테고리(약관·정보 채널) 가시화
    │
    ▼
[RPG] #RPG-약관 채널: RPG 약관동의 버튼
    │  동의 → 임시 화이트리스트 등록 및 인증 코드 생성 (API: POST /auth/pending)
    ▼
DM: "접속정보 채널에서 서버 IP를 확인하고, 인게임 /연동에 인증 코드를 입력해주세요."
    │
    ▼
마인크래프트 접속 → 인게임 /연동 <인증코드>
    │  AuthService 검증 및 역할 부여 큐 적재
    ▼
역할: 인증유저(RPG 플레이어) 부여 → RPG 화이트리스트 확정
DM: "RPG 인증 완료! 이제 RPG 채널을 이용할 수 있습니다."
```

> 포로몬도 같은 ② 패턴(포로몬 약관동의 → 포로몬 `/연동` → 포로몬 화이트리스트 + `포로몬플레이어`)을
> 따른다. 상세는 [`poromon.md`](poromon.md)(설계 선행).

### 예외 처리

| 상황 | 봇 응답 |
|---|---|
| 닉네임 이미 등록됨 | "해당 닉네임은 이미 인증된 계정입니다. 문의 채널에 제보해주세요." |
| 코드 만료 (10분) | "인증 코드가 만료되었습니다. 약관-인증 채널에서 다시 요청해주세요." |
| 코드 불일치 3회 | 10분 입력 잠금. 운영진 알림. |
| 닉네임 없음 (API) | "입력한 닉네임으로 서버 접속 기록이 없습니다." |

## 3. 슬래시 커맨드 목록

### 3-1. 플레이어 명령어 (인증유저 이상)

#### `/연동`
인게임 `/연동 <인증코드>` 완료 후 디스코드 연동 상태를 확인한다.
- 트리거: 역할 부여 큐 조회 또는 수동 상태 확인
- 응답: 인증 완료 embed 또는 대기/오류 메시지 (ephemeral)

#### `/프로필 [플레이어명]`
플레이어 카드 embed를 표시한다. 파라미터 없으면 본인. 타인 지정 시 동일한 형식으로 전체 공개.

embed 포함 항목:
```
[닉네임]  Lv.52  [자작]
클래스: 검사  |  각인: A

─── 장비 강화 ──────────────────────
무기 18강  |  방어구 평균 15강  |  평균 IL 82

─── 스탯 배분 (총 156pt / 52레벨) ─
치명: 100pt  |  특화: 50pt  |  인내: 6pt

─── 전투 스탯 ──────────────────────
공격력      1,240     체력        14,500
방어력        820     이동속도       110%
치명타 확률  35.0%    치명타 피해량  176%
스킬 피해   15.0%    받는 피해 감소  0.9%

─── 최근 보스 클리어 ────────────────
균열왕  7분 03초  (2026-05-23)
균열 파수꾼  4분 51초  (2026-05-22)
────────────────────────────────────
```

스탯은 장비 강화·잠재·스탯 포인트 합산 최종값. 최근 보스 클리어는 최대 2건.

#### `/영지`
본인 영지 현황 embed (ephemeral).

embed 포함 항목:
- 현재 작위 / 영지 크기
- 시설 현황 (약초 재배지·광물 채굴기·공방 가공기)
- 공방 가공 대기열 (완료 예상 시간)

#### `/보스`
현재 보스 현황 embed.

embed 포함 항목:
- 현재 활성 필드보스 (리젠 남은 시간)
- 시즌보스 마지막 클리어 (날짜·클리어 타임·파티 구성)
- 이번 주 시즌보스 클리어 횟수

#### `/클래스선택`
버튼 UI로 무기 클래스 역할 선택 (ephemeral).
- 기존 클래스 역할 자동 제거 후 선택한 역할 부여

#### `/알림설정`
토글 버튼으로 알림 구독 역할 추가·제거 (ephemeral).
- 현재 구독 상태 표시 (초록 = 구독 중)

#### `/강화계산 <강화단계>`
목표 강화단계까지 필요한 골드와 강화석 기댓값을 계산한다 (ephemeral).
- 입력 예: `/강화계산 18`
- 응답: 현재 강화 단계 입력 → 목표까지 필요 평균 비용 embed
- 계산 방식: 봇 내부 계산 (강화 비용 표 하드코딩). API 호출 없음. (DL-030)

#### `/파티모집`
`#파티모집` 채널에 구조화된 파티 모집 embed를 게시한다.

- 트리거: 모달 팝업 (목적지·요구 IL·한마디 입력)
- 전송 후 [참여하기] 버튼이 embed에 붙음
- 참여자 클릭 시 embed의 인원 수 자동 갱신 (1/3 → 2/3 → …)
- 3/3 달성 또는 주최자가 [모집 완료] 버튼 클릭 시 embed 잠금

```
[파티 모집] 균열왕
──────────────────────────────
주최: 닉네임  (검사 / IL 85)
요구 IL: 75 이상
한마디: 조용히 빠르게 가요

현재 인원: ■■□  2 / 3명
참여자: 닉네임A (도끼전사), 닉네임B (지팡이)

[참여하기]  [모집 완료]
```

#### `/랭킹 <종류>`
시즌 랭킹 embed를 표시한다.

| 종류 | 내용 |
|---|---|
| `강화` | 평균 IL 상위 10명 |
| `골드` | 골드 보유량 상위 10명 |
| `보스` | 시즌보스 클리어 횟수 상위 10명 |

```
[랭킹] 강화  (2026-05-23 기준)
──────────────────────────────
1위  닉네임A   평균 IL 94  (무기 22강)
2위  닉네임B   평균 IL 91  (무기 21강)
3위  닉네임C   평균 IL 88  (무기 20강)
...
```

#### `/일정`
현재 운영 일정과 보스 리젠 정보를 표시한다 (ephemeral).

embed 포함 항목:
- 현재 시각 (서버 기준)
- 다음 필드보스 리젠 예정 시각 (보스명·필드)
- 운영진이 등록한 이벤트 일정 (최대 3건)
- 시즌 남은 일수

```
[서버 일정]  2026-05-23 21:00
──────────────────────────────
⚔ 다음 필드보스
  폐광 골렘  →  22:15 예정 (광산 필드)

📅 예정 이벤트
  05-25  골드 드랍 2배 이벤트
  05-28  시즌보스 도전 주간 시작

⏳ 시즌 종료까지  22일 남음
```

#### `/버그제보`
버그 제보 모달을 띄운다. 상세는 4절 참조.

### 3-2. 운영진 전용 명령어

전용 채널(`#운영로그`)에서만 실행 가능.

| 명령어 | 기능 |
|---|---|
| `/공지 <내용>` | `#공지` 채널에 공식 embed 게시 |
| `/점검 <시작시각> <예상시간>` | 점검 안내 embed + 업데이트알림 역할 멘션 |
| `/보스알림 <보스명> <채널>` | 수동 보스 등장 알림 전송 |
| `/역할부여 <유저> <역할>` | 수동 역할 부여 (테스터·운영진 등) |
| `/유저조회 <닉네임>` | 플레이어 상세 정보 (관리자용, ephemeral) |
| `/골드지급 <닉네임> <금액> <사유>` | 골드 지급 (API 경유, 운영로그에 자동 기록) |
| `/이벤트등록 <날짜> <내용>` | `/일정` embed에 표시할 이벤트 최대 5건 등록·수정·삭제 |

## 4. 버그 제보 시스템

### 4-1. 제보 흐름

```
/버그제보 실행
    │
    ▼
모달 팝업 (4개 필드)
  ① 제목 (필수, 최대 100자)
  ② 재현 방법 (필수, 최대 500자)
  ③ 기대 결과 vs 실제 결과 (최대 300자)
  ④ 심각도 선택: [크래시] [기능오류] [표시버그] [기타]
    │
    ▼
PoroRPG API가 poro.db bug_report 테이블에 INSERT
(id AUTOINCREMENT → 접수번호로 사용)
    │
    ▼
[#버그제보] 채널에 embed 전송
    │  포함: BUG-{id} / 제보자 / 심각도 / 시각
    │  크래시·기능오류 시: @운영진 멘션
    ▼
제보자에게 DM
  "BUG-{id} 접수 완료. 확인 후 채널에서 상태를 업데이트합니다."
```

접수번호 형식: `BUG-{id}` (예: BUG-1, BUG-42). `id`는 `bug_report.id` AUTOINCREMENT 값. DB 테이블 설계는 `db_event_log_spec.md` 참조.

### 4-2. 버그 embed 형식

```
[BUG-42] 기능오류
──────────────────────────────
제보자: 플레이어명 (@디스코드)
시각: 2026-05-23 21:34
심각도: 🟡 기능오류

제목: 강화 시도 시 골드 차감 안 됨

재현 방법:
1. 강화 GUI 진입
2. 15강 → 16강 강화 시도
3. 골드 잔액 확인

기대 결과: 27,000G 차감
실제 결과: 차감 없이 성공 처리됨

─────────────────────────────
상태: 🔵 접수됨
```

### 4-3. 상태 관리

운영진이 버그 embed에 붙은 버튼으로 상태를 변경한다.

| 상태 | 의미 | 버튼 |
|---|---|---|
| 🔵 접수됨 | 자동 초기 상태 | — |
| 🟡 확인 중 | 운영진이 확인 시작 | [확인 중으로 변경] |
| 🟢 수정 완료 | 패치 반영됨 | [완료 처리] |
| ⚫ 중복/기각 | 이미 알려진 버그 or 비버그 | [기각] |

상태 변경 시 제보자에게 DM 알림 전송.

## 5. 알림 시스템

### 5-1. 필드보스 알림

PoroRPG 스케줄러가 API를 통해 트리거. 봇이 `#필드보스-알림` 채널에 embed 전송.

| 이벤트 | 트리거 | 멘션 |
|---|---|---|
| 5분 전 예고 | 리젠 타이머 -5분 | `@필드보스알림` |
| 등장 | 리젠 타이머 0 | `@필드보스알림` |

클리어 알림 없음.

```
[필드보스 등장] @필드보스알림
──────────────────────────────
보스명: 폐광 골렘 (Field_Boss_2)
위치: 광산 필드 남쪽
권장 IL: 40 이상
────────────────────────────── 
```

### 5-2. 시즌보스 알림

운영진이 `/보스알림` 명령어로 수동 전송 또는 PoroRPG 자동 트리거.

```
[시즌보스 모집] @시즌보스알림
──────────────────────────────
보스명: 균열왕
모집 인원: 최대 3인 파티
권장 IL: 80 이상 / 클리어 타임 제한: 10분
────────────────────────────── 
```

### 5-3. 점검·업데이트 알림

`/점검` 명령어 실행 시 `#공지` + `#업데이트` 채널에 전송.

## 7. API 연동 범위

봇이 호출하는 PoroRPG API 엔드포인트 목록.

| 기능 | 메서드 | 엔드포인트 |

| 영지 현황 조회 | GET | `/api/island/:uuid` |
| 보스 현황 조회 | GET | `/api/boss/current` |
| 필드보스 현황 | GET | `/api/field/boss-status` |
| 골드 지급 | POST | `/api/admin/gold-grant` |
| 역할 큐 확인 | GET | `/auth/role-queue` |
| 역할 부여 완료 확인 | POST | `/auth/role-granted` |
