# 03. 디스코드 온보딩 / 봇 계획

> **[STATUS: REFERENCE]** — 디스코드 온보딩과 봇 MVP 참조. 공식 운영 방향은 `../final_master_plan.md`가 우선.

## 운영 방식

디스코드 선입장 방식으로 간다.

```text
디스코드 입장
→ 약관 동의
→ 마크 닉네임 입력
→ 임시 화이트리스트 등록
→ 마크 접속
→ 코드 인증
→ 최종 연동
→ 인증 유저 역할 부여
```

## 역할

- 미인증
- 접속대기
- 인증 유저
- 테스터
- 운영진

권한 역할과 칭호 역할은 분리한다.

## 현재 채널 구조 활용

기존: 서버소개, 규칙, 공지, 업데이트, 접속정보, faq, 잡담, 스크린샷, 파티모집, 문의, 버그제보.

접속정보 채널은 약관 동의 후 역할(접속대기 이상)을 받은 유저에게만 열람 가능하게 설정한다. 서버 IP/도메인은 이 채널에서만 안내한다.

추가 추천:

- 약관-인증
- 필드보스-알림
- 시즌보스-알림
- 운영로그
- 밸런스메모

## 디스코드 봇 기능 개요

상세 명세: [`discord_bot_spec.md`](discord_bot_spec.md)

### 역할 체계
- 권한 역할 (자동): 미인증 → 접속대기 → 인증유저 → 테스터 → 운영진
- 클래스 역할 (선택): 검사 / 도끼전사 / 창기사 / 석궁사 / 낫사냥꾼 / 지팡이술사
- 알림 구독 역할 (선택): 필드보스알림 / 시즌보스알림 / 업데이트알림

### 주요 명령어
- `/연동` — 인게임 코드 인증
- `/프로필` — 레벨·작위·스탯·강화 현황·최근 보스 클리어 카드
- `/영지` — 영지·시설 현황
- `/보스` — 보스 현황
- `/클래스선택` — 클래스 역할 선택
- `/알림설정` — 알림 구독 토글
- `/강화계산` — 강화 비용 계산기
- `/파티모집` — 구조화 파티 모집 embed (참여 버튼·인원 자동 갱신)
- `/랭킹` — 강화·골드·보스 클리어 TOP 10
- `/일정` — 보스 리젠·이벤트 일정·시즌 남은 일수
- `/버그제보` — 버그 제보 모달

### 알림
- 필드보스 5분 전·등장
- 시즌보스 모집 공지
- 점검·업데이트 공지
