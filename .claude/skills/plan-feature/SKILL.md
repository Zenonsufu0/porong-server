# plan-feature

주제: $ARGUMENTS

목표:
- 포로 서버 기준에 맞는 기능 기획 초안을 짠다.
- 과한 확장보다 1차 버전을 먼저 제안한다.
- 기존 기준과 충돌하면 먼저 충돌 지점을 적는다.

반드시 반영:
- 기존 장비/강화/잠재/각인/상태/태그 체계를 존중
- 이미 확정된 설정은 임의로 뒤집지 않기
- 애매한 내용은 오픈 질문으로 남기기
- 문서는 한국어로 짧고 명확하게 작성

출력 형식:
1. 개요
2. 왜 필요한가
3. 핵심 사용자 경험
4. 핵심 규칙
5. 범위
6. 비범위
7. 1차 버전 제안
8. 오픈 질문
9. 다음 작업
