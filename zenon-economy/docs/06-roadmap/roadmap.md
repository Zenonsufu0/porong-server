## 2. 구현 로드맵 — 착수 순서 (수직 슬라이스 우선 · DL-E074·E133)

> 원칙: Phase 1 수직 슬라이스로 핵심 재미(노동=소비 루프) 먼저 증명 → 정치·UI 폴리시 나중. 단계마다 플레이 가능 유지(1인 개발).

- **Phase 0 — 셋업:** NeoForge 1.21.1 + Create(큐레이션) + KubeJS + **임베디드 DB(SQLite/H2)** + 자동설치기 골격 + **🔑 헤드리스 시뮬 하네스 골격**(MC 없이 CORE 틱 fast-forward) + **화폐보존 매틱 assert** + **config 핫리로드**(P1 계수 외부화). ⏤ 배관은 AI 페어로 가속(E139, §3).
- **Phase 1 — 수직 슬라이스 MVP ⭐:** 주민 집계 엔진(가계·노동·인구 동학, 토대) + 회사 설립·생산(Create)·산출물 박스 + 시장 추상 정산(호가·수요곡선) + 중앙은행 현금·거래세. → "땅 사서 회사 짓고 팔아 돈 벌고 주민 소비로 노동=소비 루프가 도는" 최소 경제. **30명 테스트로 핵심 재미 먼저 검증.**
- **Phase 2 — 경제 확장:** 기술 트리·시대(마을 진화 해금)·분야 특화(연구원) + 자본시장(IPO·현물·배당, *M&A는 시즌2*) + 계약(B2B 납품박스) + 대시보드 풀 GUI.
- **Phase 3 — 정치·창발:** 창발 지표(오염·범죄·CPI)·안건 트리거 + 의회(발의·투표·정당·서킷브레이커). (의회는 마을 성장 후 등장 → 자연 후순위)
- **Phase 4 — 폴리시·시즌:** 풀 텍스쳐 UI + 커스텀 완성품(2~3군) + 디스코드 봇 알림 + 시즌 정산·랭킹 + P1 밸런싱(플레이테스트).
- **시즌2+ — 확장:** 대도시·국가(완전 자동화·대외무역)·M&A·사회모델 풀·빌리저 풀표현·친환경/원자력·55안건 풀·커스텀 5~6군·개방경제. (설계 E124~E135 완성·유지, 구현만 후순위)

## 3. 1차 출시 인프라 방침 (DL-E139 — 1인+AI 보조 전제)

> 4대 루프(노동소비·생산·의회·자본시장)는 **비전이라 안 깎음**(사용자). 대신 비전과 무관한 *배관*만 1차 출시용으로 경량화하고, 튜닝 도구를 추가한다.

| 항목 | 1차 출시 방침 | 근거 |
|---|---|---|
| **🔑 헤드리스 시뮬 하네스** | **추가**(Phase 0~1) — CORE 틱 = 순수함수(월드 비의존), MC 서버 없이 14일치 수 초 fast-forward. 가상 유저 행동으로 진화 도달·인플레·화폐보존 검증 | 2주 시즌=데이터 1점/2주 → 창발 거시 튜닝 불가. 단일 최고가치 도구(E138 검토) |
| **화폐보존 검산** | 매 틱 `Σ(현금+예금)=통화량` assert + 인구 비음수·rate cap | 누수 버그 즉시 검출(DL-E094 코드화) |
| **config 핫리로드** | P1 전 계수 외부 데이터파일·재컴파일 없이 조정 | 라이브 튜닝(DL-E084) 전제 |
| **DB** | **임베디드(SQLite/H2)** — 외부 Postgres/MySQL 불요 | 30명·집계 인구 = 쓰기량 적음. 디코봇이 DB 직접 안 읽음(아래) |
| **디스코드** | **인증 연동 훅만** — 봇 본체는 **zenon-discord 별도 프로젝트**, economy는 인증 상태 연동 API만 | 디코봇 풀 기능 ≠ 본 서버 범위(사용자) |
| **월드** | **2 고정 차원 유지**(메인=마을+야생 / 회사 슈퍼플랫, DL-E079·E059) — 통합 안 함 | 회사부지 vs 야생·마을 = 자본제약(E120)·노동소비 공간 분리 = **디자인 핵심**(사용자). 이미 2개 고정=최소. 에이전트 "2D→1D" 권고는 확정 설계 미인지 → 기각 |
| **커스텀 아이템** | **기존 모드(Create·IE·Croptopia)+품질 태그 우선** 시작, 커스텀 텍스처 풀세트는 폴리시(Phase 4) | 텍스처=주 노동 부담 → 뒤로(2~3군 E136 유지하되 출시 임계 아님) |

> **완주 등급 재평가(E139):** 에이전트 검토는 "AI보조 0·무조건 완주" 가정으로 🔴(1인 수개월~1년)였으나, **1인+AI 페어 전제 → 배관은 보일러플레이트로 가속, 진짜 병목은 *튜닝*** → 하네스로 해소 → **완주위협 🔴→🟡**. 비전(4루프)은 보존, 칼질은 배관에만.

## 4. 착수 제작 체크리스트 — 커스텀 아이템 (DL-E073)

> 기존 모드/바닐라로 안 되는 것만 커스텀. 각 = **등록(코드) + 텍스처(16x16) + 레시피(KubeJS 가공 체인)**. 시즌1 = 2~3군만(나머지 시즌2).

**커스텀 = 완성 소비재 5~6군** (나머지 식품·원료·부품=기존 모드, 배=바닐라·기차·물류=Create 본체·가구=Macaw's Furniture, DL-E146):

| 품목군 | 레시피(가공 체인 개략) | 시즌 |
|---|---|---|
| 의류·신발 | 원단(섬유)+노동 → 옷·신발 | 1 |
| 운송(자동차·자전거) | 강철·엔진·부품 → 차체 → 자동차 | 1(전기) |
| 가전(냉장고·세탁기·TV) | 부품·전자회로 → 가전 | 1(전기) |
| 전자완성(PC·스마트폰) | 반도체·회로·부품 → 전자제품 | 2(자동화) |
| 의약·화장품 | 화학원료 → 의약/화장품 | 2 |

작업 3종 + 규모:
- [ ] **등록(코드):** 아이템 등록 + 품질 속성 hook (가벼움)
- [ ] **텍스처:** 수십 개 16x16 ⚠️ 주 일거리 → 외부 킷·AI·직접
- [ ] **레시피(제작법):** KubeJS = 가공 체인(B2B 원료→완성품) — 2층 시장·품질 전파·시대 게이팅 구현 지점
- [ ] **CORE 연동:** 산출물 박스 업종-품목 인식·시장 등록·소비유형 태그
- 규모 감각: ~5~6군 × 품질/티어 변주 = 수십 아이템 (텍스처 부담 핵심, 레시피는 KubeJS로 빠름). **시즌1은 2~3군으로 칼질(E136).**

- **가구 = 마을 장식이 아님.** B2C **8대 완성 소비재**(식료품·기호식품·의류·운송·가전·전자·의약화장품·**가구**, DL-E066) 중 하나 = **내구재**(경기 민감, design §207). **주민(집계)이 시장에서 구매**하는 거래 상품 — 회사가 생산→시장 판매→주민 소비.
- **"가구 모드" 채택 이유:** 다른 완성 소비재는 커스텀(5~6군)이지만, 가구는 기성 **가구 모드 아이템 모델을 빌려 텍스처 제작 부담을 던다**(식품을 Croptopia로 조달하는 것과 동일 논리, DL-E069 "기존 모드로 되는 건 커스텀 X").
- **결정(DL-E146): Macaw's Furniture 채택**(v3.4.1, 1.21.1 NeoForge ✅). 가구 완성품 모델을 기성 모드로 조달 → CORE는 품질·시장·소비유형 태그만 연동(커스텀 텍스처 0). 식품(Croptopia)·재료(IE)와 동일 "기존 모드 조달" 패턴.
