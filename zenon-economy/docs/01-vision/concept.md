## 한 줄 컨셉
유저는 1차 산업부터 고부가가치 산업까지 원하는 회사를 세워 경제를 일구고, **살아 움직이는 제논마을 주민**과 함께 마을을 도시·국가로 키워가는 경제 시뮬레이션 서버.

> "제논"은 서버 이름 접두사, "마을"은 **마을 → 소도시 → 도시 → 대도시 → 국가**로 진화하는 경제 공동체를 가리킨다. 마을은 **유저들이 주도해 만들어간다**. 핵심 차별점은 주민이 단순 NPC가 아니라 농사·노동·소비·투자하는 **살아있는 노동·소비 주체**라는 점 — 유저가 이들을 어떻게 대우하느냐가 곧 경기로 돌아온다.

## 장르·정체성 (확정)

**마인크래프트로 하는 경제발전 시뮬레이션 (계급 구조 경제).**
빅토리아3 / Capitalism Lab / 심시티 계열이되, **결정적 차이는 자본가 계급을 실제 유저가 플레이한다**는 것이다.

| 계급 | 누가 | 역할 |
|---|---|---|
| **자본가·기업가** | **유저(사람)** | 자본 투입·회사 설립·고용·가격/품질 결정 → 경제의 **엔진** |
| **노동·소비 계급** | **주민(빌리저) 집계** | 노동·소비·출산·투자 → 경제의 **몸통(거시 집계)** |
| **대표 빌리저** | NPC 수십 명 | 위 집계의 **얼굴** — 계층 상태를 마을에 시각적으로 표현 |

- "돈 벌기"는 단순 그라인딩이 아니라 **발전하는 경제에서 자본을 굴리는 행위**이며, 그 행위는 주민 계급의 건강(소비력)에 묶여 있다(착취 자기조정, §4·핵심 기둥).
- 발전 사다리(마을→국가)는 집계 지표(인구·학력·산업다양성·인프라)로 게이팅되고, **거시경제 사이클은 별도 엔진이 아니라 주민 집계 상태에서 창발하는 결과**다(§8).

> **시즌1 스코프 (확정):** **내수 중심 경제 모델** — 판매 수요의 주체는 마을 내부 주민이며, 대외 수출 시장·본격 무역 시스템은 다루지 않는다(시즌2+ 후보). (원료 **수입**은 비용 요소로 §9에 잔존하되, 무역 자체를 콘텐츠화하지는 않음.)

## 플레이어의 최종 목표
**돈을 많이 버는 것.** 모든 산업·창업·투자·정치 활동은 결국 부의 축적을 향한다. 시스템(토지·주민·정부·시장)은 이 목표를 추구하는 과정에서 서로 맞물려 돌아간다.

## 게임 프레임 (확정 · DL-E053)
- **규모:** 지인 운영 서버, 최대 **~30명**. 1인 1표 의회·시장 경쟁·담합이 의미 있는 소규모 사회 — **서로 다 아는 사이라 정치질이 곧 재미**.
- **시즌제:** 한 시즌 **2주(14일)**(실시간), 종료 시 정산. (영속 아님 → 승자독식 장기화 없음.)
- **승리 조건:** **시즌 종료 시 보유 부(현금+자산+지분 평가액)가 최대인 사람이 승리.** 회사 인수, 비상장 100% 오너십 유지 등 경로는 자유.
- **의회 = 부를 둘러싼 이익 정치:** 의회는 중립적 경제 조정이 아니라, 유저들이 **자기 부를 위해 정당을 만들고 모의(합종연횡)해 이해관계대로 찬반을 던지는 정치 게임**이다(§6). 거물은 법인세 인하를, 자산가는 누진세 반대를, 뒤처진 다수는 부자 증세를 민다.
- **자연 밸런스(고무줄):** 선두 독주 시 **뒤처진 다수가 의회에서 연합해 증세·규제로 견제**(1인1표 머릿수) → 정치가 곧 승자독식 견제 장치. 짧은 시즌과 함께 고인물을 막는다.

## 플레이어 경험 — 게임 루프·온보딩 (확정 · DL-E054)

**시즌 아크 (2주, E133·E134) — 기술 시대·마을 진화가 곧 페이싱**
| 시기 | 경제 단계 | 플레이 초점 |
|---|---|---|
| **Week 1** (D1-8) | 마을·소도시 (수공업·증기) | 부트스트랩·고용·생산→판매·의회 가동·B2B/계약. 첫 자본 축적 |
| **Week 2** (D8-14) | 도시 (전기) | 자동차·가전·**자본시장 활성·상장·거래** + 의회 정치 격화 → **부 경쟁 클라이맥스·정산** |
| ~~Week 3-4~~ | **시즌2+** | 대도시(완전 자동화·M&A)·국가(대외무역) |

**일상 루프 (매 세션):** 생산 관리(회사·자동화·고용) → 시장(산출물 박스 가격·시세·계약) → 대시보드(산출·품질·재무·주가) → 자본 운용(주식·M&A·대출·자산배분) → 의회(안건 뜨면 토론·정당 모의·투표, 하루 1회) → 이벤트 대응(산재·고장·범죄·거시 조정).

**온보딩 (학습곡선 완화):**
- 시작 = 소액 자본 + 작은 땅 1구획(또는 저리 대출).
- 가이드 퀘스트(FTB Quests §7): ① 농장 짓기(Create 기본) ② 주민 1명 고용 ③ 산출물 박스 설치·생산 ④ 시장 판매로 **첫 수익** ⑤ 대시보드 읽기.
- **단계적 해금 = 기술 게이팅이 곧 온보딩(DL-E020):** 처음부터 다 안 열림. 초반엔 생산·판매(단순)만, 시대 진행·마을 진화로 자동화→의회→자본시장이 순차 등장 → Create+경제+의회 복잡도 점진 노출(압도 방지). **시즌 아크 자체가 학습 곡선** — 별도 튜토리얼 단계 불필요.

## 페이싱 — 2주 시즌 역산 (확정 원칙 · DL-E055)

**원칙: 클라이맥스에서 역산.** 시즌 재미 절정(Week2 자본시장·의회 정치 격화)이 **확실히 도달**되도록 임계·속도를 시즌 길이에서 역산한다. 콘텐츠를 다 못 펼치느니 핵심 아크 도달을 보장.

**목표 진행 곡선 (14일 기준, 예시 — 수치 P1 튜닝):**
| Day | 마을 단계 | 기술 시대 | 열리는 것 |
|---|---|---|---|
| D1–4 | 마을 | 수공업 | 1차 산업·시장 판매·의회 출범 |
| D4–8 | 소도시 | 증기 | 2차 가공(강철·중공업)·B2B·계약 |
| D8–14 | 도시 | 전기 | 자동차·가전·**자본시장·부 경쟁 클라이맥스·정산** |
| 시즌2+ | 대도시·국가 | 자동화·대외무역 | 완전 자동화·M&A·반도체·PC |

- **가속/감속:** 초반 빠른 보상(첫 수익·첫 진화 일찍 = 몰입), 후반 도전(**국가·완전자동화는 stretch**).
- **튜닝 레버(P1):** 연구 진척 속도·진화 임계·생산성 계수·시작 자본·안건 빈도 — 전부 시즌 길이 역산.
- **우선순위:** 시즌1은 마을·소도시·도시(전기) 3단계 → **자본시장·의회 정치(클라이맥스)는 반드시 도시(Week2) 도달**하게 우선 튜닝. 대도시·국가·완전자동화 = 시즌2.
- **검증:** 정확한 값은 플레이테스트 필수(소규모라 1~2시즌 돌려 조정). 구상에선 곡선·역산 원칙만 고정.

## 시스템 한눈에 보기 (통합 맵)

세 개의 맞물린 루프가 서버를 굴린다. 모든 시스템은 결국 **"돈 벌기"** 라는 유저 목표로 수렴한다.

**① 성장 루프 (실물경제)**
```
교육(학교) → 고학력 주민 → 연구 → 기술 트리 → 자동화·신산업
   → 생산성↑ → 유저 매출↑·경제 성장 → 인구 증가(출산)·세수↑
   → 마을 진화(토지·인프라 확장) → 다시 교육·산업 확대 …
```

**② 시장 루프 (미시경제)**
```
유저 생산(품질·가격 결정) → 주민 수요(필수품/사치품·소득탄력) → 매출
   → 재무제표 평가 → 주식 상장·주가·M&A → 자본 조달 → 재투자 …
   (전력·운송 인프라가 받쳐줘야 판매 성립)
```

**③ 정치 루프 (유저 의회 — 거시 조정)**
```
게임/DB가 발전 상황·지표 감지(예: 환경오염↑) → 의회에 안건 투척
   → 유저 토론 → 찬/반 투표(과반 가결) → 정책 발효(수치 트레이드오프)
   → 경제·환경·발전에 타격 또는 부양 → 새 지표가 다음 안건을 부른다 …
```

**④ 노동=소비 루프 (자기조정)**
```
유저가 임금↑·고용↑ → 주민 소비력↑ → 물건 더 팔림 → 경기 호조
유저가 임금↓·해고(단가 절감) → 주민 소비력↓ → 물건 안 팔림 → 경기 악화
```

> **연결 고리:** 주민은 모든 루프를 관통하는 핵심 자원 — ①의 노동·연구자, ②의 소비자·투자자다. **정치는 유저(의회)가** 안건 투표로 주도한다. 따라서 **"살아있는 주민"의 구현 수준**과 **안건/의회 시스템 설계**가 전체를 좌우하는 두 축이다.

## 핵심 기둥 (후보)
- **산업 사다리:** 1차(농사·광질) → 2차 가공 → 고부가가치 산업으로 올라가는 산업 티어. 유저는 원하는 어떤 회사든 세울 수 있다.
- **연구·교육:** 신기술은 재료가 아니라 **고학력 주민의 연구**로 발전 트리를 찍어야 도입된다. 학교 등 교육 인프라가 인적 자본을 길러 산업 한계를 끌어올린다.
- **토지·부동산:** 회사는 **마을 토지를 구매한 곳에만** 세울 수 있다(마을 내부 한정). 가용 토지(발전 가능 칸)는 마을 발전에 따라 넓어지고, 의회 의결로 규제·축소될 수 있다.
- **살아있는 주민:** 주민은 유저가 만든 마을의 **노동력이자 소비자·투자자**다. 농사·노동·출산·소비·주식 매수를 하지만 정치는 하지 않는다(정치는 유저 몫).
- **회사·자본시장:** 유저가 회사를 설립·경영하고 재무제표로 평가받으며, 주식 상장·거래·M&A가 가능한 자본시장. 물건·주식의 **수요자는 주민**이다.
- **유저 의회(정부):** 정부는 **유저들의 의회**다. 게임 시스템이 발전 상황·DB 지표에 따라 **안건을 던지면**, 유저들이 토론하고 **찬/반 투표(과반 가결)** 로 마을 정책을 결정한다.
- **노동=소비 루프:** 주민의 임금·고용이 곧 마을의 소비력이다. 단가를 낮추려 월급을 깎고 해고하면 물건이 안 팔리고 경기가 나빠진다 — 착취의 자기조정.
- **시장 수요·품질:** 물품은 주민의 수요·필수품/사치품 여부·공급과 수요에 따라 팔린다. 유저는 가격을 조정하고 **품질**로 경쟁한다.
- **인프라:** 판매에는 전력·운송망이 필요하며, 마을 발전·의회의 에너지 정책에 따라 비용이 달라진다.
- **거시경제 사이클:** 호황→인플레이션, 과열→대공황(수요 붕괴), 신용경색(과잉 대출·부채 위기) 등 거시 위기와 회복. (은행은 중앙은행 단일 — §3·DL-E039.)
- **거래/시장:** 시장(=상점, 추상 정산)·회사 간 계약·시세 변동. **산출물 P2P 직거래는 차단**(시장·계약 경유, §11·§11-B·DL-E026).
- **경제 밸런스:** 화폐 발행·회수(싱크/소스), 물가 관리, 인플레이션 억제 장치.

## 경제학적 위치 (모델 비교 · DL-E123)

> 우리 모델 = **포스트케인지안 + 민스키 + 내생적 성장의 하이브리드**(이단·현실주의 거시). 주류 신고전 DSGE(합리적 기대·시장청산)가 아니라, 화폐·신용·분배·사이클을 *현실적으로* 잡은 쪽.

| 부문 | 우리 설계 | 대응 이론 |
|---|---|---|
| 화폐/통화 | 대출=발권·monobank·QE | **내생화폐론** + 화폐수량설 |
| 노동/소비 | 노동=소비 루프·계층 MPC | **케인지안 승수 + 임금주도성장**(Kalecki) |
| 재정 | 공공지출·공공고용·서킷 | 케인지안 재량 + **자동안정장치** |
| 금융/신용 | 가계부채 사이클·금리 인상기 위기 | **민스키 금융불안정 + 피셔 부채-디플레** |
| 성장 | 진화·인적자본(교육·연구) | **내생적 성장**(Romer·Lucas) |
| 가격 | 수급 + 주민회사 앵커 | 유연가격 + 관리가격 |

- **강점(교과서적):** 내생화폐(대부분 게임이 화폐 외생/고정인데 우리는 신용창조)·민스키 신용 사이클·노동소비 승수·분배가 거시를 움직임·자동안정 vs 재량 구분.
- **의도적 생략:** 개방경제(무역·환율 → 시즌2+)·상업은행(monobank 단순화 DL-E039)·기대형성·가격경직성·총생산함수.
- **게임 고유 왜곡:** 계급 비대칭(유저=자본가 엔진/주민=집계 몸통)·시즌제(14일, 장기균형 없음)·마크 무한공급 대응(자본 제약=시간·규모, DL-E120).
- **역설:** 주류 DSGE보다 **2008 금융위기를 더 잘 설명**(DSGE는 금융 무시, 우리는 민스키 사이클 중심) → 게임인데 이단 거시의 강점 보유.

## 열린 질문 (미확정)

> 우선순위 분류: **P0**(다른 결정의 전제가 되는 기반 결정) → **P1**(핵심 시스템 설계) → **P2**(운영·정책 디테일). P0부터 좁히면 하위 질문이 따라 정리된다.

> 〔해소됨〕 마을 구조(→ 단일 마을 확정, 4개 분할 폐기) · 정치 주체(→ 유저 의회 확정) · 인프라/기술 방향 결정권(→ 유저 의회 안건) · **주민 구현 수준(→ 하이브리드: 집계가 진짜·대표 빌리저가 얼굴, §4 확정)** · **조작 방향성(→ MC답게: 회사는 직접 건설·자동화 모드는 core가 제어·운영은 대시보드, §2 확정)**. 아래는 남은 미결.

### P0 — 기반 결정 (먼저 정해야 나머지가 풀림)
- **② 의회·안건 시스템 설계 (구조·원칙 해소, 안건 알맹이만 잔존):** 〔해소〕 의회 구조 = 전원 의석 직접민주제(1인 1표) + 발의권 CORE 전속(DL-E006) / 정당 = 무권한 소셜·가치축 결사 / 다수결 그대로(소수 보호장치 없음, 정족수만) / 의회 목적 = 경제모델 집단선택의 재미(실정치 X) / 권한 경계 = 시장 개입 O(가격상한·강제분배·법인세)·자산 파괴 X(국유화·강제분할 금지, 독점은 국유 주민회사 신설로)(DL-E007·E008). 〔해소〕 ⓐ **안건 카탈로그** 12카테고리 55안건(agenda_catalog.md, DL-E010~E025·§6분리 E130). ⓑ **운영 규칙** = 하루 1회 정기 발의·동시 1건·1시간 투표 창·참여자 중 과반+정족수·3단계 에스컬레이션(§6, DL-E025). 〔잔존(P1)〕 트리거 임계·수치 효과·안건별 기술 선행조건·정족수 수치 = 밸런싱.
- **③ 플랫폼 〔해소 → NeoForge 1.21.1 모드 루트, DL-E030〕:** 순수 NeoForge 1.21.1 모드(하이브리드 서버 회피). 자동화 = Create(증기·기차·물류 본체) + Crafts&Additions(전기), 게이팅 = KubeJS + FTB Quests, 재료·화학·전기 = **Immersive Engineering**(+Petroleum, 확정 재료 모드 DL-E082) · 가구 = Macaw's Furniture(DL-E146) · 고급 완전자동화 = AE2/Mekanism(시즌2 선택). **CORE(주민 집계·시장·의회·재무)는 커스텀 NeoForge 모드 + KubeJS로 직접 구현.** 진입은 자동설치기. 버전·호환은 착수 시 재확인.

### P1 — 핵심 시스템 설계
- **인구 동학 모델:** 〔해소 → population_model, DL-E041·E042·E048〕 출산(다인자 역U자)·사망·이주(난민 포함)·계층이동·자기조정 확정. 〔잔존 P1〕 계수·임계.
- **마을 진화 기준:** 〔해소 → §5·DL-E051〕 다인자 임계(인구·GDP·다양성·학력·인프라) + 단계별 해금 + 쇠퇴(역진화). 〔잔존 P1〕 임계 수치·계수.
- **거시경제 엔진:** 〔해소 → §8·DL-E049〕 주민 집계에서 창발(별도 엔진 X). 인플레·경기·신용경색·거품이 집계 지표의 readout.
- **노동=소비 루프:** 〔해소 → population_model 가계·노동, DL-E038·E040·E044〕 다인자 모델 확정(소비자 신뢰·공공 고용 포함). 〔잔존 P1〕 계수(착취 처벌 강도 균형).
- **수요·품질 모델:** 〔해소 → §11, DL-E013·E026·E044〕 소득탄력·사치=고품질·수요곡선·유저 호가 확정. 〔잔존 P1〕 함수 계수.
- **연구·교육 모델:** 〔해소 → §7, DL-E020·E021·E031〕 서버 공통 트리·시대 게이팅·학력 단계·연구 안건·연구 방향(의회 밖). 〔잔존 P1〕 양성 속도·진행 계수.
- **주식 시장 구조:** 〔해소 → §3, DL-E033·E034〕 주민 집계 풀+유저 혼합·IPO·거래소·배당·M&A·지배구조 확정. 〔잔존 P1〕 유동성·시장조성 함수.
- **정당제 여부:** 〔해소 → DL-E007〕 정당 결성 허용하되 무권한(소셜·가치축 결사) + 안건별 직접민주제.
- **자동화·노동 결합 모델(§2):** 어떤 자동화 모드/플러그인을 쓰고 core가 어디까지 제어하나? "채용 주민 수 ↔ 설비 가동·산출량" 함수는? 자동화가 인건비를 얼마나 절감하되 노동력 의존을 유지시키나?
- **산재·노동안전 모델(§2):** 감독관 배치·안전규칙 준수의 게임적 표현, 산재 발생 확률·트리거, 산재의 비용/평판/생산성 페널티 수위(착취 억제하되 너무 가혹하지 않게).
- **주민회사 운영 모델(§11-A):** 취급 품목·시장 점유율 상한, 개입/철수 발동 조건, 기준가 산정 공식, 운영 재원(마을 재정?)과 의회 통제 여부.
- **서비스·금융 분야 수익 모델:** 〔해소 → design §3·DL-E072〕 수수료·자본·서비스 수익 — 금융(투자 수익·중개·M&A 자문, 하이리스크)·물류(운송 대행)·전력(발전 판매)·연구 위탁(옵션). 제조 없이 부 굴리는 대안 트랙.
- **식품 시장 집계 수준(DL-E070 ⓐ):** Croptopia 수백 식품을 개별 시장재로 둘지 "식료품" 카테고리로 집계할지(잘게 쪼개면 거래 의미 희박). 소수 핵심만 개별 + 나머지 집계 권장.
- **완성품 가공 체인 깊이(DL-E070 ⓑ):** 자동차·PC 등 깊은 체인의 단계 수 — 너무 깊으면 초반 공급 부족(B2B 의존). 페이싱과 균형.

### P2 — 운영·정책 디테일
- **토지·부동산 경제:** 토지 공급 확장 속도, 가격 결정(경매·고정가), 그린벨트 발동 조건, 투기·거품 처리.
- **인프라 모델:** 〔해소〕 소유 = 하이브리드(유저 인프라 회사 + 공공 인프라 안건, §12·DL-E019). 〔잔존〕 전력·운송 부족의 판매 페널티 수치·예비율 임계·요금↔비용 함수.
- **행정 진화:** 〔해소 → DL-E056〕 의회 출범 전(Week1 초반)엔 CORE/운영자 기본 관리, 마을 성장 후 의회 정치(DL-E006). 가결 정책 집행 = CORE 자동(수치 적용), 기록 = agenda·indicator_snapshot 로그.
- **"돈 벌기" 목표의 게임화:** 〔해소 → 게임 프레임·DL-E053〕 승리 = 시즌 종료 시 최대 부. 독주 견제 = 의회 이익 정치(다수 연합 증세) + 짧은 시즌. 〔잔존 P1〕 부 평가액 산정·랭킹 UI·시즌 정산 절차.
- **건설형 안건 자동건설:** 스케메틱(.schem) paste vs 구조물(NBT)? 부지 선정·지형 정리·철회 롤백 규칙. (플랫폼 P0-③과 직결.)
- **대외 무역 (시즌2+ 후보):** 시즌1은 내수 중심으로 제외. 도입 시 수출 시장·환율·타 마을/서버 교역 설계.
- **다른 Zenon 서버(rpg·mon)와의 경제 연동:** 〔해소 → DL-E056〕 **격리**(별도 경제·화폐). 시즌제 economy 화폐를 영속 rpg/mon과 통합 시 밸런스·환율 문제 → 통합은 시즌2+ 후보.
- **시즌제 vs 영속 경제:** 〔해소 → DL-E053·E056〕 시즌제 14일(2주, E133). 시즌마다 부·자산 리셋(공정 재경쟁), **시즌 간 계승은 non-pay-to-win만**(명예의 전당·칭호·코스메틱, 시작 우위 없음). 〔확정 E133〕 14일 = 마을·소도시·도시 3단계.
- **부정행위 대응:** 〔해소 → DL-E056〕 지인 서버 경량 모델 — 구조적 방지(거래 전부 CORE 경유 감사 로그·산출물 박스 무한자원 차단·P2P 차단) + 주가 조작은 모니터링·작전 단속 안건(§11) + RMT/멀티는 지인이라 위협 낮음 → 운영자 재량·사회적 신뢰. 정교한 안티치트는 과함.
