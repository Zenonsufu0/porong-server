# 경제 설계 (Economy)  — 초안(DRAFT)

> 허브 시장 구역(`hub_design.md` §2) + 짐/배틀타워 보상 + 티켓 구매를 잇는 재화 흐름.
> 연동: `RewardManager` / `EconomyBridge` / `PlayerProgress`(`../03_poromoncore/database_schema.md`) / `economy.json`(`config_structure.md`).
> ⚠️ 초안. `TBD`/`예시`는 검토·확정 대상.

## 1. 목표 / 원칙
- 자유 플레이(채집·전투)와 엔드게임(티켓·메가스톤) 사이의 **재화 순환** 설계.
- **밸런싱 기준(확정): 골드 총 유통량은 크게 중요치 않다.** 유저 간 거래가 없어(경매장·트레이드 없음, §7) 골드가 플레이어 사이를 돌며 인플레를 일으키지 않기 때문. 따라서 **인플레 방지보다 "메가진화 등 목표에 적당한 노력이 들도록" 비용/보상을 책정**하는 데 초점.
- 그래도 너프/버프 판단을 위해 **유통량·소모량은 추적**(§6). 자동 조정은 아니고 운영 판단 근거.
- 밸런스 값(가격·보상량)은 하드코딩 금지 → `economy.json`.
- 서버 권위: 모든 잔액 변경은 서버 검증(클라 신뢰 금지).

## 2. 화폐 — **골드(Gold) 단일제 (확정)**
- 단일 화폐 **골드**만 사용. BP 등 보조 화폐 없음.
- 외부 경제 모드 없음 → **PoroMonCore 내부 경제**로 구현, `EconomyBridge`로 추상화(향후 외부 연동 대비).
- 저장: `PlayerProgress.balance`(gold).

## 3. 획득처 (Faucet) — 확정
- **바닐라 농사 / 광질**: 산출물을 판매소에 팔아 골드 (자유 플레이 주 수입원).
- **관장 초회 클리어**: 골드 (초회만, `gym_badge_design.md` §6).
- **배틀타워**: 층별 초회 클리어 보상 골드(높을수록↑, `league_season_design.md` §3).
- **야생 포켓몬 처치/포획**: 골드 지급.
  - **야생만 인정(확정).** 다른 유저의 포켓몬 처치(친선/대전)는 **보상 없음**.
  - **레벨 비례(확정): 포켓몬 레벨이 높을수록 더 많이 지급.** 동일 종 감소·쿨다운 **없음**(단순하게). 고레벨 포켓몬 사냥/포획이 자연스러운 수입원.
  - 예: `골드 = floor(레벨 × 계수)` (계수는 §9 가격표 초안에서 조정).
- (향후) 퀘스트/이벤트.

## 4. 소모처 (Sink)

**원칙(확정): 제작 가능한 건 "편의 판매", 가격 책정은 "제작 불가/진행 아이템" 위주.**
- 몬스터볼(에이프리코트 제작)·회복약(약초·베리 제작)은 Cobblemon에서 **제작 가능** → 상점은 **편의용**(직접 만들기 귀찮은 사람용), 가격 비중 낮음. 핵심 골드 sink로 보지 않음.
- 골드 sink의 무게중심은 **제작이 안 되거나 진행에 중요한 아이템**에 둔다.

**핵심 sink (가격 책정 집중):**
- **전설 조우권**: 핵심 sink (티켓 분류별, `legendary_encounter.md`). 배지 게이트.
- **메가팔찌 / 테라 해금 비용(확정)**: 진행도(배지) **+ 골드**. 골드를 게이트(cap)로 활용해 포켓몬+바닐라 활동 유도.
- **메가스톤**: 기본/고급(리자몽나이트 X/Y 등 후반 고가, `mega_tera_unlock.md`).
- **이상한 사탕(Rare Candy)**: 레벨업 단축 → 의미있는 가격(주간 제한 X, 가격으로 조절).
- **학습장치(Exp. Share 류)**: 육성 보조.
- **진화의 돌(번개의 돌/불꽃의 돌 등)**: 진화 소재.
- **기술머신 TM/TR(SimpleTMs)**: 타입별/고성능 TM — 배지·관장 클리어 게이트(`shop_design.md` §3.4).
- **포켓몬 알(Eggs Addon)**: 알 구매 = 골드 소모처 + 희귀 획득 루트(`shop_design.md` §3.5).
- **실전 육성 아이템**: 비타민/민트/Ability Capsule/Hyper Training Candy 등(밸런스 검토 후).
> ⚠️ 위 진행 아이템들은 Cobblemon에서 **제작/채굴 등 다른 획득 경로가 있는지 확인 필요**(있으면 상점은 편의·고가, 없으면 순수 sink). item id·획득 경로 검증 = TBD.

**편의 판매 (저비중):**
- 몬스터볼 / 회복약: 일상 소모. 제작 대체 가능하므로 가격은 가볍게.

## 5. 시장 구역 상점 (hub_design §2)
| 상점 | 방향 | 구분 |
|---|---|---|
| 광물 판매소 | 광물→골드 | 획득 |
| 농작물 판매소 | 작물→골드 | 획득 |
| 베리 판매소 | 베리→골드 | 획득 |
| 몬스터볼 상점 | 볼 구매 | 소모(편의·제작가능) |
| 회복약 상점 | 회복약 구매 | 소모(편의·제작가능) |
| 성장/실전 육성 상점 | 이상한 사탕/학습장치/진화의 돌/비타민/민트 | 소모(핵심 sink) |
| 기술머신 상점(SimpleTMs) | TM/TR | 소모(핵심 sink) |
| 알 상점(Eggs Addon) | 포켓몬 알 | 소모(핵심 sink) |
| (연구소) 메가스톤·메가팔찌 | 메가스톤/팔찌 해금 | 소모(핵심 sink) |
| 전설 제단 매표 | 조우권(custom) | 소모(핵심 sink) |

- 상점 구현: NPC 또는 GUI. 거래는 **서버 검증 트랜잭션**(잔액 확인→차감→지급, 중간 실패 롤백).
- **가격 정책(확정): 고정가.** 동적/자동 시세 변동은 도입 안 함. 단 **운영자가 지표(§6)를 보고 `economy.json` 가격을 수동 조정** → 리로드. (자동 변동 아님, 운영 판단 기반)

## 6. 텔레메트리 / 골드 흐름 추적 (가격 조정 근거) — 확정 방향
모든 골드 거래가 `EconomyBridge`를 통과하므로 가볍게 추적 가능.
- **골드 흐름**: faucet/sink를 **출처 태그별 집계**(판매소/관장/배틀타워/야생포켓몬 ↔ 티켓/스톤/해금/TM/알/육성/볼·회복약). 시즌·일자별 유입·유출 총량.
- **판매 빈도 / 사용 빈도**: 상점 거래가 핸들러를 거치므로 **아이템별 카운터** 누적(많이 팔리는/사는 품목 파악).
- **저장(단계적)**:
  1. 집계 카운터 = `PersistentState`(`EconomyStats`, `database_schema.md`) + 거래 이벤트 = `AuditLog`. → **무거운 DB 없이 가격 조정 근거 확보(권장 1차).**
  2. 정교한 쿼리/대시보드 필요 시 **SQLite**(번들 JDBC) 선택 도입. 외부 DB(Mongo 등) 불필요.
- 운영 루프: 지표 확인 → 가격 과다/과소 판단 → `economy.json` 조정 → `/poromon admin reload`.

## 7. 유저 거래 / 경매장 — **경매장 도입 안 함 (확정)**
- 경매장은 **만들지 않는다.** 올릴 물건은 직접 제작하거나 상점에서 사고팔면 되므로 불필요(어뷰징·시세조작 관리 부담도 회피).
- 유저 간 거래는 **바닐라 방식**(직접 건네기/상자 등) + Cobblemon 기본 포켓몬 트레이드로 충분.
- 향후 수요가 크면 단순 직거래 GUI 정도만 재검토(경매장은 여전히 보류).

## 8. PoroMonCore 연동
- `EconomyBridge` — 골드 잔액 조회/증감 API(내부 구현, 외부 모드 대비 추상화). **모든 in/out은 출처 태그와 함께 기록(§6 텔레메트리)**.
- `RewardManager` — 짐/배틀타워/포켓몬 처치·포획/이벤트 보상 지급 시 EconomyBridge 호출.
- 저장: `PlayerProgress.balance`(gold) (`database_schema.md` 스키마 추가).
- 로깅: 큰 금액 거래·티켓 구매·보상 지급 = `AuditLog`.
- 명령(향후): `/poromon admin economy give/set <player> <amount>`, 잔액 조회.

## 9. 초기 가격표 초안 (예시 — "메가 진화에 적당한 노력" 기준으로 역산)

> 밸런싱 출발점은 **"무엇을 팔고, 무엇을 얼마에 사는가"**. 아래는 *예시 수치*이며, 운영 후 §6 지표로 너프/버프.
> 앵커: **메가팔찌 ≈ 캐주얼 플레이 수 시간 분량의 골드**가 모이도록 보상/가격을 역산.

**판매(획득) — sellPrices 예시**
| 분류 | 아이템(예) | 판매가(골드) |
|---|---|---|
| 광물 | 석탄/철/금/다이아/에메랄드 | 1 / 5 / 8 / 40 / 30 |
| 농작물 | 밀/감자/당근/호박 | 1 / 1 / 1 / 2 |
| 베리/에이프리코트 | apricorn 류 | 3 |

**구매 — 편의 품목(제작 가능, 저비중) shopPrices 예시**
| 분류 | 아이템(예) | 구매가(골드) |
|---|---|---|
| 몬스터볼 | Poké / Great / Ultra | 50 / 120 / 300 |
| 회복약 | Potion / Super / Hyper | 30 / 100 / 250 |
> 제작으로 대체 가능 → 가격은 "직접 만들기 vs 사기"가 비슷하게 느껴질 정도면 충분.

**야생 포켓몬 보상 — 레벨 비례 예시**
- 처치: `레벨 × 2` 골드 / 포획: `레벨 × 1` 골드 (계수 조정 대상)

**핵심 sink — 제작 불가/진행 아이템 예시 ("적당한 노력" 앵커)**
| 항목 | 비용(골드) | 비고 |
|---|---|---|
| 메가팔찌 해금 | 20,000 | + 배지 게이트 |
| 기본 메가스톤 | 8,000 | |
| 고급 메가스톤(리자몽나이트 X/Y) | 25,000 | + 후반 해금 |
| 이상한 사탕(Rare Candy) | 800 | 레벨업 단축 |
| 학습장치(Exp. Share) | 5,000 | 육성 보조 |
| 진화의 돌(번개의 돌 등) | 2,000 | 진화 소재 |
| 전설 기본권(legendary_basic) | 15,000 | |
> 위 진행 아이템은 **Cobblemon 획득 경로(제작/채굴) 확인 후** 순수 sink인지 편의·고가인지 확정. item id 검증 필요.

### economy.json (스케치)
```jsonc
{
  "configVersion": 1,
  "currency": { "display": "골드" },
  "startingBalance": 0,
  "pokemonRewards": {            // 야생만, 레벨 비례, 감소/쿨다운 없음
    "wildOnly": true,
    "defeatPerLevel": 2, "capturePerLevel": 1
  },
  "sellPrices": { "minecraft:diamond": 40, "minecraft:iron_ingot": 5, "cobblemon:apricorn_red": 3 },
  "convenienceShop": {            // 제작 가능 → 편의 판매
    "pokeball": { "cobblemon:poke_ball": 50, "cobblemon:great_ball": 120 },
    "heal": { "cobblemon:potion": 30 }
  },
  "sinkShop": {                   // 제작 불가/진행 아이템 → 핵심 sink (item id 검증 필요)
    "cobblemon:rare_candy": 800,
    "cobblemon:exp_share": 5000,
    "cobblemon:thunder_stone": 2000
  },
  "ticketPrices": { "legendary_basic": 15000 },
  "unlockPrices": { "mega_bracelet": 20000 },
  "megaStonePrices": { "basic": 8000, "charizardite_x": 25000 }
}
```
> **가격의 단일 출처(권장):** 모든 가격은 `economy.json` 한 곳에서만 관리한다. `tickets.json`·`gyms.json` 등은 가격을 직접 적지 말고 **id로 참조**(중복 정의 시 값 불일치 위험 방지).

## 10. 확정 / 미정 정리

**확정:**
- 골드 단일 화폐 / 야생 포켓몬만 보상, **레벨 비례(감소·쿨다운 없음)**
- **밸런싱 기준: 골드 유통량 자체는 비중요(거래 없음) → "메가진화에 적당한 노력" 기준으로 가격/보상 책정**
- **제작 가능 아이템(볼·회복약)은 편의 판매(저비중), 가격 책정은 제작 불가/진행 아이템 위주**
- 상점 고정가 + 운영자 지표 기반 수동 조정(자동 시세 X)
- 골드 흐름·판매/사용 빈도 텔레메트리 추적(너프/버프 근거; 카운터+로그 1차, SQLite 선택)
- 메가팔찌/테라 해금에 골드 비용 — 골드를 게이트(cap)로 활용
- 경매장 도입 안 함

**미정(TBD) — 수치 튜닝 / 검증:**
1. 야생 레벨 계수(defeatPerLevel/capturePerLevel)
2. 판매/구매 가격표 실제 품목·수치 확정(§9 초안 → 운영 튜닝)
3. 해금/티켓 비용 최종값(메가팔찌 앵커 기준)
4. **진행 아이템(이상한 사탕/학습장치/진화의 돌 등) Cobblemon item id·획득 경로 검증** → 순수 sink vs 편의·고가 결정
5. (권장) 가격 단일 출처(`economy.json`) 적용 — id 참조
