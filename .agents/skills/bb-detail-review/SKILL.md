# Blockbench Detail Review

Use this skill when the user wants a focused review of detail quality after structure or ornament work.

## Goal
Evaluate whether the current asset's detail work is strong, readable, and structurally supported.

## Review checklist

### 1. Structural support
Check whether:
- decorative parts are attached to believable structural blocks
- gems are embedded rather than floating
- trims feel layered rather than pasted flat
- ornaments do not replace the main structure

### 2. Surface quality
Check whether:
- large surfaces are too flat
- detail is too sparse
- detail is too noisy
- repeated patterns feel unnatural
- some zones need one more pass of refinement

### 3. Readability
Check whether:
- the weapon is still readable at Minecraft scale
- the silhouette remains clear
- ornament does not clutter the overall shape
- color contrast supports readability rather than harming it

### 4. Symmetry and consistency
Check whether:
- left/right ornament is properly mirrored when intended
- gold trim thickness is consistent
- gem placement is consistent with the design language
- detail density is balanced across blade, guard, handle, and pommel

### 5. Pixel detail quality
Check whether:
- pixel detail uses intentional clusters
- random mosaic-like noise is avoided
- 2-3 step value transitions are used where helpful
- micro detail does not overpower large forms

## Output format
Report in this structure:

### Strong areas
- list 2-5 areas that are working well

### Weak areas
- list 2-5 areas that are underdeveloped or awkward

### Overly flat zones
- name specific parts that need more depth or support detail

### Overly noisy zones
- name specific parts where detail should be reduced or simplified

### Suggested next pass
- recommend the single most valuable next detail pass
- specify whether the next pass should be:
  - secondary structure
  - ornament
  - pixel detail and polish

## Rules
- Do not directly edit files unless the user explicitly asks.
- Prefer precise part names from the outliner or design spec.
- Be critical but concise.
- Focus on actionable improvements, not vague praise.
