# 서버 생애주기 상세 설계 (T21)

> **[STATUS: DRAFT]** — 2026-06-06. 시즌제 서버의 준비→활성→종료 운영을 구현 가능 수준으로 명세.
> 레지스트리 스키마 = [`data_model.md`](data_model.md) §2.1 `servers`. 요약 결정 = [`task.md`](task.md) §11.
> 가시성 정책 = task.md §10. 봇 관여 경계 = [`architecture.md`](architecture.md)·[`integration_contract.md`](integration_contract.md).

## 0. 경계 — 봇은 디스코드 측만 만진다
- `/서버종료`는 **게임 서버를 끄지 않는다.** 게임 시즌 인스턴스 기동/종료는 게임 호스팅 소관(봇 권위 아님, DL-133).
- 봇이 하는 것: **레지스트리 상태 전이 + 디스코드 카테고리/역할 가시성 + 명령·토글 게이팅 + 운영로그**.
- 게임 화이트리스트는 시즌 리셋 포함 **게임서버가 관리**. 봇은 디스코드 역할만 부여/회수.

## 1. 상태 모델

```
  /서버신설            /서버시작            /서버종료
없음 ───────▶ prep ───────────▶ active ───────────▶ ended (terminal)
              (준비)            (활성)              (아카이브 보존)
```

- `prep` 준비: 카테고리·역할 생성됨, 비공개. 유저 접근 불가.
- `active` 활성: 카테고리 공개(접근역할 가시), 명령·토글 동작.
- `ended` 종료: **완전 비활성**(모든 명령·토글 거부). 카테고리·역할은 **아카이브 보존**(삭제 X). 종착 상태 — 재오픈 안 함(새 시즌은 새 행).
- 모든 전이는 **운영자 명령으로만**, 전이마다 `mod_log` 적재 + `#운영로그` 게시.

## 2. 운영자 명령 (admin · `requires_permission`)

| 명령 | 입력 | 효과 | 멱등/검증 |
|---|---|---|---|
| `/서버신설` | `domain, season_no, display_name` | `servers` prep 행 생성 + 카테고리 템플릿 생성(T17) + 접근역할 생성·지정 | `UNIQUE(domain,season_no)` 중복 거부 |
| `/서버시작` | `server_id`(또는 domain+season) | prep→active. 카테고리 가시성 ON, 이모지 서버선택 패널 등록, 명령 게이팅 통과 허용 | 이미 active면 noop+안내 |
| `/서버종료` | `server_id, 사유` | active→ended. 카테고리 아카이브(숨김), 패널에서 제거, 명령·토글 차단. 카테고리·역할 보존 | 이미 ended면 noop+안내 |
| `/서버목록` | — | 전체 서버 상태(prep/active/ended) 임베드 | 조회 |
| `/서버정보` | `server_id` | 단일 서버 상세(카테고리·역할·시즌·시각) | 조회 |

- 모두 ephemeral 응답 + `#운영로그` 공개 기록(누가·언제·무엇).
- 상태 변경 명령(`신설`·`시작`·`종료`)은 **사용자 명시 요청 시 실구현**(DL-130 ⑤). 지금은 인터페이스 설계까지.

## 3. 전이별 디스코드 액션

> 🟢 **전역 단일 active 모델 구현(2026-06-09, task.md §5):** db v6 전역 active 1 강제. `/서버시작`=`서버준비` 일괄 회수 + 시즌 `임시`(접근)역할 일괄 부여 + 통합 카테고리를 플레이어 역할에 공개. `/서버종료`=시즌 3역할 일괄 회수 + `서버준비` 일괄 부여 + 통합을 `서버준비`에 공개·플레이어에서 숨김. `on_member_join`=active면 그 서버 `임시`, 없으면 `서버준비` 자동 배정. `서버준비`/`통합 카테고리`=`.env`(`ROLE_서버준비_ID`·`CATEGORY_통합_ID`, 미설정 시 skip). 일괄 전이=전체 멤버 순회(defer). ✅ 온보딩 약관/인증 버튼 역할전이 DB화 완료(2026-06-09, panels.py `get_any_active` 기반 — env 매핑 SUPERSEDE).

| 전이 | 디스코드 작업 |
|---|---|
| 신설 (→prep) | 카테고리 생성(템플릿: 약관·접속정보·잡담 등) · 접근역할 생성 · 채널 권한=접근역할에게만 가시 · 접근역할은 아직 미배포(운영자/봇만) |

> 🟢 **T17 구현(2026-06-09): `modules/server_lifecycle/templates.py`** — `/서버신설 자동생성:True`(기본)가 다음을 **prep=비공개**(@everyone·3역할 view=False)로 생성:
> - **온보딩 3역할**: `<표시명> 접근`(access)·`인증전`(pending)·`플레이어`(player). 역할 부여 전이는 온보딩 흐름(T7/T9, 후속) 담당.
> - **프리픽스 카테고리 4그룹**(B안, 중첩 불가 → `<표시명> · <그룹>` 나란히): `온보딩`(약관·인증) · `정보`(공지·접속정보·가이드·FAQ) · `커뮤니티`(자유채팅·스크린샷·파티모집) · `지원·음성`(건의-문의-버그제보·`➕ 음성방 만들기`(임시음성 허브 T13)).
>   - 접속정보=서버IP·실시간 상태(T18 후속), FAQ·문의=지원 T16, 약관·인증=온보딩 패널(후속) 배선 대상.
> - 역할/카테고리 ID는 `servers`(3역할) + `server_categories`(그룹별 카테고리)에 기록. 채널은 카테고리 권한 상속(synced).
> - **가시성 전이 `apply_visibility`**: 시작=약관→접근/인증→인증전/그 외 카테고리→플레이어 공개. 종료=전부 숨김 + `[종료]` 프리픽스(채널·역할 보존). 실패/중복(`UNIQUE`) 시 `cleanup()` 롤백.
> - 공통 템플릿 1개 운영(`_TEMPLATE_GROUPS`). 도메인별 분기 필요 시 후속 확장. `자동생성:False`/수동 `category` 지정 시 자동전개 생략(단수 폴백).
| 시작 (prep→active) | 카테고리 가시성 ON · 서버선택 이모지 패널에 항목 추가 · `state=active` |
| 종료 (active→ended) | 카테고리를 아카이브 처리(이름 `[종료]` 프리픽스 또는 아카이브 카테고리로 이동, 가시성 제거하되 채널·역할 **보존**) · 이모지 패널에서 제거 · `state=ended, ended_at=now` |

> "아카이브" 정의: 채널 삭제 X. 가시성만 닫고 표식. 채팅 히스토리·구조 보존(§11 확정).

## 4. 게이팅 통합 — 3층 (core 헬퍼)

명령 실행 시 순서대로 통과해야 동작:
1. **접근역할(가시성)** — 디스코드 측. 운영자 Integrations 설정으로 접근역할 없으면 명령 자체가 목록에 안 보임.
2. **카테고리 가드** — `core` 헬퍼 `requires_category(domain)`: `interaction.channel.category_id`가 그 domain의 **카테고리 집합**(`get_active_category_ids` = 단수 `category_id` ∪ `server_categories`) 중 하나인지. 아니면 ephemeral 거부. (T17 다중 카테고리 대응 — 집합 매칭, 2026-06-09)
3. **서버 상태** — `core` 헬퍼 `requires_server_active(domain)`: 해당 domain의 active 시즌 행이 있는지. 없으면(prep/ended) 거부.

> 둘 다 접근한 유저가 엉뚱한 카테고리에서 호출 → 2층에서 차단(§10 확정). 종료 서버 → 3층에서 차단.

## 5. 새 시즌 오픈/종료 런북 (운영)

```
오픈:
 1. /서버신설 rpg 3 "RPG 시즌3"      → prep (카테고리 4그룹·채널·온보딩 3역할 생성)
 2. /약관설정 rpg                     → 약관 입력 (서버시작 자동 게시 전에 권장)
 3. (게임서버 측 시즌3 인스턴스 준비 — 봇 외부)
 4. /서버시작 <id>                    → active (서버준비 회수·임시역할 일괄부여 +
                                        온보딩 패널 자동 게시[약관·인증])
종료:
 5. /서버종료 <id> "시즌3 종료"        → ended (아카이브 + 시즌역할 회수·서버준비 부여)
```
> 🟢 **온보딩 패널 자동 게시(2026-06-09):** `/서버시작`이 OnboardingCog.publish_panels 위임으로 약관/인증 채널에 패널 게시. 약관은 시작 시점 저장본 사용 → **/약관설정을 서버시작 전에** 하면 정상 약관으로 게시(미설정 시 경고 패널). 재게시는 `/온보딩패널`(새 메시지 — 중복 주의).

- 새 **시즌** = 위 절차(코드 변경 0, 레지스트리 데이터 토글).
  - ⚠ 단서: 새 시즌이 **다른 게임서버 인스턴스(다른 API URL/키)**면 `.env` 항목 추가 + 재기동은 필요(코드 0이나 *config 0은 아님*). `servers.api_env_key`로 클라이언트가 시즌별 베이스 URL을 선택하도록 설계 — 현행 `rpg_api`의 전역 `ZENON_RPG_API_URL` 단일 참조는 멀티시즌 시 확장 필요.
- 새 **게임 종류** = 위 + `modules/<도메인>/` 모듈 코드 추가 + EXTENSIONS 등록(불가피).

## 6. 미확정
- `prep→ended` 취소 경로 허용 여부(신설했다가 안 열기).
- 종료 시 온보딩 역할(`인증유저` 등) 처리 — 게임 화이트리스트는 게임서버 시즌 리셋 소관. 디스코드 역할 회수 여부는 별도 결정.
- 이모지 서버선택 패널 갱신 = 자동(봇이 메시지 재구성) vs 수동.
- 아카이브 시즌 누적 정리(`/서버정리`, N시즌 경과) = 운영 가이드.
