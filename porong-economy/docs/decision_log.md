### DL-E135 명칭 정리 전파 + Create 기계화 ≠ 자동화 시대 구분

**배경:** DL-E134 단계 재명명(5단계) 후 흩어진 구 명칭 정리 + 사용자 질문(Create 자동화는 마을부터 되는데 왜 자동화가 시대?).

**결정:**
- **Create 기계화 ≠ "자동화 시대"(spec §7-3·design §5):** Create 기본 기계화(크랭크·벨트·기어·믹서)는 **수공업(마을)부터** = "자동으로 도는 공장". **"자동화 시대"(대도시·시즌2)는 AE2/Mekanism 완전 무인 자동화·디지털 물류**(기계 팔·시퀀스 조립·자동 분류, 인력 거의 없는 스마트팩토리). 즉 *기계화(초반)* vs *완전 자동화(최종)* 정도 차이.
- **명칭 전파(5단계: 마을·소도시·도시·대도시·국가):** design §5(진화 SoT 5단계+시대 매핑)·spec §6-2(카테고리 게이트)·§7(진화 표)·§200(자본시장 도시=깊음/대도시=M&A)·§10-1(아크)·concept(경로·Week/Day 표·시즌 14일). balance §1 = 명칭 SoT.
- **잔여:** agenda_catalog 게이트·indicators 산발 언급은 balance §1 기준 점진(대부분 "대도시=전기" 구 맥락).

**관련:** `design.md` §5, `spec.md` §6-2·§7·§7-3·§200·§10-1, `concept.md`, `balance.md` §1(SoT), DL-E134·E133·E065(Create 큐레이션)·E062(시대).

### DL-E134 진화 단계 재명명 — 시즌1 = 마을·소도시·도시(전기), 소도시 신규

**배경:** 사용자 — 14일에 전기를 *확실히* 맛보려면 전기를 마지막(대도시)에 두면 짧음. "대도시"는 이름부터 과한 종착점. → 도시=전기로 당기고, 마을~도시 사이 단계(소도시) 추가, 대도시·국가는 시즌2.

**결정 (DL-E133 단계 재배치):**
- **명칭 한 칸 당김:** 구 도시(증기)→**소도시** · 구 대도시(전기)→**도시** · 구 국가(자동화)→**대도시**(시즌2) · 신규 **국가**(대외무역, 시즌2 후반).
- **시즌1 = 마을(수공업)·소도시(증기)·도시(전기) 3단계** — 12 세부단계. 도시(전기) = D8-14 6일 = 자본시장·자동차·가전 클라이맥스 충분 체감.
- **시즌2+ = 대도시(자동화)·국가(대외무역).**
- **balance §1 = 단계 명칭 SoT** + 구 명칭 매핑. 타 문서의 구 "대도시=전기·자본시장·금융" 언급은 §1 기준 점진 정리.

**관련:** `balance.md` §1·§1-1·§1-2(SoT), `spec.md` §10-1, DL-E133(시즌 14일)·E051(진화)·E085(자본시장 단계). 페이싱 임계 P1. 타 문서(content_catalog §1·spec §6-2·design §5/§7·agenda 게이트) 명칭 점진 정리 = 후속.

### DL-E131 정합 점검 3차 — 사회모델 갱신 후행 보정 (2-에이전트: 정합·복잡도)

**배경:** E124~E130 사회모델 대량 추가 후 검토(2-에이전트). 헤드라인만 들어가고 인접 수치 미전파 "갱신 후행" 🔴 6건(E101·E117과 동형) + 복잡도 경고.

- **§1 헤더/본문 구 인구값 정정:** 600→2,000 / 800·1,400·2,400 → 2,700·4,700·8,000.
- **수용력 마을 2,500 → 3,000**(도시 진화 임계 2,700 초과 → 첫 전환 인구 게이트 도달 가능). 단계별 5,500/11,000/20,000.
- **파업 → 가동률 배선:** 외생 페널티에 **k₄ 파업 0.1~0.3**(cap 0.6), balance §3·spec §4 동시 갱신. 사회 갈등(E129)의 경제 압박 실제 배선.
- **§15 신규 5지표 트리거:** 행복도(시위·파업, ≤30 긴급 의회)·부양비·혼인율·유소년 고용률·계층 이동 정체 → 신규 안건 발의 임계 확보.

**관련:** `balance.md` §0·§1·§3·§5·§15·§16, `spec.md` §4, `population_model.md`, `agenda_catalog.md`, DL-E101/E117(동형 점검)·E125·E126·E129·E130·E074(로드맵)·E084(라이브 튜닝).

### DL-E127 사회 모델 확장 — 아동노동 평판 + 혼인율→출산 + 인구 정치 콘텐츠

**배경:** 사용자 — 인구 모델이 사회경제 모델로 확장됨(환영). 아이디어: 인건비 절감 아동노동→평판 하락 / 혼인율·출산율 / 의회 안건 풍부.

**결정 (정체성 확장 = 경제발전 → 사회경제 시뮬, 인구가 토대):**
- **아동노동↔평판(population_model·agenda §6):** 유소년 고용 = 저임금(인건비 절감) vs **평판↓·행복도↓·미래 학력↓**(교육 박탈→인적자본 손실). 착취-평판 트레이드오프(산재 §9 동형)·도덕 딜레마. 아동노동 금지 안건이 바닥 규제.
- **혼인율→출산 세분:** `출산 = 혼인율 × 가임 생산가능인구 × 출산성향`. 혼인율 = f(소득안정·주거·청년 비율) → 저출산 대응이 "결혼부터"(결혼·주거 지원 안건).
- **인구 정치 콘텐츠 확장:** 인구 모델이 의회 안건 엔진 — 아동노동·결혼/주거·고령화·정년·교육 등 다채. 의회 재미↑.
- **시즌 길이 = 검토 대상(미정):** 사회 모델 깊어지면 28일 빠듯 → 사용자 확인 후 결정(별도).

**관련:** `population_model.md`, `agenda_catalog.md` §6, DL-E126(연령 구조)·E099(산재 평판)·E042(저출산)·E020(교육). 계수 P1. **장르 = 경제발전 → 사회경제 시뮬로 자연 확장.**

**배경:** 사용자 — 시작 마을 나이 분포? 출산율? 고령화·베이비붐 안건/모델 반영됐나? 2,000명에 출산 3~4%/일이면 몇 만 폭증할 것. 점검 결과 **연령 구조·수용력·인구구조 안건 전부 부재** + 출산 +2%/일 복리가 천장 없이 폭증 위험.

**결정:**
- **연령 구조(경량, balance §0·§5·population_model):** 유소년·생산가능·노년 3구간 **비율 집계**(코호트 풀 X). 초기 20/65/15%. **노동력 = 인구 × 생산가능비율** · **부양비 = (유소년+노년)/생산가능** → 복지·의료 부담.

- **공모/증자:** 신주 발행 비율·공모가 설정 → 조달액 = 발행주 × 공모가 → 자기자본 유입. 공모가 = fair 기반 자동 산정(± 유저 조정 P1). 발행분만큼 기존 대표 지분율 희석 → **51% 가드 주의**. 유상증자(상장 후)도 동일 트레이드오프(자기자본↑ vs 희석·경영권 노출).
- **무캡 자유 플레이:** 보수적(유보·안정) ↔ 공격적(레버리지·증자·고위험) 자유 선택. **투자(주식·배당·M&A)만으로도 부 축적 가능**(생산 없는 트랙 허용, DL-E072 연장 — 인위적 캡 없음). 도산은 유한책임(DL-E029)이나 신용·건전성으로 견제.

**관련:** `spec.md` §5-5·§5-4, DL-E029(법인격·재무제표)·E033(IPO·주식)·E034(IPO 요건 이월 해소·희석)·E039(대출)·E024(건전성)·E072(서비스/금융 트랙)·E089(자본시장 메커닉). 임계·계수·공모가 산정 P1.

### DL-E089 자본시장 메커닉 (주가 형성 함수·집계 유동성·배당 환류·적대적 인수 규칙·거시 연동) — DL-E033·E034 구체화

**배경:** 세부 메커닉 ③의 시장 측. DL-E033/E034는 방향(주가=펀더멘털+수급/심리, 오너식 51% 지배, 적대적 인수)만 확정, 작동 함수·규칙 미명세. spec에 자본시장 섹션 부재. 비전 확인: ① 적대적 인수 = **부차적**(리스크 관리 잘하면 거의 안 당함) ② 투자 단독트랙 = **무캡**(자유 플레이).

**결정 (spec §5-4):**
- **주가 형성 = 펀더멘털 앵커 + 수급/심리.** ① **공정가(fair)** = `(자기자본·w_book + 정규화 이익·PER_mult·w_earn)×(1+이익추세 g)` / shares_outstanding(financials 기반, 적자→fair↓). ② **시장가** `price_{t+1} = price_t×(1 + α·(fair_share/price−1) + β·net_flow + γ·macro + ε)` — α 평균회귀(**실적 없는 거품 자연 조정** = 인위적 캡 아님, 무캡 일관), β 수급, γ 거시 사이클(거품/폭락), ε 노이즈. OHLC → `stock_price_history`.
- **유동성 = 주민 집계 풀(베이스) + 유저(능동).** 주민 자산배분(DL-E045)이 저축 일부를 주식 풀로(금리 스위치·모멘텀·불평등), 매수 = **시총비례 인덱스식 + 펀더멘털 추종**·passive(개별 심리 시뮬 X, DL-E034). 유저 = 거래소 호가(능동·집중·매집). 주민 집계 = 시총비례 유동성 공급자(체결 깊이). net_flow가 수급항에 피드백.
- **유저 자본 = price impact(주가 능동 변동):** 유저 순매수/매도가 net_flow에 직접 → **자본으로 주가를 밀어 움직임**(펀더멘털 무관 단기 부양/하락). impact 강도 = 거래량 대비 주문 크기(시장 깊이) → **소형주는 적은 자본으로도 크게 출렁**(작전·세력), 대형주 둔감. **α 회귀가 자연 견제** → 펌프앤덤프는 단기 가능·유지엔 자본 지속 투입 필요(세력 이탈 시 하락). 무캡이되 무한 조작은 시장 메커니즘으로 막힘 → 주가 부양 방어·소형주 작전 등 자본시장 PvP 성립. 계수 P1.
- **배당 → 노동=소비 루프 환류:** `dividend_policy`(이익 중 배당 비율) → 결산 틱 지분 비례 배분. **주민풀 배당 → 가계 소득 → 자산효과 소비(DL-E044) → 노동=소비 루프.** 배당↑(보유동기·소비) vs 재투자↑(성장·미래 fair) 트레이드오프.
- **M&A·적대적 인수(부차적 위상):** 거래소 매집 → `shareholding` 누적, controller = 유저 max 지분(오너식) 역전 시 경영권·관리 GUI 이전(DL-E034). **51%+ = 인수 면역**(유동주식 49% 미만), **비상장·1인 100% = 불가**. 적대적 인수 = 51% 미만 대표 상장사 한정(방심 응징·호경기 드라마). 방어 = 지분 추가 매입(51% 영구면역)·배당 부양·유보(fair↑). **리스크 관리 잘하면 거의 안 당함.** 주민회사(DL-E088) = 정부 51% 영구 면역. 시장 거래라 자산 파괴 아님(DL-E008).
- **거시 연동:** 주가지수(시총 가중) ↔ §8 자산 사이클 양방향. 자산효과(주가·배당→자산→소비 DL-E044) → 사이클 증폭. 과열·붕괴 시 서킷 브레이커(§6-4)·거시 안건 개입.

**관련:** `spec.md` §5-4, DL-E033(주식·자본시장 원안)·E034(지배구조·오너식·적대적 인수)·E045(자산배분 유동성)·E044(자산효과 소비)·E085(상시 ON)·E088(주민회사 상장·면역)·E032(거래소 GUI)·E008(시장 파괴 X)·E090(회사 금융). 계수(α·β·γ·PER_mult)·매칭 깊이·임계 P1.

### DL-E088 주민회사 = 공기업 (정부 51%+ 영구·부분 상장·비용연동 동적 앵커) — DL-E005 구체화

**배경:** 주민회사(시장 안정 닻)가 "항상 고정 안정"이면 비현실·지루(사용자). 현실 공기업처럼 상장·변동을 넣되 앵커 정체성·독점 grief 방지를 보존해야.

**결정:**
- **주민회사 = 마을 공기업:** 정부(CORE/마을) **51%+ 영구 보유**(경영권·앵커 통제, 비매각) + **49% 미만 부분 상장** → 주민·유저 매수 가능(**passive 소수주주** — 배당·시세차익 O, 경영권 X). DL-E034 "주민 지분=passive·거래 가능" 패턴 일관, DL-E008(국유화/시장 파괴 X) 무충돌(앵커 장악 차단 = 51% 가드레일).
- **비용 연동 동적 앵커:** 기준가/기준 전기세 = f(비용). 연료 폭등 → 주민회사 비용↑ → **기준가↑(앵커 위로) 또는 적자(재정 부담) → 인프라 가격↑ or 보조금 의회 안건.** "절대 고정"이 아니라 비용 따라 움직이는 동적 기준선(DL-E005 바닥[0원 폭락 방지·최소 공급]은 유지).
- **공기업도 시장 참여:** 연료·원료 시장 구매(원가) + 주민 고용(인건비) → 재무제표·주가 반영. **주가 = 공영/인프라 건전성 신호** → 적자·주가↓ → 보조금/규제 정치 트리거.

**근거:** 동적 앵커가 "앵커 흔들림→보조금 정치" 재미. 부분 상장이 유저 인프라 투자 메타(주가=건전성 신호). 정부 51%가 경영권 장악·전력/필수재 grief 방지 가드레일(현실 공기업 모델).

**관련:** DL-E005(주민회사 닻 구체화)·E034(지분·경영권 passive)·E033(IPO·주식)·E008(시장 파괴 X)·E071(원료 앵커)·E087(공공 발전 앵커)·E012(재정 보조금)·agenda §3/§8. 점유율·기준가 산정·상장 비율·발동조건 = P1(DL-E005 열린 질문 일부 잔존).

### DL-E087 전력망 메커닉 = 원가연동 단일 전기세(cost-plus) + 공공 믹스 + 유저 발전 price-taker — DL-E081/E083 구체화

**배경:** DL-E081/E083(전력망 원칙)을 가격·정산 메커닉으로 내림. 발전소별 호가/merit-order 큐는 30명에 과하고 "유저가 가격 올리면 평균 폭등" 우려.

**결정:**
- **그리드:** CORE 블록 2종(**그리드 공급구** / **전력 인입구**) + CORE 추상 풀(차원 넘나듦, 물리 와이어 X). 발전소 FE→공급구, 설비 전기모터←인입구, CORE가 양끝 측정·송전·정산. 유저 자기 발전기 안 지어도 됨(price-taker 구매).
- **전기세 = 원가연동 단일 가격(cost-plus):** = **공공 발전 믹스의 평균 원가**(연료+주민 인건비+운영)+마진. **발전소 호가 없음 — 단일 가격.** 의회가 믹스(석탄/디젤/친환경)로 전기세를 움직임.
- **유저 발전소 = price-taker:** 전기세에 판매, **생산 단가 낮으면 마진**(수력·효율) / 높으면 peaker(디젤=전기세 높을 때만). **유저 비용은 소비자 전기세에 영향 X**(호가·조작 불가). 저비용 유저 발전이 공급 보태면 공공 고비용 가동↓ → 간접 전기세↓.
- **가동 = 믹스 비율(dispatch 큐 없음):** 공공=믹스 비율 가동, 유저=수익 시 자발 참여. 초기 화석 중심, 친환경은 전기 시대+ 해금 후 의회 안건 전환(건설형).
- **공공 발전 연료 시장 구매:** 공공 발전소도 석탄·디젤 시장 구매 → **정부=큰 연료 수요처**(광업/정유 유저 매출) + 정부 지출·적자 → 재정 정치. 연료가↑→공공 발전비↑→전기세↑ 연쇄.
- **친환경 전환 동학:** 의회 친환경 통과 → 믹스 화석↓ 친환경↑ → ① 친환경 토지 저효율 → 평균 원가↑ → **전기세↑**(전 산업 비용) ② 연료 수요↓ → **석탄·디젤 가격↓**(광업 타격·재정 연료비↓). 전기세↑ vs 오염↓ 정치(석탄 유저 반대 로비).
- **부족:** 전기세↑(증설 신호) + 의회 요금 규제/배급(agenda §8, 주민>산업).
- **발전소 산업(DL-E072):** spot(변동) or 계약(고정 §11-B). 공공 발전=앵커(기준가 천장, 붕괴·독점 grief 방지).

**근거:** cost-plus 단일 요금이 호가 조작·"가격 묶기" 문제 제거(가격 하나, 경쟁=비용/마진). 믹스+의회가 친환경 정치 동학, 공공 연료 구매가 재정·연료 경제 연결. 30명·1인 개발에 단순.

**관련:** DL-E081(전기시대·전기설비)·E083(전력망 모델)·E019(인프라 하이브리드)·E072(전력 수익)·E005/E088(공공 앵커·공기업)·E015/agenda §8/§환경·E010(오염)·E080(연료). 전기세 함수·믹스 비율·효율/오염 계수 P1.

### DL-E086 생산 코어 = 노동·품질·고장 결합 (산출물 박스 게이트 / weakest-link 품질) — 자동화·노동 결합 함수 해소

**배경:** Create는 **동력만으로 자동 가동**돼 주민 노동이 들어갈 자리가 없음(경제 시뮬↔Create 1번 난제). + 산출물 박스 우회(체스트 직결·수직 통합 내부 중간재)·품질 추적 불가·트리 shortcut exploit 우려(사용자).

**결정:**
- **물리/경제 분리:** Create는 물리 생산(체스트·박스 자유). **경제 판매 = 산출물 박스(회사 합산 창구)만** — 체스트는 창고일 뿐(판매 불가·산출템 P2P 차단 INBOX-029). Create 내부는 안 건드림, CORE가 박스에서 관측·제어.
- **노동 = 판매 속도 cap(가동률):** 회사 시장 진입(판매) 속도 = f(가동률). **가동률 = f(충원율[배치 인원/설비 요구 인원] × 학력 매칭 − 안전)**. 요구 인원 = 설비 규모 비례 → **큰 공장=많은 고용**(노동=소비 결합). **회사 단위 합산**이라 박스·체스트 수 무관(우회 불가). 0명=정지(DL-E004).
- **고장:** 충원율 기반 확률·길이로 **창구(박스) 일시 정지 → 복구까지 환금 0**(직결이면 backpressure로 기계 idle, 체스트 버퍼면 안 팔리고 쌓임=보관비·과잉). 복구 속도=배치 학력·감독관(DL-E047). per-회사 상태.
- **품질 = 박스 stamp(아이템 data component):** `품질 = weakest-link( 제품 레시피가 선언한 全 분야 stage ) × 학력`. **각 stage 품질 = max( 그 stage 중간재 구매 품질[②, 박스 추적], 회사의 그 분야 특화 역량[①, 연속 투자 DL-E062] )**. Create 내부 NBT 추적 회피 → 회사 집계.
  - **핵심(exploit 차단):** weakest-link는 **레시피 메타데이터가 선언한 모든 가치사슬 stage**에 걸쳐 평가 — "유저가 찍은 트리"가 아님. **미투자+미구매 stage = base-low가 디폴트.** 예: 자동차=[소재+제조+전자], 소재 특화 안 찍고 철강 자체생산(저품질, 내부라 CORE가 철 아이템은 못 봄) → CORE는 레시피로 "소재 stage 존재"를 알아 **회사 소재 역량(base-low)**을 그 stage에 박음 → weakest-link로 최종 cap. **내부 중간재를 안 봐도 미투자 stage가 자동 저품질.**
  - 역량은 **연속(투자량)** = binary 해금 아님 → "살짝 찍고 고품질" 불가. 수율(DL-E064)도 동일 식.
- **트리·분업:** 레시피(KubeJS)가 재료 강제(자동차=철강+부품) → 자체생산(분야 노드) or 시장 구매. **분야는 병렬(특화 자유) — 강제 트리 의존 X**(강제 시 수직통합 강요·거래 죽음). 제조만 특화하고 철강은 전문가한테 고품질 구매 가능(weakest-link가 보상) → **분업 유리**, 수직통합=전 분야 특화 불가로 어중간(DL-E067 분업 의도 달성).

**근거:** 노동을 "기계 물리 가동"이 아니라 "회사 판매 속도·품질·고장"에 묶으면 Create 자동화와 충돌 없이 노동이 진짜 입력이 됨. weakest-link+연속 역량이 shortcut exploit 차단. 레시피 게이팅+병렬 분야가 분업 유도. 전부 CORE가 박스에서 처리(Create 포크·믹스인 불필요, 1인 개발 현실적).

**관련:** DL-E004/E040/E046/E047(노동·고용·학력)·E013/E064(품질·전파·수율)·E062/E067(특화·분업)·E026/E029(산출물 박스)·E080(채굴 설비). `task.md` 열린 질문 "자동화·노동 결합 함수" 해소. 충원율·가동률·고장 확률·품질 가중치·요구 인원 계수 = P1.

**관련:** DL-E063(공급과잉)·E004(가동=노동)·E005/E071(주민회사 앵커)·E064(품질전파)·E079(월드)·E028(P2P/산출물 박스). 효율 계수·설비 산출함수 P1.

### DL-E076 안건 발의 우선순위 (심각도 + 노화 + 쿨다운, 긴급 큐 우회) — spec §6-2 해소

**배경:** 의제 설정은 CORE 전속(DL-E006·E025)이고 정기 발의는 하루 1건(design §6)인데, 트리거 충족 후보가 동시에 여럿일 때 **어느 안건을 먼저 올리는지 선택 기준이 기획 미정**(spec.md 착수 시 `[질문]`). 하루 1건 병목이라 선택 규칙이 페이싱·정치 재미·파국 방지를 좌우.

**결정 (`spec.md` §6-2):** 후보 큐에서 **우선순위 점수**로 1건 선정 — `점수 = 정규화 심각도(임계 초과폭, 主) + 노화 가산(대기 일수) − 재발의 쿨다운 감점`.
- **심각도 주:** 임계를 가장 크게 초과한(정규화) 안건 우선 → 급한 문제 방치→파국 방지.
- **노화 가산:** 대기 일수 비례 가산 → 만성 약위반 starvation 방지.
- **재발의 쿨다운:** 가결/부결 안건은 N일 감점(P1) → 같은 안건 도배·단조 방지. 지표 계속 악화 시 쿨다운 후 재등장.
- **긴급 우회:** 매우 심각(긴급 임계)=긴급 소집(3단계 에스컬레이션 2단계), 파국 임계=서킷 브레이커(3단계)로 정기 큐 우회 → 정기 우선순위는 일상 발의용.

**근거:** 순수 심각도순은 약위반 starvation·단조, 순수 FIFO는 급한 문제 지연. 하이브리드가 30명·3~4주 시즌에서 ① 파국 방지 ② 다양성 ③ 페이싱 양립. 정규화식·노화 계수·쿨다운 길이·긴급/파국 임계 = P1.

**관련:** `spec.md` §6-2·§6-4, `agenda_catalog.md`(트리거), DL-E025(트리거 발의)·E006(CORE 의제 전속)·E015(서킷 브레이커)·E075(틱 감시). 계수 P1.

### DL-E023 중앙은행 = 경제의 화폐 중추 (재정+통화 통합점) + 2층 은행 구조 〔2층→단일 정정: DL-E039〕

### DL-E015 통화정책 = 의회 안건(A) + CORE 비상 안전장치(서킷 브레이커)

**배경:** 통화정책 통제 주체 결정. 유저는 경제를 잘 모르고 단기 부양(금리↓·주가↑)을 선호해, 순수 의회 통제 시 파국(초인플레·거품 붕괴)까지 갈 수 있다.

**결정:**
- **통화정책 = 의회 안건(A안).** 금리·QE·지급준비율을 CORE 발의 → 유저 찬반(다른 안건과 동일). 컨셉(집단 선택의 재미)과 일관하고, 남발의 대가(인플레·거품·위기)가 자연 처벌.
- **CORE 비상 안전장치(서킷 브레이커):** 핵심 지표가 **돌이킬 수 없는 임계**(초인플레·신용 시스템 붕괴·재정 디폴트 직전)에 닿으면 CORE가 2단계 개입 — ① **경고=거부권**(파국으로 미는 안건 무효) ② **파국 임박=긴급명령**(능동 개입, 예: 강제 금리 인상). 3원칙: ⓐ 극단에서만(평범한 악수엔 불개입, DL-E007) ⓑ 위험 구간 대시보드 사전 경고(규칙처럼 예측 가능) ⓒ 소수 보호가 아니라 시스템 붕괴 방지(DL-E007 무충돌).
- 적용 범위: 통화정책뿐 아니라 파국을 부르는 모든 안건(예: 무한 국채)에 일반 적용.

**관련:** `concept.md` §6(의회 구조)·지표 §3, DL-E007(다수결 원칙)·DL-E016(거시 안건).

### DL-E013 사치품 = 고품질 소비재(품질 속성 재사용) + 소득탄력성 소비 + 원료 희귀도 티어

**배경:** "사치품을 어떻게 다루나 / 사치품 전업 유저가 있을까"(사용자 제기). 별도 사치 아이템·직업을 만들면 인위적이다.

**결정:**
- **사치품 = 일반 품목의 고품질 상층.** 별도 아이템이 아니라 이미 정한 **품질(quality) 속성**의 재사용 — 같은 빵도 저품질=필수품(빈곤층), 고품질=사치품(부유층). 누구나 품질을 올리면 사치 시장 진입(품질 경쟁 = 사치 시장 공략). → "사치품 전업 유저" 불필요.
- **소비 = 소득탄력성으로 창발.** 주민 계층 소득 분포에서 사치 수요가 나온다(가난→필수, 부유→사치). 경제 성장으로 부유층↑ → 고품질 수요↑.
- **원료 희귀도 티어:** 1차 자원에 흔함→희귀(다이아 등 바닐라보다 희귀) 티어. 희귀 원료 = 고품질 소비재(사치품)를 만드는 고부가가치 산업의 비싼 입력. 다이아=소비 사치품이 아니라 사치품의 *원료*.

**관련:** `concept.md` §11(시장)·§1(산업 티어), INBOX-013(품질 경쟁 promote). 품질 속성·노동=소비 루프와 연결.

### DL-E012 마을 재정 모델(세수 원천 + 부채 stock·이자) + 재정 안건

**배경:** 환경·치안 등 모든 안건이 세수를 소모 → 그 돈줄(재정)을 정의해야 트레이드오프가 닫힌다. 재정은 창발이 아니라 회계 항등식.

**결정:**
- **재정수지 = 세수 − 공공지출.** 적자는 국채로 메우고, **부채 = stock 누적 + 이자** → 방치 시 눈덩이(부채 위기 → 긴축 안건 / 거시 신용 위기).
- **세수 원천(S1 내수):** 법인세(회사 이익)·소득세(주민 임금)·**거래세(시장 거래액 %)**·재산세(토지·부동산). 관세 없음. 거래가 전부 시스템 경유(§1-A)라 **거래세 자동 징수 용이**(설계 시너지).
- **공공지출:** 인프라·교육·복지·치안·환경 + **주민회사 운영 재원**(→ DL-E005의 "주민회사 재원" 열린 질문 해소: 마을 재정에서 충당).
- **세율 = 정책 상태값**, 의회 안건(증세·감세)이 바꿈.
- **안건 축:** 성장·자유(감세·긴축=작은정부) vs 분배(증세·복지). **재정 안건 6개**(증세·감세·국채·긴축 + **누진세**[불평등 직접 레버, 자본가 유저 직격] + **산업 보조금**[연구 완료 기술 사용 공장에 지급 → 교육↔재정↔산업 루프]). 소득세에 누진성(progressivity) 속성. 안건표 `agenda_catalog.md` §3.

**관련:** `concept.md` §10·§11-A·지표 §4, `agenda_catalog.md` §3, DL-E005(주민회사 재원 해소)·DL-E008(법인세 등 시장 개입)·거시 카테고리(부채 위기→신용경색).

### DL-E011 치안·범죄 모델(활동 창발) + 안건(단속 vs 예방 축)

**배경:** 치안 안건을 짜려면 범죄율을 무엇으로 발생시키나가 먼저 필요. 환경오염과 같은 창발 원칙 적용.

**결정:**
- **범죄율 = 활동 창발:** `f(실업률↑, 빈곤율↑[욕구충족↓], 불평등↑)`. 별도 변수가 아니라 이미 추적하는 주민 지표에서 나온다.
- **구현 = 집계 %** (개별 범죄자 NPC 없음, 하이브리드 §4). 거래 시스템 경유(§1-A)로 유저 물리 범죄 여지 적음. **유저 = 피해자+의회 대응자**: 범죄율↑ → 산출물/상점 박스 재고 도난·매출 누수·설비 파손·부동산↓·행복도↓(대시보드 "도난 손실" 표시). 교도소 트리거 수감자 = 범죄율×인구×검거율.
- **유저 경제 범죄(탈세·담합·조작)는 별개 트랙**(P2 부정행위/시장 감시, 담합은 주민회사 견제) — 범죄율과 분리.
- **안건 축:** 단속(경찰서·교도소·순찰 = 질서) vs 예방(복지 = 분배). 건설형(경찰서·교도소)은 가용 토지↓+세수 소모 트레이드오프. 안건표는 `agenda_catalog.md` §2.

**관련:** `concept.md` 지표 §11, `agenda_catalog.md` §2, DL-E009(지표)·DL-E010(창발 패턴)·DL-E005(건설형 주민회사 무관, 건설형 안건은 §6).

### DL-E010 환경오염 모델(활동 창발 stock) + 업종 마스터 테이블 확정

**배경:** 환경 안건을 짜려면 "환경오염"과 "고오염 산업"을 무엇으로 측정하나를 먼저 정해야 했다(사용자 제기). 임의 수치로 두면 "거시는 집계에서 창발" 원칙과 어긋난다.

**결정:**
- **환경오염 = 활동에서 창발하는 누적 stock.** `오염 += 유입 − 흡수`. 유입 = Σ(회사 생산량 × 업종 오염계수) + (전력 생산량 × 발전원 오염계수). 흡수 = 자연정화(녹지 비율) + 정화사업(안건). **시즌1은 마을 전체 단일 수치**(지역별 오염은 후보).
- **오염의 대가:** 오염↑ → 주민 건강↓ → 행복도↓·노동생산성↓ → 경제 부메랑(옵션: 공단 주변 부동산↓). 장식 숫자가 아니라 노동력을 갉아먹음.
- **업종 마스터 테이블 = 산업 설계 척추.** 회사는 설립 시 업종 1개 등록. 각 업종 = 속성 묶음(오염계수·산재위험도·부가가치·노동집약도·경기민감도…)을 가진 데이터 테이블 행. **"고오염 산업" = 오염계수 ≥ 임계 업종**(탄소세 안건 자동 대상). 같은 테이블이 산재·산업다양성·거시 민감도도 굴린다.
- **업종 인식 = 산출물 게이트(방향):** 생산을 물리적으로 막지 않고 **산출물 박스에서 업종-품목 일치 검사**(엉뚱한 품목은 환금 불가). 박스 세부·P2P 차단은 INBOX-029.

**관련:** `concept.md` §1-A·지표 §7, `agenda_catalog.md` §1(환경), INBOX-029. DL-E009(지표 마스터)·DL-E004(산재)·DL-E008(탄소세 등 시장 개입) 연결.

### DL-E009 시즌1 = 내수 중심 경제 + 경제 지표 마스터 리스트(11 카테고리) 확정

**배경:** 안건 카탈로그(P0-② ⓐ)를 짜려면 CORE가 감시할 지표 체계가 먼저 필요. 동시에 시즌1 경제 범위를 좁혀 설계 면적을 줄여야 했다.

**결정:**
- **시즌1 = 내수 중심 경제 모델.** 판매 수요 주체 = 마을 내부 주민. **대외 수출 시장·본격 무역은 시즌1에서 제외**(시즌2+ 후보). 원료 수입은 비용 요소로만 잔존(§9), 무역 자체를 콘텐츠화하지 않음.
- **경제 지표 마스터 리스트 11 카테고리 확정**(`concept.md` "경제 지표 마스터 리스트"): ①주민·인구 ②시장·물가 ③거시·경기 ④재정 ⑤산업·기업 ⑥인프라 ⑦환경 ⑧연구·교육 ⑨부동산·토지 ⑩금융·자본시장 ⑪치안·범죄. ★=시즌1 1차 핵심, (관)관측/(정)정책 구분.
- **치안·범죄** 신규 채택 — 범죄율·치안 수준이 건설형 안건(경찰서/교도소)을 부르고, 가결 시 **유저 가용 토지↓ + 세수 소모** 트레이드오프로 연결.
- **행복도**는 단일 변수가 아니라 **하위 지표 가중 합성**(욕구충족·실질소득·고용안정 − 노동시간·산재·불평등·범죄)으로 모델링.

**관련:** `concept.md` 지표 마스터 리스트·시즌1 스코프, INBOX-025·027. 다음: 지표 → 안건 매핑(P0-② ⓐ).

### DL-E008 안건의 시장 개입 경계 = 경쟁 촉진 O · 시장 파괴 X

**배경:** CORE 안건이 시장에 어디까지 손댈 수 있나(P0-② ⓔ). 너무 약하면 정치가 무의미하고, 너무 세면 유저가 쌓아올린 자산을 파괴해 플레이 의욕을 꺾는다.

**결정:** 원칙은 **"경쟁은 붙이되 시장은 무너뜨리지 않는다."**
- **허용(시장 개입):** 가격 상한·강제 분배·법인세 확장 등 시장에까지 영향을 미치는 정책.
- **금지(자산 파괴):** 유저가 쌓아올린 것을 한순간에 무너뜨리는 정책. ① **국유화 불가**(유저 회사를 통째로 빼앗지 않음). ② **독점금지 명목의 회사 강제 분할(동강내기) 불가.**
- **독점 대응 방식:** 회사를 깨는 대신 **해당 품목에 국유 주민회사를 신설해 경쟁을 주입**한다(§11-A, DL-E005와 연결) — 독점은 깨되 유저 자산은 보존.

**관련:** `concept.md` §6(안건 권한 경계)·§11-A(주민회사 3역), INBOX-024, DL-E005.

### DL-E007 의회 운영 원칙 = 경제모델 집단선택의 재미 · 다수결 그대로 · 정당 무권한

**배경:** 의회를 실제 정치 놀이로 키우면 경제 시뮬에서 벗어나고, 소수 보호나 정당 권한을 넣으면 복잡도·장악 위험이 다시 생긴다.

**결정:**
- **의회 목적:** 실제 정치가 아니라 **앞으로의 경제 모델을 유저가 집단으로 선택·예측하는 재미.** 이 서버는 어디까지나 경제 시뮬레이션.
- **다수결 그대로:** 단순 과반으로 발효, **소수 보호장치를 두지 않는다**(정족수 정도만). 다수결의 무게(소수 압박 포함)는 게임적 선택의 결과로 함께 진다.
- **정당:** 가치 축(성장·자유 vs 분배·환경)을 같이하거나 **함께 시작한 친구들의 모임** 수준의 **무권한 소셜/정체성 결사.** 표를 강제하면 장악 통로가 열리므로 권력 기구로 만들지 않는다.

**관련:** `concept.md` §6, INBOX-023, DL-E006(구조).

### DL-E006 의회 구조 = 전원 의석 직접민주제 + 발의권 CORE 전속 (반(反)장악 설계)

**배경:** P0-②의 최우선 리스크는 "소수 유저가 의회를 장악"하는 것. 의석 경쟁·의제 설정권이 유저에게 있으면 담합·매수로 정치가 사유화된다.

**결정:** 소수 장악을 **구조로 차단**한다.
- **① 전원 의석 직접민주제(1인 1표):** 마을 일정 성장 후 의회 출범 시 **모든 플레이어가 의석을 가진다.** 의석 수 제한이 없어 의석 독점 경쟁 자체가 없다. 자본 규모와 무관하게 1표 평등(경제 권력 견제).
- **② 의제 설정권 = CORE SYSTEM 전속:** **유저는 안건을 발의할 수 없고 무엇을 안건으로 올릴지도 못 정한다.** CORE가 경제·환경·재정 등 지표를 감시해 조건 충족 시 안건을 던지고, 유저는 **찬/반만** 한다. agenda-setting 권력을 시스템에 둬 장악 통로를 봉쇄.
- **정당:** 결성 허용하되 **의석·표 배분·강제 없음.** 당론 추종은 자유, 정당은 권력 기구가 아니라 가치 축(성장·자유 vs 분배·환경)을 대표하는 자발적 결사·여론체.

**관련:** `concept.md` §6, INBOX-018(발의 자동화 구체화)·INBOX-022. 후속 P0-② 잔존: 안건 트리거·카탈로그, 정족수/가결기준, 정당 실효 기능, 다수결 소수보호, 의회 권한 vs 시장 자유 경계.

### DL-E005 주민회사 — 시장 안정 닻(anchor) 도입 (원칙 채택)

**배경:** 유저 회사들이 시장에 공급을 시작하면 유저들은 마을 밖 자원 노가다를 그만두려 한다. 이때 소수 유저의 **담합**이나 **공급 공백 → 포롱마을 경제 붕괴** 위험이 생긴다.

**결정:** 마을이 직접 운영하는 **공영 NPC 기업(주민회사)** 을 둔다. 기본 품목을 **항상 중간 품질·기준 가격**으로 공급해 시장 가격의 닻 역할을 한다 — 담합 천장·덤핑 바닥·공급 공백 최소선을 받친다. 유저는 산출물 **품질이 중간보다 높으면 더 비싸게 받아 경쟁력**을 가진다(안정과 경쟁 양립). 세부 파라미터(점유율·발동/철수 조건·기준가 산정·재원)는 미정 → P1.

**관련:** `concept.md` §11-A, INBOX-021. 후속 P1 미결.

### DL-E004 조작 방향성 = MC답게 (직접 건설 + 자동화 core 제어 + 산재 + 대시보드)

**배경:** "회사"라는 추상 메뉴 게임이 아니라 마인크래프트다운 손맛을 살릴지 결정 필요.

- 경영 판단은 **대시보드**로: 현재 산출량·평균 품질·시장 평균가·주민 선호도·시장 안정 공급 가능 여부.

**관련:** `concept.md` §2, INBOX-020. 후속 P1: 자동화·노동 결합 함수, 산재 모델 수위.

### DL-E003 주민 구현 = 하이브리드 2층 구조 (집계가 진짜, 빌리저는 얼굴) — P0-① 해소

**배경:** "살아있는 주민" ↔ "거시경제의 일원"이 서로 다른 방향으로 당겨 컨셉이 두루뭉실했다. 개체 기반(개별 빌리저 AI)은 몰입은 최고지만 서버 부하로 인구 수백 명 한계 → "마을→국가(수만 인구)" 비전과 충돌. 순수 집계는 부하·확장은 좋지만 "눈에 보이는 빌리저" 맛이 약함.

**결정:** **하이브리드.** ① **집계 경제(진짜)** — 인구를 계층(strata) 풀로 관리(인구수·평균학력·임금·소비성향·욕구충족·출산기여), 합이 곧 총 노동공급·품목별 수요·세수·실업률·생활수준. 집계 연산이라 **수만 인구도 부하 없이** 확장. ② **대표 빌리저(얼굴)** — 계층당 수십 명만 마을에 가시화, 상태가 계층 집계의 거울. 유저의 "NPC 고용"은 "계층에서 N명 고용"으로 번역. 결과적으로 **거시 자체가 살아있는 생물**이고 빌리저는 그 맥박.

**관련:** `concept.md` §4(전면 재작성)·§8(거시=집계 창발), INBOX-006(PROMOTED). 플랫폼(P0-③) 부하 제약 해소.

### DL-E002 장르 정체성 = 마인크래프트 경제발전 시뮬레이션 (계급 구조 경제)

**배경:** 시스템을 11개 기둥으로 나열했으나 "결국 뭐 하는 게임이냐"가 한 문장으로 안 떨어져 모호했다.

**결정:** **빅토리아3 / Capitalism Lab / 심시티 계열의 경제발전 시뮬**로 못 박되, 결정적 차이는 **자본가 계급을 실제 유저가 플레이**한다는 것. **유저 = 자본가·기업가 계급(경제 엔진)**, **주민 집계 = 노동·소비 계급(경제 몸통)**, **대표 빌리저 = 그 얼굴.** "돈 벌기"는 그라인딩이 아니라 발전하는 경제에서 자본을 굴리는 행위이며 주민 계급의 건강(소비력)에 묶인다(착취 자기조정). 발전 사다리(마을→국가)는 집계 지표로 게이팅, 거시 사이클은 집계에서 창발.

**관련:** `concept.md` "장르·정체성", DL-E003(하이브리드)와 한 쌍.

### DL-E001 단일 마을 + 유저 의회 모델로 전환 (4개 마을·주민 주권 정치 폐기)

**배경:** 초기 안은 ① 4개 마을(포롱·퍼렁·푸룽·파랑) 분할·소속 ② 정치는 주민의 영역(유저 출마 불가, 진보 vs 보수 주민 투표)이었다. 정치 주체를 재검토하며 두 축이 함께 무너졌다.

**결정:** **단일 마을 + 유저 의회.** 4개 마을 분할 폐기(정치 주체가 유저로 바뀌며 4개 진영의 의미 소멸) → 유저들이 함께 키우는 단일 마을(마을→도시→대도시→국가), 재정 단일화. 정치 주체를 **주민 → 유저 의회로 역전**: 게임/DB가 지표를 보고 안건 자동 발의 → 유저 토론 → 찬/반 과반 가결 → 정책 발효(수치 트레이드오프). 주민은 노동·소비·투자만, 정치는 안 함. 진보/보수는 명시 정당 대신 안건 성향(성장·자유 vs 분배·환경)으로 잔존.

**관련:** `concept.md` §6·§10, INBOX-018(대체), INBOX-010·011·012·015(폐기). 커밋 6e64545. 잔존 미결: 의회 권한 cap·안건 시스템 설계(P0-②).
