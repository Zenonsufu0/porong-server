## 경제 지표 마스터 리스트 (CORE 안건 입력값)

> CORE SYSTEM이 감시하다 임계 조건이 맞으면 안건을 발의하는 지표 풀(§6). 모든 지표는 집계 모델(§4)에서 계산 가능해야 한다.
> 표기 — ★ = 시즌1 1차 핵심 / 그 외 = 후보. **(관)** = 관측 지표(안건을 *부름*) / **(정)** = 정책 상태값(안건이 *바꿈*). 둘은 피드백 루프로 물린다.

**1. 주민·인구** (경제의 몸통 — 가장 중요)
- ★ 총인구 / 인구증가율·출생률 (관) · ★ 실업률 (관) · 평균 임금 / ★ 임금 중위값 (관)
- ★ 소득 불평등(지니계수) (관) · 중간층 비율 (관) · 평균 노동시간 (관) · ★ 욕구충족도(필수품/사치품)→빈곤율·생활수준 (관)
- ★ 행복도 = 합성지표 (아래 식)
- → **주민 안건**(최저임금·노동시간·복지·산재·출산·이주)이 §4 노동=소비 루프를 의회가 직접 조정. 안건표 agenda_catalog §6.

**2. 시장·물가**
- ★ 물가지수(CPI)·인플레이션율 (관) · ★ 품목별 수급(공급÷수요)→품귀/과잉 (관)
- ★ 독점도(품목별 1위 점유율·HHI) (관) → 독점금지 안건 → 국유 주민회사 신설(§11-A) · 평균 품질 분포 (관)

**3. 거시·경기**
- ★ 마을 총생산(GDP)·성장률 (관) · 경기 국면(호황/과열/침체, 합성) (관)
- ★ 신용 건전성·대출 부실률·기업 도산율 (관) → 신용경색 위기 안건 · 자산 거품(주가÷실적) (관)
- 기준금리·통화량·신용(대출) 기준 (정) — **통화정책은 의회 안건**(A안), 단 파국 임계에서 **CORE 비상 안전장치**(§6) 발동. 안건표 agenda_catalog §5.

**4. 재정** (모델 §10)
- ★ 정부 부채(stock+이자) / 부채-세수 비율 (관) · 세수 (관) / ★ 재정수지(흑·적자) (관)
- 현재 세율(법인·소득·거래·재산세) (정) · 공공지출 배분(인프라·교육·복지·치안·환경+주민회사 재원) (정)

**5. 산업·기업**
- ★ 회사 수 / 산업 다양성 (관) · 가동률 (관) · ★ 산재 발생률 (관, DL-E004 연결) → 안전규제 안건(주민 §6)
- 도산율 / 평균 수익성 / 티어 분포(1·2·3차) (관)
- → **산업 안건**(자동화 규제/장려·부실기업 구제/정리·전략 산업 육성·중소기업 보호) agenda_catalog §7.

**6. 인프라**
- ★ 전력 공급/수요 예비율→정전 (관) · 수도 공급 (관) · ★ 운송·물류 용량 vs 물동량(병목) (관)
- 전기세·요금 (정, 에너지 정책 연동)

**7. 환경**
- ★ 환경오염 수치 (관, **누적 stock = 유입−흡수 / 모델은 §1-A**) · 녹지/발전칸 비율(개발 vs 보전) (관·정)

**8. 연구·교육 (인적자본)**
- ★ 평균 학력·고학력 비율 (관) · ★ 학교 수용량 vs 수요 (관) · 연구 진척도(기술 트리) (관)
- → **교육 안건**(학교 건설·R&D 예산·무상교육) agenda_catalog §9. 연구 방향은 의회 밖 별도 시스템(§7). **기술 트리가 전 안건을 게이팅**(§7).

**9. 부동산·토지**
- 가용 토지(발전칸) / 공실률 (관·정) · 토지 가격·투기(거래 회전율) (관)
- → **부동산 안건** agenda_catalog §10 (토지 공급·투기·공공임대·조닝). 그린벨트는 환경 §1.

**10. 금융·자본시장**
- 주가지수·변동성 (관) · 주민 저축률·가계부채 (관)
- → **금융 안건**(주식 규제·가계부채·자본이득세) agenda_catalog §11. 중앙은행=화폐 중추 단일(monobank, §10·DL-E039).

**11. 치안·범죄**
- ★ 범죄율 / 지역별 치안 수준 (관) · 경찰·교정 시설 수용 vs 수요 (관)
- **범죄율 = 활동 창발:** 별도 변수가 아니라 `f(실업률↑, 빈곤율↑[욕구충족↓], 불평등↑)` — 이미 추적하는 주민 지표에서 나온다(오염과 같은 창발 패턴).
- **구현 = 집계 % (유저 개인 범죄 아님):** 범죄율은 주민 인구 대비 비율로 집계(하이브리드 §4 — 개별 범죄자 NPC를 돌리지 않음). 거래가 전부 시스템 경유(§1-A)라 유저의 물리적 범죄 여지 자체가 구조적으로 적다.
- **대가 = 유저 지갑에 손실로 체감:** 유저는 가해자가 아니라 **피해자+의회 대응자**. 범죄율↑ → 산출물/상점 박스 재고 일부 **도난**(박스에서 % 차감) · 매출 누수 · 설비 파손(가동률↓) · 부동산↓ · 행복도↓ → 경제 부메랑. 대시보드에 "도난 손실 %" 표시 → 치안 안건 찬성 동기. (교도소 트리거 수감자 = `범죄율 × 인구 × 검거율`.)
- **유저 경제 범죄는 별개 트랙:** 절도가 아니라 탈세·담합·시장 조작 — 범죄율(주민 현상)과 분리해 부정행위/시장 감시(P2)에서 다룬다(담합은 주민회사로 견제, §11-A).
- → **안건 축:** 단속(경찰서·교도소·순찰 = 질서) vs 예방(복지 = 분배). 건설형 안건(경찰서/교도소)은 **가용 토지↓ + 세수 소모** 트레이드오프(§6 건설형 안건·§9 토지).

### 행복도 — 단일 마법 변수 대신 하위 지표 합성
```
행복도 = w1·욕구충족도 + w2·실질소득(임금/물가) + w3·고용안정
        − w4·노동시간 − w5·산재율 − w6·불평등 − w7·범죄율
```
→ "행복도 하락 → 복지/치안 안건 발의"가 **이미 추적 중인 숫자들에서 자동으로** 나온다(별도 변수 불필요).
