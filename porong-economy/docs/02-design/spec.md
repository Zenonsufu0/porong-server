| **콘텐츠 모드** | 식품(Croptopia+FD)·**재료=Immersive Engineering**(+Petroleum)·가구 모드 | DL-E069·E082 |

## 2. 시뮬레이션 엔진

**갱신 = 주기 틱**(실시간 N분, 정확 길이 P1) — DL-E075. 매 경제 틱마다 아래 순서로 일괄 갱신(순서는 기획 인과 사슬 따름: 생산→시장→소득→소비→지표).

| # | 단계 | 내용 | 근거 |
|---|---|---|---|
| 1 | 생산 집계 | 산출물 박스 입고분 → 회사 재고·시장 공급 반영. **가동률 = 충원·학력·안전·외생 페널티 + 전 틱 전기세 게이트(lag)** | DL-E029·E026·E086 |
| 2 | 시장 정산 | 품목별 수급 + 유저 호가 → 수요곡선으로 판매량·매출·시세(B2C 완성품 / B2B 중간재·연료) | DL-E026·E027·E067 |
| 3 | 주민 동학 | 가계 흐름(소득→소비→저축→자산배분)·노동/고용/임금·인구(출산·사망·이주). **자산효과 소비·배당 소득 = 전 틱 주가/배당(lag)** | DL-E038·E040·E041·E044·E045 |
| 4 | 창발 지표 | 오염·범죄율·CPI·실업·GDP·지니·행복도·산재율 산정 → `indicator_snapshot` 기록 | DL-E049, indicators |
| 5 | 재정·중앙은행 | 세수(법인·소득·거래·재산)·공공지출·금리·통화량·부채 이자 | DL-E012·E023·E039 |
| 6 | 전력 정산 | 그리드 수급·발전 믹스 → 전기세 갱신(이번 틱 연료가 기준). 다음 틱 생산(1) 게이트에 적용 | DL-E083·E087·E093 |
| 7 | 자본시장 | 상장사 fair 재계산·net_flow·주가 갱신(OHLC)·배당 배분·controller 51% 판정. 다음 틱 주민동학(3)에 적용 | DL-E089·E090·E093 |
| 8 | 마을 진화 체크 | 다인자 임계 → 시대/단계 해금(또는 쇠퇴), `village_state` 갱신 | DL-E051·E062 |
| 9 | 의회 트리거 | 지표 임계 → 안건 후보. 정기 발의는 하루 1회(틱과 별개 스케줄) | DL-E025 |
| 10 | 영속화·push | DB 주기 저장 + 클라 패킷 push | DL-E035·E032 |

- **거시 사이클은 별도 엔진 없음** — 위 3·4단계의 집계 결과로 창발(인플레·경기·거품). design §8·DL-E049.
- **1틱 lag 원칙(DL-E093):** 전기세(6)·주가/배당(7)은 틱 후반 갱신 → **다음 틱**의 생산 게이트(1)·자산효과 소비·배당 소득(3)에 적용. 같은 틱 내 순환 의존(소비↔주가, 생산↔전기세) 회피.
- **해소 상태(아래 섹션):** 산출물 박스 입고 시점 = **입고 즉시 처리 / 시장 공급 반영은 틱**(§4·§2-1) · 메모리↔DB 동기 주기 = §3 [권장] · 계약 정산 주기 = **틱 배치**(§4, DL-E028 해소).
- **부트스트랩 day-0 점화(DL-E098):** 시즌 시작 시 **시드 초깃값**으로 노동=소비 루프 점화 — ① 시작 주민 인구 풀(유저당 가용 노동 충분) + 시작 고학력 풀(연구원 즉시) ② 시드 국고(세수 0에서도 N일 공공·주민회사 임금·이전소득) ③ 시드 주민 저축(1~2틱 필수소비 버퍼). **핵심 인프라(학교 1·1차 발전소·경찰서) = CORE day-0 자동 배치**(의회 인질 해방, 의회는 증설·전환만) + 주민회사 day-0 즉시 가동(주민 자영소득 부재 보완). 시드 수치 P1, 점화 구조는 결정.
- **집계 평활화(DL-E099):** 소비·고용 등 집계 갱신에 이동평균/변화율 cap → 단일 행위자(대량 해고·폐업)의 1틱 전면 충격으로 30명 소규모 경기가 톱니 진동하는 것 완화. 계수 P1.

## 3. 데이터 계층

- **DB = SQL 서버(Postgres/MySQL)** — DL-E035. 엔티티 전체(`data_schema.md`: 핵심 맵 17종 + 위성 6종 = **23**)는 그대로 구현: player·company·industry·financials·inventory·population_stratum·employment·item·market_listing·market_state·shareholding·stock_price_history·contract·techtree_node·treasury·agenda·vote·party·land_plot·deposit·loan·bond·indicator_snapshot.

## 4. 생산·시장 파이프라인

> **자본 제약 = 시간·규모·고급자원에서 발생(원자재 가격 아님) — DL-E120.** 마크 무한공급으로 *저가 원자재*는 무자본 부트스트랩(손채굴) 가능하나, **부는 가공·자동화·규모에서 나옴** → 큰 산출 = 설비(자본) + 시즌 28일 한정(시간 희소). **자본 = 시간·규모를 사는 것.** 자본 희소성 유지 3장치: ① 손채굴 저효율(DL-E080, 규모는 설비 필요) ② 고급 자원(반도체 소재·희토류)은 야생에 없음 → 부지 고급 설비/수입(자본 집약) ③ 시즌 시간 제약.

**생산 (회사 → 산출물):**
- Create 가공으로 생산. 레시피·가용 기계는 **KubeJS + 시대 게이팅**(DL-E031·E061·E062). 채용 주민 없으면 설비 정지(DL-E004), 학력 매칭이 효율·고장률·복구 결정(DL-E047).
- **오프라인·청크 언로드 생산(DL-E095):** 유저 오프라인/청크 언로드 시 **CORE가 마지막 가동률·고용 기준으로 추상 대체 생산**(물리 Create 미가동이어도 박스 입고를 집계로 대체) → **청크 강제 로딩 불요**(1인 서버 부하 회피)·라이트유저 불공정 도산 방지. **원료 재고 소진 시 추상 생산도 정지**(공짜 생산 X). 레이트 계수 P1.
- **원료·가동(§11):** 원료 = 야생 손채굴(부트스트랩) + **부지 무한 채굴 설비**(주력, DL-E080). 설비 **가동 = 노동(DL-E004) AND 전력(전기 시대, DL-E081) 게이트** — 부족 시 산출↓·전기세↑.
- **산출물 박스(입고 즉시 처리)** — DL-E029·E026: ① 업종-품목 검사(불일치=환금 불가) ② 품질 산정(품질 전파, DL-E064) ③ 오염 유입 기록(DL-E010) ④ 회사 재고 등록. **①~④는 입고 즉시, 시장 공급 반영은 매 틱(§2-1).** 유저가 **호가(가격) 설정**(DL-E027).

**생산 코어 — 노동·품질·고장 (DL-E086):** Create는 동력으로 물리 생산(체스트·박스 자유). **경제 판매 = 산출물 박스(회사 합산 창구)만** — 체스트는 창고일 뿐(판매 불가). Create 내부 안 건드림, CORE가 박스에서 처리.
- **판매 속도 = 회사 가동률 cap:** 가동률 = f(충원율[배치 인원/설비 요구 인원] × 학력 매칭 − 안전) **× (1 − 외생 페널티)**. 요구 인원 = 설비 규모 비례 → **큰 공장=많은 고용**(노동=소비 결합). **회사 단위 합산**이라 박스·체스트 수 무관(우회 불가). 0명=정지(DL-E004).
  - **외생 페널티 슬롯(DL-E093·E131 — 외생 지표→생산 배선):** 마을 외생 지표가 가동률을 깎는 통합 채널 = `외생 페널티 = k₁·오염 초과분 + k₂·범죄율 + k₃·누적 산재 + k₄·파업 강도`. **오염 → 노동생산성↓**(design §1-A "경제 부메랑" 배선), **범죄 → 설비 파손·매출 누수**(indicators §11 배선), 산재 평판(위 §산재), **파업 → 노동 거부·생산 차질**(사회 갈등 DL-E129 배선). **행복도는 합성지표(오염·범죄·산재 포함)라 슬롯에서 제외**(이중계산 방지) — 안건 트리거·이주·파업 발동으로 작용. 계수 kᵢ = P1.
- **고장:** 충원율 기반 확률·길이로 창구 일시 정지 → 복구까지 환금 0(직결이면 backpressure로 기계 idle, 체스트 버퍼면 안 팔리고 쌓임=보관비·과잉 DL-E063). 복구 속도=배치 학력·감독관(DL-E047). per-회사 상태(우회 불가).
- **품질(박스 stamp, data component):** `품질 = weakest-link( 레시피가 선언한 全 분야 stage ) × 학력`. **각 stage = max(구매 품질[②, 박스 추적], 회사 분야 특화 역량[①, 연속 투자 DL-E062])**. Create 내부 NBT 추적 회피 → 회사 집계.
  - **exploit 차단:** weakest-link는 **유저가 찍은 트리가 아니라 레시피가 선언한 全 가치사슬 stage**로 평가. **미투자+미구매 stage = base-low 디폴트** → 예: 소재 특화 X·철강 자체생산(저품질, 내부라 CORE가 철은 못 봄)이어도, 레시피가 "소재 stage 존재"를 알려 **회사 소재 역량(base-low)**으로 채점 → 최종 cap. 내부 중간재 안 봐도 미투자 stage 자동 저품질. 역량=연속(살짝 찍기 불가). 수율(DL-E064) 동일.
- **트리·분업:** 레시피(KubeJS)가 재료 강제(자동차=철강+부품) → 자체생산(분야 노드) or 구매. **분야 병렬(특화 자유)·강제 트리 의존 X** → 제조만 찍고 철강 고품질 구매 가능(weakest-link 보상) → **분업 유리**, 수직통합=전 분야 특화 불가로 어중간(DL-E067).

**산재·노동안전 (DL-E092 — 생산코어 `−안전` 항 구체화, DL-E004 환류):** 고장(물적 차질)과 별개 채널 = **인적 피해**. "안전 안 쓰고 빡세게 굴리기"(착취)의 확률적 비용.
- **산재 발생 = 확률 이벤트(per-회사, 매 생산 틱):** `산재 확률 = 업종 산재위험도 × 가동 강도 − 안전 투자`.
  - **업종 산재위험도** = 업종 마스터 테이블(design §1-A) 속성 → 위험 업종(광업·중공업)일수록 높음.
  - **가동 강도** = 충원율 × 노동시간(과로, agenda §6) × 생산 레이트 → **무리하게 굴릴수록(고레이트·장시간·과충원) ↑**.
  - **안전 투자** = 감독관 배치(고학력 주민) + 안전설비(비용·연구) → 확률 ↓. **감독관 미배치 = 기본 위험(높음, DL-E004).**
- **환류 4채널(착취 → 비용·평판 압력):**
  1. **즉시 비용:** 산재 → 보상금·의료비(손익 비용 −, DL-E029 산재 항).
  2. **노동력 손실:** 다친 주민 일시 이탈(충원율↓ → 가동률↓·고장↑, 생산코어 환류) / 중상 = 영구 이탈 / 최악 = **초과 사망**(인구 동학, DL-E048).

## 5. 금융·재정

**중앙은행 = 단일 통합점(monobank)** — DL-E039·E023. `treasury` **단일 상태 행**(data_schema)이 재정(국고·부채·세율·지출배분)과 통화(통화량·기준금리·신용기준)를 한곳에 보유 → 모든 화폐 흐름이 중앙은행 경유(design §10). 예금·대출·발권·금리·국고·국채를 전부 담당, 민간 은행 없음.

- **재정수지 = 세수 − 공공지출** (회계, 매 틱 §2-5에서 계산).
- **세수 4종(S1 내수, 관세 없음) — design §10:**

  | 소득세(누진성 속성) | 주민 임금(`population_stratum`·employment) | 틱 가계 흐름(§2-3) |

### 5-2. 통화 (중앙은행)

- **발권 = 통화량 소스/싱크:** `treasury.통화량` 증감. **소스** = 중앙은행 대출 실행·국채 매입·긴급 유동성(§6 거시 안건). **싱크** = 세금·수수료 징수·대출 상환. 인플레는 통화량(소스>싱크) 또는 수요 초과에서 창발(design §11·indicators). **(확정 — DL-E078):** 위 소스/싱크 매핑 채택 — 기획은 발권=소스/싱크·세금=싱크만 명시 → 대출 실행·국채 매입·긴급 유동성 = 소스 / 세금·수수료·상환 = 싱크로 확정.
- **기준금리(`treasury.기준금리`):** 통화정책 레버 — 대출 차입비용·예금 금리·주민 자산배분 스위치(예금↔주식, DL-E045)에 작용. 정책 상태값(의회 안건 §6, 파국 시 서킷 브레이커 강제).
- **예금/대출/신용:**
  | 엔티티 | 주체 | 비고 |
  |---|---|---|
  | `deposit` | 주민(stratum)·유저(player) | 잔액·적용금리, 중앙은행 예치 |
  | `loan` | 회사·유저 대출 | 원금·금리·잔액, `treasury.신용(대출)기준`이 한도·기준 조절(가계부채·신용 규제 안건) |

- **통화 보존 항등식(DL-E094):** 통화량 = Σ(주민현금 + 회사현금 + 유저현금 + 예금). 발권(소스)·세금/상환(싱크) 외 **어떤 정산도 화폐 총량을 바꾸지 않음** — 매 틱 항등식 검산(불일치=누수 버그). 시장·배당·계약·임금은 전부 이전(transfer).

### 5-3. 틱 처리 (§2-5 세부)

매 경제 틱 5단계에서: ① 세수 집계(법인·소득·재산; 거래세는 거래 시 누적분 합산) → ② 공공지출 집행(배분 비율대로 차감, 주민회사 운영비 포함) → ③ 재정수지·부채·국채 이자 갱신 → ④ 통화량 소스/싱크 정산 → ⑤ 금리·신용 정책 상태값은 의회 의결 시에만 변경(틱은 적용만). 결과 지표(인플레·부채-세수비율·신용 건전성)는 §2-4에서 `indicator_snapshot` 기록 → §6 안건 트리거.

### 5-4. 자본시장 (주가·유동성·배당·M&A — DL-E033·E034·E085·E088·E089)

**상시 ON**(DL-E085) — 거래소 = 커스텀 모드 GUI(실시간 캔들·호가창, DL-E032). 진입 게이트 = IPO 재무 요건(§5-5)뿐, stage-gate 아님. **상품 = 현물 주식 매매 + 배당 + (선택) 회사채만 — 파생(옵션·선물) 미채택**(30명·주식 초보 다수 환경에서 복잡도만 올리고 핵심 재미와 무관, DL-E091).

**(A) 주가 형성 = 펀더멘털 앵커 + 수급/심리** — 매 경제 틱 상장사 주가 갱신.
1. **공정가치(fair, 펀더멘털 앵커)** = 재무 기반 내재가치 — `fair = (자기자본·w_book + 정규화 최근이익·PER_mult·w_earn) × (1 + 이익추세 g)`. financials(자본·손익·성장률)에서 산출, 적자 지속 → fair 하락. **주당 공정가 = fair / shares_outstanding.** 계수 P1.
2. **시장가 갱신** = 공정가 회귀 + 순주문흐름 + 거시심리 + 노이즈 —
   `price_{t+1} = price_t × (1 + α·(fair_share/price_t − 1) + β·net_flow + γ·macro + ε)`
   - **α(평균회귀):** 시장가가 공정가로 수렴 → **실적 없는 거품은 자연 조정**(인위적 가격 캡 아님 — 무캡 자유 플레이 일관).
   - **β·net_flow(수급):** 순매수 압력(아래 B), 단기 모멘텀.
   - **γ·macro(거시):** §8 사이클 — 호황 자산거품(+)·폭락(−). 주가지수 ↔ 거시 양방향(E).
   - **ε:** 소폭 노이즈(시세 생동·캔들). OHLC → `stock_price_history` 기록.

**(B) 유동성·수급 = 주민 집계 풀(베이스) + 유저(능동)** — net_flow = 주민 순매수 + 유저 순매수, 거래량 정규화.
- **주민 집계(주력 베이스, DL-E045):** population_model 자산배분이 저축 일부를 주식 풀로 — 주식 비중 = f(금리 스위치[예금↔주식]·시장 모멘텀·불평등). 매수 배분 = **시총 비례 인덱스식 + 펀더멘털 추종**(고fair·고배당 선호), 개별 종목 심리 시뮬 X. **passive**(배당·차익만·무의결, DL-E034). 저금리·강세 → 주식 스위치 → 전 종목 net_flow(+); 고금리·약세 → 환매(−).
- **유저(능동):** 거래소 지정가/시장가 호가 → net_flow 직접. 특정 종목 집중·M&A 매집 가능.
- **호가 매칭 = 즉시 체결 단순화(DL-E093):** 틱 넘어 존속하는 미체결 지정가 주문장(order book) 대신, **주민 집계가 시총비례 유동성 공급자**로 상대편 체결 깊이를 즉시 제공 → 유저 주문은 그 깊이에 즉시 체결(대기 주문 엔티티 불필요). 체결가가 price 수급항에 피드백. 깊이·임팩트 계수 P1.
- **유저 자본 = price impact(주가 능동 변동, 무캡 일관):** 유저 순매수/순매도가 net_flow에 직접 반영 → **자본으로 주가를 밀어 움직일 수 있음**(펀더멘털 무관 단기 부양·하락). impact 강도 = **거래량 대비 주문 크기(시장 깊이)** → **유동성 얇은 소형주는 적은 자본으로도 크게 출렁**(작전·세력 용이), 대형주는 둔감. **α 평균회귀가 자연 견제:** 인위 부양분은 공정가로 회귀 → **펌프앤덤프는 단기 가능하되 유지하려면 자본을 계속 묶어야 함**(지속 비용), 세력 이탈 시 하락. 무캡이되 무한 조작은 시장 메커니즘으로 막힘(인위적 차단 아님) → 주가 부양 방어(M&A §D)·소형주 작전·펌프앤덤프 등 자본시장 PvP 성립. impact 계수·깊이 함수 P1.
- **정보 비대칭 = 주민 스마트머니 / 유저 모멘텀 (DL-E091):** 유저는 재무제표(PBR·financials)를 정독 못/안 함 → 대신 **주민 집계가 fair를 자동 추종(스마트머니)해 주가에 반영** → **주가 추세 자체가 펀더멘털 신호**, 유저는 분석 없이 추세 추종·증폭(모멘텀). **반영 lag**(주민이 펀더멘털 반영하는 지연) → 발빠른 유저는 주민보다 먼저 선반영해 차익 = **깊이 파는 유저 학습 보상**(무캡). 중앙은행·정부 = **직접 매매자 아님** = 금리(시장 전체 유동성 수도꼭지)·정책으로 환경만 설정. lag·계수 P1.
- **시장 깊이 = 주민 주식 풀 규모(인구·소득·경기, DL-E091):** 주식 풀 = 주민 인구 × 1인당 저축 × 주식배분비율. **깊으면**(후반·호황) 펀더멘털 강반영·거래 활발·유저 농단 불가 / **얕으면**(초반·불황) 주가 둔감·묶인 자본·**대주주 버티기**(매수자 없어 못 팖)·M&A 매물 부족. **주민 총자본 vs 유저 자본 비율 = 자본시장 밸런스 근본 다이얼**(주민 ≫ 유저 → 펀더멘털 우위 / 주민 ≈ 유저 → 유저 작전 용이). 인구 = 모든 집계(노동·소비·자본·세수)의 분모(DL-E003·population_model).

**(C) 배당 → 노동=소비 루프 환류** — `dividend_policy`(이익 중 배당 비율, 회사 선택), 결산 틱마다:
- 배당총액 = 이익 × dividend_policy → 지분 비례 배분(`shareholding`). **주민풀 배당 → 가계 소득**(population_model ① 자산소득 → 가처분소득 → 소비) → **노동=소비 루프 환류.** 유저 배당 → 개인 지갑(`cash`). **배당 = 실현 소득 채널**이지 자산효과(미실현 평가익) 아님 → 자산효과 산식(E)에서 제외(이중계산 방지, DL-E093).
- 트레이드오프: 배당↑ = 보유동기·주가 매력·소비 자극 / 재투자↑(배당↓) = 자기자본↑→성장→미래 fair↑. 주가에 양방향(즉시 매력 vs 미래 펀더멘털).

**(D) M&A·적대적 인수 (DL-E034 — 위상: 부차적·리스크 관리형)**
- **매집:** 유저가 거래소에서 타 상장사 유동주식(주민풀 보유분 + 타 유저 매도분) 매수 → `shareholding` 누적.
- **경영권 = controller = 유저 holder 중 max(shares)**(오너식). 매집으로 max 역전 시 **경영권 이전** → 관리 GUI·권한이 새 최대주주에게, 기존 대표 주주로 강등(DL-E034).
- **안정 지배 51%+ = 인수 면역**(유동주식 49% 미만 → 과반 매집 불가). **비상장·1인 100% = 유동주식 0 → 인수 불가.**
- **적대적 인수 성립 = 51% 미만 대표의 상장사 한정**(지분 관리 방심 = 본인 책임). 호경기 유동성·변동 활발 시 매집 기회↑(응징 드라마).
- **방어:** 대표 지분 추가 매입(**51% 도달 = 영구 면역**)·배당으로 주가 부양(매집 비용↑)·유보로 fair↑(주가↑). → **리스크 관리(51% 확보) 잘하면 거의 안 당함**(부차적 위상). 주민회사(DL-E088) = 정부 51% 영구 → 인수 면역(앵커 가드레일). 시장 거래라 자산 파괴 아님(DL-E008 무충돌).

**(E) 거시 연동 — 자산효과·거품/폭락**
- **주가지수(시총 가중)** = 거시 자산 사이클 지표. 호황→거품→자산효과 소비↑→과열 / 폭락→자산효과 역전→수요붕괴(신용경색, DL-E085 정정). **자산효과 = 주가(미실현 평가익)만** → 주민·유저 보유주식 평가액↑ → 소비↑(DL-E044) → §8 사이클 증폭(양방향). **(배당은 소득 채널 §C — 자산효과에서 제외, 이중계산 방지 DL-E093.)** 서킷 브레이커(§6-4)·거시 안건이 과열·붕괴 시 개입.
- **자본시장 활성화 = 창발(시장 깊이 연동, DL-E091·E085 일치):** 마을 성장(인구·자본↑)·호황에 따라 주식 풀이 커져 시장이 자연히 깊어짐 → **마을=얕음(IPO 자금조달 위주)→도시(전기)=깊음(트레이딩 클라이맥스, 시즌1)→대도시=M&A·작전(시즌2, E134).** 인위 stage-gate 아니라 풀 규모의 창발. 경기 사이클이 풀을 출렁(호황 팽창·불황 수축 → 대공황 시 주가↓+유동성 실종 이중 동결). 의회 인구정책(이민·출산 DL-E017·E043)이 인구→풀→시장 깊이까지 작용 = 정치·거시가 자본시장에 닿음.
  - **투자 트랙 Week1 진입 보장(DL-E140):** 깊이는 창발이되 **첫 상장 매물 시점**은 보장 — IPO 영업요건 완화(3→2일, balance §7)로 **소도시(D4~5) 첫 상장** 가능 → 순수 투자자도 Week1 놀거리 확보(없으면 2주의 절반이 공백). 마을=얕은 IPO 자금조달, 소도시부터 거래 트랙 진입, 도시=깊은 트레이딩 클라이맥스.

**(F) 틱 처리:** ① 상장사 fair 재계산(financials) → ② 자산배분(주민)·유저 호가에서 net_flow 산출 → ③ price 갱신·OHLC 기록 → ④ 결산 틱이면 배당 배분(주민 가계·유저) → ⑤ controller 재계산(지분 변동 시)·51% 판정 → ⑥ 주가지수·자산효과를 §2-4 `indicator_snapshot`·가계 자산에 반영.

### 5-5. 회사 금융 = 자본구조 (조달 3경로·IPO·증자 — DL-E029·E033·E090)

회사 성장 자금 조달 = **3경로 트레이드오프**(자본구조 선택):

| 경로 | 수단 | 대가 | 리스크 |
|---|---|---|---|
| **내부유보** | 이익 재투자(배당↓) | 성장 둔화·주주 불만 | 낮음(자기자본만) |
| **부채** | 중앙은행 `loan` / 회사채 `bond`(DL-E029) | 이자 비용 | 도산·레버리지(건전성 견제 DL-E024) |
| **자기자본** | IPO 공모·유상증자 | 지분 희석·경영권 노출 | 적대적 인수 매물↑ |

- **부채:** 중앙은행 대출(DL-E039) 또는 회사채 발행(보유자=주민/유저, §5-2). 이자 = `treasury` 금리 연동. 부채비율↑ → 도산 위험·신용 규제 안건(§6) 노출. 레버리지로 성장 가속하되 거시 신용경색 시 취약.
- **IPO(상장):** 재무 요건 충족 시 상장 신청(상시 ON, DL-E085).
  - **상장 요건 항목(DL-E034 이월 해소 — 항목 확정, 임계 P1):** ① 흑자 지속(최근 N틱 누적이익 > 0) ② 자기자본 ≥ 최소 자본금 ③ 영업 기간 ≥ 최소 ④ 자산 건전성(부채비율 ≤ 상한). `financials`에서 자동 판정.
  - **공모:** 신주 발행 비율·공모가 설정 → 조달액 = 발행주 × 공모가 → 자기자본 유입. 공모가 = fair 기반 자동 산정(± 유저 조정 범위 P1). 발행분만큼 기존 대표 지분율 **희석** → 51% 가드 주의.
- **유상증자(상장 후 추가 조달):** 신주 발행 → 자기자본↑(fair·성장 재원), 기존 주주 희석. 대표 51% 미만 하락 시 경영권 노출(트레이드오프).
- **자본구조 = 부 전략 레버(무캡):** 유저는 부채 레버리지·증자·배당 조합을 자유 선택 — 보수적(유보 위주·안정) ↔ 공격적(레버리지·증자·고성장 고위험). **투자(주식·배당·M&A)만으로도 부 축적 가능**(생산 없는 트랙 허용, DL-E072 연장 — 인위적 캡 없음). 도산은 유한책임(DL-E029)이나 신용·건전성으로 견제.
- **무자력 분배 가드(DL-E094 — 발권강탈·채권/계약 먹튀 공통 차단):** **부채 미상환·자본잠식 회사는 배당·소유주 분배 금지**(자본적정성 이상에서만). 도산 청산 = **잔여현금 → 채권자(중앙은행·회사채) 우선변제**, 도산 직전 N틱 대규모 소유주 분배는 **clawback**. → 대출(발권)→배당 빼돌림→고의도산→유한책임 악용 차단. 임계 P1.
- **틱 처리:** 부채 이자 = §5-2 통화 틱 차감. IPO·증자·배당 = 결산 틱 이벤트. 도산 판정(자본잠식·디폴트) = §2 동학.

## 6. 의회·안건 엔진

**구조 확정 — 반(反)장악 설계(design §6, DL-E006·E007·E025):** 의회 = **부를 둘러싼 이익 정치의 무대**(실세계 이념 아님). 두 축으로 소수 장악 차단: ① **1인 1표 전원 의석**(의석 경쟁 없음) ② **의제 설정권 = CORE 전속**(유저는 발의 불가, 찬/반만).

### 6-1. 출범

**의회 출범 = 마을 단계**(첫 생산·시장 루프 작동 후 — 안건이 의미 있으려면 지표·경제가 돌아야 하므로 day-0 아님; DL-E006·E085). 대도시 진화 안 기다림. 그 전까지 정책 = CORE/운영자 관리(DL-E056). **점진 성장:** 안건 카테고리·종류가 마을 성장에 따라 단계적 해금(의회 자체가 자라는 온보딩·정치 심화, §6-2).

### 6-2. 의제 설정 = CORE (트리거 → 안건 후보)

- **감시:** CORE가 매 틱(§2-7) `indicator_snapshot` 지표를 `agenda_catalog`의 안건별 트리거 조건(지표·임계)과 대조 → 충족 시 **안건 후보 큐**에 등록.
- **안건 데이터(`agenda`):** category · trigger_indicator · threshold · **effect(JSON)** · prereq_tech · status · proposed_at.
- **기술 게이팅:** 모든 안건은 서버 공통 기술 트리(§7)의 **선행 기술(prereq_tech) 충족** 시에만 후보(예: 친환경 전환 = 친환경 발전 연구 후). 미충족 안건은 큐 진입 불가. 세부 선행조건 매핑 P1.
- **카탈로그:** 11개 카테고리(환경·치안·재정·시장물가·거시통화·주민인구·산업기업·인프라·연구교육·부동산·금융자본시장) = `agenda_catalog`. 임계·효과 수치 = **P1 placeholder**.
- **안건 가용성 = 3층 게이트(DL-E085):** ① **진화 단계**(카테고리 해금 — 마을=재정·**노동·인구복지**(실업·이주, E138)·환경·치안 → **소도시**=+산업·인프라·시장물가 → **도시**=+금융·자본시장(기본)·거시통화 → 대도시·국가(시즌2)=+부동산·금융 디테일·전체, §6 분리 DL-E130·명칭 E134·경계 E138, P1) ② **기술 선행**(prereq_tech, 위) ③ **지표 트리거**(위). 셋 다 충족해야 후보 큐 진입 → **의회가 마을과 함께 성장**(온보딩·정치 심화).
- **후보 선택 규칙(DL-E076 해소):** 하루 1건 발의 병목에서 후보가 여럿이면 **우선순위 점수**로 1건 선정 —
  `점수 = 정규화 심각도(임계 초과폭, 主) + 노화 가산(대기 일수) − 재발의 쿨다운 감점`.
  - **심각도 주(主):** 임계를 가장 크게 초과한(정규화) 안건 우선 → 급한 문제 방치 방지.
  - **노화 가산:** 대기 일수에 비례 가산 → 만성 약(弱)위반 starvation 방지(언젠가는 올라옴).
  - **재발의 쿨다운:** 가결/부결된 안건은 N일 쿨다운 감점(P1) → 같은 안건 도배·단조 방지. 단 지표가 계속 악화하면 쿨다운 후 재등장.
  - **긴급 우회:** 매우 심각(긴급 임계)은 정기 큐를 거치지 않고 **긴급 소집**(6-4 2단계), 파국 임계는 **서킷 브레이커**(3단계) → 정기 우선순위는 일상 발의용. 정규화식·노화 계수·쿨다운 길이 = **P1**.

### 6-3. 운영 규칙 (발의·소집·투표 — design §6 확정)

| 항목 | 규칙 | 비고 |
|---|---|---|
| 정기 발의 | **하루 1회**(실시간), 대기 큐에 안건 있으면 **1건** | 틱과 별개 스케줄(§2-7) |
| 동시 안건 | 항상 **1건**만 진행 | 동시 표결 없음 |
| 소집·투표창 | 소집 시 **1시간** 투표 창 | 정기 소집 = 고정 시간대(예: 저녁) 권장 |
| 정족수 | **활성 플레이어(근 2일 내 접속)의 50%** 이상 참여 | 미달 시 **다음날 이월**(폐기 아님) |
| 가결 | 정족수 달성 시 **단순 다수결**(미참여=기권) | |
| 동표 | **합산 재산 많은 쪽 가결** | 자본 쪽 타이브레이커(경제 서버 정체성) |

- **정족수 동적 완화(DL-E098):** 안건 **N회 이월 시 정족수 임계 점감**(50→40→30%) + 데드라인 후 **CORE 기본값 채택**(DL-E056) → 만성 표류로 건설형·게이트 안건이 인질 되는 것 차단. N·점감폭 P1.
- **안건 페이싱 동적(DL-E099):** **위기**(다지표 동시 악화) 시 정기 발의 1일 2~3건 임시 확대(빠른 붕괴 대응) / **가뭄**(트리거 무발) 시 소프트 정책 안건(세율 미세조정·예산 재배분) 주기 발의 → 정치 무대(핵심 재미) 유지.

### 6-4. 3단계 에스컬레이션 (design §6·DL-E015)

1. **정기**(하루 1회) — 위 운영 규칙.
2. **긴급 소집** — 지표가 매우 나쁘면 CORE가 정기와 별개로 긴급 의회 소집(임계 P1).
3. **서킷 브레이커**(CORE 비상 안전장치) — 돌이킬 수 없는 임계(초인플레·신용 붕괴·재정 디폴트 직전)에서만: ① **경고=거부권**(통과돼도 무효) ② **파국 임박=긴급명령**(강제 금리 인상·비상 긴축 등 능동 개입). 3원칙: ⓐ 극단에서만 ⓑ 위험 구간 대시보드 사전 경고 ⓒ 소수 보호 아닌 **시스템 붕괴 방지**. ⚠️ 표시 안건(금리 인하·양적완화)이 주 발동 대상.
4. **실물·인구 서킷브레이커(DL-E096 — 투표 불요·위험밴드 점증):** 통화(위 3)에 더해 **실업률 파국·인구 붕괴 속도**까지 자동 안정자. **파국 임계가 아니라 "위험 밴드"에서 점증 발동**(절벽 단발 충격 회피) — 실업급여 자동 비례 확대·**공공 자동 고용 바닥**·디플레 시 자동 긴급유동성(대칭). 통화 서킷의 강제긴축 충격을 실물 서킷이 완충(구제가 일 속도 의회에만 의존하던 비대칭 해소). **인구 앵커:** 인구 붕괴 속도 반응 **강제 이주 유입**(매력 무관) + **틱당 초과사망·유출 상한(rate limit)** → 비가역 0 수렴 차단(인구=전 집계 분모 DL-E003). 자산효과→소비 계수는 작게 cap(전염 차단). 밴드·계수 P1.

### 6-5. 정당 (무권한 소셜 — `party`)

가치축(성장·자유 / 분배·환경) 표식 또는 친구 모임. **의석·표 배분·당론 강제 없음**(강제 순간 장악 통로) — DL-E007. 단 멤버 자발적 모의·합종연횡하는 **이익 카르텔**로 기능(표 강제 X, 이익 맞으면 같이 던짐). data: name · 가치축 · member player_ids.

### 6-6. 안건 효과 적용 (가결 시)

- **정책 상태값 변경:** effect(JSON)을 대상 상태값에 적용 — 대부분 `treasury`(세율·금리·지출배분·신용기준) 또는 시장·노동·토지 파라미터. 이후 틱부터 동학에 반영(§2). **부결/방치 = 현행 유지** + 트리거 지표 악화 지속(피드백 루프).
- **건설형 안건**(학교·발전소·경찰서·교도소 등): 가결 즉시 **구조물 에셋을 부지 좌표에 자동 건설**(스케메틱 .schem paste 또는 구조물 NBT — NeoForge), 철회 시 롤백. 외부 에셋 서버사이드 로드가 관건([[solo-dev-external-assets]]).
  - **부지·지형 = 예약 슬롯(DL-E077 해소):** 중앙 공용 마을(국유지, §9·design §9)에 **진화 세부단계(16단계 §7)마다 해금되는 공공 건물 슬롯 그리드** — 미리 평탄화·구획된 국유지. 가결 시 CORE가 **건물 유형에 맞는 빈 슬롯**에 paste(슬롯 규격 = 에셋 규격) → **지형 정리 불필요**(슬롯 사전 평탄화). **슬롯 소진 시** 진화로 동심원 새 구역 해금 또는 토지 공급 안건(agenda §10)으로 확장. 동심원 진화(DL-E059)와 자연 통합·좌표 충돌/지형 훼손 없음(1인 개발 친화). 슬롯 수·규격·유형 매핑 = P1.
  - **공공 건물 한계효용·안건 재발 대응(DL-E106):** ① **한계효용 체감** — 같은 종류 N개째 효과 < 1개째(경찰서 2개째 −15%·3개째 −8%…) → 무한 건설 비효율. ② **단속형(경찰서·교도소)은 증상 억제만** — 범죄 근본(실업·빈곤 창발, DL-E011) 미해결 시 **재발은 의도된 신호** → 의회가 *단속(경찰서) vs 예방(복지·일자리, agenda §2)* 노선을 선택하는 정치. ③ **쿨다운(DL-E076)** 이 같은 안건 도배 차단(지표 계속 악화 시 쿨다운 후 재등장) + **슬롯 유한** → 한 유형 독점 건설 = 타 인프라 기회비용. 계수 P1.
- **권한 경계 — 경쟁 O · 시장 파괴 X(DL-E008):** 가격 상한·강제 분배·법인세 확장은 가능하되 **유저 자산 일순 파괴 금지** — ⓐ **국유화 불가** ⓑ 독점 대응도 **강제 분할 없이** 해당 품목에 **국유 주민회사 신설**로 경쟁 주입(design §11-A). CORE가 안건 effect 검증 시 이 경계 강제.
  - **독점/공백 자동 신설(DL-E097, 투표 불요):** HHI>임계(독점) or 공급<임계(공백) 탐지 시 **CORE가 주민회사 신설·경쟁 주입을 자동 발동**(의회 가결 불요, 서킷브레이커식) → **카르텔이 정치적으로 부결시켜도 "경쟁 O" 작동**(사치재·클라이맥스 품목 독점 폭리·B2B 공백 커버). DL-E008 무충돌(분할 X·경쟁 주입 O).

## 7. 기술 트리·마을 진화

**2D 매트릭스(DL-E061·E062):** 세로=**시대(공동 뼈대)** × 가로=**분야(개인 특화)**. `techtree_node`(data_schema): id·era·prereq_node_ids·unlocked·research_progress.

### 7-1. 시대 축 = 마을 진화가 자동 해금 (공동 뼈대)

- **별도 공동 연구 grind 없음(DL-E062):** 마을 진화 다인자 임계 도달 → 해당 시대 노드 일괄 `unlocked`. 시대: 수공업 → 증기 → 전기 → 자동화.
- **마을 진화 게이트 = 다인자 임계(DL-E051, design §5):** 인구 규모 + GDP + 산업 다양성 + 평균 학력 + 인프라 수준 종합 임계 → 단계 상승. 매 틱 §2-6에서 평가.
  | 단계 | 시대 | 해금 | 시즌 |
  |---|---|---|---|
  | 마을 | 수공업 | 1차 산업·기초 인프라 / 성장 후 **의회 출범**(§6) | 1 |
  | 소도시 | 증기 | 2차 가공(강철·중공업)·B2B·인프라 확장·토지↑ | 1 |
  | 도시 | 전기 | 자동차·가전·금융·자본시장 활성·토지↑ (클라이맥스) | 1 |
  | 대도시 | 자동화 | **완전 자동화**(AE2/Mek 무인)·반도체·PC·M&A | 2+ |
  | 국가 | 대외무역 | 전 티어·거시 완전체·최대 토지 | 2+ |
- **세부단계:** 각 단계 × 4 서브레벨. **시즌1(마을·소도시·도시) = 12세부, ~1.17일/레벨**(14일, DL-E133·E134). 명칭 SoT = balance §1. 서브레벨마다 가용 면적·산업·안건 범위 점증(§9).
- **쇠퇴(역진화, DL-E051):** 대공황·인구 붕괴·재정 파탄으로 임계 아래 **지속 시 강등**(도시→마을). 지속 기간 임계 = P1. **위기도구 예외(DL-E096):** 강등이 안건 게이트에서 **금융·거시통화 카테고리를 잠그면 위기 때 금리·유동성 도구가 무장해제** → 거시통화·금융 카테고리는 한 번 해금 후 역진화로 잠그지 않음 + 서킷 브레이커 권한은 진화 단계 무관 상존.

### 7-2. 분야 축 = 개인 특화 (연구원 고용)

- **6분야:** ①농업·식품 ②광업·소재 ③제조·공산품 ④에너지·화학 ⑤전자·첨단 ⑥서비스·금융.
- **해금 메커니즘(DL-E062):** 유저가 회사에 **연구원(고학력 주민) 고용** → `research_progress` 누적 → 임계 도달 시 분야 노드 `unlocked`(내 회사 기술 우위·차별화). **병목 = 인적 자본**(재료 아님). 분야 노드는 prereq로 **해당 시대 노드 선행**(2D: era E 분야 노드 = era E 시대 해금 필요).
- **의회 밖·출범 전부터 작동:** 연구 방향은 개인이 결정, 의회는 출범 후 R&D 예산 안건(agenda §9)으로 **가속만**.
- **구현 도구(확정 — DL-E078):** Civ식 해금(연구원 진척도)을 `KubeJS`로 CORE 연구 진척도에 레시피 해금 연결 = **기본 채택**(CORE 단일 권위·게이팅 일관, 추가 모드 의존 없음). 전용 tech-tree 모드는 트리 UX 부족 시 후보(design §7). 진척 함수·임계 P1.

### 7-3. Create 큐레이션 + 산출품 심화

- **화이트리스트(시대별, DL-E065·E031, KubeJS 게이팅):** 수공업(크랭크·물레방아·맷돌·분쇄휠·프레스·믹서·벨트·기어) / 증기(증기기관·Steam'n'Rails·고속가공) / 전기(Crafts&Additions 전기) / 자동화(기계식 팔·시퀀스 조립·패키저+AE2). 제외=장식·exploit·트리 밖. 트리에 매핑된 부분집합만 레시피 개방.
- **⚠️ Create 기계화 ≠ "자동화 시대"(DL-E135):** Create 기본 기계화(크랭크·벨트·기어·믹서)는 **수공업(마을)부터** 가능 — "자동으로 도는 공장". **"자동화 시대"(대도시·시즌2)는 AE2/Mekanism 완전 무인 자동화·디지털 물류 네트워크**(기계 팔·시퀀스 조립·자동 분류, 인력 거의 없는 스마트팩토리). 즉 *기계화(초반)* vs *완전 자동화(최종)* 구분 — 같은 "자동"이라도 정도 차이.
- **시대 = 부의 천장 상승(DL-E068):** 수공업(식료품·기본의류·도구) → 증기(가공식품·대량의류·기차·배) → 전기(가전·자동차·의약·화장품) → 자동화(PC·스마트폰·고급차·첨단). 초반 저마진 필수재 → 후반 고마진 내구/사치재. Week3~4 고부가 완성품 = 부 경쟁 클라이맥스.
- **산출품 다양화 3레버(DL-E061):** ① 시대×분야 매트릭스 산출품군 ② 가공 체인 깊이(원료→중간재→완제품) ③ 품질 변주(§4·DL-E013).

### 7-4. 교육 인프라

학교(건설형 안건 §6·agenda §9) → 고학력 주민 양성(`population_stratum` education_level 상향) → 연구원 공급 + 마을 평균 학력↑(시대 진행 기여). **인적 자본이 양쪽에 작용**(평균학력→시대 / 연구원→분야).

### 8-2. 거래 UI (시장·거래소 — 주식 트레이드창 스타일, DL-E115)

- **시세 표시:** 현재가 + **등락률**(▲/▼ %, 기준 **1일·7일**) + **캔들 차트**(거래소, DL-E032). 어제·일주일 대비 얼마나 올랐/내렸는지 한눈에.
- **거래 = [매수]/[매도] 버튼**(방향은 클릭 아닌 명확한 버튼 — 오클릭 방지).
- **수량 = 입력창 + 빠른 버튼**(`×1 · ×10 · 한 스택 · 전량/예산만큼`) — 좌클/우클은 ±1 미세조정 보조.
- **체결:** 시장 스팟 = 즉시(주민 집계 깊이 공급, §5-4) → 수령 방식 소량 인벤·대량 수령박스(§4). 주식 = 호가 즉시 체결(DL-E093). 거래세 자동(§5-1).

### 8-3. 렌더·갱신

- **도형+텍스트 직접 렌더**(차트·게이지·수치), 서버 **패킷 push로 실시간 갱신**(DL-E032). DB 폴링 아님 — CORE 메모리 권위 상태(§3)를 push.
- **push 정책(확정 — DL-E078):** **GUI 연 클라이언트만 구독 등록 → 매 틱 갱신분 push + 중요 이벤트(가결·체결·도산) 즉시.** 미연결 클라엔 push 안 함(대역폭 절약). 순수 기술 사항.

### 8-4. 비주얼 (DL-E058)

- **풀 텍스쳐 폴리시 지향**, 1인 개발 비용은 **재사용 테마 세트**로 관리 — 9-slice 프레임/배경 1~2종(stretch) + 버튼 sprite(기본/호버/눌림) + 탭 아이콘. 한 세트로 6패널 통일.
- **제작 순서:** 기능 먼저 텍스처 0(도형+텍스트+바닐라 아이템 아이콘) 완성 → 풀 텍스쳐 폴리시 패스(아트가 기능을 막지 않게 분리). 아트 소싱 = 외부 킷/AI/직접, 외부 우선([[solo-dev-external-assets]]).

### 8-5. 게임 밖 알림 = 디스코드 봇

안건 발의·주가 급등·순위 변동을 디스코드 push(DL-E057) — 친구들이 이미 쓰는 채널, 웹 대시보드 대체. DB reader(§3 아키텍처).

### 9-1. 소스 매핑 (DL-E069·E070)

| 구분 | 출처 |
|---|---|
| 식품 | Croptopia + Farmer's Delight (수백 종 → "식료품" 카테고리 집계, 소수만 개별 시장재 — DL-E070 ⓐ) |

### 9-2. 커스텀 아이템 = 4작업 (DL-E073, task.md 체크리스트)

각 커스텀 = ① **등록(코드)** 아이템 등록 + **품질 속성 hook**(DL-E064 품질 전파 대상) ② **텍스처** 16x16 수십 개 — ⚠️ **주 일거리**, 외부 킷/AI/직접(외부 우선, DL-E058·[[solo-dev-external-assets]]) ③ **레시피(KubeJS 가공 체인)** = B2B 원료→완성품, **2층 시장·품질 전파·시대 게이팅의 구현 지점** ④ **CORE 연동** = 산출물 박스 업종-품목 인식·시장 등록·**소비유형 태그**(필수/내구/사치, DL-E066).

- **품질 부여 범위(DL-E064):** 원자재 포함 전 산출물에 품질. 기존 모드 아이템은 CORE 사이드에서 item별 품질 메타 관리(§4 산출물 박스 품질 산정 지점).
- **규모 감각:** 5~6군 × 품질/티어 변주 = 수십 아이템. 텍스처가 부담, 레시피는 KubeJS로 빠름.

### 9-3. 가공 체인 예시 (레시피 = 가치사슬)

의류: 원단(섬유)+노동 → 옷·신발 / 운송: 강철·엔진·부품 → 차체 → 자동차 / 가전: 부품·전자회로 → 가전 / 전자: 반도체·회로·부품 → PC·스마트폰 / 의약: 화학원료 → 의약·화장품. 깊은 체인 = 분업·B2B 거래·품질 전파 유도(§4·§7).

### 9-4. 대표 빌리저 = 집계의 얼굴 (표현 레이어, DL-E124)

**개념(design §4):** "집계가 진짜, 빌리저는 얼굴." 수십 명 실제 엔티티가 **집계의 거울**(개체 AI 아님, 집계 상태를 행동·외형으로 매핑). 숫자 대시보드를 *살아있는 마을*로 보완 → "스프레드시트 시뮬"이 아니라 "살아있는 도시".

- **수·매핑:** **~40명**(집계 600의 샘플), **계층 비율 반영**(저소득 20·중산 14·고학력 6 등). 인구 성장해도 빌리저 수 고정(샘플), **비율만 변동**(계층 이동 시 외형 전환).
- **상태 시각화(집계 거울):**
  | 집계 지표 | 빌리저 표현 |
  |---|---|
  | 고용/실업 | 회사로 출근 vs 배회·실업 |
  | 행복도 | 표정·활기·파티클(웃음 / 먹구름) |
  | 빈곤·욕구충족 | 복장(깔끔/남루)·구걸 |
  | 산재 | 붕대·절뚝 |
  | 학력 | 복장(노동자/정장/학자) |
  | 소비 | 상점·시장 방문 |
- **행동 루틴(집계 샘플링):** 거주(집) → 출근(고용된 회사 부지) → 소비(상점) → 여가 → 귀가. 시간대·집계 상태로 변화. **개체 의사결정 아니라 집계 상태를 행동으로 표현.**
- **상호작용:**
  - **우클릭 = 대화:** 현재 처지·불만·요구 코멘트(집계 플레이버 + **안건 힌트**) — "임금이 너무 낮아요"·"일자리가 없어요" → 곧 뜰 의회 안건 예고.
  - **고용 = 집계 번역(design §4):** 유저가 빌리저를 회사에 배치(직관적) → 내부 "해당 계층 N명 고용"으로 번역. 빌리저는 그 계층 출근으로 표현(직접 끌기는 시각 인터페이스, 실제는 집계).
- **특수 빌리저:** ① **대표/촌장** = CORE 의제설정 의인화(안건 발의 시 등장·공지) ② **상인** = 주민회사 운영 얼굴 ③ **불만 군중** = 행복도 낮을 때 시위·구걸 떼(거시 경고의 시각화).
- **성능:** 수십 명만 실제 엔티티 렌더, 집계 수만은 가상(DL-E003 하이브리드 핵심). 위치 = 중앙 공용 마을(§11) 거주 + 유저 부지 출근.

## 10. 운영 (시즌·정산·디스코드·부정행위)

### 10-2. 시즌 정산·계승 (DL-E056)

- **정산:** 종료 시 부 랭킹 산정(§8 '내 자산' **순자산** 기준, DL-E094) → 우승자 확정. **부 평가식 확정(DL-E094):** 순자산 = 현금+자산+**지분평가 − 부채**. 비유동 지분 = **min(시장가, fair_share) 또는 N틱 VWAP**(셀프펌핑 종이부 차단). 임계·N P1.
- **계승 = non-pay-to-win만:** 부·자산은 **시즌 리셋**(공정 재경쟁). 영구 보상 = **명예의 전당·칭호·코스메틱**만(시작 우위 없음 → 고인물 방지).

## 11. 월드·토지·인프라

**월드 = 2 고정 차원(DL-E079, DL-E059 갱신):** CORE 모드가 자체 번들 데이터팩으로 등록(3rd-party 멀티월드 모드 불필요 — Multiworld·Superflat Dimensions는 백업).

| 차원 | 역할 | 규칙 |
|---|---|---|
| **메인 월드**(자연지형) | 포롱마을(중앙 공용) + 주변 야생 | 마을=보호·동심원 확장 / 야생=채굴만 |
| **회사 슈퍼플랫** | 유저 부지 건설·자동 생산 | 부지 구매 후 설치, 무한 채굴 설비+Create 자동화 |

사회성 유지(슈퍼플랫에서 다같이 건설·구경, 마을=civic), 차원 고정 2개라 폭증 무관.

### 11-1. 포롱마을 (메인 월드 중앙 공용)
- **중앙 공용 마을:** 의회·공공 인프라·대표 빌리저·건설형 안건 결과물 시각화(design §9). CORE/운영 관리(유저가 짓는 게 아님).
- **보호:** 마을 영역은 설치·파괴 불가(공용 무대 보존, 그 밖=야생만 가능).
- **예약 슬롯(§6-6, DL-E077):** 진화 16 세부단계(§7)마다 해금되는 공공 건물 슬롯 그리드(사전 평탄화). 건설형 안건 가결 → 유형 맞는 빈 슬롯에 자동 paste.
- **동심원 진화:** 마을→소도시→도시→대도시→국가(§7)로 야생을 잠식하며 동심원 확장(시즌1=마을·소도시·도시까지). **확장 예정지 = 국유지**(구매·채굴 불가, DL-E059).

### 11-2. 야생 (메인 월드 주변)
- **역할:** 초반 손 채굴 온보딩 + 초중반 소량 원료 보충(야생→부지→Create→공동체 서사).
- **채굴만 허용(DL-E080):** Create·산출물 박스·체스트 등 **기능 블록 설치 불가**(자동화·대량반출·난개발 원천 차단), 채굴 보조(횃불·발판)만. 손 채굴물은 일반 아이템 → 인벤·운반 허용.
- **리셋:** 시즌 단위(전체) — 손채굴 비효율로 시즌 중 고갈 느림(필요 시 마을 보호영역 제외 리셋).

### 11-3. 회사 부지 (슈퍼플랫 월드)
- **plot-grid·부동산:** 격자 구획, **초기 무상 소구획**(예 32x32, DL-E059) → 부동산으로 확장 구매(골드 싱크·부 경쟁). **부지 안에서만 설치·생산.** 부동산 UI=대시보드 탭(그리드 타일).
- **가용 면적 = f(마을 진화 세부단계) − 의회 정책**(그린벨트·인프라 점유, design §9).
- **무한 채굴 설비(DL-E080):** Create: Ore Excavation/InfiniDrill 등 = 광맥 고갈 없이 원료 산출(산출 = 주민·전력·가동률·기술 제한).

### 11-4. 원료·채굴 모델 (DL-E080)
- **야생 손채굴(비효율·부트스트랩) vs 부지 무한설비(효율·시장 주력).** 효율 격차 → 손채굴은 시장 대량공급 불가 → 부지 자동화가 1차 시장 주력 → **부동산 의미·1차 회사 경제 무결성 유지.**
- **부트스트랩:** 시작 무상 부지 + 시작 키트(바닐라 도구) + 주민회사 기준가 원료(DL-E005/E071) → Create 미숙 유저도 사서 시작.

### 11-5. 전력·인프라 (DL-E081·E083·E087 / design §12)
- **동력 2층:** Create 회전력(기계 물리 구동) / 경제 전력(CORE 추상 그리드, FE 연동). **전기 시대부터(나):** 초반 부지 자급 회전력(Create 빌드) → 전기 시대 → 전력망 전환.
- **그리드(DL-E083):** CORE 블록 2종 — 발전소 FE → **그리드 공급구**, 공장 전기모터 ← **전력 인입구**. CORE 추상 풀이 양끝 측정·송전·정산(물리 와이어 X, 차원 넘나듦). 유저 자기 발전기 안 지어도 됨(price-taker 구매). 가동 = 노동(DL-E004) AND 전력 게이트.
- **전기세 = 원가연동 단일 가격(cost-plus, DL-E087):** = **공공 발전 믹스의 평균 원가**(연료+주민 인건비+운영)+마진. **발전소 호가 없음 — 단일 가격.** 의회가 믹스(석탄/디젤/친환경)로 전기세를 움직임.
- **유저 발전소 = price-taker:** 전기세에 판매, **생산 단가 낮으면 마진**(수력·효율) / 높으면 peaker(디젤). **유저 비용은 소비자 전기세에 영향 X**(조작 불가). 저비용 유저 발전 보태면 공공 고비용 가동↓ → 간접 전기세↓.
- **가동 = 잔여수요 충당(dispatch 큐 없음, DL-E093 정합):** 공공이 **믹스 비율(석탄·디젤·친환경 구성)을 유지하며 잔여수요(총수요 − 유저 발전 공급)를 충당** → 유저 저비용 발전↑ → 공공 총가동↓(특히 고비용 peaker) → 평균 원가↓ → **간접 전기세↓**(위 price-taker 항 정합). 유저=수익 시 자발 발전. 초기 화석 중심, 친환경은 전기 시대+ 해금 후 의회 안건 전환(건설형).
- **발전원 티어(IE, DL-E082):** 수력/풍력(무연료·청정·저출력 320 SU) → 석탄 증기 화력(연료·오염↑) → **디젤(석유, IE 최상위)**. 원자력=시즌2+ 보류.
- **공공 발전 연료 시장 구매:** 공공 발전소도 석탄·디젤 시장 구매 → **정부=큰 연료 수요처**(광업/정유 유저 매출) + 정부 지출·적자 → 재정 정치. 연료가↑→공공 발전비↑→전기세↑ 연쇄.
- **친환경 전환 동학(DL-E087):** 의회 친환경 통과 → 믹스 화석↓ 친환경↑ → ① 친환경 토지 저효율 → 평균 원가↑ → **전기세↑**(전 산업 비용) ② 연료 수요↓ → **석탄·디젤 가격↓**(광업 타격·재정 연료비↓). 전기세↑ vs 오염↓ 정치(석탄 유저 반대 로비).
- **부족:** 전기세↑(증설 신호) + 의회 요금 규제/배급(agenda §8, 주민>산업).
- **발전소 산업(DL-E072):** spot(변동 전기세) or 계약(고정 §11-B). 공공 발전=앵커(기준가 천장, 붕괴·독점 grief 방지). 운송망 동일(design §12). 전기세 함수·믹스 비율·효율/오염 계수 = P1.
- **공공 발전 자동 provision(DL-E098):** 공공 baseload 발전 그리드 = **전기 시대 해금 시 CORE 자동 활성**(`power_grid` init, 의회 가결 불요) → 전기설비 전환 유저가 발전소 안건 스톨에 인질 안 됨. 의회 안건은 *증설·믹스 전환·친환경*만. 공공 연료도 단일 공급자 독점 시 앵커 동적 커버(DL-E097).
