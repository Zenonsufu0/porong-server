## 심화 비전 — 살아있는 경제 공동체 시뮬레이션 (후보)

> 단순 거래 서버를 넘어, **유저의 산업 활동과 살아 움직이는 주민이 서로를 키우며 마을을 국가로 진화시키는** 경제 시뮬레이션을 지향하는 방향 후보. 아래는 풀 비전 스케치이며 전 항목 미확정.

### 0. 핵심 루프 — 함께 자라는 경제
- 초기에는 유저도, 마을도 작다. 유저는 **1차 산업(농사·광질)** 부터 시작하고 주민 수도 적다.
- 유저가 채집·생산하고 **자동화·과학을 발전**시키면 경제가 커지고, 그에 따라 **주민이 늘고(출산) 마을이 진화**한다.
- 마을이 커지면 더 많은 노동력·소비·법·인프라가 생겨 유저의 고부가가치 산업을 떠받친다 → 다시 경제가 커지는 **선순환**.
- 즉 "유저 산업 ↔ 살아있는 마을"이 서로의 성장 동력이 되는 것이 이 서버의 핵심 루프다.

### 1. 산업 사다리 — 무엇이든 만들 수 있다
- 유저는 **원하는 어떤 회사든** 세울 수 있다: 농장, 광산, 2차 가공(원료 수입 → 가공품), 고부가가치 산업(금융·기술·서비스) 등.
- 산업은 **티어**를 이룬다: 1차(원료) → 2차(가공) → 3차/고부가가치. 상위 티어는 하위 티어의 산출물과 주민 노동력·기술을 필요로 한다.
- **원료 희귀도 티어:** 1차 자원은 흔함(철·곡물)부터 **희귀(다이아 등 — 바닐라보다 희귀하게 조정)**까지 티어가 있다. 희귀 원료는 고기술·고숙련 노동과 결합해 **고품질 소비재(사치품, §11)**가 되는 고부가가치 산업의 비싼 입력 — 산업 사다리(1차→고부가가치)에 의미를 준다.
- **자동화·공장 가동에는 포롱마을 주민(노동력)이 필요**하다 — 인력 없이는 규모 있는 생산이 불가능하다.
- **산출품 카탈로그 (원료→부품→완성 소비재 · 개요. 구체 매트릭스·체인·소스·커스텀 = [`content_catalog.md`](content_catalog.md) DL-E102):**
  | 분야 | 원료·중간재 | **완성 소비재 (주민 구매)** |
  |---|---|---|
  | 농업·식품 | 곡물·축산·식자재·밀가루 | 식료품·가공식품·기호품(술·과자·커피) |
  | 광업·소재 | 광석 → 강철·합금·반도체 소재 | (중간재 — 타 산업 원료) |
  | 제조·공산품 | 원단·실·엔진·차체·부품 | **옷·신발·패션 / 자동차·배·자전거 / 가전·가구·생활용품** |
  | 에너지·화학 | 석탄 → 연료·플라스틱·화학원료 | 의약품·화장품·연료·생활화학 |
  | 전자·첨단 | 반도체·회로·부품 | **PC·스마트폰·가전·전자제품** |
  | 서비스·금융 | — | 금융·연구·물류·엔터 서비스 |
  - 가치사슬 예: 광석→강철→차체·엔진→**자동차** / 반도체 소재→반도체→회로→**PC** / 면화→원단→**옷** / 강철→선체→**배**.
  - 분야×티어 + 품질 변주 + 가공 체인 깊이로 산출품 수십 종. 분업·거래 유도(소재 유저 강철 → 제조 유저 자동차 원료), 품질 전파(§11)로 고품질 강철 → 더 좋은 자동차.
- **소스 매핑 (DL-E070):** 식품·원료·중간재·전자부품 = **기존 모드**(Croptopia+FD / ~~Mekanism·IE·MI 중 1~~ → **IE 확정 DL-E082** / Create) → 커스텀 거의 불필요. **커스텀 = 최종 완성 소비재 5~6군**(의류·자동차·자전거·가전·PC/스마트폰·의약/화장품; 배=바닐라 보트, 기차=Steam'n'Rails, 가구=가구 모드). 서비스·금융은 아이템 아닌 CORE 시스템 산출.
- **경제 작동 검토 결과 (DL-E070):** 분업→B2B→완성품→주민 소비 루프 성립, 다각화 충분 — 단 3 보강: ⓐ **식품 과잉 다양성**(Croptopia 수백 종 → "식료품" 카테고리로 집계, 소수만 개별 시장재) ⓑ 완성품 **체인 깊이 페이싱**(깊으면 초반 공급 부족 → B2B 의존, P1) ⓒ **서비스·금융 분야 수익 모델 미설계**(아이템 안 팖 → 금융 수수료·투자수익·물류 대행 수수료 등, 보강 필요).

### 1-A. 업종 마스터 테이블 + 오염 모델 (산업 설계의 척추 · 확정)

- **업종 마스터 테이블:** 회사는 설립 시 **업종을 하나 등록**한다. 각 업종은 속성 묶음을 가진 데이터 테이블 행 — **오염계수 · 산재위험도 · 부가가치 · 노동집약도 · 경기민감도** 등. 이 한 테이블이 오염·산재·산업 다양성·거시 민감도 등 여러 지표를 같은 구조로 굴리는 **산업 설계의 척추**다.
- **업종 인식 = 산출물 게이트(방향):** 생산을 물리적으로 막기보다, **산출물을 시장에 보내는 길목(커스텀 "산출물 박스")에서 업종-품목 일치를 검사**한다 — 업종 외 품목은 만들어도 환금 불가. 이 박스가 생산이 경제 시뮬과 만나는 단일 지점(검사·품질 산정·오염 기록·시장 등록). (세부·P2P 차단은 idea_inbox.)
- **환경오염 모델 — 활동에서 창발하는 누적 stock:**
  - `오염 += 유입 − 흡수`
  - `유입 = Σ(회사 생산량 × 업종 오염계수) + (전력 생산량 × 발전원 오염계수)`
  - `흡수 = 자연정화(녹지 비율 비례) + 정화사업(안건)`
  - **시즌1 = 마을 전체 단일 오염 수치**(지역별 오염은 후보).
  - **대가(실제 배선, DL-E093):** 오염↑ → ① **노동생산성↓** = 가동률 외생 페널티 슬롯(`×(1−k₁오염)`, spec §4) ② 주민 건강↓ → **행복도↓**(합성지표 −오염 → 이주·안건 트리거 경유, 소비엔 직접 X) ③ **초과사망↑**(인구 동학, population_model) → 경제에 부메랑(+옵션: 공단 주변 부동산↓). 오염은 장식 숫자가 아니라 노동력=경제 몸통을 갉아먹는다.
- **"고오염 산업"** = 오염계수 ≥ 임계인 업종(중공업·제련·화력 등) → **탄소세 안건의 자동 대상**(별도 정의 불필요).

### 2. 회사 설립·경영 — 직접 짓는 MC, 보는 대시보드 (방향 확정)

**조작 방향성: MC답게.** 회사는 추상 메뉴가 아니라 **유저가 땅을 사서 직접 짓는 실제 건축물**이다. 땅을 구매하고(§9) 거기서 어떤 회사를 할지(주민 노동 기반 자동 농사, 자동화 모드/플러그인으로 철 자동 생성 등) 유저가 손으로 짓는다.

- **자동화 = core 모드가 제어하는 자동화 모드/플러그인:** 자동화 모드/플러그인(예: **Create 모드** — 채택 시 모드 플랫폼 전제, P0-③)을 쓰되, **core 모드(서버 자체 로직)가 그 위에 올라타 제어**한다:
  - **채용된 주민이 없으면 설비가 안 돌아간다** — 자동화도 노동력(고용된 계층 인원)이 전제. 인력 없는 무인 가동 불가.
  - **감독관·안전규칙 미준수 시 산업재해(산재):** 감독관을 두지 않거나 안전 규칙을 어기면 주민이 다치고 산재가 발생한다 — 노동 안전이 비용·평판·생산성으로 되돌아오는 압력.
- **법인격 (확정):** 회사는 **법인** — 회사 자금과 유저 개인 지갑이 분리되고 **유한책임**(도산해도 유저 개인 파산 X). **단독(1인 주주) + 공동 소유/창업(지분 분할)** 모두 가능(다수 유저가 한 회사 지분 공유). 주식 상장·지분 거래·M&A(§3)가 이 법인격 위에서 성립. (대다수는 1인 1회사로 예상되나 시스템은 둘 다 지원. 유한책임의 도덕적 해이는 신용(대출) 규제로 견제, agenda §11·거시 §5.)
- **회사 평가 = 재무제표 3종:** **손익계산서**(매출[시장+계약] − 비용[인건비·원료·전기·세금·위약금·산재] = 이익) · **재무상태표**(자산[현금·토지·설비·재고] = 부채[대출] + 자본[자본금·이익잉여금]) · **현금흐름표**(영업·투자·재무). + 식별(이름·지분·업종·설립일)·고용(주민 계층별 인원·임금)·주식(상장·발행·주가). 이 항목들이 대시보드·법인세·주가 평가·안건 트리거(도산율·수익성)의 입력값.
- **업종별 경기 민감도**가 다르다(제조업은 공황에 취약, 금융업은 시스템 리스크 등).
- **운영 = 대시보드로 파악:** 건설·생산은 MC에서 손으로 하되, 경영 판단은 대시보드로 본다 — **현재 산출량 · 평균 품질 · 시장 평균가 · 주민 선호도 · 시장에 안정적으로 공급 가능한지** 등을 한눈에 확인.
- **대시보드 = 인게임 네이티브 커스텀 GUI (확정 · DL-E057):** 바닐라 chest-GUI(아이콘+lore 호버)도, 웹뷰(브라우저)도 아닌 **모드 자체 Screen(GuiGraphics)**. 흐름: **핫바 "경영 단말기" 아이템 우클릭(또는 단축키) → 메인 메뉴 → 탭 [회사][시장][거래소][의회][마을][내 자산] → 클릭하면 해당 패널.** 차트·게이지·수치는 **도형+텍스트로 직접 렌더**, 서버 push로 **실시간 갱신**.
  - **비주얼 = 풀 텍스쳐 폴리시 지향 (DL-E058):** 최종 목표는 예쁜 단말기 룩. **재사용 테마 세트**로 1인 개발 비용 관리 — 9-slice 프레임/배경 1~2종(어떤 크기 패널에도 stretch) + 버튼 sprite(기본/호버/눌림) + 탭 아이콘. 한 세트로 6개 패널 통일감.
  - **제작 순서:** 기능을 텍스처 0(도형+텍스트+바닐라 아이템 아이콘)으로 먼저 완성 → 풀 텍스쳐 폴리시 패스로 입힘(분리해 아트가 기능을 막지 않게).
  - **아트 소싱:** 외부 GUI 텍스처 킷·템플릿 / AI 생성 / 직접(9-slice 1장) — 1인 개발이라 외부 에셋 우선. 클라 모드 설치(DL-E030)라 가능.
  - **외부 웹·MCEF(인게임 웹뷰) 비채택:** 캐주얼 지인 30명은 브라우저 안 켬 → 외부 웹 무의미, MCEF는 무겁고 리스크. 모든 플레이어 UI는 인게임 네이티브.
  - **게임 밖 알림은 디스코드 봇:** 안건 발의·주가 급등·순위 변동을 디스코드로 push(친구들이 이미 쓰는 채널). 웹 대시보드 대체.

### 3. 주식·자본시장
- **주가 모델:** 주가 = **펀더멘털 앵커(재무 실적·성장)** + **수급/심리(매수·매도 압력·경기)**. 호황·거품 시 과열, 침체 시 폭락(거시 §8 사이클과 연동).
- **투자자 = 주민 집계 풀 + 유저 (혼합 · 확정):** 주식 수요자는 주민(집계)이 주축 — 주민 저축이 투자 풀로 유입, 재무·주가 추세·경기·저축률에 따라 매수/매도 압력을 **집계 함수**로 생성(개별 심리 시뮬 아님, 하이브리드 §4 일관). **유저끼리도 거래 가능**(자본시장·M&A 성립). (INBOX-002 해소.)
- **상장(IPO):** 재무 요건(자본·이익·기간) 충족 시 상장(요건은 금융 안건 §11로 조정). 공모가·발행 주식 수 설정 → **자본 조달(현금 유입)** 대신 **지분 희석·경영권 노출**. (상장의 구체 재무 기준·자본 건전성 지표·임계는 **재무제표/데이터 스키마 단계**에서 정의.)
- **거래소 = 커스텀 GUI(실시간 차트, DL-E032):** 유저 호가 주문 + 주민 집계 유동성이 받쳐줌. 캔들 차트·실시간 시세.
- **배당:** 이익 일부를 주주(주민 풀·유저)에게 배당 → 보유·투자 동기. 회사는 **배당 vs 재투자**(주주 환원 vs 성장) 선택.
- **M&A:** 시장에서 지분 매집 → 과반 시 경영권. **상장사는 적대적 인수 대상**(비상장·1인 100% 회사는 불가). 호경기에 활발(§8). 시장 거래라 **자산 파괴 아님**(DL-E008 무충돌).
- **지배구조 = 지분(오너식) · 주주총회 없음:** 경영권(대표)은 **최대 유저 지분 보유자**가 가진다(별도 주주총회·CEO 선임 투표 없음 — 과함). **51% 이상 = 안정 지배**(적대적 인수 면역), **51% 미만 = 불안정 지배**(대표지만 경쟁 유저가 지분을 더 매집하면 경영권 탈취). **주민 지분 = passive(비의결·배당만)이되 거래 가능한 유동주식** → 적대적 인수의 매물(주민 비정치 §6과 일관). **경영권 탈취 시 회사 관리 GUI·권한이 새 최대주주에게 이전, 기존 대표는 주주로 강등**(지분 관리 실패는 본인 책임). 공동 창업도 최대 지분자가 대표.
- **은행 = 중앙은행 단일 (확정 · DL-E039):** 민간 은행 없음 — 예금·대출·발권·금리·국고를 모두 **중앙은행(CORE 단일, monobank)** 이 담당. 주민·유저 예금과 회사·유저 대출이 전부 중앙은행 경유. 유저 금융업은 예금은행이 아니라 **투자·증권 중개**로 한정. 민간 은행 부재로 '리만식 연쇄 도산'은 빠지고, 거시 위기는 자산 거품·부채·신용경색·수요붕괴로 발생(§8).
- → **금융 안건**(주식 규제·가계부채·자본이득세) agenda_catalog §11. 통화정책·신용경색 대응은 거시 §5.
- **서비스·금융 분야 수익 모델 (확정 · DL-E072):** 이 분야는 아이템을 안 만들고 **수수료·자본·서비스 수익**으로 번다 — 제조 없이 부를 굴리는 대안 플레이 트랙(승리=최대 부, 제조 안 하고 투자·서비스로 참여):
  - **금융(증권·투자):** 주식 투자 수익(시세차익+배당)·거래 중개 수수료·M&A 자문료. 경기 사이클·자산 거품을 타는 **하이리스크**(호황 대박/폭락 위험).
  - **물류·운송:** 운송 용량·물류 대행 B2B 판매(수수료, Steam'n'Rails 기차) — 인프라 하이브리드(§12).
  - **전력·인프라:** 발전소로 전력 시장/계약 판매(§12).
  - **연구 위탁(옵션):** 연구원 보유사의 위탁 연구·기술 라이선스 수수료.
  - 스타일: 금융=변동성 하이리스크 / 물류·전력=안정 인프라 수익.

### 4. 살아있는 주민 — 노동·소비·투자의 주체 (하이브리드 2층 구조 · 확정)

주민은 **"집계가 진짜, 빌리저는 얼굴"** 인 2층 구조로 표현한다. (P0-① 구현 수준 결정 = 하이브리드.)

**1층 — 집계 경제 (실제로 굴러가는 것, 비가시):** 인구를 **계층(strata) 풀**로 관리한다. 각 계층은 `인구수 · 평균학력 · 임금 · 소비성향(MPC) · 욕구충족도(필수품/사치품) · 출산 기여` 같은 속성을 가진다(예: 농민 계층 1,200명·학력1·소비성향0.8 / 공장노동 800명 / 고학력 150명·연구가능 / 실업 300명). 이 풀들의 합이 곧 **총 노동공급 · 품목별 총 소비수요 · 세수 · 실업률 · 평균 생활수준**이며, 여기서 거시 사이클이 창발한다(§8). 집계 연산이라 **수만 인구(국가 단계)도 부하 없이** 확장된다.

**2층 — 대표 빌리저 (눈에 보이는 얼굴, 수십 명):** 각 계층을 대표하는 빌리저 몇 명만 마을에 실제로 돌아다닌다. 이들의 가시 상태(잘 먹음/굶주림/일하는 모습)는 **자기 계층 집계의 거울**이다. 유저가 "이 노동자 NPC 고용" 하면 내부적으로 "해당 계층에서 N명 고용"으로 번역된다 — 추상 숫자를 살아있게 만들되 개체 AI를 수만 개 돌리지 않는다.

> 이 구조가 "살아있는 주민" ↔ "거시경제의 일원" 긴장을 해소한다: **거시 자체가 자라고·병들고·회복하는 살아있는 생물**이고, 빌리저는 그 맥박이다. 주민은 노동·소비·출산·주식 매수를 하지만 **정치는 하지 않는다**(정치는 유저 의회의 몫, §6).

- **인구 동학:** 경제가 발전하면 출산율이 오르고 인구(집계)가 는다. 인구는 노동 공급이자 소비 수요이며, 고용에는 **인건비**가 든다.
- **노동=소비 루프(핵심):** 물건과 주식의 **수요자가 곧 주민**이므로, 유저가 단가를 낮추려 **임금을 깎고 주민을 해고하면** 마을의 소비력이 줄어 **자기 회사 물건도 안 팔리고 경기가 나빠진다.** 반대로 임금·고용을 늘리면 소비가 살아 경기가 부양된다 — 착취가 스스로를 처벌하는 자기조정 구조.

### 5. 마을의 진화 — 마을 → 도시 → 대도시 → 국가 (확정 · DL-E051)
- **진화 게이트 = 다인자 임계**(인구 하나가 아니라): 인구 규모 + 경제 규모(GDP) + 산업 다양성 + 평균 학력 + 인프라 수준이 종합 임계를 넘으면 한 단계 진화.

  | 단계 | 진화 조건(다인자 임계) | 해금 |
  |---|---|---|
  | **마을** | 시작 | 1차 산업·기초 인프라 / **의회 출범**(마을 단계, 첫 경제 루프 후 — DL-E006·E085) + 안건 카테고리 점진 해금 |
  | **도시** | 인구·GDP·다양성 1차 임계 | 2차 산업(가공)·인프라 본격 확장·가용 토지↑·안건 범위↑ |
  | **대도시** | 2차 임계 + 학력·인프라 | 고부가가치 산업(금융·기술·서비스)·자본시장 **활성**(개방 아님 — 상시 ON·IPO 요건, DL-E085)·토지/인프라 대폭↑ |
  | **국가** | 최종 임계 | 전 산업 티어·거시경제 완전체·최대 토지 (시즌2+ 대외무역) |

- **마을 진화 = 시대 뼈대 해금 (확정 · DL-E062):** 마을 발전(인구·경제·인프라·평균 학력)이 곧 다음 기술 시대(공동 뼈대, §7)를 연다 — 별도 공동 연구 단계 없음(시간 이중 소모 제거). 분야 특화만 개인이 연구원 고용해 진행(§7).
- **쇠퇴(역진화):** 대공황·인구 붕괴·재정 파탄으로 임계 아래에 오래 머물면 **강등**(도시→마을). 진화는 일방통행이 아니라 유지해야 하는 긴장 — 노동=소비·재정 방치의 진짜 대가.
- **세부 단계:** 4대 단계 각각 **4 서브레벨**(마을1~4 → 도시1~4 → 대도시1~4 → 국가1~4 = **16단계**) → 부드러운 해금·페이싱(28일에 ~1.7일마다 레벨업, DL-E055). 서브레벨마다 가용 사유 면적·산업·안건 범위 점증(§9).
- 진화는 인프라·산업 다양성·행정 구조를 함께 끌어올린다(아래 §6). (임계 수치·계수는 P1.)

### 6. 정치 — 유저 의회와 안건 투표 (구조 확정 · 반(反)장악 설계)

**의회의 목적**은 **부를 둘러싼 이익 정치의 무대**다 — 유저들이 자기 이익(내 부)을 위해 정당을 만들고 모의(합종연횡)해 안건에 찬반을 던진다(실세계 이념 롤플레이가 아니라 "내 돈을 위한" 정치). 30명 지인 서버에서 이 정치질이 핵심 재미이자 **승자독식의 고무줄 견제 장치**(뒤처진 다수가 연합해 선두 증세, 게임 프레임 참조). 그 위에서 **소수 유저가 의회를 장악할 수 없게** 두 가지로 차단한다.

- **① 전원 의석 직접민주제 (1인 1표):** 정부는 유저 전원의 의회다. 마을이 일정 성장 후 의회가 출범하면 **모든 플레이어가 의석을 가진다** — 의석 수 제한이 없어 의석을 "차지"하는 경쟁 자체가 없다. 거대 자본가도 신규 유저도 정치적으로는 1표로 동등(경제 권력에 대한 견제). 주민은 노동·소비·투자만, 정치는 안 한다.
- **② 의제 설정권 = CORE SYSTEM 전속:** **유저는 안건을 발의할 수 없고, 무엇을 안건으로 올릴지도 정하지 못한다.** 의회 출범 이후 **CORE SYSTEM이 경제·환경·재정 등 다양한 지표를 감시하다 조건이 맞으면 안건을 던지고**, 유저는 그 안건에 **찬/반만** 한다. (담합·매수해도 의제를 못 만들어 장악이 무력화되는 두 번째 축.)
- **정당 (= 가치 축·친구들의 모임):** 뜻이 같은 유저나 함께 시작한 친구끼리 **정당을 결성할 수 있다.** 단 정당은 의석·표를 배분받거나 강제하지 못한다 — 당론 추종은 자유. **정당은 의도적으로 기계적 권한이 없는 소셜/정체성 결사**(성장·자유 vs 분배·환경 가치 축의 표식, 또는 함께 시작한 친구 모임)일 뿐이다. 표를 강제하는 순간 다시 장악 통로가 열리므로 **권력 기구로 만들지 않는다.** 단 멤버가 자발적으로 모의·합종연횡하는 **이익 카르텔**로 기능한다(표 강제는 없되 이익 맞으면 같이 던짐) — 소규모 서버 정치 재미의 핵심.
- **토론 → 찬/반 투표 → 단순 과반 가결:** 유저들은 CORE가 던진 안건을 두고 토론하고 찬·반 표를 던진다. **단순 과반**대로 정책이 발효된다. **다수결의 소수 압박은 게임적 선택의 무게로 그대로 둔다**(별도 소수 보호장치 없음, 정족수 정도만) — 이건 게임이고, 집단 선택의 결과를 함께 지는 것이 재미의 일부다.
- **운영 규칙 (발의·소집·투표 · 확정):**
  - **정기 발의:** 하루 1회(실시간 기준), 대기 안건이 있으면 CORE가 **1건** 발의.
  - **동시 안건 불가:** 항상 한 번에 **1건**만 진행(여러 안건 동시 표결 없음).
  - **소집 → 1시간 투표 창:** 의회가 소집되면 모든 플레이어가 **1시간 안에** 의회로 와서 안건 확인·투표. 정기 소집은 참여율 높은 **고정 시간대(예: 저녁)** 권장(1인 1표라 참여율이 정당성).
  - **정족수:** **활성 플레이어(근 2일 내 접속)의 50%** 이상 투표 참여. **미달 시 다음날로 이월**(폐기 아님).
  - **가결 기준:** 정족수 달성 시 **단순 다수결**(미참여 = 기권). **동표 시 합산 재산이 더 많은 쪽 가결** — "돈을 모으는 경제 서버"라는 정체성 반영(1인 1표에서 살짝 자본 쪽으로 기우는 타이브레이커).
  - **3단계 에스컬레이션:** ① 정기(하루 1회) → ② **긴급 소집**(지표가 매우 나쁘면 CORE가 정기와 별개로 긴급 의회 소집) → ③ **서킷 브레이커**(돌이킬 수 없는 임계에서 CORE 거부권/긴급명령, 위 항목·DL-E015).
- **안건은 수치 트레이드오프다.** 모든 선택에 득과 실이 있어 정답이 없다 — 마을의 현재 상태와 유저들의 이해관계에 따라 판단이 갈린다.

  | 예시 안건 | 찬성 시 (가결) | 반대 시 (부결) |
  |---|---|---|
  | 친환경 에너지 전환 | 발전 가능 칸 **−30**, 전기세 **+50골드/kWh**, 환경오염 **−50** | 환경오염 **계속 증가** |

- 이렇게 **마을의 상태(지표)가 안건을 부르고, 유저 의결이 다시 지표·경제를 바꾸는 피드백 루프**가 정치의 핵심이다. (정치가 의미를 가지려면 마을이 먼저 어느 정도 발전해야 한다.)
- **안건의 권한 경계 — 시장 경쟁 O · 시장 파괴 X (확정):** CORE 안건은 **가격 상한·강제 분배·법인세 확장 등 시장에까지 개입**할 수 있다. 단 절대선이 있다 — **유저가 쌓아올린 자산을 한순간에 무너뜨리는 정책은 금지.** ⓐ **국유화 불가**(유저 회사를 통째로 빼앗지 않는다). ⓑ **독점 대응도 회사를 강제 분할(동강내기)하지 않고, 대신 해당 품목에 국유 주민회사를 신설해 경쟁을 붙인다**(§11-A). 원칙: **경쟁을 붙이되 시장을 무너뜨리지 않는다.**
- **건설형 안건의 실행 (기술 후보):** 학교·발전소·경찰서·교도소처럼 건축물을 세우는 안건이 가결되면, 미리 준비한 **구조물 에셋을 부지 좌표에 자동 건설**한다 — 가결 즉시 마을에 실제 건물이 들어선다(철회 시 롤백). 방식은 스케메틱(.schem) paste(Paper: FAWE / 모드: WorldEdit 모드) 또는 구조물(NBT) 등 양 플랫폼 모두 가능. **1인 개발이라 건물을 직접 짓기 어려워 외부 에셋 활용 가능성이 크고, 따라서 "외부 구조물 에셋을 서버사이드에서 로드"할 수 있느냐가 플랫폼 선택의 관건**(P0-③). 부지 선정·지형 정리는 미확정(아래 열린 질문).
- **CORE 비상 안전장치 (서킷 브레이커):** 평상시 결정은 의회 몫이지만, 핵심 지표가 **돌이킬 수 없는 임계**(초인플레·신용 시스템 붕괴·재정 디폴트 직전)에 닿으면 CORE가 2단계로 개입한다 — ① **경고**: 파국으로 미는 안건에 **거부권**(통과돼도 무효) ② **파국 임박**: **긴급 명령**으로 능동 개입(예: 강제 금리 인상·비상 긴축). 3원칙: ⓐ 극단(존재적 위협)에서만 — 평범하게 나쁜 결정엔 개입 안 함(그건 다수결의 무게, DL-E007). ⓑ **위험 구간은 대시보드로 사전 경고**해 규칙처럼 예측 가능. ⓒ 소수 보호가 아니라 **시스템 붕괴 방지**(DL-E007과 무충돌). 이유: 유저는 단기 부양(금리↓·주가↑)을 선호해 파국까지 갈 수 있어 최소 안전판이 필요.
- **가치 축:** 안건마다 유저들은 대체로 **성장·자유(개발·규제완화)** 와 **분배·환경(복지·보전)** 사이에서 갈린다 — 정당이 이 축을 대표한다.

### 7. 연구·교육·기술 트리 — 인적 자본이 곧 발전
- **2층 구조 (마을 발전 뼈대 + 개인 특화 · 확정 · DL-E062):** 시대 progression(**뼈대**)은 **마을 발전(진화)이 자동 해금** — 별도 공동 연구 grind 없음(마을 발전과 공동 연구를 따로 굴리면 시간 이중 소모). 마을 진화 다인자(인구·경제·인프라·**평균 학력**)가 다음 시대를 연다. 거기서 갈라지는 분야 특화(**가지**)는 **개인이 연구원(고학력 주민) 고용**해 연구 → 내 회사만의 기술 우위·차별화. 인적 자본은 양쪽에 작용(마을 평균학력→시대 / 개인 연구원→분야).
- **연구 트리 UX:** Civ 과학 발전 트리 스타일(노드 그래프 + 의존성 선)을 지향. **FTB Quests**로 시각화 가능하나 기본 해금이 '퀘스트/아이템 제출'이라, 우리 'Civ식 = 고학력 주민 연구로 점수 쌓아 해금'은 **해금 조건을 CORE 연구 진척도에 KubeJS로 연결**해 구현(또는 전용 tech-tree 모드 검토).
- **기술 트리 = 게이팅 (확정):** 생산 가능 업종·발전원·자동화 수준 **+ 발의 가능 안건**이 현재 기술 시대에 묶인다. **시대 progression**: 수공업 → 석탄·증기(산업혁명) → 전기·내연 → 자동화·친환경(현대). 예: **친환경 에너지 전환 안건은 친환경 발전 기술 연구 후에야 해금**(석탄발전·증기기관 등 선행 필요). → agenda_catalog의 모든 안건에 "선행 기술" 전제가 깔린다(세부 매핑은 P1). **마을 진화(§5)가 곧 시대 뼈대를 연다**(별개 축 아님, 통합 — DL-E062).
- **기술 트리 구조 = 시대 × 분야 2D 매트릭스 (확정):**
  - **세로(시대) = 공동 뼈대 (마을 발전이 해금):** 수공업→증기→전기→자동화. 별도 연구 grind 없이 마을 진화(§5)가 시대를 연다 (16 서브레벨 페이싱).
  - **가로(분야) = 개인 특화:** ①농업·식품 ②광업·소재 ③제조·공산품 ④에너지·화학 ⑤전자·첨단 ⑥서비스·금융. 의회 밖 연구 시스템에서 **개인이 어느 분야를 키울지 선택**(특화 vs 다양성 전략 — 짧은 시즌이라 다 못 찍음 → 산업 다양성·전략 산업 안건과 연결).
  - 각 (시대×분야) 노드 해금 = 새 업종·공정·**산출품군** + 큐레이트된 Create 기계. **개인 특화로 산업 다양성·차별화·거래가 자연 발생**(다들 다른 품목 → 직접 경쟁 완화, 계약·시장 활발).
- **Create 큐레이션 (방대함 길들이기):** Create 전체 노출 X — **우리 트리에 매핑된 부분집합만** KubeJS 레시피 게이팅으로 열고 나머지는 잠금/숨김. 시대별 화이트리스트(1차 예시, P1 조정):
  - **수공업:** 손 크랭크·물레방아·맷돌(millstone)·분쇄휠·기본 프레스·믹서·벨트/깔때기·기어/축
  - **증기:** 증기기관(flywheel+furnace)·Steam'n'Rails(기차·증기 물류)·고속 가공
  - **전기:** Crafts&Additions(전기 모터·발전기·와이어·충전)·전기 구동 기계
  - **자동화:** 기계식 팔·시퀀스 조립·패키저 + AE2(저장·자동 크래프팅) = 완전 자동 라인
  - **제외/잠금:** 순수 장식(copycat·기차 외관), 경제 우회 exploit, 트리 밖 콘텐츠
- **시대별 산출품 심화 (마을 발전 = 부의 천장 상승 · 확정 DL-E068):**
  | 시대 | 새로 가능(누적) | 대표 완성 소비재 | 소비 유형 |
  |---|---|---|---|
  | 수공업 | 1차 채집·기초 가공·수공품 | 식료품·기본 의류·도구·가구 | 필수재 위주 |
  | 증기 | +강철·중공업·대량 생산·철도 | 가공식품·대량 의류·기차·배·기본 기계 | 필수+내구 |
  | 전기 | +전기·전자·화학 | 가전·자동차·전기제품·의약·화장품 | 내구·사치↑ |
  | 자동화 | +반도체·첨단·정밀 | PC·스마트폰·고급차·첨단기기·정밀의약 | 내구·사치(최고 부가) |
  → 시대 진행 = **부 천장 상승**: 초반 저마진 필수재 → 후반 고마진 완성 내구/사치재(자동차·PC). 시즌 페이싱(DL-E055)과 일치 — **Week3~4 고부가 완성품 시장이 부 경쟁 클라이맥스**. 시대를 빨리 올리는 게 간접 경쟁력(마을 발전 기여).
- **산출품 조달 = 기존 모드 + 커스텀 소수 (확정 DL-E069):**
  - **기존 콘텐츠 모드로 대량 공급:** 식품=**Croptopia + Farmer's Delight**(수백 종), 소재·화학·부품=**Immersive Engineering**(+Petroleum, 〔DL-E082 확정 — 기존 "Mekanism/IE/MI 택1"에서 좁힘〕), 가공=Create(+애드온). → 원료·중간재·식품 완성품은 거의 커스텀 불필요.
  - **커스텀 = 최종 완성 소비재 소수:** 자동차·PC·스마트폰·옷 등 전용 모드 드문 것만 커스텀(아이콘+레시피).
  - **⚠️ 큐레이션 필수:** 대형 테크 모드를 다 넣으면 복잡도·부하·콘텐츠 중복 폭발 → **Create(중심) + 식품 모드 + 재료 모드 1개 + 커스텀 완성품**으로 골라 쓰고, 우리 트리/KubeJS로 시대 게이팅(레시피를 시대에 매핑).
- **산출품 다양화 — 3 레버:** ① **시대×분야 매트릭스**가 곧 산출품군(예: 농업×수공업=곡물 / 농업×전기=가공식품·기호품[사치] / 소재×증기=강철 / 전자×자동화=고급 전자제품). ② **가공 체인 깊이**(원료→중간재→완제품, Create 가공 단계마다 산출품, 깊을수록 부가가치↑). ③ **품질 변주**(DL-E013)로 같은 품목도 시장 분화.
- **개인 분야 연구 = 의회 밖, 연구원 고용:** 분야 특화 연구·방향 선택은 연구 시스템에서 직접 이뤄진다(의회 밖) — 마을 초기(의회 출범 전, §6·§10)부터 작동. 의회는 출범 후 **R&D 예산 안건**(agenda_catalog §9)으로 가속만, 방향은 개인이 정한다. **재료가 아니라 고학력 주민(연구원) 투입이 분야 기술을 연다** — 인적 자본이 병목이자 산업 한계(자원 아님).
- **교육 인프라:** 학력을 높이려면 **학교 등 사회기반시설**이 필요하다. 학교가 고학력 주민을 길러내고, 그 주민이 연구를 가능케 하는 사슬.
- 신기술은 생산성↑·새 산업 해금·자동화를 가져온다. 서버 방향성에 따라 자동화·생산용 **모드/플러그인**을 도입할 수 있다(플랫폼 결정과 연동).
- 학교·사회기반시설 건설과 기술 트리 방향은 **유저 의회의 안건 투표**(§6)로 결정된다 — 어디에 예산·발전 칸을 쓸지가 곧 정치다.
- **연구 → 재정 보조금 루프:** 한 기술의 연구가 완료되면, 그 기술을 쓰는 공장에 **산업 보조금 안건**(재정, agenda_catalog §3)이 발의될 수 있다 — 신기술 채택을 가속하는 교육↔재정↔산업 연결 고리.

### 8. 거시경제 사이클 — 주민 집계에서 창발
- **별도 엔진이 아니다.** 인플레·호황·대공황·신용경색은 주사위가 아니라 **주민 집계 상태(총임금·고용률·평균학력·저축·소비성향)의 readout**이다(§4). 유저의 집단 행동(임금·고용·투자)이 집계를 흔들면 사이클이 따라온다.
- 회사 수 ↑, 발행 주식 ↑, 마을 성장 → **호경기**: 인수 증가, 그리고 **인플레이션**.
- 경기 과열·자산 거품 → 조정. 제조업은 **대공황(수요 붕괴)** 을 맞을 수 있다.
- **신용경색**(과잉 대출·부채 위기·자산 거품 붕괴) → 경기 침체. (은행은 중앙은행 단일이라 민간 은행 연쇄 도산은 없음 — §3·DL-E039. 위기는 부채·거품·수요붕괴 경로.)

### 9. 토지·부동산 — 2 고정 차원(메인 + 슈퍼플랫) + 중앙 공용 마을 + 유저 구획 (DL-E059 → DL-E079 갱신)
> 〔갱신 DL-E079: **2 고정 차원** — 메인(자연지형: 중앙 공용 마을 + 야생) / 회사 슈퍼플랫(유저 부지). 아래 plot-grid·부동산은 슈퍼플랫 월드 적용. 채굴=DL-E080·전력=DL-E081. 상세 spec §11.〕
- **2 고정 차원(DL-E079):** 메인(중앙 공용 마을 + 야생) + 회사 슈퍼플랫(유저 부지). 같은 서버에서 걸어다니며 **다른 유저의 회사·공장을 구경**하고(사회성) 의회·시장에 모인다. (차원 2개 고정 — 무한 차원 폭증 회피.)
- **중앙: 공용 주민 마을 (다 같이 보는 무대):** 의회·공공 인프라(발전소·학교·경찰서)·대표 빌리저·건설형 안건 결과물이 여기 시각화. 중앙 건물 저택→청사로 진화, **동심원으로 마을→도시→대도시→국가** 확장. CORE/운영이 관리(유저가 짓는 게 아님). **마을 확장 예정지 = 국유지**(구매 불가).
- **주변: 유저 사유 구획 (plot-grid):** 유저 회사·건물은 격자 구획에 짓는다 — **안 꼬이되 서로 보임.** 초기 구획 부여(예: 300x300 중 30x30 사용 — 수치 P1), 부동산으로 확장.
- **가용 사유 면적 = f(마을 발전 세부단계 §5) − 의회 정책**(인프라 점유·그린벨트가 총 한도 깎음 = 개발 vs 보전 트레이드오프).
- **격리 아님이라 경제 그대로:** 시장 추상 정산(DL-E026)·계약·의회는 GUI/집계라 구획 분리와 무관. 산출물 박스만 내 구획에 설치.
- 외부 자원(원료) **수입**은 비용 요소로 가능(§1), 생산 거점은 내 구획 위에.
- **부동산 UI(대시보드 탭):** 그리드 타일(산 땅 / 살 수 있는 땅 / 사용 중 색 구분) → 타일 클릭 구매·설치. 도형 그리드(텍스처 0).
- → **부동산 안건**(토지 공급 확대·투기 억제·공공 임대/분양·용도 구역 지정) agenda_catalog §10. 그린벨트는 환경 §1.

### 10. 마을 재정
> 〔변경〕 초기의 "4개 마을(포롱·퍼렁·푸룽·파랑) 분할·소속" 안은 **폐기**되었다 — 유저들이 함께 만들어가는 **단일 마을**로 통합. (사유: 정치 주체가 주민→유저 의회로 바뀌면서 4개 진영 분할의 의미가 사라짐.)
- 마을은 **하나의 재정**을 가진다: 세수를 걷고, 필요하면 빚을 낸다.
- **재정 모델(회계 항등식, 창발 아님):** `재정수지 = 세수 − 공공지출`. 적자면 **국채로 메우고, 부채는 stock으로 누적 + 이자** → 방치 시 눈덩이(부채 위기 → 긴축 안건 또는 거시 신용 위기). **국채 매수자 = 주민 저축(자산배분, population_model 가계 흐름)** + 유저 — 재정 자금 조달이 주민 금융과 연동.
- **세수 원천(S1 내수):** 법인세(회사 이익) · 소득세(주민 임금) · **거래세(시장 거래액 %)** · 재산세(토지·부동산). 관세 없음. → **모든 거래가 시스템 경유(§1-A)라 거래세 자동 징수가 용이**(설계 시너지).
- **공공지출:** 인프라 · 교육 · 복지 · 치안 · 환경 + **주민회사 운영 재원**(§11-A, 마을 재정에서 충당).
- **세율은 정책 상태값** — 의회 안건(증세·감세·누진세·산업 보조금, agenda_catalog §3)이 바꾼다. **소득세는 누진성(progressivity) 속성**을 가져 누진세 안건이 이를 조정(불평등 직접 레버, 자본가 유저에 직격). **산업 보조금**은 연구 완료 기술(§7)에 연동 — 해당 기술 사용 공장에 보조금.
- **중앙은행 = 경제의 화폐 중추 (행정부급 · 확정):** 마을엔 단일 중앙은행(CORE 운영)이 있어 ① **국고**(세수 저장) ② **공공지출 통로**(인프라·복지 등 모든 지출이 여기서) ③ **발권**(화폐 발행·회수 = 통화량 소스/싱크) ④ **기준금리**(통화정책 §6·DL-E015) ⑤ **국채** 발행·이자 ⑥ **예금·대출**(주민·유저 예금, 회사·유저 대출 — 민간 은행 없음, DL-E039)을 모두 담당한다. **재정(세수·지출)과 통화(금리·발권)의 통합점** — 모든 화폐 흐름이 중앙은행을 경유해 "경제가 은행에 달려있다". 정책 결정은 의회 안건, 집행은 중앙은행.
- 재정 상태(빚·세수·발전 칸·환경 등 DB 지표)는 **의회 안건의 입력값**이 된다 — 지표가 나빠지면 게임이 관련 안건을 발의하고, 유저 의결이 다시 재정·지표를 바꾼다(§6).

### 11. 시장 수요·가격·품질 경쟁
- 유저 회사의 물품은 **마구잡이로 팔리지 않는다.** 주민이 그 물품을 **얼마나 필요로 하는지**, **사치품인지 필수품인지**, 전체 **공급과 수요**가 어떤지에 따라 팔림새가 정해진다.
- 따라서 유저는 **가격을 조정**하고, 경쟁사보다 **품질을 높여야** 한다.
- **품질(quality)** 은 산출품의 핵심 속성으로 존재한다 — 같은 품목도 품질에 따라 가격·선호·수요가 갈린다(품질 경쟁이 곧 차별화 수단).
- **사치품 = 고품질 소비재 (별도 아이템 아님):** 사치품은 따로 만드는 품목·직업이 아니라 일반 품목의 **고품질 상층**이다 — 같은 빵도 저품질=필수품(빈곤층 소비), 고품질=사치품(부유층 소비). **누구나 품질을 올리면 사치 시장에 진입**(품질 경쟁 = 사치 시장 공략).
- **소비는 소득탄력성으로 창발:** 주민 계층 소득 분포에서 사치 수요가 나온다 — 가난하면 필수(기본품), 부유하면 사치(고품질). 경제가 성장해 부유층이 늘면 고품질 수요가 자란다(또 하나의 창발).
- **소비 유형 × 경기 탄력 (확정 DL-E066):** 완성 소비재(§1 카탈로그)는 유형별로 경기 민감도가 다르다 — **필수재**(식료품·기본 의류·의약)=경기 둔감(불황에도 삼) / **내구재**(자동차·PC·가전·가구)=**경기 민감**(호황 구매↑, 불황 연기) / **사치재**(고급차·명품·고급 전자·기호품)=고소득·경기 민감. → 불황 시 자동차·PC가 안 팔려 제조·전자 유저 직격, 식품 유저는 버팀. 거시 사이클(§8)·업종 경기 민감도(§2)와 직결.
- **가격·물가 형성:** 품목 가격은 수급(공급÷수요)에서 형성(수요>공급 → 가격↑). **물가지수(CPI)** = 필수품 바스켓 가중 평균, **인플레이션** = 통화량(소스>싱크) 또는 수요 초과에서 창발. **독점도(HHI)** = 품목별 회사 점유율 제곱합. 시장 개입 안건은 agenda_catalog §4(전부 DL-E008 "경쟁 O·파괴 X" 준수).
- **2층 시장 구조 (확정 · DL-E067):**
  - **B2C(주민 수요):** 주민은 **완성 소비재만** 산다(식료품·옷·자동차·PC 등). 집계 수요가 품질·가격·소득탄력·소비유형으로 흡수(시장=상점, 아래).
  - **B2B(회사↔회사):** 중간재·부품·원료(강철·반도체·원단·엔진)는 **회사 간 거래** — 시장(스팟·변동가) 또는 계약(고정가, §11-B). 특화 분업의 핵심.
  - → 가치사슬 = 공급 네트워크: 원료사 →B2B→ 부품사 →B2B→ 완성품사 →B2C→ 주민. 특화(§7)로 분업·B2B 활발, 품질 전파로 상류 품질이 하류로.
- **시장 = 상점 (추상 자동 정산 · 확정):** 주민이 집계(§4)라 개별 손님 NPC가 상점에 오지 않는다. 유저는 **산출물 박스(§1-A)에 물량·가격을 등록**하면 **집계 주민 수요가 품질·가격·소득탄력에 따라 자동 흡수 → 정산·매출**. 별도 소매점 NPC 없음. 유저-유저도 P2P 차단이라 시장(또는 계약) 경유 — **"시장이 곧 상점"**.
- **공급 과잉·도산 대응 (CORE 무한매입 ❌ · 확정 DL-E063):** 공급 과잉으로 가격이 급락해 유저가 망할 수 있다 — **CORE/은행 무한 매입은 안 한다**(무한 화폐 발행=인플레, 시장 신호 죽음, 리스크 제거 시 부 경쟁 무의미). 도산은 게임의 일부. 완충 4: ① 주민회사 가격 **바닥**(DL-E005, 0원 폭락 방지하되 낮은 기준가 — 원가 이하면 여전히 손해=감산 신호) ② **유한책임**(DL-E034, 회사 도산해도 유저 재기 가능) ③ 시장 신호+**계약 헤지**(대시보드 사전 경고 → 감산·전환·B2B 고정가 계약 DL-E028) ④ 자기조정(급락→신규 진입 X·감산→공급↓→회복). 개인 특화(DL-E062)가 "다들 자체 생산"을 완화(분업·거래).
- **품질 전파 (quality propagation · 확정 DL-E064):** 품질(DL-E013)을 **원자재 포함 전 산출물**에 부여. **고품질 투입 → 산출 품질↑ AND/OR 수율↑**(산출량↑·불량↓). 예: 고품질 철 → 기계 품질↑·생산 효율↑. 효과 — 공급 체인 전체가 품질 경쟁, 원자재 공급자도 차별화(저품질 양산 대신 고품질 프리미엄 = 과잉 폭락 회피 활로), 가공 체인 깊이(§7)와 시너지.
- **가격 결정 = 유저 호가 + 수요곡선 (확정):** 유저가 박스에 **가격을 직접 호가**하면 집계 수요곡선이 반응 — 시세보다 비싸면 적게(재고↑), 싸면 빨리(마진↓), **품질 높으면 같은 값에 더(또는 더 비싸도) 팔림**. 시세(수급 형성)는 기준선, 주민회사 앵커(DL-E005)가 천장·바닥. **가격 경쟁 + 품질 경쟁이 둘 다 작동**(시장가 고정 시 가격 레버가 죽어 단조 → 호가제 채택).

### 11-A. 주민회사 — 시장 안정 닻 (anchor)
- **문제의식:** 유저 회사들이 시장에 공급을 시작하면 유저들은 마을 밖에서 자원을 끌어오는 노가다를 그만두고 싶어진다. 이때 **소수 유저의 담합**이나 **공급 공백 → 포롱마을 경제 붕괴** 위험이 생긴다.
- **해법 = 주민회사(공영/NPC 기업) 출범:** 마을이 직접 운영하는 회사가 기본 품목을 **항상 중간 품질·기준 가격으로 공급**한다.
- **닻(anchor) 역할:** 주민회사가 시장 가격의 기준선을 잡아준다 — 담합으로 가격을 띄우려 해도 주민회사 물량이 천장을, 덤핑해도 바닥을 받친다. 공급이 끊겨도 최소선은 주민회사가 메운다(붕괴 방지).
- **유저의 경쟁 여지:** 유저가 만든 산출물의 **품질이 주민회사(중간)보다 높으면 더 비싼 값을 받아 경쟁력**을 가진다. 즉 주민회사는 바닥/기준을 깔고, 유저는 품질로 그 위를 차지하는 구조 — 안정과 경쟁을 동시에.
- **독점 대응 수단(겸용):** 특정 유저가 한 품목을 독점하면, 회사를 강제 분할하는 대신 **그 품목에 국유 주민회사를 신설해 경쟁을 붙인다** — 독점은 깨되 유저 자산은 보존한다(§6 안건 권한 경계와 연결). 즉 주민회사는 ① 가격 닻 ② 공급 공백 방지 ③ **반(反)독점 경쟁 주입기**의 3역.
- **공급망 바닥 보장(④번째 역 · DL-E071):** 취급 범위에 **기본 필수재(빵 등) + 핵심 원료(곡물·기본 광석)** 포함. 다들 고부가만 쫓아 1차를 버려도 주민회사가 **최소선·기준가로 받쳐** 주민 아사·공급망 완전 붕괴 방지. 단 중간 품질·제한 물량이라 유저가 더 싸고 많게 하면 이김(대체 아니라 바닥).
- **운영 재원:** 마을 재정에서 충당(§10) — 주민회사 운영비도 공공지출의 일부.
- **열린 질문:** 주민회사 시장 점유율·발동 조건(언제 개입/철수?)·기준가 산정 방식은 미정(아래 P1).

### 11-B. 회사 간 계약 (B2B 안정 공급 채널 · 확정)
- 시장(스팟·변동가, §11)과 **별개 채널**로 회사끼리 안정 공급 계약을 맺는다.
- **체결:** 계약 GUI에 조건 등록 → 상대 수락(또는 **공개 오퍼 게시판**). 조건 = 공급 레이트(분/시간당)·개당 대금·계약 기간·위약금.
- **자동 이행:** **납품박스(공급자) ↔ 수령박스(수요자)** 설치+계약 연결 → 자동 아이템 이송 + 대금 정산(중앙은행 경유·거래세). 정산 주기 = 실시간 vs 배치(실시간은 부하 우려, **구현 시 결정**).
- **위약:** 공급 미달 자동 감지 → 미달분 위약금 차감, 누적/심각 시 자동 파기. 중도 해지는 위약금 물고 가능.
- **시장 vs 계약 트레이드오프:** 계약=고정가·안정 공급(예측 가능, 시세 급등 못 누림), 시장=변동가·유연(불안정). 2차 산업 원료 확보·공급자 매출 예측에 쓴다.
- 공개 오퍼 + 사적 직접 제안 둘 다.

### 12. 인프라 — 전력·운송
> 〔갱신 DL-E081: 전력 = (나) **전기 시대부터** 등장 / 회전력(자급·저효율) vs 전기 설비(Crafts&Additions 전기모터, 전력망 의존·고효율) 트레이드오프 / 경제 전력 = 실제 **FE 미터링 비용**(전기세)·위기 배급, 물리 가동 차단 아님. 상세 spec §11-5.〕
- 물품을 **팔려면 전력 공급과 운송망**이 받쳐줘야 한다 — 생산만으로는 부족하고, 판매에 필요한 인프라가 전제된다.
- 인프라는 **마을 발전에 연동**된다(마을이 클수록 전력·물류 역량↑).
- 인프라는 **정부 정책의 영향**을 받는다 — 예: 친환경 에너지 전환 정책을 펴면 **전기세가 오를** 수 있다(정책 ↔ 생산 비용의 연결고리, 성장·자유 vs 분배·환경 가치축과 맞물림).
- **소유 = 하이브리드 (확정):** 유저도 **발전소·운송 회사**를 차려 전력·운송을 시장에 공급할 수 있고(인프라=산업 기회), **기간망·대형 공공 인프라는 의회 안건으로 공공 건설**(재정·건설형 안건). 전력/운송 부족 시 판매 페널티가 증설 압력이 된다. 발전소 증설의 **발전원 선택(화력 vs 친환경)은 오염 모델(§1-A)과 직결**. 인프라 안건은 agenda_catalog §8.
