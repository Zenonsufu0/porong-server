## 1. 시스템 아키텍처

**플랫폼:** NeoForge 1.21.1 서버 + 클라 모드(자동설치기 배포) — DL-E030.

**구성요소:**
| 요소 | 역할 | 근거 |
|---|---|---|
| **CORE 모드**(커스텀, 서버 권위) | 경제 엔진 — 주민 집계 동학·시장 정산·재정/중앙은행·의회/안건·마을 진화·산출물 박스 처리. **단일 writer** | DL-E035 |
| **KubeJS** | 커스텀 레시피(가공 체인)·Create 큐레이션·시대 게이팅 규칙 | DL-E031·E061·E073 |
| **Create(+Crafts&Additions·Steam'n'Rails)** | 자동화·생산·물류 (CORE가 산출물 박스로 경제에 연결) | DL-E030·E065 |
| **콘텐츠 모드** | 식품(Croptopia+FD)·재료 1개(Mekanism\|IE\|MI)·가구 모드 | DL-E069 |
| **DB(SQL 서버, Postgres/MySQL)** | 영속 저장(엔티티) | DL-E035 |
| **클라 GUI**(모드 Screen) | 네이티브 대시보드, CORE가 패킷 push로 실시간 | DL-E057·E032 |
| **디스코드 봇** | 게임 밖 알림(안건·주가·순위) | DL-E057 |

**데이터 흐름:** CORE(메모리 권위 상태) ⇄ DB(영속) / CORE → 클라(패킷 push) / Create 산출 → 산출물 박스 → CORE / KubeJS = 레시피·게이팅 규칙.

## 2. 시뮬레이션 엔진

**갱신 = 주기 틱**(실시간 N분, 정확 길이 P1) — DL-E075. 매 경제 틱마다 아래 순서로 일괄 갱신(순서는 기획 인과 사슬 따름: 생산→시장→소득→소비→지표).

| # | 단계 | 내용 | 근거 |
|---|---|---|---|
| 1 | 생산 집계 | 산출물 박스 입고분 → 회사 재고·시장 공급 반영 | DL-E029·E026 |
| 2 | 시장 정산 | 품목별 수급 + 유저 호가 → 수요곡선으로 판매량·매출·시세 산정(B2C 완성품 / B2B 중간재) | DL-E026·E027·E067 |
| 3 | 주민 동학 | 가계 흐름(소득→소비→저축→자산배분)·노동/고용/임금·인구(출산·사망·이주) | DL-E038·E040·E041·E044·E045 |
| 4 | 창발 지표 | 오염·범죄율·CPI·실업·GDP·지니·행복도 산정 → `indicator_snapshot` 기록 | DL-E049, indicators |
| 5 | 재정·중앙은행 | 세수(법인·소득·거래·재산)·공공지출·금리·통화량·부채 이자 | DL-E012·E023·E039 |
| 6 | 마을 진화 체크 | 다인자 임계 → 시대/단계 해금(또는 쇠퇴) | DL-E051·E062 |
| 7 | 의회 트리거 | 지표 임계 → 안건 후보. 정기 발의는 하루 1회(틱과 별개 스케줄) | DL-E025 |
| 8 | 영속화·push | DB 주기 저장 + 클라 패킷 push | DL-E035·E032 |

- **거시 사이클은 별도 엔진 없음** — 위 3·4단계의 집계 결과로 창발(인플레·경기·거품). design §8·DL-E049.
- `[질문]` 후보(다음 섹션에서 확인): 산출물 박스 입고 시점(즉시 vs 틱 집계)·메모리↔DB 동기 주기·계약 정산 주기(DL-E028 "구현 시") — §3·§4에서 다룸.

## 3. 데이터 계층

- **DB = SQL 서버(Postgres/MySQL)** — DL-E035. 엔티티 16종은 `data_schema.md` 그대로 구현(player·company·industry·financials·population_stratum·employment·item·market_listing·market_state·shareholding·stock_price_history·contract·techtree_node·treasury·agenda·vote·party·land_plot·deposit·loan·bond·indicator_snapshot).
- **권위·동기:** CORE 메모리 = 런타임 권위 상태, DB = 영속. 클라는 push 받은 사본(표시용).
- **시계열:** `stock_price_history`·`indicator_snapshot`은 틱/주기마다 append(차트·웹용) — DL-E049·E033.
- `[권장 — 확인 요망]` **DB 동기 주기:** 순수 기술 사항(게임 동작 아님). 권장 = **매 틱 변경분 영속화 + 중요 이벤트(거래·도산·시즌 정산) 즉시**. 다른 정책 원하면 알려줘.

## 4. 생산·시장 파이프라인

**생산 (회사 → 산출물):**
- Create 가공으로 생산. 레시피·가용 기계는 **KubeJS + 시대 게이팅**(DL-E031·E061·E062). 채용 주민 없으면 설비 정지(DL-E004), 학력 매칭이 효율·고장률·복구 결정(DL-E047).
- **산출물 박스(입고 시 처리)** — DL-E029·E026: ① 업종-품목 검사(불일치=환금 불가) ② 품질 산정(품질 전파, DL-E064) ③ 오염 유입 기록(DL-E010) ④ 회사 재고 등록. 유저가 **호가(가격) 설정**(DL-E027).

**시장 정산 (매 틱, §2-2) — 2층(DL-E067):**
- **B2C(주민=완성 소비재):** 집계 수요 = f(품질·가격·소득탄력·소비유형[필수/내구/사치]) × 유저 호가 → 판매량·매출·시세. 시장=상점 추상 정산(DL-E026·E027·E066).
- **B2B(회사간=중간재·부품):** 시장 스팟(변동가) 또는 계약. 특화 분업의 통로(DL-E067).
- **주민회사 앵커:** 가격 천장/바닥, 필수재·핵심 원료 최소 공급(붕괴 방지) — DL-E005·E071.
- **가격·시세:** 수급(공급÷수요)에서 형성, 유저 호가 + 수요곡선(DL-E014·E027).

**계약(B2B 채널) — DL-E028:**
- 계약 GUI(공급 레이트·개당 대금·기간·위약금) → 납품박스↔수령박스 연결.
- **정산 = 틱 배치**(DL-E075 일관, DL-E028 해소): 매 틱 이송분 정산(중앙은행 경유·거래세). 공급 미달 자동 감지 → 위약금 차감, 누적 시 파기.

**거래 무결성:** 산출템은 인벤 불가·박스↔박스만 이동(P2P 차단, INBOX-029) → 전 거래 CORE 경유(감사·과세).
