## 1. 시스템 아키텍처

**플랫폼:** NeoForge 1.21.1 서버 + 클라 모드(자동설치기 배포) — DL-E030.

**구성요소:**
| 요소 | 역할 | 근거 |
|---|---|---|
| **CORE 모드**(커스텀, 서버 권위) | 경제 엔진 — 주민 집계 동학·시장 정산·재정/중앙은행·의회/안건·마을 진화·산출물 박스 처리. **단일 writer** | DL-E035 |
| **KubeJS** | 커스텀 레시피(가공 체인)·Create 큐레이션·시대 게이팅 규칙 | DL-E031·E061·E073 |
| **Create(+Crafts&Additions·Steam'n'Rails)** | 자동화·생산·물류 (CORE가 산출물 박스로 경제에 연결) | DL-E030·E065 |
| **콘텐츠 모드** | 식품(Croptopia+FD)·재료 1개(Mekanism\|IE\|MI)·가구 모드 | DL-E069 |
| **DB(SQL 서버, Postgres/MySQL)** | 영속 저장(엔티티) | DL-E035 |
| **클라 GUI**(모드 Screen) | 네이티브 대시보드, CORE가 패킷 push로 실시간 | DL-E057·E032 |
| **디스코드 봇** | 게임 밖 알림(안건·주가·순위) | DL-E057 |

**데이터 흐름:** CORE(메모리 권위 상태) ⇄ DB(영속) / CORE → 클라(패킷 push) / Create 산출 → 산출물 박스 → CORE / KubeJS = 레시피·게이팅 규칙.

## 2. 시뮬레이션 엔진

**갱신 = 주기 틱**(실시간 N분, 정확 길이 P1) — DL-E075. 매 경제 틱마다 아래 순서로 일괄 갱신(순서는 기획 인과 사슬 따름: 생산→시장→소득→소비→지표).

| # | 단계 | 내용 | 근거 |
|---|---|---|---|
| 1 | 생산 집계 | 산출물 박스 입고분 → 회사 재고·시장 공급 반영 | DL-E029·E026 |
| 2 | 시장 정산 | 품목별 수급 + 유저 호가 → 수요곡선으로 판매량·매출·시세 산정(B2C 완성품 / B2B 중간재) | DL-E026·E027·E067 |
| 3 | 주민 동학 | 가계 흐름(소득→소비→저축→자산배분)·노동/고용/임금·인구(출산·사망·이주) | DL-E038·E040·E041·E044·E045 |
| 4 | 창발 지표 | 오염·범죄율·CPI·실업·GDP·지니·행복도 산정 → `indicator_snapshot` 기록 | DL-E049, indicators |
| 5 | 재정·중앙은행 | 세수(법인·소득·거래·재산)·공공지출·금리·통화량·부채 이자 | DL-E012·E023·E039 |
| 6 | 마을 진화 체크 | 다인자 임계 → 시대/단계 해금(또는 쇠퇴) | DL-E051·E062 |
| 7 | 의회 트리거 | 지표 임계 → 안건 후보. 정기 발의는 하루 1회(틱과 별개 스케줄) | DL-E025 |
| 8 | 영속화·push | DB 주기 저장 + 클라 패킷 push | DL-E035·E032 |

- **거시 사이클은 별도 엔진 없음** — 위 3·4단계의 집계 결과로 창발(인플레·경기·거품). DL-E008-X… §8.
- `[질문]` 후보(다음 섹션에서 확인): 산출물 박스 입고 시점(즉시 vs 틱 집계)·메모리↔DB 동기 주기·계약 정산 주기(DL-E028 "구현 시") — §3·§4에서 다룸.
