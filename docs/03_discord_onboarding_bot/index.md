# 03. 디스코드 온보딩 / 봇 계획

> **[STATUS: REFERENCE]** — 디스코드 온보딩과 봇 MVP 참조. 공식 운영 방향은 `../final_master_plan.md`가 우선.

## 운영 방식

디스코드 선입장 방식으로 간다.

```text
디스코드 입장
→ 약관 동의
→ 마크 닉네임 입력
→ 임시 화이트리스트 등록
→ 마크 접속
→ 코드 인증
→ 최종 연동
→ 인증 유저 역할 부여
```

## 역할

- 미인증
- 접속대기
- 인증 유저
- 테스터
- 운영진

권한 역할과 칭호 역할은 분리한다.

## 현재 채널 구조 활용

기존: 서버소개, 규칙, 공지, 업데이트, 접속정보, faq, 잡담, 스크린샷, 파티모집, 문의, 버그제보.

접속정보 채널은 약관 동의 후 역할(접속대기 이상)을 받은 유저에게만 열람 가능하게 설정한다. 서버 IP/도메인은 이 채널에서만 안내한다.

추가 추천:

- 약관-인증
- 필드보스-알림
- 시즌보스-알림
- 운영로그
- 밸런스메모

## 디스코드 봇 MVP

- 약관 동의 버튼
- 닉네임 입력 모달
- 임시 화이트리스트 등록
- `/연동`
- `/프로필`
- `/영지`
- `/보스`
- 필드보스 리젠 알림

## 알림

- 필드보스 5분 전/등장
- 시즌보스 공지
- 가공 완료
- 점검/업데이트
