## 3. 운영 미결 결정사항

운영 담당자가 결정해야 하며 기술 구현과 무관하게 별도 추적한다.

| # | 항목 | 옵션 / 메모 |
|---|---|---|
| O-1 | 실제 오픈일 | 미정 |
| O-2 | 45일 종료 예정일 | O-1 확정 후 결정 |
| O-3 | 종료 후 연장 여부 판단 기준 | 활성 유저 수, 콘텐츠 완료율 등 기준 미정 |
| O-4 | 시즌 종료 후 성장 데이터 초기화 여부 | 초기화 vs 보존 선택 |
| O-5 | 치장/칭호/컬렉션 기록 보존 여부 | O-4와 연동 |
| O-6 | 커피값 후원 안내 시점 | 오픈부터 vs 이후 시점 |
| O-7 | 인증 유저 표시 칭호 최종명 | 예: "개척자" 또는 다른 명칭 |
| O-8 | 채널명 최종 표기 | 약관-인증, 필드보스-알림 등 확정 필요 |

## 4. 기술/콘텐츠 미결 결정사항

구현 전에 반드시 결정해야 하는 항목이다.

| # | 항목 | 옵션 / 메모 |
|---|---|---|
| T-1 | 임시 화이트리스트 만료 기간 | 3일 vs 7일 |
| T-2 | 닉네임 오입력 수정 가능 횟수 | 무제한 vs 3회 등 |
| T-3 | 보스 알림 기본값 | ON vs OFF (유저 설정 override 가능 여부) |
| T-4 | 시즌 최종보스 포함 시점 | 오픈 시 포함 vs 시즌 중 업데이트 |
| T-5 | 모험 도감 / 컬렉션북 UI 최종 명칭 | 두 명칭 중 선택 또는 병기 |
| T-6 | 마력 동력 / 전력 용어 병기 여부 | "마력 동력(전력)" 형태 또는 단독 사용 |
| T-7 | 영지 등급 명칭 최종 확정 | 현재 예시: 개척지/정착지/발전 영지/군수 영지/제국 공방 |
| T-8 | 시즌보스 입장 방식 세부 | 파티 인원 제한, 주간 입장 횟수, 쿨타임 구조 |
| T-9 | 기여도 산정 방식 | 딜 기여도만 vs 힐/버프 포함 가중치 |
| T-10 | 공방 가공기 대기열 한도 | 동시 처리 슬롯 수 |

## 5. 구현 우선순위 요약

```
1순위 (오픈 필수, 수치 확정 없으면 Phase 3~4 진행 불가)
  - 무기별 스킬 쿨타임 / 피해량 공식
  - 강화 성공률 표 / 비용
  - T1 장비 기본 스탯

2순위 (Phase 5~6 진행 전)
  - 영지 레벨업 비용
  - 기계별 마력 소비/생산량
  - 필드보스 / 시즌보스 HP / 보상

3순위 (오픈 직전 밸런스 조정)
  - 골드 드랍량 / 경제 수치
  - 잠재 옵션 풀 및 확률
  - 큐브 수급 밸런스

운영 결정(O-1~O-8)은 기술 구현과 독립적으로 추적한다.
기술 결정(T-1~T-10)은 Phase 2~3 시작 전 순서대로 결정한다.
```
