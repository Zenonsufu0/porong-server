## 기존 포로 서버 기준과의 충돌 점검

CLAUDE.md 원칙에 따라 본 설계가 기존 확정 기준과 충돌할 수 있는 지점을 먼저 나열한다.

- **스킬 키 세팅 축 (2026-04-19 확정)**: 이전 결정 "Q·F·우클릭·Shift+우클릭 + Shift+F + 핫바 9번" 6키 안은 **이번 E 채택으로 일부 덮어씀**. 본 문서는 **Q·E·F·우클릭·Shift+우클릭 5키** 축으로 재정렬한다. Shift+F는 후속 검토용으로 남겨두되 이번 HUD 1차에서는 쓰지 않는다. 충돌 지점이므로 오픈 질문 §9에 명시한다.
- **비전투 상태 바닐라 동작 유지 원칙**: `poro:weapon_type` PDC 태그 단일 게이트로 Q(아이템 버리기) / F(SwapHand) / E(인벤 열기) 바닐라 동작이 유지된다. HUD 레이어 2(스킬 목록)도 **커스텀 무기 장착 시에만 활성화** 규약을 따른다.
- **CustomModelData 7자리 인코딩 / 도메인 예약 구간 (확정)**: UI 글리프는 `A=3` / 3000000~3099999 구간. 본 문서에서 레이어별 하위 예약을 제안하되, 기존 구간은 깨지 않는다.
- **리소스팩 3단계 배포 정책 (확정)**: 자동 → 디스코드 수동 → 체험 월드 강등. **본 HUD는 리소스팩 필수**이므로 체험 월드 강등 플레이어에게 3레이어가 보이지 않는다. 체험 월드 폴백 규약을 §5/§10에 명시한다.
- **월드 사운드 제거 a안 (확정)**: 월드 앰비언트만 제거, **UI 클릭·액션바 사운드 유지**. HUD 알림음은 기존 규약과 동일 범위.
- **전투 상태 판정 (확정)**: 필드 = 최근 피격 10초 이내 / 보스전 인스턴스 = 항상 전투 상태. 본 HUD는 이 판정을 **소비자**로 쓰며 새 정의를 추가하지 않는다.
- **기존 `HealthHudFormatter` / `HealthHudListener`**: 현재 `player.sendActionBar`에 텍스트 형태로 HP 게이지를 그린다. 본 설계는 **바닐라 HP 제거 + 3레이어 병행**을 전제하므로 기존 클래스는 **액션바 단독 렌더가 아닌 "레이어 1(HP 바) 기본 텍스트 폴백" 용도로만 유지**하고, 정식 렌더는 리소스팩 font glyph 오버레이로 교체한다. §7 참고.

충돌 지점 정리: 이전 6키 안 → 5키 축으로 재정렬 + HealthHud 기존 클래스의 역할 축소. 둘 다 본 문서에서 처리 방향만 제시하고 최종 적용은 구현 PR에서 확정한다.

## 1. 기능 개요

바닐라 체력·배고픔 바를 전면 비표시로 돌리고, 화면 하단 중앙을 **아래 → 위 3레이어 세로 구조**의 커스텀 HUD로 재구성한다.

- **레이어 1 (최하단)**: 커스텀 HP 바 (바닐라 하트 치환)
- **레이어 2 (중간)**: 스킬 목록 + 쿨다운 표시 (Q·E·F·우클릭·Shift+우클릭)
- **레이어 3 (최상단)**: 버프·디버프 아이콘 목록

리소스팩 font glyph 오버레이 + Adventure `player.sendActionBar()` / BossBar 보조 조합으로 구현한다. HUD는 **커스텀 무기 장착 중**에만 레이어 2가 활성화되고, 레이어 1·3은 상시 렌더한다.

---

## 2. 목표 (왜 필요한가)

- 바닐라 하트·쉴드·허기 UI는 포로 서버의 고정 HP 20/마력 기반 스탯 구조와 맞지 않는다. 커스텀 스탯(방어력·받는 피해 감소·상태이상 저항 등)이 하트 단위 "칸"으로 환산되지 않아 체감 왜곡이 크다.
- 스킬 바인딩 키 5종(Q·E·F·우클릭·Shift+우클릭)이 확정됐지만, **현재 쿨다운·사용 가능 여부·바인딩 현황을 화면에서 바로 볼 수단이 없다**. 메뉴 진입 없이 1~2초 안에 판단할 수 있어야 한다.
- 버프/디버프 3 카테고리(상태이상·제어·표식)와 일반 버프(음식·연금술·강화 보조)를 한 축으로 묶어야 플레이어가 보스전·필드전에서 리스크를 관리할 수 있다. 현 Shift+메뉴 버프 탭은 "사후 확인" 용도라 실시간성이 부족하다.

### 3.2 플로우

- 로그인 직후: 레이어 1 렌더 시작(풀 HP). 커스텀 무기 미장착이면 레이어 2는 비활성(회색). 버프가 없으면 레이어 3은 빈 프레임(또는 숨김 택1).
- 전투 진입(피격 또는 보스 존): 레이어 1 색상이 HP 구간별 변색. 레이어 2 활성.
- 스킬 사용: 해당 키 아이콘이 쿨다운 숫자/원형 게이지로 전환. 사용 불가는 흐림.
- 버프/디버프 획득: 레이어 3 좌측부터 아이콘 추가. 만료 3초 전부터 아이콘 점멸.

## 4. 핵심 규칙

### 4.1 바닐라 HP·배고픔 비표시 (체감 "제거")

- **배고픔**: `player.setFoodLevel(20)` + `player.setSaturation(20)`을 **주기 스케줄러(1~5초)로 고정 유지**. 허기 바는 기본 HUD에 남지만 "항상 꽉 찬 상태"로 의미를 제거한다.
  - 완전 제거 대안: 리소스팩 font glyph로 `icons.png`의 허기 아이콘 영역을 투명 픽셀로 덮어쓰기. 리소스팩 필수 + 체험 월드 강등자는 바닐라 허기 노출. (권장: 리소스팩 덮어쓰기 + 서버측 포만도 고정 2중)
- **체력**: `player.setHealthScale(0.0)` 호출은 하트 개수를 "0"으로 만들어 HUD에서 제거한다. Paper에서 동작 확인됨. 내부 판정 `player.getHealth()`는 정상 유지.
  - 대안 1: 리소스팩으로 `icons.png` 하트 스프라이트 영역 투명화. `setHealthScale(0)`과 병행하면 "어느 쪽이 실패해도 보이지 않음"이 보장된다.
  - 대안 2: `setHealthDisplay(hidden)` 류의 Paper 옵션은 **버전에 따라 유무**가 갈린다(오픈 질문 §9).
- **기본 원칙**: 서버측 호출(1차 방어) + 리소스팩 글리프 덮어쓰기(2차 방어) **병행**. 한쪽 실패 시 다른 쪽이 커버한다.

### 4.2 레이어 1 — 커스텀 HP 바

- **렌더 매체**: font glyph 오버레이 1장 + Adventure 컴포넌트 1줄.
  - 글리프 1장 = HP 바 배경 + 포맷 + 수치 프리셋. 수치 텍스트는 별도 `Component.text`로 겹침.
  - 바닐라 하트 위치(경험치 바 위)에 **음수 스페이스로 좌표 밀어넣기**. 기존 Shift+메뉴 UI 글리프와 같은 방식 재사용.
- **채널 선택**: `player.sendActionBar()` 사용. 액션바는 포로의 기존 HealthHudFormatter가 이미 쓰던 채널이고, 1초 이내 갱신 주기에서 깜빡임이 가장 적다. BossBar는 보스전 전용이므로 HP 바에 차용하지 않는다.
- **갱신 트리거**:
  - `EntityDamageEvent` / `EntityRegainHealthEvent` (기존 `HealthHudListener` 훅 재활용)
  - `PlayerJoinEvent`
  - `PlayerRespawnEvent`
  - 최대 체력 변동(Attribute) — 각인·세트 효과 갱신 시
  - 보험용 2초 주기 스케줄러(내부 상태 변동 누락 방지)
- **색상 규약**:
  - HP 70% 이상 = 기본색(에메랄드 계열)
  - HP 30~69% = 주황
  - HP 30% 미만 = 빨강 + 가벼운 점멸
  - HP 10% 미만 = 빨강 강점멸 + 액션바 가장자리 warning 글리프
  - 색상은 리소스팩 글리프 4종 세트(`hp_bar_full` / `mid` / `low` / `critical`)로 분리, 서버가 상황에 맞춰 선택.
- **표기 포맷**: `██████████░░░░░░  1,480 / 2,000` (10칸 블록 + 현재/최대 수치). 기존 `HealthHudFormatter`의 `BAR_LENGTH=10` 유지.
- **최대 체력 변동 반영**: 각인·세트 효과로 최대 체력이 바뀌면 칸 수가 아닌 글리프 1장을 유지하고 **수치만 재계산**. 칸 수 변경은 글리프 교체 리스크가 커서 1차 범위 외.

### 4.3 레이어 2 — 스킬 목록 + 쿨다운 표시

- **키 슬롯 5종 고정 순서**: 좌 → 우 `[Q] [E] [F] [우클릭] [Shift+우클릭]`.
- **아이콘 = CustomModelData font glyph**. 직업·무기별 스킬 아이콘은 플레이어 무기의 `poro:skill_enabled` + 슬롯 바인딩 테이블을 참조.
- **상태 표시 규약**:
  - 사용 가능: 풀컬러 + 테두리 강조
  - 쿨다운 중: 회색조 + 남은 초 카운트(소수점 첫자리) + 원형 게이지(글리프 8단계 프레임)
  - 자원 부족(마력/스태미너 부족): 파랑 흐림 + `!` 마크
  - 비활성(해당 키 미바인딩): 빈 슬롯 글리프(점선 테두리)
  - 비전투 + 커스텀 무기 미장착: 레이어 2 전체 **회색 흐림 30%** 또는 숨김(§9 오픈)
- **렌더 채널**: 액션바는 레이어 1이 점유하므로 레이어 2는 **BossBar 경량 채널 1개(title 폭 0, progress 0)를 "빈 bossbar + 글리프 title"로 차용**. 또는 `player.sendActionBar()`를 레이어 1과 레이어 2가 **하나의 긴 컴포넌트**로 공유(멀티라인 글리프 배치, 음수 스페이스로 위·아래로 나눔).
  - 권장 1차: **액션바 단일 컴포넌트**에 레이어 1·2·3을 **음수 스페이스로 수직 배치**. BossBar는 보스전 전용으로 보존.
  - 대안: Paper 1.21.10의 `Title` subtitle slot 활용 (1초마다 리프레시 비용 필요).
- **쿨다운 데이터 소스**: EmpireRPG 스킬 엔진이 제공하는 `SkillCooldownService`(미구현 — 오픈 질문 §9)가 `(playerUuid, skillId) → remainingTicks`를 반환. 본 HUD는 **소비자**이며 엔진을 새로 만들지 않는다.
- **비전투 상태 Q/F/E 바닐라 동작 보장**: 레이어 2가 회색/숨김일 때 Q/F/E 키 이벤트는 **가로채지 않는다**(listener에서 early return). 커스텀 무기 장착 시에만 이벤트 후킹 활성.

### 4.4 레이어 3 — 버프/디버프 아이콘

- **최대 표시 수 1차 = 10칸**. 초과 시 `+N` 글리프로 말줄임.
- **정렬 순서 1차 = 만료 임박 순**. 곧 끝나는 버프가 좌측, 방금 획득한 오래 남은 버프가 우측. (대안: 중요도 순, 획득 순 — §9 오픈)
- **카테고리 색상**:
  - **버프**: 녹색 테두리
  - **디버프 — 상태이상(DEBUFF_STATUS)**: 보라 테두리
  - **디버프 — 제어(DEBUFF_CONTROL)**: 빨강 테두리
  - **디버프 — 표식(DEBUFF_MARK)**: 주황 테두리
  - 기존 `10_seed_and_config_tables` 상태 registry 4 카테고리 그대로 재사용.
- **지속 시간 표시**:
  - ≥ 10초: 숫자 없이 아이콘 채도 유지
  - 3~10초: 초단위 숫자 카운트
  - < 3초: 숫자 + 테두리 점멸
  - 영구/충전식(예: 데카): `∞` 또는 스택 숫자 표기
- **스택**: 우측 하단 작은 숫자 글리프로 표시(예: 데카 3, 출혈 5).
- **클릭/호버 툴팁**: 1차 범위 외. 툴팁은 Shift+메뉴 버프 탭에서 제공(기존 규약).
- **렌더 채널**: 레이어 1·2와 합쳐 **액션바 단일 멀티라인 컴포넌트**로 구성. 아이콘은 font glyph 1개 = 1 버프.
- **갱신 주기**: 1초 스케줄러 + `PlayerEnhanceBuffAppliedEvent` / `PlayerEnhanceBuffConsumedEvent` 등 기존 이벤트 훅 재활용. 파티·음식 버프 추가 시 같은 이벤트 시그니처 확장.

### 4.5 레이어별 CustomModelData 예약 구간 제안

`A=3` UI 글리프 네임스페이스 내부에서 3-layer HUD 하위 공간을 아래와 같이 예약한다. 기존 `3001001` 등은 유지하고, HUD는 **3002000번대 블록**을 새로 받는다.

- **레이어 1 (HP 바)**: `3002000 ~ 3002099`
  - `3002000` = `hp_bar_full` 배경 글리프
  - `3002001` = `hp_bar_mid` (주황)
  - `3002002` = `hp_bar_low` (빨강)
  - `3002003` = `hp_bar_critical` (강점멸)
  - `3002010 ~ 3002019` = 칸 채움 블록(0~10칸)
  - `3002020 ~ 3002029` = 수치 텍스트 프레임 예비
- **레이어 2 (스킬 바)**: `3002100 ~ 3002299`
  - `3002100` = 스킬 바 배경 프레임
  - `3002101 ~ 3002105` = 슬롯 테두리(Q/E/F/우/S우)
  - `3002110 ~ 3002119` = 원형 쿨다운 게이지 8단계 프레임
  - `3002200 ~ 3002299` = 직업·무기별 스킬 아이콘 세트(전사 3무기 × 5슬롯 × 예비)
- **레이어 3 (버프/디버프)**: `3002300 ~ 3002499`
  - `3002300` = 버프 아이콘 슬롯 프레임(공통)
  - `3002301 ~ 3002304` = 카테고리 테두리(buff/status/control/mark)
  - `3002310 ~ 3002319` = 숫자 카운트 글리프 0~9
  - `3002400 ~ 3002499` = 개별 버프·디버프 아이콘(기존 상태 registry 1:1 매핑)
- **공통 폰트 오프셋**: `3002900 ~ 3002999` = 음수 스페이스·세로 정렬 보정용 예비 글리프

전체 예약은 기존 `3001xxx` 구간(월렛 UI 등)과 겹치지 않는다. CSV 레지스트리(`poro_custom_model_data_registry.csv`)에 등록 필요.

## 9. 오픈 질문

- **E 키 vs Shift+F 재확정**: 2026-04-19 상위 결정은 "R 대체 = E 확정"이지만, 같은 날짜 스킬 키 묶음에 "Shift+F 대체"도 병존 기록이 있다. 어느 쪽이 최신인지 마스터플래닝에 단일 기록으로 고정 필요.
- **비전투 상태 레이어 2 표시 방식**: 회색 흐림 30% vs 완전 숨김 — 어느 쪽이 포로 "HUD 미니멀" 기조에 맞는가.
- **레이어 3 정렬 기본값**: 만료 임박 순(권장) vs 중요도(카테고리 가중) vs 획득 순. 1차 기본값 확정 필요.
- **레이어 3 최대 표시 수 10칸이 가로 폭(182px)에 실제로 들어가는가** — 아이콘 14px × 10개 + 간격 2px = 160px로 계산상 여유. 실측 필요.
- **바닐라 HP 제거 방식 호환성**: `setHealthScale(0)`이 Paper 1.21.10 + 타 플러그인(Citizens / MythicMobs / BossBar용 플러그인)과 충돌 없는지 검증.
- **리소스팩 미적용 유저 UX**: 체험 월드 강등자에게 레이어 2·3을 완전 숨기고 "리소스팩 안내 1줄"만 보여줄지, 아니면 기존 `HealthHudFormatter` 텍스트 게이지만 보여줄지.
- **`SkillCooldownService` 실서비스 상태**: 현재 EmpireRPG에 구현되어 있는지, 본 PR에서 스텁을 써야 하는지. 없으면 스킬 엔진 트랙 선행 필요.
- **기존 `HealthHudListener`/`HealthHudFormatter` 처리**: 폴백 유지(권장) vs 전면 삭제 후 신규 모듈 단일화 중 어느 쪽을 1차에 쓸 것인가.
- **보스전 진입 시 레이어 1·3 자동 축소**: 보스전 BossBar와 시각 충돌 방지를 위해 투명도 70% 자동 적용이 필요한가.
- **허기 시스템의 향후 재해석**: 포만도 고정 정책이 시즌1 내내 유지되는가(스태미너 기획 트랙과의 관계).
