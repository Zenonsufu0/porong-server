# 강화 보조 물약 ENHANCE_SLOT 버프 UI 표시 규격 (초안 v0.1)

## 1. 기능 개요
강화 보조 물약(장인의 집중제/숙련된 집중제/장인의 영약)은 ENHANCE_SLOT 단일 슬롯을 점유하는 1회 소모형 버프다. 플레이어가 언제 버프를 사용했고, 강화 시 실제 적용 성공률이 얼마인지, 95% 상한에 걸렸는지를 직관적으로 파악할 수 있도록 UI 표기 규격을 정의한다. 월드 사운드가 제거된 포로 서버 UX 원칙에 따라 UI·파티클·액션바·토스트 중심으로 연출한다.

## 2. 왜 필요한가
- 물약은 비용을 지불하고 소모하는 자원이므로, "지금 버프가 켜져 있는가"를 명확히 알려야 한다.
- 95% 상한 클램프 때문에 저강 구간에서는 버프 효과가 일부 무효화될 수 있다. 이 사실을 강화 직전에 미리 보여줘야 환불 항의가 줄어든다.
- Shift+좌클릭 메뉴, HUD, 강화 패널 세 영역이 이미 존재하므로, 어디에 표시할지 기준을 먼저 정해두지 않으면 정보가 중복되거나 누락된다.

## 3. ENHANCE_SLOT 버프 상태 표시 위치 (3안 비교)

### (A) 강화 패널 내부 상단 배지
- 장점: 강화 행위와 정보가 같은 화면에 있어 맥락이 명확하다. HUD 공간을 차지하지 않는다.
- 단점: 강화 패널을 열기 전까지 버프 활성 여부를 알 수 없다. 버프를 켠 채로 이동·전투하다 깜빡 잊고 시간이 흐를 수 있다.
- UX 정합성: 중간.

### (B) 상시 HUD 우측 배지
- 장점: 항상 보이므로 "내가 지금 버프 중이다"를 잊지 않는다. 등급별 아이콘 색(NORMAL 은색 / ADVANCED 청색 / RARE 금색)으로 즉시 구분된다.
- 단점: HUD에 상시 요소가 하나 더 늘어 피로도가 커진다. 보스전 BossBar, 체력·허기 바와 시각 충돌 가능.
- UX 정합성: 포로는 "HUD 미니멀, 필요한 순간에만 알림" 원칙이므로 상시 노출은 부담.

### (C) Shift+좌클릭 메뉴 `캐릭터 > 버프` 탭
- 장점: 기존 메뉴 체계에 자연스럽게 편입된다. 장기 버프(향후 음식/문양 등)와 함께 한 곳에서 관리 가능.
- 단점: 메뉴를 열어야 확인 가능해 즉시성이 떨어진다. 단독으로는 부족하다.
- UX 정합성: 높음. 다만 단독 채택은 부적합.

### 추천안: (A) + (C) 병행, (B)는 보류
강화 패널 상단 배지로 즉시성을 확보하고, Shift+메뉴 버프 탭으로 전체 조회 기능을 보장한다. 버프 활성/소비 순간에는 액션바 1줄 알림으로 보완한다. HUD 상시 배지는 1차 범위에서 제외하고 플레이어 피드백을 본 뒤 재검토한다.

## 4. 강화 UI 내부 표기 규격
강화 패널(로웬·테마 장인 대화 또는 별도 GUI) 상단 정보 영역에 아래 순서로 표기한다.

- 기본 성공률: `현재 기본 성공률: 45%`
- 버프 비활성: `강화 보조 물약: 없음`
- 버프 활성: `장인의 영약 적용 → 54% (×1.20)` 형식으로 적용 후 수치와 배수를 함께 노출
- 상한 클램프: 기본×배율이 95%를 초과하면 `95% 상한 도달 — 물약 효과 일부 무효` 회색 경고 1줄 추가
- 소모 경고: 강화 버튼 위에 `이 강화 시도 후 물약 효과는 소비됩니다` 상시 표기
- 배치 강화(연속 n회): `물약 효과는 1회차에만 적용됩니다` 문구를 배치 설정 옵션 옆에 고정 노출

## 5. 버프 소멸 시점 및 알림
- 물약 사용 순간: 자동 적용(추가 확인 팝업 없음). 액션바 `장인의 영약 활성 — 다음 강화 1회에 적용`.
- 강화 시도 직전: 별도 확인 팝업 없음. 강화 패널 상단 배지 + 강화 버튼 툴팁으로 충분.
- 강화 성공/실패 직후: 액션바 `강화 보조 물약 효과 소비됨` 1.5초 노출.
- 강화 패널 종료 후 버프 유지: 유지한다. 플레이어가 패널을 닫고 재료를 더 가져오는 동선을 고려한 결정이다. 로그아웃 시 유지 여부는 오픈 질문.

## 6. 중복 사용 / 교체 UX
- 이미 버프 활성 중 신규 물약 사용 시도: 확인 팝업 `기존 [장인의 집중제] 효과를 [장인의 영약]으로 교체하시겠습니까?` 노출.
- 교체 승인 시 기존 물약은 소비 처리. 환불 없음.
- 동일 등급 물약 재사용: `이미 같은 효과가 활성 중입니다` 액션바 안내, 소비 없음.
- 수동 해제 기능: 1차 버전에서 제공하지 않는다. 버프 제거는 강화 시도 또는 교체로만 발생.

## 7. DB 플래그 / 이벤트 연계
플래그 저장소 v0.1 `player.*` 루트를 준수한다.

- `player.enhance_buff.active_consumable_id` (문자열, 없을 시 null)
- `player.enhance_buff.applied_at` (epoch millis)
- `player.enhance_buff.multiplier` (실수: 1.05 / 1.10 / 1.20)
- `player.enhance_buff.grade` (NORMAL / ADVANCED / RARE)

이벤트:
- `PlayerEnhanceBuffAppliedEvent`: 물약 사용 성공 직후 호출. 교체 케이스에서도 호출된다.
- `PlayerEnhanceBuffConsumedEvent`: 강화 시도 판정 직후, 성공/실패와 무관하게 호출.
- 두 이벤트 모두 Bukkit 이벤트 버스로 노출해 디스코드 봇·웹 조회 모듈이 구독 가능하게 한다.

## 8. 파티클·사운드 연출
- 버프 적용: 플레이어 주변 금빛 광휘(RARE) / 연청색 아지랑이(ADVANCED) / 은색 반짝임(NORMAL) 1.5초.
- 강화 시도 시 성공: UI 내부 축하 이펙트 + 액션바 녹색 강조.
- 강화 시도 시 실패: UI 내부 감쇠 이펙트(회색 파편) + 액션바 회색 톤.
- 월드 사운드 금지 원칙 준수. 허용되는 것은 UI 클릭음(GUI 내부), 액션바 텍스트, 화면 토스트뿐.

## 9. 파티·영지 공유
- 파티 공유: 1차 버전에서는 개인 한정. 공유는 추후 검토.
- 영지 올인원 조합대 대량 제작: 1차에서는 비범위. 물약 제작은 연금술 라인(올인원 조합대 연금 Lv 7/11/15)에서 담당.
- 거래 정책: NORMAL/ADVANCED 거래 가능, RARE 귀속은 "귀속 유력"으로 표기하되 밸런스 검토 전까지 확정하지 않는다.

## 10. 범위 (1차 버전)
- ENHANCE_SLOT 단일 슬롯 버프 적용/소비/교체 로직.
- 강화 패널 상단 배지 + Shift+메뉴 버프 탭 표시.
- 액션바 알림 3종(적용·교체·소비).
- 플래그 저장 및 두 이벤트 발행.
- 기본 파티클 연출 3등급.

## 11. 비범위 (1차 버전)
- HUD 상시 배지.
- 파티 공유, 영지 대량 제작.
- 수동 해제, 지속시간 제한(현재는 "다음 강화 1회"까지 유지).
- 버프 히스토리 로그 웹 조회 UI.

## 12. 선행조건
- 플래그 저장소 v0.1 `player.*` 루트 운영.
- 강화 패널(로웬/테마 장인) 공통 GUI 컴포넌트에 상단 정보 영역 슬롯 확보.
- Shift+좌클릭 메뉴 `캐릭터 > 버프` 탭 스켈레톤.

## 13. 구현 우선순위
1. 플래그 4종 + 이벤트 2종 정의.
2. 물약 아이템 사용 핸들러(적용/교체 로직).
3. 강화 판정 훅에서 버프 읽기·소비·클램프 처리.
4. 강화 패널 상단 배지 + 성공률 표기 문자열.
5. 액션바 알림 3종.
6. Shift+메뉴 버프 탭 연동.
7. 파티클 연출 3등급.

## 14. 오픈 질문
- 로그아웃 시 버프 유지 여부(유지/초기화 중 어느 쪽이 포로 운영 기조에 맞는가).
- 배치 강화에서 "1회차 적용" 외에 "성공할 때까지 유지" 옵션을 장기적으로 열 것인가.
- RARE 등급 귀속 확정 여부와 제작 난이도.
- 향후 음식·문양 버프가 추가될 때 `캐릭터 > 버프` 탭 정렬 기준.
- 95% 클램프로 효과가 일부 무효화된 경우, 물약 일부 환불 정책을 둘 것인가(현재는 금지 권장).
