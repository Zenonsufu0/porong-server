# 1막 커스텀 UI 초안 — 테마 보드 · 메인/서브 트래커

## 개요
이 문서는 1막 시점에 플레이어가 마주하는 **커스텀 UI** 초안이다.
1막 스토리라인 진행을 한 눈에 보여주기 위한 화면 설계 + 공용 대화 UI 아키텍처를 함께 다룬다.
(2026-04-19: 기존 독립 문서 `poro_dialogue_ui_architecture_draft.md`를 본 문서 §대화 UI 아키텍처 섹션으로 흡수 — 원본 파일 삭제 완료.)

## 목표
- 7개 테마 중 하나를 선택하는 "테마 선택 보드"를 명확히 한다.
- 1막 메인 11단계 + 테마 초입 메인 + 서브 2개의 진행도를 한 화면에 표시한다.
- HUD 우측에 상시 노출되는 축약 트래커를 정의한다.

## 핵심 규칙
- 1막 UI는 **테마 선택 보드**, **스토리 보드**, **서브 트래커 HUD**의 3개 레이어로 구성한다.
- 3개 레이어 모두 토글 가능해야 하며, 전투 중에는 반투명/자동 축소로 전환한다.
- 테마 선택 보드는 `메인 7. 갈라진 징조` 완료 후에만 열린다.

---

## 1. 테마 선택 보드

### 구성
- 중앙에 제국 지도 이미지 (7개 테마 위치 아이콘)
- 각 테마 아이콘에 hover/focus 시 우측 패널에 상세 표시
- 하단에 "선택 확정" / "취소" 버튼

### 표시 정보 (테마당)
- 테마 이름, 한 줄 감각 (예: "동부: 길도, 형상도 거짓이 되어가는 숲")
- 초입 필드 난이도 (별 1~5)
- 핵심 기믹 태그 (분신판별 / 가짜쉼터 / 과열 등)
- 명성 / 길드 신뢰 현황
- "선택 중" 하이라이트

### 텍스트 목업
```
┌─ 파견 지도 ────────────────────────────────────────┐
│                                                    │
│     [북부 설혼]                                    │
│          ▲                                         │
│  [서부] ──┼── [수도]* ── [동부]                    │
│          │                                         │
│     [남부 화맥]     [사하르]     [아르카논]        │
│                                                    │
├── 선택 정보 ───────────────────────────────────────┤
│  동부 침식 숲                                      │
│  "길도, 형상도 거짓이 되어가는 숲"                 │
│  난이도: ★★★☆☆    핵심: 분신판별 / 침식           │
│  명성 0%  |  길드 신뢰 5%                          │
│                                                    │
│        [ 선택 확정 ]     [ 취소 ]                  │
└────────────────────────────────────────────────────┘
* = 수도 허브(현재 위치)
```

## 2. 스토리 보드 (메인 메뉴 내부)

### 구성
- 좌측: 1막 메인 진행도 (11단계 체크리스트)
- 중앙: 선택한 테마의 초입 메인 진행도 (7단계 체크리스트)
- 우측: 활성 서브 2개 카드 (진행도 바, 완료 조건 3줄 요약)

### 진행도 표시
- `●` 완료 / `◐` 진행 중 / `○` 미시작
- 단계별 보상 미리보기 (명성 %, 신뢰 %, 재화)

### 텍스트 목업
```
┌─ 1막 스토리 보드 ─────────────────────────────────────────────┐
│ [수도 메인]            | [동부 초입]         | [서브 활성]    │
│ ● 낯선 수도            | ● 열차 하차         | ◆ 같은 길…     │
│ ● 조사자의 자격        | ● 숲지기 브리핑     |   진행 2/3     │
│ ● 정착의 권리          | ◐ 침식 수림 외곽    | ◆ 수액을 마신… │
│ ● 살아남는 방식        | ○ 필드보스 조우     |   진행 1/5     │
│ ● 표시된 길            | ○ 장비 15강+        |                │
│ ● 준비된 자만 떠난다   | ○ 첫 현장 보고      |                │
│ ● 갈라진 징조          |                     |                │
│ ● 파견 열차            |                     |                │
│ ◐ 첫 현장 보고         |                     |                │
│ ○ 장비를 벼려라        |                     |                │
│ ○ ???의 편지           |                     |                │
└────────────────────────────────────────────────────────────────┘
```

## 3. 서브 트래커 HUD (상시 노출)

### 구성
- 화면 우측, 미니맵 아래
- 최대 2줄 (활성 서브 2개) + 1줄 (현재 메인 단계)
- 각 줄은 아이콘 + 축약 텍스트 + 진행도

### 텍스트 목업
```
  ▼ MAIN: 첫 현장 보고 (일반 12/20, 부산물 4/6)
  ◆ SUB : 같은 길, 다른 얼굴 (지점 2/3)
  ◆ SUB : 수액을 마신 나무들 (고목 1/5)
```

### 규칙
- 전투 중 3초 이상 피격 시 자동 반투명 50%
- 보스 UI 또는 대화 UI 활성 시 자동 숨김
- `/hud sub` 슬래시 커맨드로 토글

---

## 4. 상호작용 흐름

1. `메인 7. 갈라진 징조` 완료 시 테마 선택 보드가 강제 팝업
2. 테마 확정 시 열차 NPC에게 자동 텔레포트는 하지 않음 (스스로 이동)
3. 초입 비석 활성화 시 스토리 보드 자동 오픈 (1회성)
4. 서브 수락은 기존 BetonQuest 대화 UI를 통해 진행, HUD에만 반영

---

## 5. 범위
- 1막 한정 커스텀 UI 3종(테마 보드 / 스토리 보드 / 서브 트래커 HUD)
- 기존 대화 UI, 보스 허브 UI와의 역할 분리

## 6. 비범위
- 2막 이후 UI (표층 조사/세력 제안 UI는 별도 문서)
- 미술 톤, 아이콘 컬러팔레트, 폰트 선정
- 성능 최적화 수치 (렌더 주기, 동기화 간격)

## 7. 구현 포인트
- **렌더링 주체**: `empire-rpg` 플러그인 내부 `Act1UiService`를 신설해 3종 UI를 관장한다.
- **데이터 소스**: 메인/서브 진행도는 BetonQuest 변수 + 자체 KV 저장소의 해시키 조합.
- **테마 선택 이벤트**: 확정 시 `Act1ThemeChosenEvent`를 발행해 열차 안내관·비석 활성화와 연결.
- **UI 구성 기술**: Bukkit 인벤토리 GUI + 타이틀바 + Bossbar 조합 (ProtocolLib 최소화).
- **HUD 토글**: `/hud sub`, `/hud main`, `/hud board` 3개 슬래시 커맨드.

## 8. 오픈 질문
- 테마 선택 확정 후 **재선택 가능 여부** (한 번 확정하면 1막 내 고정? 아니면 1회 변경 허용?)
- 스토리 보드에 "추천 서브" 목록을 자동 띄울지, 플레이어가 수동 고정하게 할지
- HUD에서 서브 2개 초과 시 표기 (대기열 방식? 상위 2개만 노출?)
- 디스코드 봇에 동일 보드를 `/act1board` 명령으로 공개할지 여부 (기존 open-access 원칙과 정합)

## 9. 대화 UI 아키텍처 (구 `poro_dialogue_ui_architecture_draft.md` 흡수, 2026-04-19)

### 9-1. 목표
Citizens NPC 클릭 시 **채팅형 대화가 아니라 EmpireRPG 커스텀 UI**가 열리도록 전환한다.

### 9-2. 핵심 원칙
- Citizens는 NPC 실체와 클릭 진입점만 담당
- EmpireRPG가 대화 UI·선택지·트리거·퀘스트 시작을 담당
- 대화는 **seed 기반**으로 관리
- 대화 종료 후 quest / estate / boss entry 등 trigger를 EmpireRPG가 직접 실행

### 9-3. 추천 패키지 구성
- `ui.dialogue`
- `ui.framework`
- `dialogue.seed`
- `dialogue.session`
- `dialogue.trigger`

### 9-4. 핵심 클래스
- `DialogueUiService`
- `DialogueSessionService`
- `DialogueRegistry`
- `DialogueNode`
- `DialogueChoice`
- `DialogueTriggerResolver`
- `DialogueUiStateStore`

### 9-5. Seed 필드
- `dialogue_id`
- `npc_seed_id`
- `node_id`
- `speaker_name`
- `portrait_id`
- `text`
- `next_node_id`
- `choice_text`
- `choice_next_node_id`
- `finish_trigger_type`
- `finish_trigger_value`

### 9-6. 1차 구현 우선순위
- 수도 레오니드 대화
- 영지 관리관 대화
- 보스 입장 NPC 대화

### 9-7. 한 줄 요약
**대화는 Citizens 클릭 → EmpireRPG Dialogue UI open → 선택지 진행 → finish trigger 실행 구조로 통일한다.** BetonQuest 제거 + 엘마르 4탭 `dialogue_tab` / `dialogue_node` / `dialogue_choice` / `npc_tab_state` 스키마와 정합.
