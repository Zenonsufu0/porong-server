## 3. 영지 시설 (3종)

### 3.1 시설 배치 방식

- **물리 블럭 설치 없음** — 영지설정 GUI에서 슬롯 클릭으로 배정
- 배정 결과는 DB `island_facility_slots` 테이블에 저장
- EmpireRPG `MachineProductionScheduler`가 20분마다 DB 기준으로 생산 처리

### 3.2 시설 종류

| 시설 | item_id | 생산 주기 | 최대 레벨 | 생산물 |
|---|---|---|---|---|
| 약초 재배지 | `estate_herb_plot` | 20분 | 3 | 제국 약초 / 제국 정수(레어) |
| 광물 채굴기 | `estate_ore_extractor` | 20분 | 3 | 마도철 원석 / 은 원석(레어) |
| 공방 가공기 | `estate_workshop` | 즉시(대기열) | 1 | 레시피 결과물 |

### 3.3 레벨별 산출량

**약초 재배지**

| 레벨 | 제국 약초/사이클 | 레어 보너스 (제국 정수) | 해금 조건 |
|---|---|---|---|
| 1 | 2~3개 | — | 개척지 시작 |
| 2 | 3~4개 | 10% 확률 1개 | 남작령 자동 |
| 3 | 4~6개 | 30% 확률 1개 | 백작령 자동 |

**광물 채굴기**

| 레벨 | 마도철 원석/사이클 | 레어 보너스 (은 원석) | 해금 조건 |
|---|---|---|---|
| 1 | 2~3개 | — | 개척지 시작 |
| 2 | 3~4개 | 10% 확률 1개 | 남작령 자동 |
| 3 | 4~6개 | 30% 확률 1개 | 백작령 자동 |

> 레어 보너스는 기본 산출물 대체 아님 — 사이클마다 독립 확률로 추가 지급

### 3.4 공방 가공기

- 1대 = 대기 슬롯 3개
- 슬롯 1개당 동일 레시피 최대 8세트 대기
- 대기열 선입선출(FIFO)
- 재료 차감: 대기열 등록 즉시 (§14.0 창고→인벤 순 적용)
- 완료 결과물: **영지 창고 자동 입금**
- 오프라인 완료 시 다음 접속 때 알림 전달

## 4. 영지 저장고

- DB 기반 가상 저장고 (물리 상자 없음)
- `island_storage` 테이블에 `{player_uuid, item_id, quantity}` 형태 저장

| 조작 | 동작 |
|---|---|
| 좌클릭 | 64개 출금 (잔량 < 64이면 잔량 전부) |
| 우클릭 | 1개 출금 |
| 전체 입금 버튼 | 인벤토리 전체 아이템 입금 (바닐라 아이템 포함) |
| 자동 입금 | 편의 해금 후 습득 시 자동 저장 |

## 6. 기본 광물 생성기

플레이어가 **울타리 + 빈칸 + 물**을 직접 설치해 구성하는 바닐라 광물 드랍 시설.  
물이 내려와 울타리에 닿을 때마다 아래 확률표에서 광물 1개 생성.  
영지 작위에 따라 확률 변동 (EmpireRPG `BlockFormEvent` 가로채 커스텀 드랍 처리).

| 아이템 | 개척지 | 기사령 | 준남작령 | 남작령 | 자작령 | 백작령 | 후작령 | 공작령 |
|---|---:|---:|---:|---:|---:|---:|---:|---:|
| 조약돌 | 45% | 38% | 32% | 26% | 20% | 15% | 7% | 5% |
| 구리 원석 | 20% | 20% | 18% | 16% | 13% | 11% | 7% | 5% |
| 철 원석 | 17% | 18% | 20% | 21% | 20% | 18% | 14% | 12% |
| 금 원석 | 5% | 7% | 9% | 12% | 15% | 18% | 22% | 20% |
| 레드스톤 | 5% | 7% | 8% | 10% | 12% | 13% | 15% | 15% |
| 청금석 | 4% | 5% | 7% | 8% | 10% | 11% | 13% | 13% |
| 다이아몬드 | 3% | 4% | 5% | 5% | 7% | 10% | 13% | 17% |
| 에메랄드 | 1% | 1% | 1% | 2% | 3% | 4% | 9% | 13% |

**자작령 이상 — 다이아/에메랄드 생성 시 보너스 드랍**

다이아 또는 에메랄드 생성 시, 석탄·금 원석·철 원석·레드스톤·청금석 중 1종 랜덤 추가 지급.

| 행운 레벨 | 보너스 수량 |
|---|---:|
| 없음 | 1개 |
| 행운 I | 1~2개 |
| 행운 II | 2~3개 |
| 행운 III | 3~5개 |

## 7. DB 스키마 (영지 관련)

```sql
-- 영지 메타
island_territory (
  player_uuid     TEXT PRIMARY KEY,
  rank            TEXT,   -- PIONEER / KNIGHT / BARON_JUNIOR / ... / DUKE
  storage_cap     INTEGER,
  upgrade_ts      INTEGER
)

-- 시설 슬롯 배정
island_facility_slots (
  player_uuid   TEXT,
  slot_index    INTEGER,   -- 0~17
  facility_type TEXT,      -- HERB_PLOT / ORE_EXTRACTOR / WORKSHOP / EMPTY
  facility_level INTEGER,
  last_produced_ts INTEGER,
  PRIMARY KEY (player_uuid, slot_index)
)

-- 저장고
island_storage (
  player_uuid   TEXT,
  item_id       TEXT,
  quantity      INTEGER,
  PRIMARY KEY (player_uuid, item_id)
)

-- 공방 대기열
island_workshop_queue (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  player_uuid   TEXT,
  slot_index    INTEGER,   -- 공방 가공기 슬롯 번호
  recipe_id     TEXT,
  queued_sets   INTEGER,
  queued_ts     INTEGER,
  completed_ts  INTEGER    -- NULL이면 미완료
)
```

## 8. 생산 스케줄러 (`MachineProductionScheduler`)

- BukkitScheduler `runTaskTimerAsynchronously` — 20분(24,000 tick) 주기
- 온라인 플레이어만 처리하는 게 아니라 **오프라인 포함 전체 유저** 처리
- `last_produced_ts` 기준으로 밀린 사이클 계산 (최대 3사이클까지 보상)
- 생산 결과 `island_storage` INSERT/UPDATE
- 저장고 용량 초과분은 드랍 없이 버림 → 접속 시 경고 메시지

## 9. 재화 소모 공통 규칙 (전역 정책, §14.0)

모든 시스템(강화·큐브·공방·상점 등)에서 아이템 재료 소모 순서:

1. **영지 창고** 잔량 먼저 차감
2. 창고 잔량 부족 → **플레이어 인벤토리**에서 추가 차감
3. 합산 후에도 부족 → `§c재료가 부족합니다. (창고 + 인벤토리 합산)` 출력, 동작 중단

> 골드는 플레이어 골드 잔액에서만 차감 (창고 개념 없음)
