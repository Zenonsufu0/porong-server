# 포로 서버 영지 전력 시스템 도입 (확정)

## 문서 목적
팩토리오 스타일 전력 시스템을 **도입 확정** 후 v0.1 스프린트 실행 경로 정리. 반대 의견은 참고 메모로 보존.

## 사용자 결정 (확정)
**도입 방향 채택**. 반려 의견(포로 핵심 루프 초점 분산 우려) + 대안(영지 만족도 단일 게이지)은 참고로 보존하되, 본 문서의 v0.1 범위로 **실제 구현 경로 확정**.

---

## 1. 도입 장점
- **공장 라인 게임 레이어 격상** — 영지가 "단순 채집기 배치"에서 "전력 설계 게임"으로 확장
- 플레이어의 **영지 최적화 동기** 증가 (시간당 발전량·설비 균형)
- 테마별 발전기 보너스로 지역 정체성 강화
- 기존 설비 `에너지` 컬럼 **재해석 가능** — 새 데이터 모델 불필요

## 2. 도입 반대 의견 (반드시 고려)
- **포로 핵심 루프 = 전투·장비·강화**. 영지는 보조 → 영지 팩토리오화 시 **초점 분산**
- 전력 부족이 T1 강화석 공급 차단 → 전투 콘텐츠 진행까지 막혀 **이탈 가속**
- 팩토리오 스타일 **초보자 진입 장벽** 상승
- 영지 그리드 칸이 발전기 2~4대 추가로 소모 → 기존 12기 설비와 칸 경쟁

## 3. 대안 — "영지 만족도" 단일 게이지
**팩토리오 전력 대신 NPC·식량·에너지를 하나의 추상 게이지로 통합**.
- 구현 난이도 **1/3 수준**
- 만족도 높음 → 모든 설비 +10% 속도 / 낮음 → 모든 설비 -10% 속도
- 단일 게이지 관리로 UX 단순
- 세밀한 설계 게임성은 포기

## 4. 도입 확정 시 v0.1 범위

### 전력 생성기 2종 (v0.1)
| 발전기 | Lv1 발전 | Lv5 발전 | 설치 비용 | 특징 |
|---|---|---|---|---|
| 기본 화력 | 8 | 28 | 철 32, 2,000G | 석탄·목재 연료 소모, 모든 테마 공통 |
| 풍차 | 5 | 20 | 가공 목재 48, 철 16, 3,000G | 연료 불필요, **북부 설혼 +25%** |

v0.2 확장: 수력·마력 발전기. v0.3: 태양광·지열.

### 전력 네트워크 = 가상 링크 (v0.1)
- 좌표 등록만, 물리 블록 없음
- 영지 범위 내 발전기·설비 자동 한 네트워크 (연결 거리 제한 없음)
- 데이터 모델: `estate_id`별 발전량 합계 vs 소모량 합계 1쌍
- 커스텀 경로 블록 배선은 v0.2 이후

### 전력 부족 시 동작
- **우선순위 배분**: 1단계 채집 > 2단계 가공 > 3단계 제작 > 4단계 창고
- 가동 여부 **0/1 이원화** (완전 가동 vs 완전 중단, 50% 감속은 v0.2)
- 창고는 전력 부족해도 슬롯만 유지 (에너지 미사용 원래 규칙)
- 수동 배분은 v0.3 이후

### 집계 주기 = **분 단위** (1Hz 또는 드물게)
- 틱 단위(20Hz) = 100명 × 36 설비 × 20 = 초당 72,000회 연산 → 부하 크고 체감 차이 없음
- 분 단위 = 초당 수십 회 연산 → I/O 없으면 1ms 미만
- 설비 생산이 초 단위(3~90초)이므로 분 단위 집계로도 충분한 정확도

### 오프라인 정책 = **면제 (옵션 A)**
- 오프라인 시 전력 체크 면제, 모든 설비 정상 가동
- "오프라인 = 이상적 조건" 프레임
- 전력 관리는 **온라인 전투·파밍 동기**
- 옵션 B(오프라인 체크 적용)는 로그인 후 "아무것도 안 모였네" 체감 악화 → 비권장

## 5. 기존 설비 `에너지` 컬럼 재해석

설비 스펙 문서(`poro_factory_facility_spec_draft.md`) 기준:
- 광석 채집기 Lv5 = 에너지 4
- 커스텀 제련로 Lv5 = 5
- 마력 정제기 Lv5 = 7
- 영지 중앙 창고 Lv5 = 2
- 마력 채집기 Lv5 = **자체 +10** (발전량 재해석 가능)

**재정의**: "시간당 소모 전력" → "상시 소모 전력(단위 시간당)"

**영지 Lv5 풀업 총합 추정**: 약 37 (마력 채집기 발전량 제외)
- 발전기 1~2대 = Lv1~3 영지 충당 가능
- 발전기 3~4대 = Lv5 풀업 필요
- **균형 감각 맞음**

## 6. 테마별 발전기 보너스 (v0.1은 3지역만)
| 지역 | 보너스 |
|---|---|
| 북부 설혼 | 풍차 +25% (냉풍) |
| 남부 화맥 | 화력 +20% (지열), 연료 효율 +10% |
| 아르카논 | 마력 발전기 +30% (공명, v0.2 도입 시) |
| 서부 숲 | 풍차 +10%, 수력 +15% (v0.2 이후) |

v0.1은 북부·남부·수도 3지역만. 6지역 전부 v0.2 이후 (밸런싱 부담).

## 7. 구현 복잡도 평가
- **가상 링크 + 분 단위 집계**: 서버 부하 거의 무시 가능
- 100명 × 36 설비 × 12 = 분당 43,200회 연산 = **초당 720회** (I/O 없으면 1ms 미만)
- DB 업데이트는 설비 상태 변경(가동↔중단) 전환 시에만 (**이벤트 기반**)
- 우선순위 정렬 + 그리디 할당 = O(N log N). 문제 없음

## 8. UX 완화책
- **영지 Lv2 해금 시점에 기본 화력 발전기 1대 무상 증정** — Lv1은 전력 개념 없이 시작
- **영지 HUD에 "전력 부족 경고" 상시 표시** + 최초 발생 시 툴팁·가이드 팝업
- 발전기 이동 쿨다운 완화 (**30분**, 일반 설비 1시간 대비)

## 9. 공수
- v0.1 설계·구현·테스트 = **10~14 man-day** (플래그 저장소와 유사 규모)
- **플래그 저장소 v0.1 머지 완료 후** 착수 권장 (영지 상태 플래그 일부 활용 여지)

## 10. 리스크
- **오프라인 정합성**: 옵션 A(면제) 확정 필요, T1 오프라인 누적 100%와 충돌 방지
- **영지 그리드 크기 재검토**: 12기 설비 + 발전기 2~4기 = 16칸 내외
- **설비 이동 쿨다운 vs 전력 토폴로지 변경**: 발전기는 쿨다운 완화
- **기존 문서 용어 교체**: `에너지` → `전력 소모` 재해석 vs 전면 교체 결정

## 11. 테스트 포인트
- 총 발전량 vs 총 소모량 집계 정확도 (분 단위 갱신 후 예상값 일치)
- 전력 부족 시 4단계 우선순위 중단 순서
- 테마 보너스 계수가 발전량에만 적용 (소모량 영향 없음)
- 마력 채집기 `자체 +N`이 발전량으로 합산
- 오프라인 로그인 직후 전력 상태 재계산 중복 소모 없음
- 발전기 파괴/이동 시 설비 가동 상태 1틱 내 반영
- 동시 접속 100명 시뮬레이션 분당 집계 부하 측정

## 12. 결론 — 도입 확정 (사용자 결정)
**v0.1 최소 침습적 도입 경로 확정**:
- 화력·풍차 2종만, 가상 링크, 분 단위, 0/1 가동, 단계별 우선순위, 오프라인 면제, `에너지` 컬럼 재해석
- 영지 Lv2 해금 시점 **화력 발전기 1대 무상 증정**으로 초보자 진입 장벽 완화
- **플래그 저장소 v0.1 머지 후 착수** (영지 상태 플래그 일부 활용 여지)
- 공수 10~14 man-day

**참고 메모 (도입 반대 의견 보존)**: 포로 핵심 루프(전투·장비·강화) 초점 분산 우려 / 전력 부족 → 강화석 공급 차단 → 이탈 가속 리스크. **대안**: 영지 만족도 단일 게이지(NPC·식량·에너지 통합, 구현 1/3). 이 대안은 v0.1 도입 후 UX 체감 악화 시 **v0.2에서 하이브리드 고려**할 여지로 보존.
