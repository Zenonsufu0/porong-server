## 0. 문서 스코프와 비스코프

### 스코프
- 운영 손에 있는 가격 축 6개: (a) 수도 상점 NPC (b) 오픈 시점 경매 시드 (c) 영지 NPC 상점 (d) 치장 확정 구매 NPC (e) 랜덤박스 가격·확률 (f) 강화 보조 물약 NPC 판매.
- 가격 결정 원칙, 카테고리별 대표 가격표, 운영 시드 풀 정책 권장안, 45일차 이탈 감지 트리거, 리스크·완화안.

### 비스코프
- 플레이어 간 경매 낙찰가 개입(자율 시장 원칙 존중).
- 경매 수수료율 재설계(기존 5% 가정 유지).
- 성장 재료 직접 판매(상업화 금지선 — 본 문서에서도 금지선 유지).
- T2 완성 장비 NPC 직거래(T2는 재료 제작형, NPC 판매 대상 아님).

### 기존 기준과의 충돌 점검
충돌 없음. 본 문서는 다음 원칙을 그대로 유지한다.
- T1/T2 2단계, T2 재료 제작형 구조 존중 (T2 완성품 NPC 판매 금지).
- 강화 25강 파괴/하락 없음 존중 (강화 보조 물약은 성공률 배율 보조에 한정).
- 치장 귀속 X + 랜덤박스·확정 구매 병행 구조 유지.
- 확률 버프 축소 도입(가공·연금·요리 Lv3/4/5 = 1/2/3%, 보너스 귀속 O) 존중.
- 월 공급 제한량 X 결정 유지.
- 상업화 금지선 — 재화 직접 판매·월렛 확장권·재화 2배권 도입 없음.

|---|---|---|---|---|

### 분류 축 의도
- (a)·(d)는 **하한/상한 앵커** 역할. 자율 시장 시세가 이 선과 이탈하면 알람.
- (b)는 자율 시장 원칙과 가장 충돌하기 쉬운 축 — 본 문서에서 별도 절(§5)로 권장안 명시.
- (c)·(f)는 생활 루프·강화 루프와 직접 맞물림.
- (e)는 확률 설계가 가격 설계보다 먼저 — 확률표 고정 후 가격 역산.

## 2. 가격 결정 원칙 4개

### 2-1. 원칙 1: NPC는 "하한 앵커" — 시장가가 아래로 무너지지 않게 지지한다
- 수도 상점 NPC 기본 소비품·튜토리얼 키트 판매가는 **자율 시장 최저가의 바닥** 역할.
- 시장 평균가가 NPC 판매가보다 지속 저렴하면 해당 품목은 "공급 과잉"으로 판정.
- NPC 판매가는 시장 평균가보다 **낮게 잡지 않는다**(자율 시장 형성 방해 방지).

### 2-2. 원칙 2: 치장 확정 구매는 "상한 앵커" — 시장가가 위로 폭주하지 않게 천장을 씌운다
- 확정 구매 가격 = 랜덤박스 등급 기댓값의 **1.5~2.0배 수준**(치장 희소성 유지 + 시장 폭주 방지).
- 동일 SKU의 경매 평균가가 확정 구매가를 초과하면 "수집 수요 과열"로 판정.
- `auction_cosmetic_summary.source_ratio`가 랜덤박스 축에서 확정 구매가를 지속 상회하면 **랜덤박스 드롭표 이상** 의심 (경매 대시보드 지표 4-3과 연동).

### 2-3. 원칙 3: 신규 접근성 축 — 튜토리얼·기초 소비품은 "저가 진입"
- 일반 회복 물약·빵·스타터 T1 기본 장비는 **첫 수도 귀환 시 무료 지급분 + 저가 재구매**.
- 신규 플레이어가 5분 이내에 기초 생존 싸이클을 돌릴 수 있는 가격 허들.
- "1차 구매 1회 한정" 할인 슬롯 허용(플래그 `starter_bundle_purchased=true` 판정).

### 2-4. 원칙 4: 운영 재조정 용이성 — 가격은 CSV 한 파일로 교체
- 본 문서의 모든 가격은 `seeds/npc_shop_prices.csv` 단일 파일에 적재.
- 스키마: `shop_code, item_code, price_gold, stock, sell_to_player, buy_from_player, restock_cycle_sec, flag`.
- 런타임은 5분 TTL 캐시 + 플러그인 콘솔 명령 `/shop reload`로 핫리로드.
- 랜덤박스 확률표는 별도 CSV `seeds/lootbox_probability.csv` 분리(확률 조작 감사 대상이라 별 파일).

### 3-4. 치장 랜덤박스 — 가격 + 확률표

랜덤박스 1종 기본형 + 이벤트 1종 구성.

| box_code | 품명 | 판매가(G) | 구성 |
|---|---|---:|---|
| lootbox_cosmetic_basic | 기본 치장 상자 | 2,500 | 일반 70% / 고급 25% / 희귀 4.5% / 영웅 0.5% |
| lootbox_cosmetic_event | 이벤트 치장 상자 | 5,000 | 일반 50% / 고급 35% / 희귀 12% / 영웅 2.5% / 전설 0.5% |

기댓값 역산 (기본 상자):
- 일반(1,500G) × 0.70 = 1,050G
- 고급(5,000G) × 0.25 = 1,250G
- 희귀(18,000G) × 0.045 = 810G
- 영웅(60,000G) × 0.005 = 300G
- 합계 기댓값 ≈ 3,410G, 판매가 2,500G → 기댓값 대비 판매가 **73%** (=유저에게 약간 유리, 랜덤박스 구매 유인 확보).

관전 포인트: 기댓값이 판매가보다 높은 건 "랜덤박스가 확정 구매보다 기댓값 유리하되 원하는 SKU 확률이 낮아 자율 시장에서 해소"되는 구조. 원칙 2와 일치.

- 장점: 기댓값 투명 공개 → 상업화 금지선 근접 우려 완화(확률 숨김 비판 차단).
- 위험요소: 영웅·전설 확률이 너무 낮으면 피티 없이는 수집 욕구 꺾임. 피티(80회 보장 등) 검토 필요.
- 조정 제안: 이벤트 상자에 **40회 누적 시 희귀 이상 1개 보장 피티** 도입. 기본 상자는 피티 미도입(저가 축 유지).

관전 포인트: NORMAL 300G는 1회 강화 재료(조각·파편·강화석)의 평균가 대비 **5~10% 미만** 수준으로 책정. 재료 절약 효과(기본 성공률 × 1.05)의 기댓값과 정합.

- 장점: 파괴/하락 없음 구조 + 1회 소모형이라 가격 투명성 확보.
- 위험요소: ADVANCED 1,500G가 너무 싸면 플레이어가 자율 시장 제작 물약을 무시하고 NPC 독점. 연금 플레이어 수요 고사.
- 조정 제안: ADVANCED NPC 재고를 **일일 최대 10개 제한** + 재고 소진 시 다음날 복원. 자율 시장과 병존.

## 4. 가격 결정 수식 (운영 참조)

### 4-1. NPC 판매가 하한 수식
```
npc_sell_price = max(
    production_cost_estimate × 1.15,      // 제작 원가의 115% (마진 15%)
    market_floor_observed × 1.00,          // 시장 최저가 관측치 (원칙 1)
    min_accessibility_price                // 신규 접근성 최저선
)
```

### 4-2. 치장 확정 구매가 상한 수식
```
direct_buy_price = lootbox_expected_value(rarity) × multiplier
// multiplier = 1.5 (희귀 이하), 1.8 (영웅), 2.0 (전설)
```

### 4-3. 랜덤박스 판매가 역산 수식
```
box_price = sum(rarity_expected_value × probability) × discount_factor
// discount_factor = 0.70~0.80 (유저 유인 확보)
```

## 5. 경매 시즌1 오픈 시점 "운영 시드 풀" 정책

### 5-1. 찬반 비교

| 항목 | 시드 풀 도입 | 시드 풀 미도입 |
|---|---|---|
| 자율 시장 원칙 부합도 | 낮음 (운영 개입) | 높음 |
| 오픈 당일 거래 활성화 | 높음 | 낮음 (첫 낙찰 대기) |
| 가격 발견 속도 | 빠름 | 느림 |
| 시세 왜곡 리스크 | 중 (운영 시드가 기준선으로 오인) | 낮음 |
| 운영 부하 | 시드 카탈로그 관리 필요 | 낮음 |
| 판매자 경쟁 | 시드가 경쟁 압력 | 온전히 플레이어 간 경쟁 |

### 5-2. 권장안: **조건부 도입 — "시드 풀 최소 + 24시간 한시"**

조건:
- 시드 대상 **카테고리 제한**: 기초 소비품(빵·NORMAL 회복 물약)과 강화석 조각/파편만. **T1/T2 완성 장비·치장·T2 상위 조각 제외** (플레이어 유통의 핵심 축은 시드하지 않음).
- 시드 **양 제한**: 카테고리당 최대 50개 매물, 매물당 수량 10개 이하.
- 시드 **시작가**: 해당 NPC 판매가의 **90%**. 즉시 구매가 = NPC 판매가의 **95%** (NPC보다 살짝 유리하되 플레이어 제작가보다 비싸야 경쟁이 성립).
- 시드 **기간**: 오픈 후 24시간. 미낙찰 분은 24시간 후 회수.
- 시드 **판매자 UUID**: 운영 전용 UUID(`00000000-0000-0000-0000-000000000001`)로 표시. 판매자 신원 투명 공개.
- 시드 **수수료 정책**: 수수료는 정상 부과(시장 전체 일관성 유지).

### 5-3. 시드 풀 카탈로그 (1차 초안)

| item_code | 시드 수량 | 시작가(G) | 즉시구매가(G) | 비고 |
|---|---:|---:|---:|---|
| bread_basic | 50매물 × 10개 | 13 | 14 | NPC 15G의 90%/95% |
| basic_healing_potion | 30매물 × 5개 | 54 | 57 | NPC 60G의 90%/95% |
| enhance_stone_fragment_t1 | 20매물 × 50개 | 개당 45 | 개당 48 | 자율 시장 기대가 관측 후 재설정 |
| craftsman_focus_lesser | 10매물 × 1개 | 270 | 285 | NPC 300G의 90%/95% |

관전 포인트: 강화석 조각은 NPC 판매가가 존재하지 않으므로 **자율 시장 기대가 관측 후** 값을 설정해야 한다 — 1차 초안은 1일차 운영자 재량 수치. 22일차 4:1 변환 이벤트와 별개로 오픈 24시간 내 가격 발견 시드 역할.

- 장점: 오픈 당일 경매 기능 즉시 활성화, 판매자 경쟁 부담 저감, 자율 시장 원칙 훼손 최소.
- 위험요소: 시드 가격이 플레이어 기대가 대비 높으면 시드만 덩그러니, 낮으면 시드가 시장가 상한 형성.
- 조정 제안: 오픈 6시간차에 시드 낙찰률 확인 — 30% 미만이면 시작가 -10% 하향, 70% 초과면 24시간 조기 종료 + 잔여 회수.

## 6. 45일차 이탈 감지 트리거 (경매 대시보드 연동)

기존 `auction_daily_summary` / `auction_category_summary` / `auction_cosmetic_summary` 집계 테이블에서 파생.

### 6-1. 트리거 T1 — NPC 판매가 하회 지속 (원칙 1 위반)
- 조건: `auction_daily_summary.avg_price < npc_sell_price × 0.80` 이 **7일 이상 지속** (카테고리 = 기초 소비품 / 강화 보조 물약).
- 의미: NPC가 시장가 하한 지지에 실패, 공급 과잉.
- 알람: 디스코드 운영 채널 경고 + 주 1회 검토 보고서.

### 6-2. 트리거 T2 — 확정 구매가 상회 지속 (원칙 2 위반)
- 조건: `auction_cosmetic_summary.avg_price > direct_buy_price × 1.20` 이 **7일 이상 지속** (동일 등급 + 확정 구매 축).
- 의미: 수집 수요 과열, 확정 구매 노출 부족 의심.
- 알람: 확정 구매 NPC 재고·노출 재검토 트리거.

### 6-3. 트리거 T3 — 랜덤박스 공급 이상
- 조건: `auction_cosmetic_summary.source_ratio` 중 랜덤박스 축 평균가 < 확정 구매 축 평균가 가 **5일 이상 지속** (기존 지표 4-3, 경고 기준 7일을 5일로 강화).
- 의미: 랜덤박스 드롭표 이상 또는 과도한 대량 개봉.
- 알람: 랜덤박스 확률표 감사 + CSV 해시 비교.

### 6-4. 트리거 T4 — 강화 보조 물약 시세 붕괴
- 조건: `material/enhance_potion/*` 카테고리 `avg_price < npc_sell_price × 0.60` 이 **5일 이상 지속**.
- 의미: 연금 플레이어의 자율 생산이 NPC 재고를 압도, 연금 직군 수익성 고사.
- 알람: 연금 레시피 재료비 재조정 검토.

### 6-5. 트리거 T5 — 청사진 가격 대비 골드 유통량 괴리
- 조건: 영지 Lv2→Lv3 청사진 월 구매 건수 < 예상치 50% **AND** `auction_transaction.total_volume` 시즌 누적이 22일차 기준선 대비 +30% 초과.
- 의미: 플레이어가 청사진 구매를 건너뛰고 골드를 경매에 쏟음 → 영지 진척 지연.
- 알람: 청사진 판매가 -20% 임시 인하 검토 + 퀘스트 보상 경로 병행 검토.

## 7. 리스크 3개 + 완화안

### 7-1. 리스크 R1 — NPC 판매가 과다 → 플레이어가 NPC 무시 + 자율 경매만 이용
- 증상: 트리거 T1 반대 양상 — NPC 재고 장기 적체, `/shop stats`에서 일일 판매량 < 기대치 20%.
- 영향: NPC 시드 기능 붕괴, 하한 앵커 소실, 시장가 왜곡 시 회복 경로 없음.
- 완화안:
  - NPC 판매가를 **시장 평균가의 110%로 자동 추종**하는 완화 루프 도입 검토(운영 수동 승인 전제).
  - 기초 소비품은 "NPC 판매 유지 + 자율 시장 병존" 구조로 운영 보고서 주간 확인.

### 7-2. 리스크 R2 — NPC 판매가 과소 → NPC 판매가가 시장가 상한으로 고착
- 증상: 트리거 T4 — 연금 직군 수익성 고사, 플레이어 제작 루프 이탈.
- 영향: 자율 시장 형성 방해, 연금·요리 생활 시스템의 경제적 의미 상실.
- 완화안:
  - NPC 판매 재고를 **일일 최대치 제한**(ADVANCED 물약 10개/일 등 §3-5 조정 제안 적용).
  - 22일차 튜닝 시점에 자율 시장 평균가 기준으로 NPC 판매가 상향.

### 7-3. 리스크 R3 — 랜덤박스 가격·확률 조합이 상업화 금지선 근접
- 증상: 랜덤박스 기댓값 < 판매가 (discount_factor > 1.0) 또는 피티 없이 전설 확률 0.5% 미만 고정 장기화.
- 영향: "성장 재료 직접 판매 지양, 치장/편의 중심" 원칙 약화, 유저 신뢰 손상.
- 완화안:
  - 확률표 CSV 커밋 로그 + 감사 대상 지정. 변경 시 PR 리뷰 필수.
  - 이벤트 상자에 40회 피티 도입 (§3-4 조정 제안).
  - 기댓값 대비 판매가 비율(`discount_factor`)을 공개 범위로 운영 (70~80% 유지, 100% 초과 금지).
