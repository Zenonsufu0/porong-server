## 1. 큐브 시스템 (§7.1)

### 1.1 큐브 수급

- **큐브 조각**: 정예몹 드랍 + 필드보스 전 5종 확정 드랍 (DL-057) → 10개 합산 자동 전환 → 큐브 1개
- 전환은 `PlayerItemPickupEvent` 또는 인벤토리 점검 시점에 자동 처리

### 1.2 큐브 사용 비용

- 사용 비용: **500G** (고정, DL-016)
- 큐브 1개 소모 → 선택한 장비 슬롯의 잠재 옵션 재롤

### 1.3 등급 업 확률 (memorial system)

각 큐브 시도 시 현재 장비 잠재 등급보다 높은 등급이 결과로 나올 경우 등급 업 발생.  
등급 업 발생 시 **옵션 선택 강제** (현재 유지 또는 새 옵션 선택).

| 등급 | 한글 | 확률 분포 |
|---|---|---|
| COMMON | 커먼 | 가장 낮음 |
| RARE | 레어 | — |
| EPIC | 에픽 | — |
| UNIQUE | 유니크 | — |
| LEGENDARY | 레전더리 | 가장 낮음 |

> 정확한 확률 수치: `growth_potential_option_pool.csv` 스키마 `weight` 컬럼 기준  
> (`option_code, grade, value_min, value_max, weight, slot_filter`)

## 2. 잠재 옵션 풀 (§7.2)

### 2.1 잠재 등급별 옵션 수

| 등급 | 옵션 라인 수 |
|---|---|
| 커먼 | 1줄 |
| 레어 | 2줄 |
| 에픽 | 3줄 |
| 유니크 | 3줄 |
| 레전더리 | 3줄 |

### 2.2 슬롯별 옵션 풀

`slot_filter` 컬럼: `|`로 슬롯 조합 표기. 예: `WEAPON|HELMET`

| 슬롯 | 주요 옵션 특성 |
|---|---|
| 무기 (`WEAPON`) | 공격력, 크리 확률, 크리 피해, 스킬 피해, 보스 피해 |
| 투구 (`HELMET`) | 최대 HP, 받는 피해 감소, 쿨타임 감소 |
| 상의 (`CHESTPLATE`) | 최대 HP, 방어력, 받는 피해 감소 |
| 하의 (`LEGGINGS`) | 최대 HP, 방어력, 이동 속도 |
| 신발 (`BOOTS`) | 이동 속도, 방어력, 방어력 무시 |

> 전체 옵션 풀: `growth_potential_option_pool.csv` 참조 (M-7 확정 2026-05-20)

### 2.3 전승 시스템

전승 = 흔적의 등급을 대상 장비에 이전하고, 전승 실행 시 **대상 슬롯 기준** 옵션 풀에서 부가 스탯을 무작위 생성하여 적용.

> **구현 모델:** `equip_trace_*`는 `IslandTerritoryState.customItems` 카운터 재료로 저장됨 (인스턴스 아이템 아님). 부가 스탯은 흔적에 내재된 값이 아니라 전승 실행 시 `growth_potential_option_pool.csv`의 슬롯 필터 + 흔적 등급 기준으로 즉시 생성됨. `equip_trace_unidentified`는 전승 재료 불가 (식별 미구현, 1차 시즌 제외).

| 전승 방식 | 소모 | 효과 |
|---|---|---|
| 기본 전승 | 장비의 흔적 × 1 + 0G | 흔적 등급 이전 + 슬롯 기준 부가 스탯 무작위 생성 |
| 등급 전승권 사용 | 등급전승권 100,000G | 흔적 등급만 이전, 부가 스탯 무변경 |
| 세부스탯전승권 사용 | 세부스탯전승권 100,000G | 슬롯 기준 부가 스탯만 무작위 재생성, 등급 무변경 |

### 3.3 가호 천장 시스템 (pity)

- 적용 범위: 11강 이상
- 천장 공식: `ceil(2 / 성공률)` 회
- 천장 도달 시 강화 자동 성공 → **슬롯별 카운터 초기화**
- **카운터는 DB에 슬롯별 저장** (서버 재시작 후에도 유지 필수)
- 강화 GUI에서 현재 시도 횟수 / 천장 표시

```
DB 컬럼 예시:
player_enhance_pity (
  player_uuid   TEXT,
  slot          TEXT,   -- WEAPON / HELMET / CHESTPLATE / LEGGINGS / BOOTS
  current_tries INTEGER,
  ceiling       INTEGER,
  PRIMARY KEY (player_uuid, slot)
)
```

### 3.4 강화 흔적 소모품 (곱연산, DL-090)

| 아이템 | item_id | 성공률 보정(곱연산) | 사용 조건 |
|---|---|---:|---|
| 별의 흔적 | `mat_trace_star` | ×1.15 (+15%) | 10강 이상 |
| 달의 흔적 | `mat_trace_moon` | ×1.25 (+25%) | 10강 이상 |
| 태양의 흔적 | `mat_trace_sun` | ×1.30 (+30%) | 10강 이상 |

- **보정: 곱연산.** 켠 흔적들의 보너스를 **합산한 뒤 한 번 곱**한다: `유효 성공률 = 기본 성공률 × (1 + Σ보너스)`.
  - 예: 25강 기본 0.05% + 태양만 → 0.05 × 1.30 = 0.065%. 전부(별+달+태양) → 0.05 × 1.70 = 0.085%.
- **3종 동시 사용 가능** (별·달·태양 독립 ON/OFF 토글 + 일괄 토글). 전부 = ×1.70.
- 토글 ON한 각 흔적은 **단계별 요구 수량을 전량 소모**(all-or-nothing, 비례 적용 아님). 요구 수량 미달이면 그 흔적은 적용·소모 안 됨.
- 요구 수량 (흔적별 × 목표 단계, **시작값 — 밸런스 7일차 재산정 대상**):

  | 목표 강화 단계 | 별 | 달 | 태양 |
  |---|---:|---:|---:|
  | 11~17강 | 1 | 1 | 1 |
  | 18~22강 | 2 | 1 | 1 |
  | 23~25강 | 2 | 2 | 1 |

  > 곱연산에서 흔적 이득은 "1회 강화비용(골드+강화석)"에 비례하므로 수량 곡선은 완만하게 유지(목표: 흔적 포함 기대비용 ≈ 흔적 미사용의 0.85~0.9). 흔적값(마도합금·블럭 시세)에 민감 → 오픈 후 실측 재산정.
  > 천장(가호) 강제 성공으로 처리되는 시도에서는 흔적을 소모하지 않는다.

### 3.5 강화 처리 흐름

```
강화 GUI에서 장비 선택 → 강화 버튼 클릭
  → 골드·강화석 충분성 검증 (창고 + 인벤 합산)
  → 강화 흔적 토글(별/달/태양) ON 여부 확인 (요구 수량 보유분만 적용·소모)
  → 실효 성공률 = 기본 성공률 × (1 + Σ 켠 흔적 보너스)  // 곱연산, DL-090
  → 가호 카운터 확인: current_tries + 1 >= ceiling → 강제 성공
  → 0~1 사이 난수 생성 → 실효 성공률 이하면 성공
  → 성공: enhance_level += 1, pity 카운터 초기화, DB 갱신
  → 실패: pity 카운터 += 1, DB 갱신
  → GUI 업데이트 (성공률 / 천장 진행도 재계산)
```

## 5. 아이템 레벨 (IL) 시스템 (§9.3)

**IL 공식:** 강화 단계 1당 IL +5 / 평균 IL = 5슬롯 평균 강화 단계 × 5

| 필드 | 적정 평균 IL | 경고 임계 |
|---|---|---|
| 수도 외곽 평원 | IL 0~35 | IL 65 이상 (평균 13강+) |
| 폐광 지대 | IL 30~55 | IL 75 이상 (평균 15강+) |
| 오염된 수로 | IL 50~75 | IL 85 이상 (평균 17강+) |
| 무너진 초소 | IL 65~85 | IL 90 이상 (평균 18강+) |
| 고대 성벽 잔해 | IL 80~100 | 없음 |

- 경고 3회 누적 → 임시 화이트리스트 제외 (운영자 수동 해제)
- 일반몹/정예몹 30마리마다 경고 1회
