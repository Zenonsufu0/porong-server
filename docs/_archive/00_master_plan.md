## 0. 최종 방향 요약

포로 서버 1차는 장기 상시 운영 서버가 아니라, 약 45일간 운영되는 시즌제 프로젝트 서버다.  
목표는 거대한 RPG 전체를 처음부터 완성하는 것이 아니라, **수도 주변 압축형 RPG 경험 + 개인 영지 생산 + 보스 도전 + 치장/컬렉션**을 하나의 시즌 루프로 완성하는 것이다.

핵심 결론은 다음과 같다.

- 정식 오픈은 **디스코드 유입 제한형**으로 진행한다.
- 공개 테스트는 하지 않는다. 내부/소수 점검 후 바로 정식 오픈한다.
- 정식 오픈 시 1차 시즌 콘텐츠 전체 공개를 목표로 한다.
- 운영 기간은 약 45일이며, 종료/연장/데이터 처리 여부는 별도 공지한다.
- 장비는 **T1 단일 장비 체계**만 사용한다.
- **T1.5 / T2 / 세트 장비 / 공용각인**은 1차 시즌에서 제외한다.
- 직업은 사실상 **무기 클래스**로 처리한다.
- 오픈 무기는 **검 / 망치 / 창 / 석궁 / 낫 / 스태프** 6종이다.
- 무기별 스킬은 4개, 무기별 직업각인은 2개다.
- 입력 방식은 무기 장착 상태에서 **좌클릭 / 우클릭 / Shift+우클릭 / F**를 사용한다.
- 플레이어 스킬에서 **지속 장판, 표식, 받는 피해 증가, 방어 감소**는 제외한다.
- 보스는 1차에서 **바닐라 강화형 + MythicMobs + PoroRPG 보상/패턴 관리**로 간다.
- 외부 보스 모델, ModelEngine, BetterModel, FMM은 2차 확장 후보로 둔다.
- 마인팜은 **IridiumSkyblock 기반 개인 영지 + PoroRPG 영지 저장고/마력/가공** 구조다.
- 마인팜 물류는 호퍼/상자 자동화가 아니라 **DB 기반 영지 저장고**로 처리한다.
- 웹/디스코드 봇/통계는 **PoroRPG DB와 API**를 기준으로 조회한다.
- 경제가 정상적인 시장 흐름에서 벗어나면 운영진이 개입할 수 있다.
- 후원/유료 상품은 완전히 보류한다. 단, 성장 보상 없는 자발적 커피값 후원 정도는 가능하다.
- 리소스팩 스타일은 **깔끔한 RPG풍**이며, 다크판타지·고어·공포 톤은 지양한다.

## 1. 운영 정책

### 1.1 운영 톤

운영 톤은 **친근하지만 절제된 톤**으로 한다.

- 유저와 편하게 소통한다.
- 공지와 제재는 명확하고 차분하게 작성한다.
- 과하게 장난스럽거나 과하게 권위적인 톤은 피한다.
- 서버가 프로젝트 서버임을 명확히 알리되, 테스트 서버처럼 가볍게 보이지 않게 한다.

### 1.2 운영 기간

포로 서버 1차는 약 45일간 열리는 시즌제 프로젝트 서버다. 종료 예정일: **2026-09-01**, 연장 없이 확정 종료한다.

권장 공지 문구:

```text
포로 서버 1차 시즌은 약 45일간 운영되는 프로젝트 서버입니다.
운영 기간 동안 1차 콘텐츠 전체를 공개하고, 유저 플레이 데이터와 밸런스 데이터를 바탕으로 이후 시즌 방향을 결정합니다.
운영 종료, 연장 여부, 데이터 보존/초기화 여부는 종료 전 별도 공지합니다.
```

### 1.3 정식 오픈 범위

원칙은 **정식 오픈 시 1차 시즌 범위를 전부 공개**하는 것이다.  
단, 구현 일정상 반드시 컷이 필요할 경우에는 시즌 최종보스만 시즌 중 업데이트 대상으로 분리할 수 있다.

정식 오픈 필수:

- 디스코드 인증/화이트리스트
- 6무기 클래스와 스킬
- 무기별 직업각인
- T1 장비 / 강화 / 잠재
- 수도 + 필드 5개
- 일반몹 / 정예몹 / 필드보스 5개
- 시즌보스 3개
- 개인 영지 / 영지 저장고 / 마력 / 공방 가공기
- 모험 도감 MVP
- 필드보스 알림

오픈 후 확장 가능:

- 시즌 최종보스
- 고급 치장
- 웹 유저 대시보드
- 고급 통계 화면
- 커스텀 보스 모델
- 케이블/배터리/상자 물류
- 전문 가공기

### 1.4 경제 개입 원칙

서버 경제가 정상적인 시장 흐름에서 벗어날 경우 운영진은 개입할 수 있다.

개입 대상:

- 특정 재화 과잉 축적
- 골드 총량 과잉 증가
- 강화 재료 과잉 공급
- 버그/비정상 루트로 인한 재화 유입
- 특정 보스/필드/마인팜 루트의 과도한 효율

가능한 조치:

- 드랍률 조정
- 제작 비용 조정
- 생산량 조정
- 보상량 조정
- 비정상 획득 아이템 회수
- 경제 로그 기반 사후 조치

문구 원칙:

```text
경제 개입은 특정 유저를 불리하게 만들기 위한 조치가 아니라,
45일 시즌 서버의 정상적인 성장 흐름을 유지하기 위한 조치입니다.
```

### 1.5 후원/유료 상품

1차 시즌에서는 성장형 유료 상품을 판매하지 않는다.

금지:

- 골드 판매
- 강화석 판매
- 큐브 판매
- 장비 판매
- 보스 입장권 판매
- 성장 속도에 직접 영향을 주는 상품 판매

가능:

- 보상 없는 자발적 커피값 후원
- 감사 표시 수준의 명단/역할 표시

치장 유료 상품도 1차 시즌에서는 보류한다.

## 2. 디스코드 서버 구조와 온보딩

### 2.1 현재 구조 활용

현재 디스코드 서버에는 다음 구조가 있다.

- 안내: 서버소개, 규칙, 공지, 업데이트
- 정보: 접속정보 (**약관 동의 후 역할 부여 시에만 열람 가능**), faq
- 커뮤니티: 잡담, 스크린샷, 파티모집
- 지원: 문의, 버그제보
- 음성 채널
- 봇: Carl-bot, ServerStats, TempVoice, Ticket Tool, Xenon 등

이 구조는 유지하되, 1차 시즌 운영 기준으로 아래를 보강한다.

추가 추천:

- `약관-인증` 채널
- `필드보스-알림` 채널
- `시즌보스-알림` 채널
- 운영진 전용 `운영로그` 채널
- 운영진 전용 `밸런스메모` 채널

### 2.2 역할 구조

권한 역할과 칭호 역할은 분리한다.

권한 역할:

- 미인증
- 접속대기
- 인증 유저
- 테스터
- 운영진

인증 유저 초기 칭호: **모험가**

표시/칭호 역할은 이후 콘텐츠에 따라 추가 가능하다.

예:

- 모험가 (인증 유저 기본)
- 토벌대원
- 균열 도전자
- 제국 공방장
- 시즌 정복자

### 2.3 디스코드 선입장 인증 플로우

최종 플로우:

```text
디스코드 입장
→ 약관 동의
→ 마인크래프트 닉네임 입력
→ 임시 화이트리스트 등록
→ 마크 서버 접속
→ 코드 인증
→ 디스코드-마크 계정 최종 연동
→ 인증 유저 역할 부여
```

주의:

- 디스코드에서 입력한 닉네임은 임시 화이트리스트용이다.
- 최종 소유 검증은 마크 서버 접속 후 코드 인증으로 한다.
- 임시 화이트리스트는 24시간 후 만료한다.
- 닉네임 오입력 시 인증 채널에서 재입력 절차를 둔다. 재입력 횟수는 무제한이며, 변경 전 닉네임은 DB에서 즉시 제거한다.

### 2.4 디스코드 봇 기능

MVP 기능:

- 약관 동의 버튼
- 닉네임 입력 모달
- 임시 화이트리스트 등록
- 연동 코드 발급
- `/연동`
- `/프로필`
- `/영지`
- `/보스`
- 필드보스 리젠 알림

추가 기능:

- 가공 완료 알림
- 시즌보스 주간 보상 초기화 알림
- 랭킹 조회
- 컬렉션 조회

## 3. 서버 기술 구조

### 3.1 기본 환경

- 서버 버전: Paper 1.21.10
- Java: Java 21
- 개발 방식: Java + Gradle 커스텀 플러그인
- 운영 환경: Linux
- 데이터 저장: 초기 SQLite, 필요 시 MySQL/PostgreSQL 확장

### 3.2 플러그인 역할 분리

원칙:

```text
PoroRPG가 데이터와 핵심 로직을 소유한다.
다른 플러그인은 껍데기/보조/표시/월드 관리만 담당한다.
```

유지/추가 플러그인:

| 플러그인 | 역할 |
|---|---|
| PoroRPG | 서버 코어 |
| MythicMobs | 몹/보스 스폰, 기본 연출 |
| IridiumSkyblock | 개인 영지 생성, 멤버, 보호 |
| Vault | 경제/권한 연동 보조 |
| LuckPerms | 권한 관리 |
| EssentialsX | 운영 편의, 기능 제한 |
| WorldEdit | 개발/건축 |
| WorldGuard | 지역 보호 |
| Multiverse-Core | 월드 분리 |
| PlaceholderAPI | 정보 표시 연동 |
| DecentHolograms | 안내 홀로그램 |
| Chunky | 오픈 전 청크 프리젠 |
| spark | 성능 측정 |

제거/보류:

| 항목 | 처리 |
|---|---|
| BetonQuest | 제거 |
| Citizens | 제거 — 퀘스트 NPC 없음, 상점·이동 모두 GUI 메뉴로 처리 |
| ItemsAdder/Oraxen/Nexo | 보류 |
| ModelEngine/BetterModel/FMM | 2차 확장 후보 |
| MMOCore/AuraSkills/RPGItems | 보류 |
| Quests류 대형 퀘스트 플러그인 | 보류 |

- 골드/아이템 흐름 로그

## 4. 월드와 콘텐츠 구조

### 4.1 월드 구조

| 월드 | 용도 |
|---|---|
| world_main | 제국 수도 + 필드 5개 |
| world_farm | 개인 영지 |
| world_boss | 시즌보스/최종보스 공간 |
| world_test | 개발/리소스/보스 테스트 |

메인 월드는 일반 유저의 블럭 파괴를 기본적으로 막는다.

제국 수도 내 **훈련장** 구역에 딜미터기용 허수아비 몬스터를 배치한다. 필드 GUI에서 접근 가능.

### 4.2 필드 5개

1차 시즌 필드는 수도 주변 5개로 압축한다.

| 필드 | 역할 | 필드 공통 재료 | 필드보스 핵심 재료 |
|---|---|---|---|
| 수도 외곽 평원 | 초반 적응 | 야생의 가죽 | 들판 포식자의 송곳니 |
| 폐광 지대 | 강화/광물 | 폐광 철편 | 폐광 골렘의 파편 |
| 오염된 수로 | 포션/저항 | 오염된 점액 | 수로 군주의 점액핵 |
| 무너진 초소 | 장비 파편/전투 숙련 | 낡은 기사 문장 | 타락 기사장의 문장 |
| 고대 성벽 잔해 | 시즌보스 전 고난도 | 균열 파편 | 균열 파수꾼의 결정 |

필드별 몹 구성:

- 일반몹 2종
- 정예몹 1종
- 필드보스 1종

일반몹/정예몹/필드보스는 1차에서 바닐라 몹 기반으로 처리한다.

### 4.3 드랍 최소화 원칙

몹별로 드랍을 복잡하게 나누지 않는다.

일반몹:

- 골드
- 필드 공통 재료
- 낮은 확률로 T1 장비 파편

정예몹:

- 골드
- 필드 공통 재료
- T1 장비 파편
- 강화석 파편

필드보스:

- 보스 핵심 재료
- 강화석 파편
- 큐브 조각
- 치장 제작 파편

## 5. 보스 구조

### 5.1 1차 보스 원칙

1차 시즌 보스는 전부 바닐라 강화형으로 간다.

- 외부 보스 에셋은 사용하지 않는다.
- ModelEngine/BetterModel/FMM은 2차 확장 후보로 둔다.
- 보스 로직과 외형은 분리한다.
- 2차에서 외형만 커스텀 모델로 교체할 수 있게 설계한다.

### 5.2 필드보스

필드보스는 매칭으로 빼지 않고 필드 리젠형으로 둔다.

- 각 필드 1개, 총 5개
- 30~60분 리젠
- 랜덤 오차 ±10분
- 리젠 전/등장 시 디스코드 알림
- 막타 보상 중심이 아니라 기여도 기반 보상

### 5.3 시즌보스

시즌보스는 입장형/파티형으로 둔다.

| 보스 | 목표 클리어율 | 역할 |
|---|---:|---|
| 시즌보스 1 | 활성 유저 약 70% | 입문형 파티 보스 |
| 시즌보스 2 | 활성 유저 약 50% | 상태이상/이동 대응형 |
| 시즌보스 3 | 활성 유저 약 40% | 1차 시즌 상위 목표 |
| 시즌 최종보스 | 활성 유저 약 10% | 45일 시즌 최종 도전 |

### 5.4 보스 패턴 설계 원칙

보스 패턴은 개별 하드코딩하지 않고 다음으로 분리한다.

- 패턴 모듈
- 보스별 조합
- 테마 연출 스킨
- 수치 파라미터

공용 패턴 예:

- 전방 강타
- 직선 돌진
- 원형 폭발
- 부채꼴 휩쓸기
- 투사체 발사
- 소환체 호출
- 페이즈 전환
- 무력화/딜타임

필드보스는 공용 패턴 비중을 높이고, 시즌보스는 시그니처 패턴 1개 이상을 둔다.

## 6. 전투와 무기 클래스

### 6.0 무기 클래스 선택

- 최초 접속 시 튜토리얼에서 GUI로 6종 무기 중 하나를 선택한다.
- 이후 캐릭터 GUI에서 변경 가능, 변경 쿨타임 3분, 전투 중 변경 불가 (비용 없음).
- 강화는 별도 해금 없이 처음부터 사용 가능하다.
- 스코어보드에 현재 무기 클래스(직업)를 표시한다.

### 6.1 입력 방식

핫바 스킬 슬롯 방식은 사용하지 않는다.  
무기를 들고 있을 때 아래 입력으로 스킬을 사용한다.

| 입력 | 역할 |
|---|---|
| 좌클릭 | 기본기/주력기 |
| 우클릭 | 이동기/보조기 |
| Shift+우클릭 | 특수기/제어기/자기 강화 |
| F | 핵심기 |

무기를 들고 있지 않을 때는 일반 마인크래프트 조작으로 처리한다.

### 6.2 전투 설계 금지 항목

1차 시즌에서는 밸런스와 가독성을 위해 다음을 제외한다.

- 플레이어 지속 장판
- 적에게 붙는 표식
- 받는 피해 증가 디버프
- 방어 감소 디버프
- 파티 전체 피해 증가 디버프
- 복잡한 스택 폭발 구조

대신 자기 자원 중심으로 처리한다.

### 6.3 무기 6종

| 무기 | 고유 자원 | 각인 A | 각인 B |
|---|---|---|---|
| 검 | 검세 3 | 수호검 | 결투검 |
| 망치 | 충격 3 | 파쇄자 | 성벽수호자 |
| 창 | 압박 5 | 관통창 | 질풍창 |
| 석궁 | 조준 1 | 추적자 | 기계궁수 |
| 낫 | 그림자 흐름 3 | 사신 | 암영무희 |
| 스태프 | 마력 충전 5 | 화염술사 | 비전술사 |

공용각인은 1차 시즌에서 제외한다.

## 7. 장비 성장

### 7.1 T1 단일 장비

1차 시즌은 T1 장비만 사용한다.

제외:

- T1.5
- T2
- 세트 장비
- T2 제작 재료
- 세트 효과

### 7.2 강화

1차 시즌 강화는 T1 기준으로 제한한다.

- 최대 강화: **25강** (시즌 이론 최고)
- 18강: 일반 유저의 현실적 최종선
- 22강: 상위권 도전 구간
- 25강: 극한 도전 구간 (ATK 192, 기대 시도 3,226회)
- 파괴/하락 없음
- 골드 + 강화석/보조재 소모

### 7.3 잠재

1차 시즌 잠재는 유니크까지 우선한다.

- 커먼/레어/에픽/유니크
- 레전더리는 시즌2 이후 후보
- 큐브 수급과 잠재 옵션은 로그 기반으로 조정한다.

## 8. 개인 영지와 마인팜

### 8.1 구조

```text
IridiumSkyblock = 영지 생성/보호 껍데기
PoroRPG = 영지 성장/저장고/마력/기계/가공
```

유저에게는 Skyblock이라는 표현을 노출하지 않고 **개인 영지**로 표현한다.

### 8.2 영지 작위

영지는 골드/재화로 작위를 구매해 성장한다. 작위 구매 시 영지 이름이 바뀌고, 보관량·기능이 함께 확장되며, 디스코드/인게임 칭호가 지급된다.

작위 8단계:

| 작위 | 영지명 | 재료별 보관량 (초안) |
|---|---|---:|
| — | 개척지 (시작) | 500 |
| 기사 | 기사령 | 900 |
| 준남작 | 준남작령 | 1,500 |
| 남작 | 남작령 | 2,500 |
| 자작 | 자작령 | 3,500 |
| 백작 | 백작령 | 5,000 |
| 후작 | 후작령 | 7,000 |
| 공작 | 공작령 | 10,000 |

### 8.3 영지 저장고

영지 저장고는 DB 기반 가상 저장고다.

기능:

- 재료별 수량 확인
- 좌클릭 64개 출금
- 우클릭 1개 출금
- 전체 입금
- 자동 입금 권한
- 작위 등급별 최대 보관량 증가

### 8.4 기계

> **2026-05-19 확정: 발전기·마력 시스템·자동채굴기 전면 폐지. 영지는 농사 전용.**

1차 기계:

- 자동 재배기
- 공방 가공기
- 영지 저장고

### 8.6 바닐라 농사/채광

커스텀 자동 생산기:

```text
생산물 → 영지 저장고 자동 저장
```

바닐라 농사/채광:

```text
기본 → 인벤토리 획득
편의 해금 후 → 영지 저장고 자동 입금 가능
```

자동 심기는 작위 취득 후, 골드/재료로 권한을 구매해야 사용 가능하게 한다.

### 8.7 공방 가공기

1차는 올인원 단일 가공기다.

GUI 탭:

- 요리
- 포션
- 광물 정제
- 연료
- 강화 보조
- 큐브 촉매
- 치장/영지
- 보스 보상권

가공에는 제작 시간이 있고, 대기열 제한이 있다.  
재료는 영지 저장고에서 차감하고, 결과물은 기본적으로 영지 저장고에 들어간다.

## 9. 웹, 디스코드 봇, 통계

### 9.1 기본 구조


- 큐브 사용량

- 평균 클리어 시간
- 무기별 사용률
- 무기별 평균 DPS
- 영지 레벨 분포
- 마인팜 생산량
- 치장 보유율
- 컬렉션 완성률

## 10. 리소스팩과 치장 제작

### 10.1 1차 스타일

- 깔끔한 RPG풍
- 밝고 읽기 쉬운 UI
- 다크판타지 지양
- 보스 모델은 바닐라 강화형
- 무기/아이콘/GUI/기계는 2D~2.5D 중심

### 10.2 제작 우선순위

1. 재료 아이콘
2. GUI 아이콘
3. 영지/가공기 GUI
4. 마인팜 기계 외형
5. 무기 6종 치장
6. 도감 GUI

날개와 고퀄 보스 모델은 1차에서 제외한다.

### 10.3 역할 분담

- Claude/Claude Design: 구조 설계, 파츠 분해, 검수 기준
- Figma: GUI, 아이콘, 전면 패널, 게이지
- GPT 이미지: 컨셉/텍스처 초안
- Blockbench: 모델 배치, UV, display 확인
- Codex: 리소스팩 경로, json, CustomModelData, registry 연결

Codex는 텍스처 제작 담당이 아니라 병합/연결 담당이다.
