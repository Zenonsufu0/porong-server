## 개요
체험 월드(`experience_world`)는 마크 최초 접속 유저가 자동 스폰되는 **격리된 소개 공간**이다. 디스코드 봇 약관 임베드가 본인 확인·화이트리스트 pending을 완료한 뒤 유저가 `/confirm <OTP>`를 입력하면 메인 월드로 `player.teleport()`되는 게이트 역할을 한다. 구현은 별도 Paper 서버·Bungee 프록시 대신 **동일 Paper 인스턴스 내부 분리 월드**로 확정됐고, 청크·엔티티 비용이 메인 월드와 함께 돌아가는 트레이드오프가 있으므로 본 문서는 "가볍고, 짧고, 톤만 전달"을 전제로 한다.

세계관 톤은 기존 포로 서사(관찰자·세계수·수도 황성·1막 4테마)의 **프롤로그 전단계**로 위치시키되, 1막 메인 퀘스트·4테마(동서남북) 스포일러는 전면 차단한다. 노출 수위는 "여기가 어디이고, 너는 곧 어떤 이야기의 일부가 된다"까지만.

---

## 목표
1. **격리**: 메인 월드 경제·전투·영지·랭킹 어느 축과도 섞이지 않는다.
2. **10~15분 체험**: 스폰 → 관찰자 톤 서사 전달 → 디스코드 초대 링크 수령 → `/confirm` OTP 입력 안내 → 메인 월드 이관까지 한 루프가 10~15분.
3. **세계관 인상 심기**: 수도·관찰자·세계수라는 3대 아이콘을 짧게 노출해 본편 진입 시 기시감을 만든다.
4. **메인 서버 부하 최소화**: 청크 로딩·엔티티·리소스팩 비용을 메인 월드 대비 5% 이하로 유지한다.
5. **재방문 통제**: 메인 이관 후 재진입은 원칙 차단하되, 명시적 복귀 수단(`/experience`) 한 가지만 유지한다.

## 핵심 규칙
- 체험 월드는 **디스코드 연동 전 단계**이므로 계정 데이터(월렛·캐릭터·진행도)는 생성·저장하지 않는다. 캐릭터 엔티티는 "손님" 세션으로 취급한다.
- 체험 월드에서는 **월드 사운드 제거 범위 정책**(월드 앰비언트·BGM 자동 재생 제거)을 메인 월드 기준 그대로 적용한다. 광장 종·환경 BGM은 없음, 파문 링 시각 이펙트 + 채팅 서사로 대체.
- **전투·블록 파괴·아이템 드롭·경매·영지 호출 전면 차단**. WorldGuard 리전 + EmpireRPG 플러그인 훅 이중 차단.
- 관찰자 NPC는 본편 관찰자의 **거울상**(프롤로그 전용 대역)으로 분리. 본편 관찰자 플래그·대사와 연결하지 않는다.
- 디스코드 초대 NPC는 **단 1명, 웰컴 광장 중앙**에 배치. 여러 NPC에 링크를 분산하지 않는다.
- OTP 입력 안내는 **체험 월드 내부 NPC 1회 + 액션바 상시 고정** 2중 안내. 채팅 스팸 금지.
- 1막 메인 스포일러 금지선: 4테마 이름·메인 7 선택지·`???의 편지`·세력명(기사단/그림자/귀족) 모두 노출 금지. 허용 키워드는 "수도 황성", "관찰자", "세계수", "당신이 선택할 길".

## 구현 포인트

### 1. 월드 스펙 상한
- **월드 크기**: 반경 4청크(64×64 블록) 이내. 플레이어가 실질 이동 가능한 영역은 웰컴 광장 중심 32×32 블록. WorldBorder로 64×64 하드 캡 설정.
- **Y 범위 활용**: Y 60~80 구간만 실사용(세계수 상징물은 Y 90까지 예외 허용). 지하·천공 레이어 없음.
- **맵 생성 방식**: 플랫 월드 베이스 + 커스텀 스키매틱 1장(`experience_plaza.schem`). 바닐라 월드 생성기 호출 금지(청크 생성 비용·엔티티 랜덤 스폰 방지).
- **광원**: 풀 라이트 규칙(`doDaylightCycle=false`, 시간 6000 고정 정오). 횃불·랜턴 대신 `gameRule doFireTick=false` + 빛 블록(light block) 사용으로 엔티티 이펙트 최소화.
- **스카이박스**: 기본 오버월드. 리소스팩 없음(아래 경량화 원칙과 연동). 세계수 실루엣은 블록 구조물로 표현.
- **환경 룰**: `mobSpawning=false`, `weatherCycle=false`, `keepInventory=true`(아이템 없지만 안전장치).

### 2. 스폰 구조
- **최초 스폰 좌표**: `experience_world` 스폰 포인트 (0, 65, 0). 광장 중앙 관찰자 앞 2블록 거리.
- **웰컴 광장 블로킹**: 원형 광장(반경 12블록) + 석재 테두리 + 중앙 세계수 모형(잎 블록 + 빛 블록). 광장 외곽 4방향에 짧은 복도(길이 8블록) → 복도 끝은 WorldBorder.
- **NPC 배치도** (총 5명, Citizens 플러그인 기반):
  1. **관찰자(프롤로그 대역)** — 광장 중앙, 스폰 지점 앞. 세계관 첫 대사 담당.
  2. **수도의 속삭임 NPC** — 광장 북쪽 복도 끝. 수도 황성 실루엣 + 짧은 플레이버 대사.
  3. **세계수의 목소리 NPC** — 광장 동쪽 복도 끝. 세계수 모형 하단에서 읊조리는 한 줄.
  4. **길잡이 NPC(디스코드 초대)** — 광장 남쪽 복도 끝. **유일한 디스코드 초대 링크 제공자**. 이 NPC만 상호작용 필수 경로.
  5. **문지기 NPC(OTP 안내)** — 광장 서쪽 복도 끝. `/confirm <OTP>` 입력 방법 안내 + 메인 월드 이관 트리거.
- 3~5번 NPC는 투어 동선의 핵심이며 1~2번은 톤 장식용. 투어 길이 조절 시 1~2번을 우선 가감한다.

### 3. 체험 동선 (10~15분 투어)
스폰 시점부터 단계별 타이머와 기대 행동을 다음과 같이 설계한다.

1. **T+0~2분 — 각성**: 스폰 직후 관찰자 NPC가 자동 말 걸기(접근 감지 5블록). 프롤로그 3~4줄. "너는 지금 문턱에 서 있다" 톤.
2. **T+2~5분 — 세계관 인상**: 북/동 복도 NPC 2명과 자유 탐색. 클릭하지 않아도 지나치면 액션바 한 줄씩 출력(통과 트리거). 스킵 가능.
3. **T+5~9분 — 디스코드 초대**: 남쪽 길잡이 NPC 상호작용. 디스코드 초대 링크 + 약관 봇 사용법 GUI 표시(Book GUI). 유저가 디스코드 가입·약관 동의 진행.
4. **T+9~13분 — OTP 대기**: 문지기 NPC 상호작용. `/confirm <OTP>` 입력 방법 + "봇이 DM으로 보낸 6자리 코드를 채팅에 입력하세요" 안내. 화면 상단 액션바에 "OTP 대기 중" 상시 표시.
5. **T+13~15분 — 이관**: `/confirm` 성공 시 페이드 연출 2초 → `player.teleport(main_world_spawn)`. 실패 시 문지기 NPC가 재발급 안내.

15분 초과 유저에 대해서는 "30분 유휴 시 세션 종료 안내 메시지 + 60분 유휴 시 강제 로그아웃" 2단 타이머 적용.

### 4. WorldGuard 영역 설정
`experience_world` 전역에 단일 리전 `exp_all`을 생성하고 다음 플래그를 적용한다.

- `pvp: deny`
- `mob-spawning: deny` (게임룰 + 플래그 이중)
- `damage-animals: deny`, `pvp: deny`, `entity-painting-destroy: deny`, `entity-item-frame-destroy: deny`
- `block-break: deny`, `block-place: deny`
- `interact: deny` (예외 화이트리스트: Citizens NPC 우클릭 이벤트, Book GUI 트리거 블록)
- `item-drop: deny`, `item-pickup: deny`
- `use: deny` (예외: 체험 월드 전용 Book)
- `enderpearl: deny`, `chorus-fruit-teleport: deny`
- `exp-drops: deny`, `exp-gain: deny`
- `feed-amount: 20`, `feed-delay: 1`, `heal-amount: 20`, `heal-delay: 1` (HP/허기 자동 풀 유지)
- `fire-spread: deny`, `lava-flow: deny`, `water-flow: allow` (광장 연출용 분수대 허용)

EmpireRPG 훅 이중 차단 항목:
- `AuctionService.requestOpen()` → 체험 월드 world_name 필터로 조기 리턴
- `EstateService.requestTeleport()` → 동일
- `RankingService.submitScore()` → 이벤트 자체 수신 차단
- `WalletService.credit()` → 체험 월드 세션 uuid면 무시
- `CombatService.onDamage()` → 게임룰 레벨에서 막고, 남는 커스텀 피해 이벤트도 world 필터로 차단

### 5. 디스코드 초대 NPC 상호작용
**대화 스크립트 가안**(길잡이 NPC):
1. "여기서 끝은 아니야. 저 너머로 가려면 먼저 너의 목소리를 등록해야 해."
2. "아래 버튼으로 디스코드 서버로 들어가. 봇이 너를 맞이할 거야." → [디스코드 초대 링크 열기] 버튼
3. "디스코드에서 약관을 읽고 동의 버튼을 누르면, 봇이 너에게 6자리 코드를 보낼 거야."
4. "받은 코드를 들고 서쪽 문지기에게 가. 그가 길을 열어준다."

**표시 방식**:
- Book GUI에 디스코드 초대 URL을 `clickEvent: OPEN_URL` 형태로 삽입. QR 코드 표기는 리소스팩 의존이 커서 1차 범위 제외(오픈 질문).
- 링크는 서버 설정값(`config.discord_invite_url`)에서 주입, 하드코딩 금지.
- 클릭 후 5분 내 유저가 돌아오지 않으면 길잡이 NPC가 "아직 저쪽에 있니?" 리마인드 1회.

**문지기 NPC 대화 스크립트 가안**:
1. "코드를 가져왔나? 여기에 말해봐. `/confirm` 다음에 여섯 자리를."
2. (성공) "좋아. 너의 자리는 준비됐다." → 페이드 연출 후 `player.teleport()` 호출.
3. (실패) "그 코드는 여기서 울리지 않아. 다시 봇을 불러봐." → 재발급 안내(`/resend` 봇 명령 안내).

### 6. 리소스 경량화 원칙
메인 서버 영향 최소화가 최우선이다.

- **청크 프리로딩**: 체험 월드는 스폰 청크 **4×4(총 16청크)** 만 force-load. 나머지는 유저 접근 시만 로드. `spawn-chunks` 수치를 기본 11×11에서 4×4로 줄인다.
- **엔티티 수 상한**: NPC 5명 + 장식 엔티티(아머 스탠드 등) 10개 = 월드 전체 15 엔티티 이내. 몹·아이템 엔티티 0 유지.
- **리소스팩**: 체험 월드 전용 리소스팩 **미사용** 확정. 메인 월드와 동일한 기본 리소스팩만 사용하거나 아예 서버 리소스팩 강제 없이 바닐라 범위 내에서 표현.
- **파티클**: 광장 중앙 세계수 발 부근 `END_ROD` 약한 파티클만 주기 5초 단발. 그 외 금지.
- **사운드**: 월드 앰비언트·BGM 자동 재생 없음. NPC 상호작용 시 `entity.villager.yes`(vol 0.3) 1회만 허용.
- **TPS 가드**: 체험 월드 tick 비용이 메인 월드 tick의 5%를 초과하면 경량화 재검토 트리거(운영 모니터링 지표).
- **플러그인 리스너 필터**: Listener 등록 시 `event.getWorld().getName()` 체크를 상단에서 하여 체험 월드 이벤트는 불필요 플러그인이 조기 리턴하도록 한다.

### 7. 재진입 정책
- **기본 원칙**: 메인 월드로 이관된 플레이어는 체험 월드를 자동 재방문하지 않는다. 최초 접속 플래그(`player.experience.intro_seen=true`)가 true면 메인 월드 스폰으로 직행.
- **명시적 복귀 수단**: `/experience` 명령어 제공. 제한 규칙:
  - 메인 월드에서만 호출 가능(체험 월드 내부에서는 미등록).
  - 전투·던전·경매·영지 인스턴스 밖에서만 호출 가능.
  - 쿨다운 24시간(튜토리얼 복습 수요 흡수 + 남용 방지).
  - 체험 월드 진입 시 **세션 플래그**만 교체하고 계정 데이터는 건드리지 않는다.
  - 체험 월드 내부에서 `/leave` 또는 문지기 NPC 재상호작용 시 메인 월드 원 위치로 복귀. `/confirm` 재입력은 요구하지 않는다.
- **v1.1 재동의 유예 강등 유저**: 72h 재동의 초과로 체험 월드 강등된 케이스는 일반 체험 세션과 분리해 "재동의 복귀 플로우"로 라우팅(문지기 NPC가 재동의 안내 전담). 본 문서 1차 범위 밖, 2차 범위에서 다룬다.

## 레벨 디자인 대안 비교

### 대안 A — 미니멀 플랫 광장
- **형태**: 32×32 플랫 석재 광장 + 중앙 관찰자 1명 + 디스코드 NPC 1명 + 문지기 NPC 1명. 세계수·수도 상징물 없음.
- **장점**: 제작·유지 비용 최소. 메인 서버 부하 거의 없음. 구현 기간 3일 이내.
- **단점**: 세계관 인상 거의 제로. "여기가 왜 포로 서버인지" 신규 유저가 체감하지 못함. 이탈률 우려.

### 대안 B — 수도 스폰 인트로 축소판 (권장)
- **형태**: 수도 황성 입구 광장을 1/4 스케일로 재현 + 세계수 모형 + 관찰자 프롤로그 대역 + NPC 5명 + 짧은 복도 4방향.
- **장점**: 1막 메인 1 "낯선 수도" 진입 시 기시감 유발. 세계관 3대 아이콘(수도·관찰자·세계수)을 짧게 노출. 본편 스포일러는 피하면서 톤 일관성 확보. 구현 기간 1~1.5주.
- **단점**: 수도 본편과 스키매틱이 너무 닮으면 복제감. 축소·상징화 레벨 조율 필요. NPC 5명 + 장식 엔티티 관리 필요.

### 대안 C — 관찰자 공간 확장
- **형태**: 관찰자 공간(1막 메인 11 `???의 편지` 연출지)을 프롤로그용으로 별도 제작. 공허·별빛 배경 + 관찰자 + 세계수 조각 + 디스코드/문지기 NPC.
- **장점**: 본편 관찰자 공간과 시각적 연결감이 강해 "이 서버의 관찰자 서사"를 강하게 인상. 소수 유저에게 몰입 깊이 제공.
- **단점**: 본편 관찰자 연출(1막 메인 11)의 희소성·놀람 효과를 선점·소모할 리스크. 1막 메인 11의 "첫 조우" 무게가 약해진다. 4막 인간찬가 서사까지 연결되는 관찰자 스토리와 충돌 가능성.

### 권장안: **B (수도 스폰 인트로 축소판)**
- 근거 1: 세계관 3대 아이콘을 본편 스포일러 없이 안전히 소개할 수 있다. A는 톤 전달력이 약하고 C는 본편 희소성을 훼손한다.
- 근거 2: 1막 메인 1 "낯선 수도"와 톤 연결이 자연스럽다. 유저가 메인 월드 이관 직후 수도 광장을 보며 "아, 저기서 본 그곳의 본체구나" 감각을 얻는다.
- 근거 3: 제작 범위가 중간(1~1.5주)이고, 리소스팩 없이 바닐라 블록 범위에서 표현 가능해 경량화 기준(메인 월드 5% 이내)을 지킬 수 있다.

## 오픈 질문
- **디스코드 초대 링크 QR 표시**: Book GUI + `OPEN_URL` 1차 범위로 충분한지, 지도 아이템 QR(리소스팩 텍스처)까지 2차에서 볼지.
- **관찰자 프롤로그 대역 NPC와 본편 관찰자**: 완전히 다른 엔티티로 간주할지, 본편 플래그 `observer_first_met`을 체험 월드에서 set하지 않고 메인 이관 후 재set할지(현재 문서 기본: 연결 안 함).
- **OTP 입력 실패 상한**: 5회 실패 시 체험 월드 자동 킥? 10회? 봇 재발급 쿨다운과 맞물린 밸런스.
- **`/experience` 쿨다운 24h가 적정한지**: 가이드·커뮤니티 홍보 목적 재방문 수요가 많으면 12h로 완화.
- **웰컴 광장 분수대 수류 허용**: 파티클·사운드·엔티티 비용 추가 이슈. 1차 범위에서 제외하고 2차에서 재검토 권장.
- **체험 월드 세션 종료 시 인벤토리 초기화 보장 방식**: 체험 월드에서는 아이템을 지급하지 않으나, 혹시 모를 bug drop·plugin exploit 대비 이관 시점 강제 인벤토리 클리어 플래그 필요 여부.
- **v1.1 재동의 강등 복귀 플로우**: 일반 최초 체험과 UI·NPC 분리할지, 문지기 NPC가 상태 감지해 대사만 교체할지. 1차 범위 밖으로 확정하되 2차 에픽 선후 관계 정의 필요.
