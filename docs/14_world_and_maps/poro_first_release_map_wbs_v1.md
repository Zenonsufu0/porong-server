## 1. 구현 목표

1. 시즌1 오픈일 이전까지 **체험 월드 + 수도 황성 내부(24×24 청크) + 수도 외곽 필드보스 5종 존** 빌드 완료.
2. 1인 건축가 기준 **1~2개월(240~480 man-hour 총 가동 범위)** 내 QA 완료 상태로 공개 가능한 빌드 품질 확보.
3. FAWE + 스키매틱 + 리소스팩 + 공개 에셋(상업 허용 라이선스만)으로 **1막 메인 1~5(수도 내부 완결) + 수도 히든 메인 초입** 플레이가 가능한 최소 맵 커버리지 제공.
4. 리소스팩 ON/OFF 양쪽에서 수도 핵심 랜드마크 **실루엣 식별 가능** 수준의 바닐라 폴백 유지(리소스팩 미적용자는 체험 월드 강등이 기본이지만, 수도 방문 경로가 잠시라도 열려 있으면 식별성이 리스크 흡수재가 된다).
5. 스키매틱·FAWE 재사용 도구 세트를 정리해 **2차 확장(4테마 필드) 개발 속도 기반** 마련 — 영지 월드 7×7 슬롯 스키매틱 파이프라인과 명명·저장 규약 공유.
6. **모든 외부 공개 에셋의 라이선스가 CC0/MIT/Apache 2.0/CC-BY 중 하나임을 증빙 가능한 대장(ledger)** 을 공개 시점에 완비.

## 2. 선행조건

### 2.1 문서·기획 선행조건
- 체험 월드 B안 레벨 디자인 명세 확정(완료).
- 수도 NPC **11명 역할·배치 좌표 1차 맵핑** 확정(부분 확정 상태. 회중·잿빼 등 서브 2 인물의 1차 공개 포함 여부는 오픈 질문).
- 수도 외곽 필드보스 **5종 스폰 존 크기·접근 동선·배리케이드 규격** 초안.
- 1막 메인 1~5 공용 허브 씬 카드(완료) + 히든 메인 초입 트리거 좌표.
- 영지 청사진 포털 블록 존재 여부(Lv1 부여 전까지 비활성 플레이스홀더만 요구 — 포털 구현 자체는 3월드 전환 에픽).

### 2.2 기술 선행조건
- Paper 1.21 단일 인스턴스에 `experience_world`·`main_world`·`estate_world` 3월드 부팅 가능(3월드 전환 에픽 PR0~PR4 머지 이후). 본 WBS 1차 범위는 `experience_world`·`main_world` 2개만 실사용.
- FAWE 설치 + 스키매틱 저장 디렉터리(`plugins/FastAsyncWorldEdit/schematics/`) 백업 경로 확보. 네임스페이스 하위 디렉터리 분리(`capital/`, `experience/`, `estate/`, `field_boss/`).
- Citizens 플러그인 NPC 엔트리 템플릿(외형·스킨·클릭 핸들러 연결 규약) 및 EmpireRPG `interaction_profile_id` 스펙.
- WorldGuard 리전 네이밍 규약(`exp_all`, `capital_wall_*`, `capital_market`, `capital_slum`, `capital_outer_boss_*` 등).
- 리소스팩 v0.1 초안 빌드 접근 가능(해시 고정). 미도착 시 커스텀 블록 필요 지점을 **바닐라 플레이스홀더**로 선시공.
- MythicMobs 5종 필드보스 스폰 정의 스텁 존재(외형·스킬은 별도 에픽, 본 WBS는 스폰 포인트 연결만).

### 2.3 운영·리소스 선행조건
- 리소스팩 팀(별도 인원)과 **주 1회 싱크** 창구 확보.
- 1인 건축가 주당 가동 시간 확정(6~8h × 5~6일 = 주 30~48h).
- **공개 에셋 라이선스 검증 담당자** 확정(기본: 사용자 본인). 라이선스 체크리스트 서식은 본 WBS 3.7 BP-22에서 정의.

### 2.4 블로커 후보
- 리소스팩 v0.1 도착 지연 → W6 BP-19(랜드마크 체크)와 W7 BP-24(ON/OFF 스모크)가 플레이스홀더 교체 비용을 흡수. 교체 비용은 W8 폴리싱으로 흡수한다.
- FAWE 라이선스 성격(LGPL 파생) 재검토 필요 여부 — 본 WBS는 "플러그인 바이너리 사용만 하고 맵 에셋엔 FAWE 자체가 포함되지 않는다"는 1차 판단 아래 진행. 사용자 최종 확인 포인트.
- 수도 NPC 11명 중 회중·잿빛(빈민가 그림자 트랙)의 1차 공개 포함 여부 미확정 — 포함 제외 시 BP-09 빈민가 구역은 일부 동선 차단벽 + 플레이스홀더로 축소.

## 3. 작업 분해

작업 단위는 **빌드 패킷(Build Packet, BP)** 로 부른다. 각 BP는 독립적으로 완료·검증·롤백 가능한 최소 단위이며, 가능하면 FAWE `//schematic save` 1회로 아카이브 가능한 크기를 목표로 한다. 시간은 모두 **사용자 참고용 범위** — 1인 속도 편차가 큼을 전제.

영역 코드: EW(체험월드) / CW(수도외벽) / CC(수도중앙) / CB(수도건물) / CD(수도구역) / NP(NPC 배치) / FB(외곽 필드보스) / RP(리소스팩 정합) / LI(라이선스) / QA(테스트·폴리싱) / OP(운영).

## 6. 기술 리스크

### R1. 1인 병목·번아웃 (최대 리스크)
- 크리티컬 패스의 CW-01 / CB-02 / NP-02 지연은 전체 일정을 직선적으로 밀어낸다.
- **완화**: 주 작업일 고정(월~금) + 토·일 의무 휴식. 같은 주차 내 "블로킹·디테일·문서화"를 번갈아 배치.
- **완화**: BP를 2~14h 단위로 분해했기 때문에 하루 목표 1~2 BP가 가능. 달성/미달성 체감이 일일 단위로 생긴다.

### R2. 시즌1 오픈 일정 미확정
- 일정이 앞당겨지면 QA 2주가 1주로 압축 → 품질 저하.
- **완화**: W6 완료 시점에 "**최소 공개 가능 상태**"(MVP) 판정 기준을 미리 정의. 필드보스 3종·수도 6구역 → 3구역 축소로도 공개 가능한 폴백 안 유지.

### R3. 공개 에셋 라이선스 충돌 (2026-04-20 신규 핵심)
- CC0/MIT/Apache 2.0/CC-BY만 허용. CC-BY-SA("공유 동일 강제"로 서버 전체 바이럴 리스크) · CC-BY-NC(상업 서버 전환 시 문제) · GPL(바이럴) 자동 탈락.
- **완화**: LI-01~04를 상시 트랙으로 돌림. 외부 스키매틱/텍스처/폰트/사운드를 받을 때마다 "라이선스·출처·URL·저자·해시"를 대장에 즉시 기록. 미기록 에셋은 **자동 제거 대상**.
- **완화**: 라이선스 모호·탈락 에셋은 바닐라 블록 교체로 대체. 자체 제작 블로킹이 오히려 안전 경로.
- **완화**: CC-BY 저작자 표기 방식은 리소스팩 `credits.txt` 또는 서버 내 `/credits` 명령어로 통합. 맵 안에 표지판으로 심으면 톤 저해.

### R4. 외주 미포함 → 리소스팩 팀과의 분담 경계
- 픽셀 아트·NPC 외형은 건축가 영역 밖이라 리소스팩 팀 지연이 그대로 QA 지연.
- **완화**: NPC 외형은 리소스팩 미적용 시 **바닐라 플레이어 스킨 대체** 계약을 사전 합의. 맵 QA는 외형 최종본 없이도 진행 가능하게 분리.
- **완화**: 커스텀 블록 필요 지점을 BP별로 "팩 의존 ON" 태그로 마킹. 팩 도착 전에는 바닐라 플레이스홀더(광택 안산암·상아 테라코타 등) 고정.

### R5. FAWE 복제·회전 시 지형 부정합
- 외벽 스키매틱 3면 paste 시 지형 기복 차이로 해자·입구 높이가 어긋날 수 있음.
- **완화**: CW-01 스키매틱 저장 시 **바닥 기준 Y축 절대값** 고정. paste 후 FAWE `//smooth` 지형 적응은 CW-05 단계에서 일괄 처리.
- **완화**: 단일 청크 크기 이상의 `//paste`는 비동기 옵션 `-a`로 호출(메인 스레드 TPS 보호).

### R6. 24×24 청크 기준점 오판
- 384×384 블록 안에 6~8개 구역 + 도로망이 들어가면 구역당 평균 128×64~96×64 수준으로 좁을 가능성.
- **완화**: CC-01 완료 시점에서 실측 검증 후 구역 크기 재배분. 필요 시 빈민가를 외벽 안쪽 코너로 몰아 폭 확보.

### R7. 리소스팩 ON/OFF 양립
- 리소스팩에 의존한 커스텀 모델을 랜드마크 실루엣 핵심으로 사용하면 OFF 상태에서 식별 불가.
- **완화**: 랜드마크 실루엣은 **바닐라 블록 윤곽**으로 먼저 완성하고, 리소스팩은 **표면 데코**만 담당하도록 역할 분리.

### R8. NPC 배치 좌표 미확정 (OP-01 의존)
- 수도 NPC 11명의 역할별 좌표가 W4 시작 시점에 확정되어 있어야 NP-01 재작업 방지.
- **완화**: 각 건물 BP(CB-01·02·04·06·08·09) 완료 시점마다 "NPC 앵커 좌표"를 1개씩 즉시 기록. NP-01에서는 기록된 좌표를 옮기기만.

### R9. 외곽 필드보스 5존 반경 겹침
- 외곽 반경 40청크 내 5존 배치가 과밀이면 스폰 간섭·플레이어 시야 오염 발생.
- **완화**: FB-01 착수 전 좌표 5개를 종이 또는 지도 PNG에 먼저 배치하고 반경 50블록 원으로 겹침 시각 확인. FAWE `//pos1`·`//pos2` 좌표 기록 선행.

### R10. 24×24 청크 기준 수도 반경 WorldBorder 수용성
- 외곽 필드보스 5존이 수도 반경(수도 메인 월드 전용 규칙) 안에 수용 가능한지 수치 검증 필요.
- **완화**: FB-01 착수 전에 수도 반경 per-player WorldBorder 규칙 문서와 좌표 대조. 초과 시 외곽 반경 확장 또는 존 1~2개 재배치.

- NP-04 기믹: 광장 파문 링만 1차 필수, 나머지 2건은 2차 이월 가능.

### 8.4 품질 스코프
- "**완성도 80% × 전 범위 커버**"를 목표로 한다. 국소 완성도 100%보다 전 범위 커버가 QA·공개에 유리.
