# 포로 서버 1차 맵 공개 WBS (Work Breakdown Structure) v1 드래프트

> 문서 버전: 2026-04-19 초안 (implementation-reviewer)
> 선행 결정: 2026-04-19 사용자 결정 A 4건
> - A-1 인력 = **1인 건축가**
> - A-2 공개 에셋 활용, **외주 없음**
> - A-3 외부 맵 툴 미도입, **FAWE + 스키매틱 + 바닐라 편집만**
> - A-4 4테마 오픈 순서 자유(2차 확장 범위)
> 상위 문서: `poro_map_production_feasibility_review_v1.md`, `poro_experience_world_level_design_draft.md`, `../poro_master_planning.md`

> **주의**: 본 WBS의 시간 추정치는 모두 **사용자 참고용 범위 값**이다. 1인 빌더의 경험·컨디션·스키매틱 라이브러리 정리 상태에 따라 편차가 크며, 실제 스케줄은 주차별 실측 진행률로 재보정해야 한다.

---

## 1. 구현 목표
1. 시즌1 오픈일 이전까지 **체험 월드 + 수도 황성 내부(24×24 청크) + 수도 외곽 필드보스 5종 존** 빌드 완료.
2. 1인 건축가 기준 **1~2개월(240~480 man-hour)** 내 **QA 완료 상태**에서 공개 가능한 빌드 품질을 확보.
3. 공개 에셋 + FAWE + 스키매틱 + 리소스팩 정합성만으로 **1막 메인 1~5(수도 내부 완결) + 수도 히든 메인 초입** 플레이가 가능한 최소 맵 커버리지를 제공.
4. 리소스팩 ON/OFF 양쪽에서 수도 핵심 랜드마크 실루엣이 식별 가능한 수준의 **바닐라 폴백 형태** 유지.
5. 스키매틱·FAWE 재사용 도구 세트를 정리해 **2차 확장(4테마 필드)** 개발 속도 기반을 마련.

## 2. 선행조건

### 2.1 문서 선행조건
- 체험 월드 B안 레벨 디자인 명세 확정(완료).
- 수도 NPC 11명 배치 좌표 및 역할(길드/기사단/행정/시장/빈민가) 1차 맵핑(선행 문서에서 확인 필요).
- 수도 외곽 필드보스 5종의 **스폰 존 크기·접근 동선·배리케이드 규격** 초안(미확정 시 오픈 질문으로 남김).
- 1막 메인 1~5 공용 허브 씬 카드(완료) + 히든 메인 초입 트리거 위치 정리.

### 2.2 기술 선행조건
- Paper 서버 인스턴스에 `experience_world`, `main_world` 월드 파일 분리 셋업.
- FAWE 플러그인 설치 + 스키매틱 저장 디렉터리 백업 경로 확보.
- Citizens 플러그인 NPC 엔트리 템플릿(외형·스킨·클릭 핸들러 연결 규약).
- WorldGuard 리전 네이밍 규약(`exp_all`, `capital_wall_north` 등) 합의.
- 리소스팩 버전·해시 고정 정책 확정(빌드 중 텍스처 변경 시 재빌드 영향 파악).

### 2.3 팀 선행조건
- 리소스팩 팀(별도 인원)과 **주 1회 싱크** 창구 확보. 픽셀 아트·NPC 외형 병목 방지.
- 1인 건축가 주당 가동 시간 확정(6~8h × 5~6일 = 주 30~48h).

## 4. 작업 시퀀스·의존성

### 4.1 크리티컬 패스
```
EW-01 → EW-02 → EW-03 → EW-07 → EW-10 → QA-01
                                          │
CW-01 → CW-02 → CC-01 → CC-02 → CB-01~06 ─┤
                                          │
                            NP-01 → NP-02 ─┤→ QA-04 → QA-05 → QA-06
                                          │
                  FB-01 → FB-02~06 → FB-07 ┤
                                          │
                            RP-01/RP-02 ───┘
```

### 4.2 병렬 가능 구간
- **W1 동안**: EW-01~05 단독 진행. 수도는 대기.
- **W2~W3 동안**: CW-01 외벽 1면 작업 중 EW-06~10(체험 월드 마무리) 병렬 가능.
- **W4**: CB-01~06 중 서로 독립한 건물은 1일 1건 단위로 병렬 가능(다만 1인이므로 실제 병렬은 불가, "아침 1건 저녁 1건" 식 분할).
- **W5**: NP-01~02와 FB-01~02 병렬 가능(서로 다른 월드 영역).
- **RP-01/RP-02**는 수도 완성 전 부분 검수로 앞당겨도 됨.

### 4.3 병목 예상 지점
- **CW-01(외벽 프로토타입)**: 여기서 스키매틱이 확정되지 않으면 CW-02 이후 전부 지연. 가장 리스크 큰 크리티컬.
- **CB-06(빈민가)**: 좁은 골목 구조 특성상 FAWE 재사용이 어렵고 수작업 비중이 높아 지연 리스크.
- **RP-03**: 리소스팩 팀 대응 속도가 느리면 QA-05가 지연. 주 1회 싱크 엄수 필요.

## 5. 주차별 마일스톤

**총 8주(주 30~48h 기준, W7~W8는 QA·폴리싱)** 구성.

### W1 — 체험 월드 웰컴 광장 기본
- EW-01~05 완료(월드 초기화·광장·세계수·복도·관찰자 앵커).
- 산출물: `experience_plaza.schem` 스키매틱 백업.

### W2 — 수도 황성 외벽 + 주 광장
- CW-01 완료(외벽 1면 프로토타입 스키매틱화).
- CW-02 시작(3면 paste).
- EW-06~10 병렬 완료(체험 월드 전체 완성).

### W3 — 수도 주요 건물 1차
- CW-03~05 완료(성문·망루·외곽 정리).
- CC-01~02(중앙 광장·도로망) 완료.
- CB-01(기사단)·CB-02(길드) 완료.

### W4 — 빈민가·시장·농경지
- CB-03(행정)·CB-04(시장)·CB-05(농경)·CB-06(빈민)·CB-07(히든 초입) 완료.
- 수도 내부 블로킹 전 구역 완성.

### W5 — NPC 배치 + 외곽 필드보스 5종 존 기본
- NP-01~03(NPC 배치·트리거·히든 연결) 완료.
- FB-01~06(외곽 정리·5종 존 블로킹) 완료.

### W6 — 기믹 연출·폴백·스폰 테스트
- NP-04(수도 기믹 연출 2~3건) 완료.
- FB-07(스폰 테스트) 완료.

## 6. 기술 리스크

### R1. 1인 병목·번아웃
- **리스크**: 주 30~48h × 8주는 단거리 스프린트 느낌이 강해 중반(W4~W5) 생산성 급락 가능.
- **대응**: W2·W5 말 **의무 휴식일** 삽입. 주간 진행률 측정 후 W4 말 재평가 포인트 고정.

### R2. 시즌1 오픈 일정 미확정
- **리스크**: 일정이 앞당겨지면 QA 2주가 1주로 압축 → 품질 저하.
- **대응**: W6 완료 시점에서 "**최소 공개 가능 상태**"(MVP) 판정 기준을 미리 정의. 필드보스 3종만으로도 공개 가능한 폴백 안 유지.

### R3. 공개 에셋 라이선스 범위
- **리스크**: 공개 스키매틱·아트 에셋의 라이선스(CC BY, 상업 사용 여부, 재배포 조건)가 포로 서버 운영 형태와 충돌 가능.
- **대응**: 스키매틱 도입 전 라이선스 체크리스트 통과 필수(출처·라이선스명·재배포 허용). 상업 사용 불가 에셋은 **구조 참고**만 하고 블록 단위 재빌드.

### R4. 외주 미포함 → 리소스팩 팀과의 분담
- **리스크**: 픽셀 아트·NPC 외형은 건축가 영역 밖이라 리소스팩 팀 지연이 그대로 QA 지연.
- **대응**: **NPC 외형은 리소스팩 미적용 시 바닐라 플레이어 스킨 대체** 계약을 사전 합의. 맵 QA는 외형 최종본 없이도 진행 가능하게 분리.

### R5. FAWE 복제·회전 시 지형 부정합
- **리스크**: 외벽 스키매틱 3면 paste 시 지형 기복 차이로 해자·입구 높이가 어긋날 수 있음.
- **대응**: CW-01 스키매틱 저장 시 **바닥 기준 Y축 절대값** 고정. paste 후 FAWE `/smooth` 지형 적응은 CW-05 단계에서 일괄 처리.

### R6. 24×24 청크 기준점 오판
- **리스크**: 384×384 블록 안에 6개 구역 + 도로망이 들어가면 **구역당 평균 128×64** 수준으로 좁을 가능성. 빈민가·시장 좁아 보이는 체감 이슈.
- **대응**: CC-01 완료 시점에서 실측 검증 후 구역 크기 재배분. 필요 시 빈민가를 외벽 안쪽 코너로 몰아 폭 확보.

### R7. 리소스팩 ON/OFF 양립
- **리스크**: 리소스팩에 의존한 블록 커스텀 모델을 랜드마크 실루엣 핵심으로 사용하면 OFF 상태에서 식별 불가.
- **대응**: 랜드마크 실루엣은 **바닐라 블록 윤곽**으로 먼저 완성하고, 리소스팩은 **표면 데코**만 담당하도록 역할 분리.

### R8. NPC 배치 좌표 미확정
- **리스크**: 수도 NPC 11명의 **역할별 좌표**가 선행 문서에서 완전히 확정되지 않은 경우 NP-01 시작 시 재작업.
- **대응**: W3 완료 전까지 NPC 배치 좌표를 오픈 질문으로 수집 → W4 시작 시점에 확정 고정.

## 8. 1차 버전 권장안

### 8.1 반드시 1차에 포함
- 체험 월드 B안 **전체**(NPC 5명·세계수·복도 4방향).
- 수도 황성 **외벽 4면 + 4성문**.
- 수도 중앙 광장·메인 도로 4축.
- 주요 건물 **6종**(기사단/길드/행정/시장/농경/빈민).
- 수도 NPC **11명 배치** + 메인 1~5 진입 트리거.
- 수도 히든 메인 **초입 1포인트**.
- 외곽 필드보스 **3종 이상**(최소 MVP: 3종, 목표: 5종).
- 리소스팩 ON/OFF 양립 폴백.

### 8.2 FAWE·스키매틱 재사용 전략(권장)
- **외벽 1면 스키매틱화**: 동일 구조 4면 복제(CW-01 → CW-02).
- **필드보스 존 셸 스키매틱**: 48×48 기본 지형 + 배리케이드 하나 만든 뒤 5회 paste, 표층 데코만 차등.
- **성문 4종** 중 2종은 동일 스키매틱 mirror. 나머지 2종만 고유화.
- **영지 7×7 슬롯 스키매틱**(3월드 에픽)과 **스키매틱 저장/불러오기 도구 체계**를 공유. 명명 규약(`poro_<영역>_<세부>_v1.schem`) 통일.

### 8.3 품질 스코프
- "**완성도 80% × 전 범위 커버**"를 목표로 한다. 국소 완성도 100%보다 전 범위 커버가 QA·공개에 유리.
- 폴리싱은 W8에서 **눈에 띄는 실루엣 이슈 Top 10**만 집중 수정.
