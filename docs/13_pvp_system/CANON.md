# PvP 대전 시스템 설계서

> **[STATUS: CANON]** — 확정 기준: DL-069 (2026-05-28)
> 1차 시즌 포함. 구현 범위·규칙 모두 이 문서 기준.

---

## 1. 대전 종류

| 종류 | 장비 조건 | 점수 | 보상 | 진입 조건 |
|---|---|---|---|---|
| 자유대전 | 현재 착용 장비 그대로 | 없음 | 없음 | 영지에서만 |
| 정규대전 | IL 60 동일화 (§3 참고) | O (+15/−10) | 없음 (랭킹) | 영지에서만 |
| 친선대전 | 현재 착용 장비 그대로 | 없음 | 없음 | 영지에서만, 상대 수락 필요 |

---

## 2. 진입 흐름

### 자유대전 / 정규대전
1. 영지 → 대전 메뉴 → 대전 종류 선택 → 대기열 즉시 진입
2. 매칭 성사 시 수락·거절 없이 아레나로 텔레포트
3. 대전 종료 후 자신의 영지로 자동 귀환

### 친선대전
1. 영지 → 대전 메뉴 → 친선대전 → 상대 닉네임 입력
2. 상대방에게 알림: `§e[닉네임]§f 님이 친선대전을 신청했습니다. §a[수락] §c[거절]`
3. **수락 조건:** 상대가 영지에 있을 때만 수락 가능. 다른 장소에 있으면 `§c영지에서만 수락 가능합니다.` 안내.
4. 수락 → 양측 아레나 텔레포트. 거절 or 30초 무응답 → 요청 소멸.

---

## 3. 정규대전 장비 동일화

- **기준:** 무기·투구·갑옷·각반·신발 5슬롯 모두 **12강 고정 (총 IL = 60)**
- **동일화 항목:** 강화 수치(12강), 등급 세부 스탯, 잠재능력 → 기본값으로 통일
- **유지 항목:** 무기 종류(클래스), 각인, 스탯 포인트 배분 (crit/spec/endur)
- **구현 방식:** 실제 인벤토리 변경 없이 `PvpContext`에 가상 스탯을 계산해 전투 시스템에 주입. 대전 종료 후 원래 스탯으로 자동 복귀.

---

## 4. 아레나 방 구조

- **방식:** 보스룸 풀과 동일 — 사전 생성된 고정 아레나 방을 배정·반납.
- **방 크기:** 미정 (권장: 20×20 평탄 플랫폼). 인게임 테스트 후 확정.
- **방 수:** 최소 10개 (동시 5경기 기준 여유). 서버 오픈 전 1회 `/poro-genarenas` 생성.
- **좌표:** `config.yml` `pvp-room-slots` 섹션에 정의.

---

## 5. 진행 규칙

| 항목 | 규칙 |
|---|---|
| 타임아웃 | **3분** |
| 타임아웃 승자 결정 | 남은 HP 비율이 높은 쪽 승리. 동일 시 무승부(정규: 점수 변동 없음). |
| 사망 처리 | 즉시 **패배 확정** → 자신의 영지로 텔레포트 |
| 생존자 처리 | 상대 사망 확인 후 자동 영지 귀환 (3초 대기 후) |
| 대기 중 서버 이탈 | 대기열에서 자동 제거 (패배 처리 없음) |
| 대전 중 서버 이탈 | 자동 패배 처리 (정규: −10점) |
| PvP 허용 구역 | 아레나 방 내부만. 그 외 WorldGuard pvp-deny 유지. |

## 6. 정규대전 점수·랭킹

| 항목 | 기준 |
|---|---|
| 초기 점수 | 100점 |
| 승리 | +15점 |
| 패배 | −10점 (최저 0점) |
| 무승부 | 변동 없음 |
| 초기화 주기 | 시즌 종료 시 (45일) |
| 랭킹 조회 | `/대전랭킹` 커맨드 또는 대전 메뉴 내 GUI |

---

## 7. 관련 커맨드

| 커맨드 | 기능 |
|---|---|
| `/대전` | 영지에서 대전 메뉴 GUI 열기 |
| `/대전랭킹` | 정규대전 랭킹 조회 |
| `/poro-genarenas` | (관리자) 아레나 방 사전 생성 |

> `/대전`은 영지 밖에서 사용 시 `§c영지에서만 사용 가능합니다.` 안내.

---

## 8. DB 스키마 (신규)

```sql
-- 정규대전 점수
CREATE TABLE pvp_rating (
    player_id   VARCHAR(36) PRIMARY KEY,
    score       INT         NOT NULL DEFAULT 100,
    wins        INT         NOT NULL DEFAULT 0,
    losses      INT         NOT NULL DEFAULT 0,
    updated_at  TIMESTAMP   NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 대전 기록 (선택)
CREATE TABLE pvp_match_log (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    match_type  ENUM('FREE','RANKED','FRIENDLY') NOT NULL,
    winner_id   VARCHAR(36),
    loser_id    VARCHAR(36),
    draw        BOOLEAN     NOT NULL DEFAULT FALSE,
    duration_s  INT,
    played_at   TIMESTAMP   NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

## 9. 오픈 질문 (미확정)

| 항목 | 상태 |
|---|---|
| 아레나 방 크기·개수 | 인게임 테스트 후 확정 필요 |
| 정규대전 점수 최저·최고 상한 | 최저 0점 확정. 최고 상한 미정 (현재 무제한) |
| 친선대전 — 요청 수락 타임아웃 | 30초 (잠정). 인게임 피드백 후 조정 가능 |
| 랭킹 구간(티어) | 1차 시즌은 점수 숫자 그대로 표시. 티어 시스템은 2차 시즌 검토 |
