> **[STATUS: REFERENCE]** — 클리어 기록 GUI 상세 참조. 공식 리소스팩 기준은 `index.md`가 우선.

> 작성일: 2026-05-17  
> 인벤토리: 27슬롯 (3×9), 바닐라 아이템 아이콘 전용

## 1. 슬롯 레이아웃

```
행\열   1        2        3        4         5       6    7    8       9
  1    ░         ░        ░        ░          ░      ░    ░    ░       ░
  2   [보스1]   [보스2]   [보스3]    ░      [최종보스]  ░    ░    ░   [트로피]
  3   [뒤로]     ░        ░        ░          ░      ░    ░    ░   [닫기]
```

| 슬롯 | 역할 |
|---|---|
| 9 | 시즌보스 1 클리어 기록 |
| 10 | 시즌보스 2 클리어 기록 |
| 11 | 시즌보스 3 클리어 기록 |
| 13 | 시즌 최종보스 클리어 기록 |
| 17 | 균열왕의 심장 (트로피) 보유 현황 |
| 18 | 뒤로 (보스 서브 허브) |
| 26 | 닫기 |

빈 슬롯: `GRAY_STAINED_GLASS_PANE`.

## 4. 서버 통계 표기 원칙

| 항목 | 표기 여부 | 비고 |
|---|:---:|---|
| 서버 최초 클리어 날짜·플레이어 | ✅ | 최종보스 슬롯만 표기 |
| 서버 최단 기록·플레이어 | ✅ | 최종보스 슬롯만 표기 |
| 전체 유저 시즌 평균 클리어 시간 | ✅ | 보스 1~3 슬롯에 표기 |
| 개인 총 클리어 횟수 | ✅ | 모든 보스 슬롯 |
| 파티 구성원 목록 | ❌ | 개인 정보 제한 |

## 5. 구현 참조

| 항목 | 내용 |
|---|---|
| 열기 트리거 | 보스 서브 허브 슬롯 15 클릭 |
| 접근 제한 | **없음** (전투 중 포함 언제나 열람 가능) |
| DB 테이블 | `boss_clear_records` (플레이어·보스·인원·클리어 시간·타임스탬프) |
| 서버 최단 기록 | `SELECT MIN(clear_time) FROM boss_clear_records WHERE boss_id=?` |
| 트로피 보유 수 | `player_items` 또는 `player_inventory` 내 트로피 아이템 수량 |
| 갱신 주기 | GUI 열기 시 1회 조회 |
