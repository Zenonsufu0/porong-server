# 장비 패널 GUI 설계

> **[STATUS: REFERENCE]** — 장비 패널 GUI 상세 참조. 공식 장비 기준은 `../04_combat_weapon_skills/CANON.md`가 우선.

> 작성일: 2026-05-16
> 인벤토리: 54슬롯 (6×9), 비트맵 배경 `menu_equipment.png`
> 1차 시즌: 무기·투구·흉갑·바지·신발만. 반지·목걸이·보조장비 없음.

## 1. 슬롯 배치 (0-indexed 슬롯 번호)

```
행\열   1    2    3    4    5       6      7   8    9
 1     [─── 장비 ───────────] [─────── 스탯 배분 + 표시 ────────]  (장식행)
 2    [무기] [치장] [토글]  ░  [치명 __] [부효과] [-] [+] [잔여포인트]
 3    [투구] [치장] [토글]  ░  [특화 __] [부효과] [-] [+]    ░
 4    [흉갑] [치장] [토글]  ░  [인내 __] [부효과] [-] [+]    ░
 5    [바지] [치장] [토글]  ░  [ATK __]  [DEF __]  [HP __]  ░    ░
 6    [신발] [치장] [토글] [핫바] [치명확률__%] [치명피해__%] [보스피해__%] [피감__%] [닫기]
```

**배분 가능 스탯 (행 2-4)**: 레벨업으로 얻은 포인트를 치명/특화/인내 트리에 투자. 비전투 중 재배분 가능.  
**표시 전용 스탯 (행 5-6)**: 장비 강화·잠재능력·스탯 트리 부효과 합산값. 배분 불가.

> ATK/DEF/HP는 장비 강화로만 성장. 레벨별 고정 증가 없음. (`level_stat_system_v1.md §0` 확정)

## 2. 외형 토글 시스템

슬롯 11·20·29·38·47 클릭 시 아래 3가지를 순환:

| 상태 | 아이콘 CMD | 동작 |
|---|---|---|
| **치장 보이기** (기본) | 치장 아이콘 | 치장 슬롯 아이템의 모델로 외형 표시 |
| **원본 보이기** | 원본 아이콘 | 장착 슬롯 아이템의 실제 모델로 표시 |
| **안보이기** | 숨김 아이콘 | 해당 부위 외형 숨김 (투명 처리) |

### 슬롯별 외형 처리 방법

**무기 (슬롯 9/10/11):**
- 치장 보이기: 핫바 지정 슬롯에 치장무기 CMD 아이템 배치
- 원본 보이기: 핫바 지정 슬롯에 실제 무기 아이템 배치
- 안보이기: 핫바 지정 슬롯에 투명 아이템(`barrier` + `item_name="§r"`) 배치

**갑옷 (투구·흉갑·바지·신발):**
- 치장 보이기: `EntityEquipment.setHelmet()` 등에 치장 아이템(CMD 텍스처 오버라이드)
- 원본 보이기: 실제 장착 아이템
- 안보이기: 가죽 갑옷 + 완전 투명 컬러 (`Color.fromRGB(0,0,0)` + alpha 처리) 또는 빈 슬롯

> **구현 주의**: 1.21 기준 `minecraft:hide_additional_tooltip` 컴포넌트를 사용하면 클라이언트 측 갑옷 렌더를 제어할 수 있다. 단, 커스텀 텍스처 갑옷은 리소스팩 `equipment/` 레이어를 통해 처리하므로 투명 처리는 별도 투명 텍스처 PNG를 준비해야 한다.

## 3. 핫바 슬롯 선택 (슬롯 48)

클릭 시 무기 배치 핫바 슬롯을 1→2→3→…→9→1 순환.

```
아이템 이름: "무기 슬롯: §e3번"   (현재 슬롯 번호 하이라이트)
아이템 설명: "클릭으로 변경"
CMD: hotbar_slot_icon (커스텀 아이콘)
```

변경 시:
1. 기존 핫바 슬롯의 무기 아이템 제거
2. 새 슬롯에 무기 아이템 배치
3. DB에 `weaponHotbarSlot` 업데이트
4. 해당 슬롯 잠금 재적용

## 4. 스탯 표시 슬롯 (읽기 전용)

패널 열릴 때 플러그인이 실시간으로 계산해서 아이템 이름·lore에 주입:

```java
// 예시: ATK 표시 슬롯 (슬롯 40)
ItemStack atkItem = new ItemStack(Material.CYAN_STAINED_GLASS_PANE);
ItemMeta meta = atkItem.getItemMeta();
meta.displayName(Component.text("⚔ 공격력  " + playerStats.getAtk())
    .color(NamedTextColor.WHITE)
    .decoration(TextDecoration.ITALIC, false));
meta.lore(List.of(
    Component.text("  기본: " + baseAtk).color(NamedTextColor.GRAY),
    Component.text("  강화: +" + enhanceAtk).color(NamedTextColor.GREEN)
));

// 예시: 치명 트리 배분 슬롯 (슬롯 13)
meta.displayName(Component.text("⚡ 치명  " + critChance + "%")...);
meta.lore(List.of(
    Component.text("  기본값: 5%").color(NamedTextColor.GRAY),
    Component.text("  스탯 배분: +" + statCrit + "%  (" + critPts + "pt)").color(NamedTextColor.GREEN),
    Component.text("  잠재능력: +" + potentialCrit + "%").color(NamedTextColor.LIGHT_PURPLE),
    Component.text("─ 부효과 ─").color(NamedTextColor.DARK_GRAY),
    Component.text("  치명피해량: +" + critDmgBonus + "%").color(NamedTextColor.GOLD)
));
```

**스탯 배분 슬롯 (행 2-4)**

|---|---|---|
| 13 | 치명타 확률% | 기본 5% + 스탯 배분(+0.30%/pt) + 잠재 합산 |
| 14 | 치명타 피해% (부효과) | 스탯 배분 포인트 × 0.2% |
| 22 | 스킬 피해% | 스탯 배분(+0.30%/pt) + 잠재 합산 |

## 5. 치장 시스템 상세

### 치장 아이템 정의
치장 아이템 = CMD를 가진 커스텀 아이템. 스탯 없음, 외형 데이터만 보유.

```
아이템 NBT:
  poro:cosmetic_type = "weapon" | "helmet" | "chestplate" | "leggings" | "boots"
  poro:cosmetic_cmd = <custom_model_data 값>
  poro:cosmetic_texture = "poro:item/cosmetics/weapons/holy_wing"
```

### 치장 + 실제 장비 동작 규칙

| 장착 상태 | 치장 슬롯 | 토글 | 결과 |
|---|---|---|---|
| 무기 장착, 치장 없음 | 비어있음 | 치장 보이기 | 원본 표시 (치장 없으므로) |
| 무기 장착, 치장 있음 | 치장 아이템 | 치장 보이기 | 치장 외형 |
| 무기 장착, 치장 있음 | 치장 아이템 | 원본 보이기 | 실제 무기 외형 |
| 무기 장착, 치장 있음 | 치장 아이템 | 안보이기 | 아무것도 안 보임 |
| 무기 없음 | — | — | 항상 아무것도 없음 |

## 6. 플러그인 이벤트 처리

```
InventoryClickEvent (장비 패널 내부)
  ├─ 장착 슬롯 (9·18·27·36·45) 클릭
  │   ├─ 올바른 아이템 타입 확인
  │   ├─ 장착/해제 처리
  │   └─ 스탯 재계산 → 스탯 표시 슬롯 갱신
  │
  ├─ 치장 슬롯 (10·19·28·37·46) 클릭
  │   ├─ poro:cosmetic_type 일치 확인
  │   └─ 치장 등록/해제 + 외형 업데이트
  │
  ├─ 토글 슬롯 (11·20·29·38·47) 클릭
  │   └─ 상태 COSMETIC→BASE→HIDDEN 순환 + 외형 즉시 반영
  │
  └─ 핫바 슬롯 선택 (48) 클릭
      └─ 1~9 순환 + 무기 이동 + DB 저장

PlayerInventoryClickEvent (일반 인벤토리에서)
  └─ 지정된 핫바 슬롯에 다른 아이템 이동 시도 → cancel
```
