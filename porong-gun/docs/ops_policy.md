# porong-gun 운영 정책 (= 약관) — 2026-06-10

> 디스코드 인증(`/verify`) 시 이 정책에 동의한 것으로 간주. SoT: [`concept.md`](concept.md)(비전) · [`design/world.md`](design/world.md)(운영 전제) · [`design/finale.md`](design/finale.md)(와이프 이월).

## 1. 🎯 운영 철학 = 최소 개입 (확정)
**게임 안에서 벌어지는 일은 게임이다.** porong-gun은 *비정한 자유 서바이벌* 정체성이라:
- **게임 내 행위(약탈·레이드·기습·배신·기만·동맹/파기·스폰 근처 교전) = 게임의 일부 → 운영 개입 X.**
- **운영자 개입 = 시스템 밖 문제만**(핵·버그 악용·현실 괴롭힘·현금거래).
- 근거: ①비정한 톤(분쟁도 콘텐츠) ②**1인 운영 = 24/7 감시 불가** → 자동화 + 신고 기반 사후 대응이 현실적.

## 2. 🚫 금지 행위 (밴 사유 — 시스템 밖)
| 행위 | 설명 | 제재 |
|---|---|---|
| **핵·치트·매크로** | 불공정 우위(에임봇·플라이·X-ray·자동화) | 즉시 영구 밴 |
| **버그·익스플로잇 악용** | 의도 안 된 메커니즘(복제·벽 통과·밀폐 우회 등) 반복 이용 | 경고→영구(신고 시 우선 패치) |
| **현실 괴롭힘** | 욕설·신상 공개·차별·협박(게임 채팅·디스코드) | 경고→영구 |
| **현금 거래(RMT)** | 아이템·화폐·계정 현금 매매 | 영구 밴 |
| **계정 공유·대리·인증 우회** | 화이트리스트 게이트 무력화 | 인증 해제·밴 |

## 3. ✅ 허용 (게임의 일부 — 밴 아님, 분쟁 예방 명시)
- **약탈·레이드·코어 파괴·기지 털기** / **기습·백스탭·동맹 파기·기만**(구두 연합) / **스폰 근처·POI 교전·데스박스 강탈** / **눈치·정치·협박(게임 내)**.
- → "당했다"는 신고는 **개입 대상 아님**(시스템이 허용한 행위). 단 시스템 밖 괴롭힘이 동반되면 그 부분만 제재.

## 4. ⚖️ 밴 정책
- **사안별 경고→임시→영구** / **핵·RMT = 즉시 영구**(경고 없음).
- **디스코드 추방 ↔ 화이트리스트 제거 연동**([`design/economy.md`](design/economy.md) 인증 모듈) = 디스코드/게임 밴 일원화.
- **신고 + 증거 기반**(스샷·영상) — 1인 운영이라 사후 검토. 무고·악의적 신고도 제재 대상.

## 5. 🔄 시즌 운영
- **와이프 주기 = 2주 기본**([`design/world.md`](design/world.md)), 유저 합의 시 연장. 시즌 종료=결전→와이프→**새 시드**.
- **절차:** 결전 D-N 카운트다운 → 결전(평일 ~21시) → 와이프 → 새 시즌 시드·화이트리스트 유지(재인증 X).
- **이월:** 탈출=배낭 전부 / 농성 생존=보안함·명예 / 사망·미접속=빈손([`design/finale.md`](design/finale.md)).

## 6. 💾 백업·롤백
- **정기 백업**(와이프 전 + 일일 권장).
- **🔑 롤백 = 서버 장애(크래시·데이터 손상)만** — **게임 내 손실(약탈·레이드·사망)은 롤백 없음**(비정한 톤, "잃으면 잃는 것"). 핵 피해는 가해자 밴이 원칙, 개별 보상은 무보상(심각 시 시즌 조기 와이프 검토).

## 7. 🛡️ 안티치트·1인 운영 현실
- **자동화 우선:** 백업·와이프·인증·화이트리스트는 자동(봇·스케줄). 안티치트 플러그인 검토(Forge 1.20.1 호환).
- **신고 기반 사후 대응**(상시 감시 불가 전제).
- **운영자 특혜 없음:** 운영자도 플레이어로 참여 시 동일 규칙(중립성) — 관리 권한은 운영 목적만, 게임 내 우위로 안 씀.

## 8. ⚠️ 미정 / 착수 시
- 안티치트 플러그인 선택(스택 통합 게이트 재검증), 신고 채널·양식, 백업 주기·보관, 무고 신고 기준 = 운영 착수 시.
